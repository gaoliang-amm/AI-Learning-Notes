## 二、核心概念的演进主线

### 主线一：从预测到生成

| 阶段 | 笔记 | 核心思想 |
|------|------|---------|
| 下一 Token 预测 | [[强化学习与多模态/01. LLM基础与GPT-2|01. LLM基础与GPT-2]] | 给定前文，预测下一个词 — LLM 的本质 |
| 自回归生成 | [[强化学习与多模态/01. LLM基础与GPT-2|01. LLM基础与GPT-2]] | 逐 token 生成，每次将输出追加到输入 — 从"预测"到"创造" |
| 扩散去噪 | [[强化学习与多模态/06. 扩散模型与文生图|06. 扩散模型与文生图]] | 从纯噪声逐步去噪生成图像 — 与自回归同属"逐步生成"思想 |

> **直觉联系**：LLM 逐个预测 token 直到句号，扩散模型逐个去噪步骤直到清晰图像 — 都是"从无到有"的迭代生成，只是作用在不同空间（文本 vs 像素）。

### 主线二：从试错到对齐

| 阶段 | 笔记 | 核心思想 |
|------|------|---------|
| 强化学习基础 | [[强化学习与多模态/02. 强化学习基础|02. 强化学习基础]] | Agent 在环境中试错，通过奖励信号学习策略 |
| PPO | [[强化学习与多模态/03. PPO与RLHF对齐|03. PPO与RLHF对齐]] | 裁剪机制防止策略更新过大 — 用"护栏"稳定学习 |
| RLHF | [[强化学习与多模态/03. PPO与RLHF对齐|03. PPO与RLHF对齐]] | 用人类反馈训练奖励模型，再用 PPO 优化 LLM — 让 AI 听人话 |
| DPO | [[强化学习与多模态/04. GRPO与DeepSeek-R1|04. GRPO与DeepSeek-R1]] | 去掉奖励模型，直接用偏好数据优化 — 简化 RLHF |
| GRPO | [[强化学习与多模态/04. GRPO与DeepSeek-R1|04. GRPO与DeepSeek-R1]] | 去掉价值网络，用组内相对排名计算优势 — 适合推理任务 |

> **直觉联系**：RLHF 的核心矛盾是"奖励信号从哪来"。PPO 需要人训练奖励模型；DPO 说"偏好数据本身就是奖励"；GRPO 说"一组回答中相对好的就是好的"——三种思路在不断简化人类反馈的获取成本。

### 主线三：从单模态到多模态

| 阶段 | 笔记 | 核心思想 |
|------|------|---------|
| ViT | [[强化学习与多模态/05. 多模态模型|05. 多模态模型]] | 把图像切成 patch，用 Transformer 处理 — 图像也是"语言" |
| CLIP | [[强化学习与多模态/05. 多模态模型|05. 多模态模型]] | 对比学习让图文在同一空间对齐 — "这张图"和"这个描述"是一对 |
| ClipCap | [[强化学习与多模态/05. 多模态模型|05. 多模态模型]] | CLIP 提取图像特征 → GPT-2 生成文字 — 跨模态桥接 |
| DALL-E2 | [[强化学习与多模态/06. 扩散模型与文生图|06. 扩散模型与文生图]] | CLIP 文本嵌入 → Prior → UNet 扩散 → 图像 — 文生图的完整链路 |

> **直觉联系**：CLIP 是连接视觉和语言的"翻译官"。在 ClipCap 中它帮 GPT-2 看懂图片；在 DALL-E2 中它帮扩散模型理解文字。同一个 CLIP，两种用法。

## 三、贯穿全局的三个关键概念

### 3.1 损失函数：看似不同，实则同源

| 模型 | 损失函数 | 本质 |
|------|---------|------|
| GPT-2 | 交叉熵（预测下一个 token） | 分类 |
| DPO | -log σ(β · margin) | 二分类 |
| 扩散模型 | MSE（预测噪声 vs 真实噪声） | 回归 |
| CLIP | 对称交叉熵（图文匹配） | 分类 |

看似五花八门，但核心都是**让模型的输出逼近某个目标**——只是"目标"的定义不同。

### 3.2 参考模型/冻结模型：防止"走太远"

| 场景 | 冻结模型 | 作用 |
|------|---------|------|
| PPO/RLHF | SFT 模型 $\pi_{ref}$ | KL 约束，防止偏离人类语言 |
| DPO | 参考 LLM $\pi_{ref}$ | 隐式正则化，防止过拟合 |
| GRPO | 参考模型 $\pi_{ref}$ | 显式 KL 损失项 |
| DALL-E2 | CLIP + Prior | 冻结先验，只训 UNet |
| ClipCap | CLIP | 冻结视觉编码器，只训映射网络+GPT2 |

> **直觉**：训练新模型时，"忘记旧知识"比"学到新知识"更危险。冻结参考模型就是给新模型一个"锚点"，让它在学新东西的同时不要丢掉已有能力。

### 3.3 嵌入空间：让不同东西"可比较"

| 模型 | 嵌入空间的作用 |
|------|--------------|
| GPT-2 | token embedding 让离散文字变成连续向量 |
| ViT | patch embedding 让像素块变成 token |
| CLIP | 将图像和文本映射到同一空间，点积 = 语义相似度 |
| ClipCap | MLP 将 CLIP 空间映射到 GPT-2 空间 |
| DALL-E2 | Prior 在 CLIP 嵌入空间中做扩散 |
| Qwen2.5-VL | 合并器将视觉 token 压缩到 LLM 空间 |

> **直觉**：嵌入空间是 AI 理解世界的"坐标系"。不同模型的嵌入空间就像不同语言——CLIP 翻译图文，ClipCap 翻译 CLIP→GPT2，DALL-E2 的 Prior 在 CLIP 的"语言"中做扩散。

## 六、常见困惑澄清

### Q: DPO 和 RLHF 是什么关系？
RLHF 是一个流程（预训练→SFT→RLHF），DPO/PPO/GRPO 是这个流程中"RL 阶段"的三种不同算法。PPO 是最早成功的（ChatGPT 用的），DPO 是简化的（不需要奖励模型），GRPO 是最新的（适合推理任务）。

### Q: 扩散模型和 LLM 有什么关系？
它们都是生成模型，但生成方式不同：LLM 自回归（逐 token），扩散迭代去噪（逐步消除噪声）。DALL-E2 将两者结合——用 LLM 类架构（CLIP+Prior）做条件准备，用扩散模型生成图像。

### Q: CLIP 和 GPT-2 有什么关系？
没有直接关系，但都是 Transformer 家族。CLIP 用 Encoder-Only（理解），GPT-2 用 Decoder-Only（生成）。在 ClipCap 中，CLIP 负责"看"，GPT-2 负责"说"，两者通过 MLP 桥接。

### Q: 为什么 GRPO 不需要价值网络？
PPO 用价值网络（Critic）评估每个 token 的优势，但 GRPO 认为：给同一个问题生成一组回答，组内相对排名就足以判断好坏。这减少了显存开销，且对推理任务更自然——答案是对是错，相对比较就够了。
