# 强化学习基础

## 一、强化学习的核心框架

强化学习研究 **智能体（Agent）** 如何在 **环境（Environment）** 中通过与环境的交互最大化获得的奖励。

### 核心概念

| 概念 | 符号 | 说明 |
|------|------|------|
| 状态 | $S_t$ | 当前环境的状态 |
| 动作 | $A_t$ | 智能体采取的动作 |
| 奖励 | $R_t$ | 环境给出的即时反馈 |
| 策略 | $\pi(a|s)$ | 在状态 s 下采取动作 a 的概率 |
| 折扣因子 | $\gamma$ | 未来奖励的折扣系数 (0~1) |

### 交互循环

```
环境 ──状态 S_t──→ 智能体
智能体 ──动作 A_t──→ 环境
环境 ──状态 S_{t+1} + 奖励 R_t──→ 智能体
```

### 策略函数

$$\pi(a|s) = P(A_t = a | S_t = s)$$

- 输入状态，输出所有可能动作的概率分布
- 从概率分布中 **采样** 一个动作（而非直接选最大）
- **探索与利用的权衡**：总是选概率最大的动作（利用）可能导致忽略更好的策略，偶尔选概率小的动作（探索）能发现更好的长期回报

## 二、回报与价值函数

### 回报（Return）

未来奖励的折扣总和：

$$G_t = R_t + \gamma R_{t+1} + \gamma^2 R_{t+2} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k}$$

递推形式：$G_t = R_t + \gamma G_{t+1}$

- $\gamma = 0$：只看即时奖励（短视）
- $\gamma \approx 1$：重视未来奖励（远视）
- 防止连续任务中奖励发散

### 状态价值函数

$$V_\pi(s) = \mathbb{E}_\pi[G_t | S_t = s]$$

衡量在状态 s 下一直按策略 $\pi$ 行动的未来预期回报。

**贝尔曼期望方程**：

$$V_\pi(s) = \mathbb{E}_\pi[R_t + \gamma V_\pi(S_{t+1}) | S_t = s]$$

直观理解：当前价值 = 当前奖励 + 折扣因子 × 下一状态的价值

### 最优状态价值函数

$$V_*(s) = \max_\pi V_\pi(s)$$

### 马尔可夫决策过程（MDP）

MDP 定义了强化学习的数学框架：

- **状态转移**：$p(s'|s,a)$ — 在状态 s 采取动作 a 转移到 s' 的概率
- **奖励函数**：$r(s,a,s')$ — 在 (s,a,s') 下的奖励
- **马尔可夫性质**：下一状态只依赖当前状态和动作，不需要历史信息

> **奖励稀疏性问题**：如果只有成功时才给奖励（如插插头），智能体可能永远得不到奖励就放弃了。

## 三、策略梯度法（Policy Gradient）

### 核心思想

直接将策略参数化（用神经网络 $\pi_\theta$），通过梯度上升最大化期望回报：

$$\max_\theta J(\theta) = \mathbb{E}_{\tau \sim \pi_\theta} \left[ \sum_{t=0}^T G(\tau) \right]$$

### 策略梯度定理

$$\nabla_\theta J(\theta) \approx \frac{1}{N} \sum_{i=1}^N \sum_{t=0}^T G(\tau^{(i)}) \nabla_\theta \log \pi_\theta(A_t^{(i)} | S_t^{(i)})$$

**关键解读**：
- $\nabla \log \pi_\theta(a|s)$：提升某动作的概率方向
- $G(\tau)$：作为权重 —— 好轨迹（高回报）提升其中所有动作的概率，差轨迹降低动作概率
- 等价于最小化交叉熵损失：$J(\theta) = \sum G(\tau) \log \pi_\theta(A_t|S_t)$

### 代码实现要点

```python
class Agent:
    def collect_trajectory(self, env):
        # 采样一条轨迹: [states, actions, rewards]
        states, actions, rewards = [], [], []
        state = env.reset()
        while not done:
            action, probs = self.get_action(state)  # 采样
            next_state, reward, done, _ = env.step(action)
            states.append(state); actions.append(action); rewards.append(reward)
            state = next_state
        return states, actions, rewards

    def update(self, trajectory):
        states, actions, rewards = trajectory
        # 逆序计算回报 G
        G = 0
        for r in rewards[::-1]:
            G = r + self.gamma * G
        # 梯度上升：提升高回报轨迹中动作的概率
        loss = -log_prob * G  # 负号因为是梯度下降
        loss.backward()
        self.optimizer.step()
```

## 四、REINFORCE 算法

REINFORCE 是最基本的策略梯度算法：

1. 用策略网络采样得到完整轨迹
2. 逆序计算每条轨迹的回报 $G(\tau)$
3. 对所有时间步计算策略梯度并更新

```python
# 核心：对每条轨迹的每个时间步，损失 = -log_prob * G
for s, a in zip(states, actions):
    log_prob = torch.log(probs[a])
    loss += -log_prob * G
```

## 五、Actor-Critic 架构

### 动机

原始策略梯度中，用整条轨迹的回报 $G(\tau)$ 来评估每个动作 —— 方差很大。

Actor-Critic 引入 **Critic 网络**（价值网络）来更精确地评估每个状态的价值：

```
Actor（策略网络） → 输出动作概率分布
Critic（价值网络） → 输出状态价值 V(s)
```

### 优势函数

$$A(s,a) = Q(s,a) - V(s)$$

或使用时序差分误差：

$$\delta = R_t + \gamma V(S_{t+1}) - V(S_t)$$

- $\delta > 0$：比预期好，提升该动作概率
- $\delta < 0$：比预期差，降低该动作概率

### 代码要点

```python
class ActorCritic(nn.Module):
    def __init__(self, input_size, action_size):
        self.actor = nn.Sequential(
            nn.Linear(input_size, 128),
            nn.ReLU(),
            nn.Linear(128, action_size),
            nn.Softmax(dim=-1)
        )
        self.critic = nn.Sequential(
            nn.Linear(input_size, 128),
            nn.ReLU(),
            nn.Linear(128, 1)  # 输出标量价值
        )
```

## 六、时序差分学习（TD）

### TD(0)

结合蒙特卡洛（完整轨迹才更新）和动态规划（单步更新）的优点：

$$V(S_t) \leftarrow V(S_t) + \alpha [R_{t+1} + \gamma V(S_{t+1}) - V(S_t)]$$

- 只依赖单步经验，在线学习
- 比蒙特卡洛方差小，但引入偏差

## 七、关键要点总结

| 算法 | 核心思路 | 优点 | 缺点 |
|------|---------|------|------|
| 原始策略梯度 | 整条轨迹回报作为权重 | 简单直观 | 方差大 |
| REINFORCE | 策略梯度 + 完整轨迹 | 理论基础清晰 | 高方差，需要大量采样 |
| Actor-Critic | Actor 选动作 + Critic 评估 | 降低方差 | 需要训练两个网络 |
| TD(0) | 单步bootstrap更新 | 在线学习，低方差 | 引入偏差 |
