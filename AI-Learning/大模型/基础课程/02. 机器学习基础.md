## 一、ML 概述

### 1.1 什么是机器学习

机器学习是一门让计算机**从数据中自动归纳规律**，并利用规律实现预测或决策的学科。核心逻辑是打破传统"硬编码规则"的模式，转而**数据驱动**。

> **直觉类比**：传统编程是"人告诉机器规则"，机器学习是"机器从数据中自己发现规则"。

ML 最核心的三个问题：

| 问题 | 含义 | 核心任务 |
|------|------|---------|
| **学什么** | 确定用何种数学函数/模型去拟合数据规律 | 模型选择 |
| **怎么学** | 确定评价标准（损失函数），通过算法寻找最优参数 | 参数训练 |
| **学得好不好** | 验证模型的预测/决策结果是否准确 | 模型评估 |

### 1.2 发展历史

- **1950s**：图灵测试、感知机（Rosenblatt, 1958）
- **1980s**：决策树、反向传播算法、支持向量机
- **2006**：Hinton 提出深度学习（深层神经网络）
- **2012**：AlexNet 在 ImageNet 大放异彩，深度学习爆发
- **2017**：Transformer 架构提出（Attention Is All You Need）
- **2022+**：大语言模型（GPT、LLaMA、DeepSeek）时代

### 1.3 AI、ML、DL 的关系

```
人工智能（AI）
  └── 机器学习（ML）
        └── 深度学习（DL）
```

- **AI**：最广泛，包含规则系统、专家系统、ML 等
- **ML**：AI 的子集，强调从数据中学习
- **DL**：ML 的子集，使用多层神经网络自动学习特征表示

### 1.4 ML 应用领域

NLP、计算机视觉、推荐系统、语音识别、自动驾驶、医疗诊断、金融风控等。

### 1.5 ML 分类

| 类型 | 描述 | 典型算法 |
|------|------|---------|
| **有监督学习** | 有标签数据，学习输入→输出映射 | 线性回归、逻辑回归、SVM、决策树 |
| **无监督学习** | 无标签数据，挖掘数据本身的结构 | K-Means、PCA、DBSCAN |
| **半监督学习** | 少量标签 + 大量无标签，降低标注成本 | 自训练、标签传播 |
| **强化学习** | 智能体与环境交互，以奖惩为反馈学习策略 | Q-Learning、PPO |

### 1.6 核心术语

| 术语 | 定义 |
|------|------|
| **数据集（Data Set）** | 多条记录的集合 |
| **训练集（Training Set）** | 用于训练模型的数据 |
| **验证集（Validation Set）** | 用于调节超参数的数据 |
| **测试集（Test Set）** | 用于评估模型性能的数据 |
| **样本（Sample）** | 数据集中的一条记录 |
| **特征（Feature）** | 反映样本某方面性质的变量，也称属性 |
| **特征向量（Feature Vector）** | 将样本所有特征表示为向量形式 |
| **标签（Label）** | 监督学习中每个样本的结果信息，也称目标值（target） |
| **模型（Model）** | 算法与训练后参数的集合，用于预测或分类 |
| **参数（Parameter）** | 模型通过训练学习到的值（如权重和偏置） |
| **超参数（Hyper Parameter）** | 由用户手动设置，不能通过训练自动学习（如学习率、正则化系数） |

> **参数 vs 超参数**：参数是模型自己学的（如 w, b），超参数是人提前定的（如学习率 α、正则化系数 λ）。

## 二、建模流程

监督学习一般建模流程：

```
数据收集 → 数据清洗 → 特征工程 → 模型选择与训练 → 模型评估 → 模型部署
```

### 2.1 数据收集

根据应用场景收集相关行业数据，确保数据能代表实际问题的不同方向。

### 2.2 数据清洗

剔除脏数据和不可用数据（缺失值、异常值、重复值等）。

### 2.3 特征工程

特征工程（Feature Engineering）是 ML 中**极其重要**的一步，指通过对原始数据的处理、转换和构造，生成新的特征或选择有效的特征，从而提高模型性能。

> **直觉类比**：特征工程就像"给模型做菜前的食材处理"——食材（数据）处理得好，菜（模型）才好吃。

#### 2.3.1 特征选择

**定义**：从原始特征中挑选出与目标变量关系最密切的特征，剔除冗余、无关或噪声特征。**不会创建新特征，不改变数据结构**。

**目的**：减少模型复杂度、加速训练、减少过拟合。

**举例**：预测房价，原始特征有面积、朝向、房主手机号、门牌号、楼层、周边奶茶店数量。特征选择后删掉手机号、门牌号（无关噪声），保留面积/楼层/周边配套。

**常用方法**：

| 方法 | 原理 | 适用场景 |
|------|------|---------|
| **低方差过滤** | 删除方差低于阈值的特征（方差小 = 区分度低） | 快速筛选无用特征 |
| **皮尔逊相关系数** | 计算特征与标签的线性相关程度 | 线性关系筛选 |

```python
import numpy as np
from sklearn.feature_selection import VarianceThreshold

# 低方差过滤：删除方差低于0.2的特征
var_thresh = VarianceThreshold(0.2)
X_filtered = var_thresh.fit_transform(X)
```

```python
import pandas as pd
adv = pd.read_csv("advertising.csv")
x = adv.drop("Sales", axis=1)
y = adv["Sales"]

# 皮尔逊相关系数
corr = x.corrwith(y, method="pearson")
print(corr)

# 相关系数矩阵 + 热力图
matrix = adv.corr(method="pearson")
import seaborn as sns
sns.heatmap(matrix, cmap="coolwarm", fmt=".2f", annot=True)
```

#### 2.3.2 特征转换

对数据做数学/统计处理，使其适配模型输入要求。根据特征类型分为**数字型特征**和**类别型特征**。

**归一化（Min-Max Scaling）**：将数据缩放到 [0, 1] 区间。

$$x' = \frac{x - x_{\min}}{x_{\max} - x_{\min}}$$

```python
from sklearn.preprocessing import MinMaxScaler
scaler = MinMaxScaler(feature_range=(0, 1))
x_scaled = scaler.fit_transform(x)
```

**标准化（Standardization / Z-Score）**：将数据转换为均值 0、标准差 1 的分布。

$$x' = \frac{x - \mu}{\sigma}$$

```python
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
x_scaled = scaler.fit_transform(x)
```

> **归一化 vs 标准化**：归一化受异常值影响大（有 min/max），标准化更稳健。一般优先用标准化。

#### 2.3.3 标签编码（Label Encoding）

将类别型特征转换为整数编码。**注意**：整数编码会引入虚假的有序关系（如 1 < 2 < 3），仅适用于有序类别（如学历：本科 < 硕士 < 博士）。

```python
from sklearn.preprocessing import LabelEncoder
data = ['本科', '硕士', '博士', '本科']
encoder = LabelEncoder()
result = encoder.fit_transform(data)  # [0, 1, 2, 0]
```

#### 2.3.4 独热编码（One-Hot Encoding）

将类别型特征转换为二元向量，**消除虚假的顺序关系**。适用于名义变量（如颜色：红/蓝/绿）。

```python
from sklearn.preprocessing import OneHotEncoder
X = np.array([['红'], ['蓝'], ['绿'], ['红']])
encoder = OneHotEncoder()
result = encoder.fit_transform(X).toarray()
# [[1,0,0], [0,1,0], [0,0,1], [1,0,0]]
```

**避免多重共线性**：独热编码后各列之和恒为 1（完全线性相关），会导致矩阵不可逆。使用 `drop="first"` 删除每个特征的第一列来打破共线性。

#### 2.3.5 特征构造

基于现有特征创造新的、更有代表性的特征。通过组合、转换或聚合现有特征，形成更能反映数据规律的特征。

例如：已有"长度"和"宽度"，可构造"面积 = 长度 × 宽度"。

#### 2.3.6 特征降维 — PCA 主成分分析

当特征数量非常大时，PCA 通过线性变换将高维数据投影到低维空间，**保留最大方差方向**，减少计算复杂度并避免过拟合。

```python
from sklearn.decomposition import PCA

# n_components 为浮点数时表示保留的信息量比例，为整数时表示保留的维度数
pca = PCA(n_components=0.9)
x_pca = pca.fit_transform(x)
print(pca.explained_variance_ratio_)  # 各主成分解释的方差比例
```

> **直觉类比**：PCA 就像给数据"拍照片"——从最佳角度拍，用最少的维度保留最多的信息。

### 2.4 模型选择与训练

根据问题类型（回归/分类）和数据特点选择合适的模型，通过训练集拟合模型参数。详见后文经典模型部分。

## 三、损失函数

损失函数是衡量模型预测好坏的"标尺"，也是指导模型优化的"指挥棒"。

> **直觉类比**：损失函数就像打靶——子弹（预测值）离靶心（真实值）越远，损失越大。

损失函数的核心作用：
1. **评估**模型的预测精度
2. **指导**模型优化方向（训练过程就是让损失函数越来越小）

### 3.1 经验误差与泛化误差

| 概念 | 定义 |
|------|------|
| **经验误差（训练误差）** | 模型在训练集上的平均误差 |
| **泛化误差（测试误差）** | 模型在测试集上的平均误差 |

**经验风险最小化（ERM）**：当经验风险最小时，认为取到了最优模型。

### 3.2 常见损失函数

#### MSE（均方误差）

$$\text{MSE} = \frac{1}{n} \sum_{i=1}^{n}(y_i - \hat{y}_i)^2$$

- 对大误差敏感（平方放大），适用于需要惩罚大误差的场景
- **凸函数**，存在全局唯一最小值，便于求解
- 线性回归默认损失函数

#### MAE（平均绝对误差）

$$\text{MAE} = \frac{1}{n} \sum_{i=1}^{n}|y_i - \hat{y}_i|$$

- 对异常值不敏感，解释直观
- 适用于数据包含异常值的场景

#### RMSE（均方根误差）

$$\text{RMSE} = \sqrt{\frac{1}{n} \sum_{i=1}^{n}(y_i - \hat{y}_i)^2}$$

- 量纲与目标变量一致，更直观
- 过度追求低 RMSE 可能导致过拟合

#### 交叉熵损失（Cross-Entropy Loss）

二分类（Binary Cross-Entropy / Log Loss）：

$$L = -\frac{1}{n}\sum_{i=1}^{n}[y_i \log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i)]$$

- 逻辑回归的损失函数，来源于最大似然估计（MLE）
- 衡量模型输出的概率分布与真实标签之间的差距

### 3.3 正则化

正则化（Regularization）是在损失函数中添加额外项来**惩罚过大的参数**，限制模型复杂度，避免过拟合。

$$\text{Total Loss} = \text{原损失函数} + \lambda \cdot \text{正则化项}$$

- **原损失函数**：拟合数据集
- **正则化项**：减小参数大小，降低模型复杂度
- **λ（正则化系数）**：超参数，控制惩罚强度

| 正则化 | 公式 | 效果 | 别名 |
|--------|------|------|------|
| **L1** | $\lambda \sum \|w_i\|$ | 使部分权重变为 0 → **特征选择**，产生稀疏模型 | Lasso 回归 |
| **L2** | $\lambda \sum w_i^2$ | 使所有权重变小但不为 0 → **防止过拟合**，模型更平滑 | Ridge 回归（岭回归） |
| **ElasticNet** | $\alpha L1 + (1-\alpha) L2$ | 结合 L1 和 L2，兼具稀疏性和稳定性 | 弹性网络回归 |

```python
from sklearn.linear_model import Lasso, Ridge

# L1 正则化（Lasso）
model = Lasso(alpha=0.009)

# L2 正则化（Ridge）
model = Ridge(alpha=0.009)
```

## 四、模型评估

### 4.1 回归模型评估指标

| 指标 | 公式 | 特点 |
|------|------|------|
| **MAE** | $\frac{1}{n}\sum\|y_i-\hat{y}_i\|$ | 对异常值不敏感 |
| **MSE** | $\frac{1}{n}\sum(y_i-\hat{y}_i)^2$ | 放大误差，对异常值敏感 |
| **RMSE** | $\sqrt{\text{MSE}}$ | 量纲与目标一致 |
| **R²（决定系数）** | $1 - \frac{\sum(y_i-\hat{y}_i)^2}{\sum(y_i-\bar{y})^2}$ | 越接近 1 越好 |

R² 的含义：
- R² = 1：完美拟合
- R² = 0：模型预测不如直接用均值
- R² < 0：模型比均值还差

```python
model.score(test_x, test_y)  # sklearn 中 score 计算的就是 R²
```

### 4.2 交叉验证

交叉验证通过将数据划分为多个子集，反复训练和验证，减少单次划分的随机性误差。

| 方法 | 描述 | 特点 |
|------|------|------|
| **简单交叉验证** | 70% 训练 / 30% 验证 | 受单次划分影响大 |
| **k 折交叉验证** | 分 k 折，每次 k-1 折训练、1 折验证，取平均 | 最常用，结果稳定 |
| **留一交叉验证（LOO）** | 每次留 1 个样本验证 | 适用于小数据，计算成本极高 |

### 4.3 分类模型评估指标

#### 混淆矩阵（Confusion Matrix）

|  | 预测正例 | 预测负例 |
|--|---------|---------|
| **实际正例** | TP（真正例） | FN（假负例） |
| **实际负例** | FP（假正例） | TN（真负例） |

```python
import sklearn.metrics as metrics
y_true = ["猫", "猫", "猫", "猫", "猫", "猫", "狗", "狗", "狗", "狗"]
y_pred = ["猫", "猫", "狗", "猫", "猫", "猫", "猫", "猫", "狗", "狗"]
matrix = metrics.confusion_matrix(y_true, y_pred, labels=["猫", "狗"])
```

#### 核心指标

| 指标 | 公式 | 含义 | 别名 |
|------|------|------|------|
| **准确率（Accuracy）** | (TP+TN) / 全部 | 正确预测的比例 | — |
| **精确率（Precision）** | TP / (TP+FP) | 预测为正例中实际为正例的比例 | 查准率 |
| **召回率（Recall）** | TP / (TP+FN) | 实际为正例中预测为正例的比例 | 查全率 |
| **F1 分数** | 2PR / (P+R) | 精确率和召回率的调和平均 | — |

> **直觉类比**：精确率 = "说正的里面有几成是对的"，召回率 = "真正的正例抓回来了几成"。F1 是两者的平衡。

#### ROC 曲线与 AUC

- **ROC 曲线**：以假正例率（FPR）为横轴，真正例率（TPR/召回率）为纵轴，展示不同阈值下模型表现
- **AUC（Area Under Curve）**：ROC 曲线下面积
  - AUC = 1：完美模型
  - AUC = 0.5：随机猜测
  - AUC 越大，模型区分正负类能力越强

```python
from sklearn.metrics import roc_auc_score
auc = roc_auc_score(test_y, model.predict_proba(test_x)[:, 1])
```

```python
# 评估报告
print(metrics.classification_report(y_true, y_pred, labels=["猫", "狗"]))
```

## 五、经典模型

### 5.1 线性回归（Linear Regression）

#### 基本原理

线性回归通过拟合一条直线（或超平面）来描述自变量与因变量之间的线性关系：

$$\hat{y} = w_0 + w_1 x_1 + w_2 x_2 + \cdots + w_n x_n$$

- $w_0$：截距（bias），自变量全为 0 时的基准值
- $w_i$：系数（weight），每个自变量对因变量的影响程度

**一元线性回归**：只有一个自变量 $\hat{y} = w_0 + w_1 x$

**多元线性回归**：多个自变量 $\hat{y} = w_0 + w_1 x_1 + \cdots + w_n x_n$

#### 应用场景

GDP 预测、广告效果评估、药物剂量研究、产品质量控制、政策效果评估、气候变化建模。**总结：只要关系是线性的，都可以用线性回归。**

#### 为什么用 MSE 作为损失函数

1. **凸函数**：存在全局唯一最小值，平方项处处可导
2. **解析解存在**：最小二乘法可通过矩阵运算直接求解 $\hat{w} = (X^TX)^{-1}X^Ty$
3. **统计最优**：若误差服从正态分布，MSE 对应极大似然估计（MLE）

**缺点**：对大误差敏感（平方放大效应）。

#### 求解方法

**解析法（正规方程法）**：直接计算 $\hat{w} = (X^TX)^{-1}X^Ty$，适合中小规模数据。

**梯度下降法**：迭代优化，适合大规模数据。详见下节。

```python
from sklearn.linear_model import LinearRegression

model = LinearRegression()
model.fit(X, y)
print(model.coef_)      # 系数
print(model.intercept_) # 截距
print(model.predict([[30]]))  # 预测
```

### 5.2 梯度下降法（Gradient Descent）

#### 基本原理

梯度下降是一种一阶迭代优化算法。核心思路：从初始参数出发，沿着**负梯度方向**（损失函数下降最快的方向）不停更新参数，最终收敛到极小值。

$$w_{k+1} = w_k - \alpha \cdot \nabla L(w_k)$$

- $\nabla L(w_k)$：损失函数在当前参数下的梯度
- $\alpha$：学习率（步长），控制每一步更新的幅度

> **直觉类比**：蒙着眼睛下山，每一步往最陡的方向走一步，步长由学习率控制。

#### 三种梯度下降变体

| 变体 | 每次使用数据量 | 优点 | 缺点 |
|------|--------------|------|------|
| **BGD（批量梯度下降）** | 全部训练数据 | 稳定收敛 | 计算开销大 |
| **SGD（随机梯度下降）** | 1 个随机样本 | 速度快，适合大数据 | 震荡大，难收敛 |
| **Mini-batch GD** | 一小批（如 32/64 个） | 平衡速度和稳定性 | 需选择 batch size |

> **Mini-batch GD 是最常用的方法**，深度学习中几乎都用它。

#### 学习率的选择

- **过大**：跳过最优解，甚至发散（损失越来越大）
- **过小**：收敛极慢，易陷入局部极小
- **自适应学习率**：Adam、Adagrad 等优化器动态调整

#### 特征缩放

梯度下降收敛慢的一个常见原因是特征尺度不一致。**提前对特征做标准化或归一化**可以显著加快收敛。

> **直觉类比**：如果一个特征范围是 [0, 1]，另一个是 [0, 10000]，损失函数的等高线会非常"扁"，梯度下降会来回震荡。缩放后等高线变"圆"，收敛更快。

#### 牛顿法（二阶方法）

利用二阶导数（黑塞矩阵）信息，收敛速度快但计算复杂（需要求矩阵逆）。拟牛顿法用正定矩阵近似黑塞矩阵的逆，降低计算成本。一般用于中小规模凸优化问题。

```python
# 手动梯度下降示例
import numpy as np

x = np.hstack((np.ones((10, 1)), np.array([[5],[8],[10],[12],[15],[3],[7],[9],[14],[6]])))
y = np.array([[55],[65],[70],[75],[85],[50],[60],[72],[80],[58]])
n = x.shape[0]

def grad(beta):
    return (x.T @ (x @ beta - y)) / n * 2

beta = np.array([[1], [1]])
alpha = 0.01
for i in range(10000):
    beta = beta - alpha * grad(beta)
```

### 5.3 逻辑回归（Logistic Regression）

#### 基本原理

逻辑回归用于**分类问题**（尤其二分类）。名称含"回归"但本质是分类。核心思想：将线性回归的输出通过 **Sigmoid 函数**映射到 [0, 1] 区间，表示概率。

$$\sigma(z) = \frac{1}{1 + e^{-z}}, \quad \sigma'(z) = \sigma(z)(1 - \sigma(z))$$

$$P(y=1|x) = \sigma(w^Tx + b)$$

- 输出 > 阈值 → 预测为 1 类
- 输出 ≤ 阈值 → 预测为 0 类

> **直觉类比**：线性回归输出一个任意实数，Sigmoid 把它"压"到 0~1 之间变成概率。

#### 损失函数：交叉熵

$$L = -\frac{1}{n}\sum_{i=1}^{n}[y_i \log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i)]$$

来源于最大似然估计（MLE）。为什么不用 MSE？因为 MSE 在 Sigmoid 输出上**非凸**，容易陷入局部最优；交叉熵是凸的。

#### 应用场景

信用评分、欺诈检测、垃圾邮件检测、广告点击预测、情感分析、医学诊断等。

```python
from sklearn.linear_model import LogisticRegression

model = LogisticRegression(
    solver="lbfgs",    # 优化算法（拟牛顿法）
    penalty="l2",      # 正则化类型
    C=1.0,             # 正则化强度（C 越小正则化越强）
    class_weight="balanced"  # 自动平衡类别权重
)
model.fit(train_x, train_y)
model.score(test_x, test_y)  # 准确率
```

#### 多分类扩展

**OvR（One-vs-Rest）**：训练 C 个二分类器，每个将一个类别 vs 其余类别，预测时取概率最高的。

**Softmax 回归（多项逻辑回归）**：直接建模所有类别的概率分布，概率之和为 1。

$$P(y=k|x) = \frac{e^{w_k^Tx}}{\sum_{j=1}^{C} e^{w_j^Tx}}$$

- 只需训练 1 个模型，计算高效
- 所有类别概率和为 1，分类一致性更好

```python
# Softmax 回归（sklearn 中 LogisticRegression 默认使用）
model = LogisticRegression(max_iter=10000)
model.fit(train_x, train_y)
model.predict_proba(x)  # 输出各类别概率
```

### 5.4 感知机（Perceptron）

#### 基本原理

感知机是**最简单的神经网络**，二分类模型。接收多个输入信号，乘以权重求和后，超过阈值则输出 1（激活），否则输出 0。

$$y = \begin{cases} 1 & \text{if } \sum w_i x_i + b > 0 \\ 0 & \text{otherwise} \end{cases}$$

> **直觉类比**：感知机就像一个"投票器"——每个输入信号有自己的权重（重要性），加权求和后看是否超过"门槛"。

#### 逻辑门实现

感知机可以实现与门、或门、与非门：

```python
import numpy as np

class Perceptron:
    def __init__(self, w, b):
        self.w = w
        self.b = b

    def predict(self, x):
        return 1 if self.w @ x + self.b > 0 else 0

# 与门
and_gate = Perceptron(w=np.array([0.5, 0.5]), b=-0.7)
# 或门
or_gate = Perceptron(w=np.array([0.5, 0.5]), b=0)
# 与非门
nand_gate = Perceptron(w=np.array([-0.5, -0.5]), b=0.7)
```

#### 单层感知机的局限性：异或问题

单层感知机**无法实现异或门（XOR）**，因为异或问题是**线性不可分**的——无法用一条直线将 0 和 1 分开。

> 这就是 1969 年 Minsky 指出的感知机局限性，导致了第一次 AI 寒冬。

#### 多层感知机（MLP）

通过**叠加多层感知机**可以解决线性不可分问题。异或门可以由与门、或门、与非门组合实现：

```python
def xor(x):
    s1 = nand_gate.predict(x)  # 与非
    s2 = or_gate.predict(x)    # 或
    return and_gate.predict(np.array([s1, s2]))  # 与
```

多层感知机是**深度学习的基础**——当层数和神经元增多，就变成了深层神经网络。

### 5.5 决策树（Decision Tree）

#### 基本原理

决策树通过一系列**if-else 判断**对数据进行分类或回归。树的每个内部节点是一个特征判断，每个分支是一个判断结果，每个叶节点是一个预测值。

> **直觉类比**：决策树就像"20 个问题"游戏——通过一系列问题逐步缩小答案范围。

#### 核心问题：如何选择分裂特征？

| 算法 | 分裂标准 | 适用任务 |
|------|---------|---------|
| **ID3** | 信息增益（越大越好） | 分类 |
| **C4.5** | 信息增益比（修正 ID3 偏向多值特征） | 分类 |
| **CART** | 基尼系数（分类）/ MSE（回归） | 分类+回归 |

**信息熵**：衡量数据集的"不纯度"，$H(D) = -\sum_{k=1}^{K} p_k \log_2 p_k$

- 熵越大 → 数据越混乱 → 越不纯
- 熵 = 0 → 所有样本属于同一类 → 完全纯净

**基尼系数**：$Gini(D) = 1 - \sum_{k=1}^{K} p_k^2$，含义与熵类似，但计算更简单。

#### 剪枝

| 方法 | 描述 |
|------|------|
| **预剪枝** | 训练时限制（最大深度、最小样本数、最小信息增益） |
| **后剪枝** | 先生成完整树，再自底向上删除无用子树 |

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 可解释性强（白盒模型） | 容易过拟合 |
| 无需特征缩放 | 对噪声敏感 |
| 能处理非线性关系 | 不稳定（数据微小变化可能导致完全不同的树） |
| 能处理类别型特征 | 贪心算法，不保证全局最优 |

```python
from sklearn.tree import DecisionTreeClassifier, DecisionTreeRegressor

# 分类
clf = DecisionTreeClassifier(max_depth=5, min_samples_split=10)
clf.fit(X_train, y_train)

# 回归
reg = DecisionTreeRegressor(max_depth=5)
reg.fit(X_train, y_train)
```

### 5.6 KNN（K 近邻）

#### 基本原理

给定一个新样本，找到训练集中与它**最近的 K 个邻居**，投票决定其类别（分类）或取平均值（回归）。

> **直觉类比**：KNN 就像"近朱者赤，近墨者黑"——你跟谁最近，你就属于谁的类别。

**三个关键要素**：
1. **距离度量**：欧氏距离、曼哈顿距离、余弦相似度
2. **K 值选择**：K 太小 → 过拟合（对噪声敏感）；K 太大 → 欠拟合（边界模糊）
3. **投票策略**：多数投票（分类）/ 加权投票（距离越近权重越大）

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 简单直观，无需训练 | 预测时需遍历全部训练数据，O(n) |
| 天然支持多分类 | 对高维数据效果差（维度灾难） |
| 非参数方法，无需假设数据分布 | 对不平衡数据敏感 |

```python
from sklearn.neighbors import KNeighborsClassifier

knn = KNeighborsClassifier(n_neighbors=5, weights='distance')
knn.fit(X_train, y_train)
knn.predict(X_test)
```

### 5.7 SVM（支持向量机）

#### 基本原理

SVM 的目标是找到一个**最大间隔超平面**，将不同类别的样本分开。距离超平面最近的样本点称为**支持向量**。

$$\min \frac{1}{2}\|w\|^2 \quad \text{s.t.} \quad y_i(w^Tx_i + b) \geq 1$$

> **直觉类比**：SVM 就像在两堆人中间画一条线，不仅要画对，还要让线离两边的人都尽可能远（最大间隔）。

#### 核心技巧

**核函数（Kernel Trick）**：将低维线性不可分的数据映射到高维空间，使其线性可分。关键是**不需要显式计算高维映射**，只需计算核函数。

| 核函数 | 公式 | 适用场景 |
|--------|------|---------|
| **线性核** | $K(x,z) = x^Tz$ | 线性可分 |
| **多项式核** | $K(x,z) = (x^Tz + c)^d$ | 中等复杂度 |
| **RBF（高斯核）** | $K(x,z) = \exp(-\gamma\|x-z\|^2)$ | **最常用**，复杂非线性 |

**软间隔**：允许部分样本违反间隔约束（通过惩罚参数 C 控制）。C 大 → 间隔小，过拟合风险高；C 小 → 间隔大，欠拟合风险高。

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 小样本表现好 | 大数据集训练慢（$O(n^2)$~$O(n^3)$） |
| 核函数处理非线性 | 对特征缩放敏感 |
| 全局最优（凸优化） | 不直接支持多分类（需 OvR/OvO） |

```python
from sklearn.svm import SVC

svm = SVC(kernel='rbf', C=1.0, gamma='scale')
svm.fit(X_train, y_train)
```

### 5.8 集成学习

#### 核心思想

组合多个弱学习器，形成一个强学习器。"三个臭皮匠，赛过诸葛亮"。

#### Bagging（Bootstrap Aggregating）

**思路**：从训练集中有放回地随机采样多个子集，分别训练多个模型，**投票/平均**取结果。

**代表算法——随机森林（Random Forest）**：
- 基学习器：多棵决策树
- 随机性 1：样本随机（Bootstrap 采样）
- 随机性 2：特征随机（每棵树分裂时只考虑部分特征）
- 最终结果：分类取多数投票，回归取平均值

> **直觉**：随机森林 = Bagging + 决策树 + 特征随机。每棵树看问题的角度不同，综合起来更全面。

**优点**：降低过拟合（方差减小）、可并行训练、特征重要性可解释。

```python
from sklearn.ensemble import RandomForestClassifier, RandomForestRegressor

rf = RandomForestClassifier(n_estimators=100, max_depth=10, random_state=42)
rf.fit(X_train, y_train)
rf.feature_importances_  # 特征重要性
```

#### Boosting

**思路**：串行训练多个弱学习器，每一轮重点关注前一轮**预测错误**的样本。

| 算法 | 核心思想 | 特点 |
|------|---------|------|
| **AdaBoost** | 调整样本权重（错误样本权重增大） | 对噪声敏感 |
| **GBDT** | 拟合前一轮的**残差**（负梯度） | 表达能力强，但串行训练慢 |
| **XGBoost** | GBDT + 正则化 + 二阶泰勒展开 + 列采样 | 精度高，速度快 |
| **LightGBM** | 直方图近似 + 叶子优先生长 | 速度更快，适合大数据 |

**GBDT 核心思想**：每棵新树拟合的是前一轮模型的**负梯度**（回归问题中就是残差 = 真实值 - 预测值）。

```python
from sklearn.ensemble import GradientBoostingClassifier
from xgboost import XGBClassifier

# GBDT
gbdt = GradientBoostingClassifier(n_estimators=100, learning_rate=0.1)
gbdt.fit(X_train, y_train)

# XGBoost
xgb = XGBClassifier(n_estimators=100, learning_rate=0.1, max_depth=6)
xgb.fit(X_train, y_train)
```

#### Bagging vs Boosting 对比

| 特性 | Bagging | Boosting |
|------|---------|----------|
| 训练方式 | 并行 | 串行 |
| 样本权重 | 等权 | 关注错误样本 |
| 目标 | 降低方差 | 降低偏差 |
| 代表 | 随机森林 | GBDT/XGBoost |
| 过拟合风险 | 低 | 较高（需调参） |

## 六、面试高频问题

### Q1：过拟合 vs 欠拟合，如何解决？

| | 欠拟合 | 过拟合 |
|--|--------|--------|
| **表现** | 训练误差和测试误差都高 | 训练误差低，测试误差高 |
| **原因** | 模型太简单，没学到数据规律 | 模型太复杂，学到了噪声 |
| **解决** | 增加模型复杂度、增加特征、减小正则化 | 增加数据、正则化（L1/L2）、Dropout、交叉验证、减少特征 |

> **直觉类比**：欠拟合 = 考试前没复习好，什么都不会；过拟合 = 只背了答案不理解原理，换题就不会。

### Q2：偏差（Bias） vs 方差（Variance）

- **偏差**：模型预测值与真实值的偏离程度。高偏差 → 欠拟合。
- **方差**：模型对训练数据变化的敏感程度。高方差 → 过拟合。
- **偏差-方差权衡**：模型越复杂，偏差越低但方差越高；反之亦然。目标是找到两者的最佳平衡点。

### Q3：梯度下降学习率过大/过小会怎样？

- **过大**：步长太大，可能跳过最优解甚至发散（损失越来越大，NaN）
- **过小**：步长太小，收敛极慢，可能卡在局部极小
- **实践建议**：从 0.01 或 0.001 开始尝试，配合学习率衰减策略；或使用自适应优化器（Adam）

### Q4：逻辑回归 vs 线性回归

| | 线性回归 | 逻辑回归 |
|--|---------|---------|
| **任务** | 回归（预测连续值） | 分类（预测类别概率） |
| **输出** | 任意实数 | [0, 1] 概率（经 Sigmoid） |
| **损失函数** | MSE | 交叉熵（Log Loss） |
| **关系** | — | 逻辑回归 = 线性回归 + Sigmoid |

### Q5：精确率 vs 召回率，如何选择？

- **精确率优先**：关注"预测为正的里面有几个是对的"。如垃圾邮件检测——宁可漏掉垃圾邮件，也不能误判正常邮件。
- **召回率优先**：关注"所有正例抓回来了几个"。如癌症筛查——宁可误报，也不能漏掉真正的患者。
- **F1 分数**：精确率和召回率的调和平均，适合两者都重要的场景。

### Q6：L1 vs L2 正则化

- **L1（Lasso）**：产生稀疏解（部分权重为 0），自带特征选择功能
- **L2（Ridge）**：使权重变小但不为 0，模型更平滑
- **选择**：特征多且怀疑有冗余 → L1；特征间有共线性 → L2；两者都要 → ElasticNet

### Q7：为什么逻辑回归不用 MSE 作为损失函数？

Sigmoid + MSE 是**非凸函数**，有多个局部最优，梯度下降可能卡在局部极小。交叉熵是凸函数，保证全局最优，且梯度形式简洁（梯度 = 预测值 - 真实值）。

### Q8：随机森林 vs GBDT？

- **随机森林**：Bagging，并行训练，降低方差，不易过拟合，对超参数不敏感
- **GBDT**：Boosting，串行训练，降低偏差，表达能力强，但需小心调参防过拟合
- **选择**：数据量大、需要快速基线 → 随机森林；追求精度、愿意调参 → XGBoost/LightGBM

### Q9：SVM 的核函数怎么选？

- 特征数多、样本少 → 线性核（避免过拟合）
- 特征数少、样本多 → RBF 核（非线性映射）
- 不确定 → 先试 RBF，再调 γ 和 C

## 七、背诵版总结

机器学习让计算机从数据中自动学习规律，核心三问：学什么（模型选择）、怎么学（损失函数+优化算法）、学得好不好（模型评估）。建模流程：数据收集→清洗→特征工程（选择/转换/降维）→训练→评估。关键概念：偏差-方差权衡、过拟合用正则化（L1稀疏/L2平滑）解决。损失函数：回归用MSE，分类用交叉熵。评估：回归看R²，分类看精确率/召回率/F1。经典模型：线性回归（连续值预测）、逻辑回归（Sigmoid映射到概率做分类）、感知机（最简神经网络，多层可解非线性）、决策树（if-else规则，可解释性强）、KNN（最近邻投票）、SVM（最大间隔+核技巧）。集成学习：Bagging降方差（随机森林），Boosting降偏差（GBDT/XGBoost）。梯度下降是核心优化算法，Mini-batch GD最常用，学习率需仔细调节。
