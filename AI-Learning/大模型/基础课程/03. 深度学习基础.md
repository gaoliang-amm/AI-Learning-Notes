## 一、PyTorch 基础

### 1.1 PyTorch 简介

PyTorch 是 Meta 团队开发的开源深度学习框架，核心优势：

- **类 NumPy 张量计算** + GPU/MPS 硬件加速
- **内置自动微分**（autograd），原生支持反向传播
- **动态计算图**：运行时逐步构建，调试灵活（对比 TensorFlow 静态图）

> 直觉：PyTorch = NumPy + GPU加速 + 自动求导，是连接深度学习理论与工程实践的桥梁。

### 1.3 张量（Tensor）创建

张量是 PyTorch 的核心数据结构，本质是 NumPy 数组的升级版（支持 GPU 加速 + 自动微分）。

#### 基于内容创建

```python
import torch

# 0维（标量）：存储损失值、准确率等单一数值
scalar = torch.tensor(10)          # shape: torch.Size([]), dim: 0

# 1维（向量）：存储特征向量、序列数据
vec = torch.tensor([1, 2, 3, 4])   # shape: torch.Size([4]), dim: 1

# 2维（矩阵）：存储批量数据、特征矩阵（最常用）
mat = torch.tensor([[1, 2, 3], [4, 5, 6]])  # shape: torch.Size([2, 3])

# 3维：单张彩色图像 [C, H, W]、时间序列
# 4维：批量彩色图像 [B, C, H, W]
```

**核心规律**：嵌套层数 = 张量维度数。N 层嵌套列表 → N 维张量。

#### 基于形状创建

```python
torch.Tensor(2, 3, 5)       # 大写T，创建未初始化张量，默认float32
torch.zeros(2, 3)            # 全0
torch.ones(3)                # 全1
torch.full((2, 3), 10)       # 全填充指定值
torch.eye(3)                 # 单位矩阵
torch.empty(3)               # 未初始化（内存残留值，效率最高）
```

#### 根据数据区间创建

```python
torch.arange(0, 10, 2)          # 固定步长: [0, 2, 4, 6, 8]
torch.linspace(0, 10, 6)        # 固定数量: 步长自动算，含端点
torch.logspace(1, 3, 3)         # 对数间隔: [10, 100, 1000]
```

#### 随机创建

```python
torch.rand(2, 3)                        # [0,1) 均匀分布
torch.randn(2, 3)                       # 标准正态分布 N(0,1)
torch.normal(5, 2, size=(2, 3))         # 自定义正态 N(5, 4)
torch.randint(1, 100, size=(2, 3))      # 整数均匀分布
torch.randperm(10)                      # 随机打乱 [0,10)

torch.manual_seed(42)                   # 固定随机种子，保证可复现
```

#### tensor 与 Tensor 的区别

| | `torch.tensor()`（小写，函数） | `torch.Tensor()`（大写，类） |
|---|---|---|
| 创建方式 | 基于具体数据 | 基于形状 |
| dtype | 自动推断（int→int64，float→float32） | 固定 float32 |
| 推荐度 | **推荐** | 不推荐（易类型混乱） |

### 1.5 张量数值计算

#### 基本运算

```python
t1 = torch.tensor([[1, 2, 3], [4, 5, 6]])

# 非原地（返回新张量，原张量不变）
t1 + 10          # 运算符重载
t1.add(10)       # 方法调用

# 原地操作（直接修改原张量，训练时慎用——会破坏计算图）
t1.add_(10)      # 下划线后缀 = 原地操作
```

#### 元素级运算（Element-wise）

```python
t1 + t2     # 逐元素加
t1 * t2     # 逐元素乘（哈达玛积，非矩阵乘法！）
t1 / t2     # 逐元素除
```

#### 矩阵相乘

```python
t1 = torch.tensor([[1, 2, 3], [4, 5, 6]])       # (2, 3)
t2 = torch.tensor([[10, 20, 30, 40], [40, 50, 60, 70], [40, 50, 60, 70]])  # (3, 4)

t1 @ t2           # 运算符 @ → (2, 4)
t1.mm(t2)         # mm() 仅支持2维
t1.matmul(t2)     # matmul() 通用（支持高维批量矩阵乘法）
```

> 高维批量矩阵乘法：仅最后两个维度满足矩阵乘法条件，前面的维度视为批量维度。

#### 统计运算

```python
t1.sum(dim=0)          # 按维度求和（降维）
t1.float().mean(dim=0) # 按维度求均值（需float类型）
t1.float().max(dim=1)  # 返回 (values, indices)
t1.float().var(dim=1)  # 方差
t1.float().std(dim=1)  # 标准差
t1.float().unique()    # 去重
t1.float().sort(dim=2) # 排序，返回 (sorted_values, indices)
```

### 1.7 张量形状操作

#### 维度交换

```python
t1 = torch.randint(1, 10, (3, 2, 6))

t1.T              # 转置所有维度: (3,2,6) → (6,2,3)
t1.mT             # 仅转置最后两个维度: (3,2,6) → (3,6,2)
t1.transpose(0, 1)  # 交换指定两个维度: (3,2,6) → (2,3,6)
t1.permute(1, 0, 2) # 自定义所有维度顺序（最灵活）
```

#### 调整形状

```python
t1.reshape(36)          # 展平，-1可自动推导
t1.reshape(3, 1, -1)    # -1表示自动计算该维度
t1.view(3, -1)          # 同reshape，但要求内存连续
```

#### 增删维度

```python
t1.unsqueeze(dim=2)     # 在指定位置增加大小为1的维度
t1.unsqueeze(dim=0).squeeze()  # squeeze删除所有大小为1的维度
```

#### 拼接

```python
# cat: 按指定维度拼接（维度不变，拼接维度大小相加）
torch.cat([t1, t2], dim=1)

# stack: 堆叠（新增一个维度，要求所有张量形状完全相同）
torch.stack([t1, t2], dim=0)  # 等价于 cat([t1.unsqueeze(0), t2.unsqueeze(0)], dim=0)
```

## 二、神经网络基础

### 2.1 全连接层（nn.Linear）

全连接层 = 线性变换，数学公式：**y = xW^T + b**

```python
import torch.nn as nn

# 2个输入神经元，3个输出神经元
linear = nn.Linear(in_features=2, out_features=3, bias=True)

# 可训练参数
linear.weight.shape   # (3, 2) — 注意是 W^T 的形状
linear.bias.shape     # (3,)

# 前向传播（两种等价写法）
x = torch.tensor([[1.0, 2.0], [3.0, 4.0]])
y = linear(x)          # 推荐：__call__ 自动调用 forward
y = linear.forward(x)  # 显式调用
```

### 2.2 激活函数

激活函数 = 给线性变换引入**非线性**，让网络能拟合复杂映射。

| 激活函数 | 公式/特性 | 输出范围 | 适用场景 | 缺点 |
|---------|----------|---------|---------|------|
| **Sigmoid** | σ(x)=1/(1+e^(-x)) | (0, 1) | 二分类输出层 | 梯度消失 |
| **Tanh** | (e^x-e^(-x))/(e^x+e^(-x)) | (-1, 1) | 浅层隐藏层 | 梯度消失 |
| **ReLU** | max(0, x) | [0, +∞) | **隐藏层首选** | 神经元死亡 |
| **LeakyReLU** | max(αx, x), α=0.01 | (-∞, +∞) | ReLU效果不佳时替代 | 增加少量计算 |
| **GELU** | x·Φ(x) | 平滑近似ReLU | **Transformer默认** | 计算稍慢 |
| **Softmax** | e^xi / Σe^xj | (0, 1)，和为1 | **多分类输出层** | 需指定dim |

```python
x = torch.tensor([0.7, -1.2, 2.5, 0, -3.1])

# 三种调用方式（等价）
torch.sigmoid(x)       # 全局函数
x.sigmoid()            # 张量方法
nn.Sigmoid()(x)        # nn层类（适合放入Sequential）

# ReLU
torch.relu(x)          # [0.7, 0, 2.5, 0, 0]

# Softmax（必须指定dim）
torch.softmax(x, dim=0)

# GELU（Transformer标配）
nn.GELU()(x)
```

**选择原则**：
- **隐藏层**：首选 ReLU，效果不佳时试 LeakyReLU / GELU
- **输出层**：二分类→Sigmoid，多分类→Softmax，回归→无激活

### 2.3 搭建神经网络

#### 方式一：自定义 nn.Module

```python
import torch
import torch.nn as nn

class MyModule(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear1 = nn.Linear(3, 4)
        self.linear2 = nn.Linear(4, 4)
        self.out = nn.Linear(4, 2)

    def forward(self, x):
        x = torch.tanh(self.linear1(x))
        x = torch.relu(self.linear2(x))
        y = torch.softmax(self.out(x), dim=-1)
        return y

model = MyModule()
y = model(torch.randn(10, 3))  # model(x) 自动调用 forward(x)
```

> 直觉：`__init__` 定义"积木"（各层），`forward` 定义"拼法"（数据流向）。

#### 方式二：nn.Sequential（顺序结构专用）

```python
model = nn.Sequential(
    nn.Linear(3, 4),
    nn.Tanh(),
    nn.Linear(4, 4),
    nn.ReLU(),
    nn.Linear(4, 2),
    nn.Softmax(dim=-1)
)
```

> 优点：代码简洁。缺点：仅适用于顺序结构，复杂网络（ResNet等）仍需自定义Module。

#### 查看模型参数

```python
# 遍历所有参数（带名称）
for name, param in model.named_parameters():
    print(f"{name}: {param.shape}")

# 状态字典（模型保存/加载的基础）
state_dict = model.state_dict()
```

### 2.4 模型保存与加载

```python
# 保存（只保存参数，不保存模型结构）
torch.save(model.state_dict(), 'model.pth')

# 加载（必须先重建相同结构的模型）
loaded_model = nn.Sequential(
    nn.Linear(3, 4), nn.Tanh(),
    nn.Linear(4, 4), nn.ReLU(),
    nn.Linear(4, 2), nn.Softmax(dim=-1)
)
loaded_model.load_state_dict(torch.load('model.pth'))
```

## 三、神经网络训练

### 3.1 损失函数

| 损失函数 | PyTorch API | 适用任务 | 说明 |
|---------|-------------|---------|------|
| MAE (L1) | `nn.L1Loss()` | 回归 | 平均绝对误差 |
| MSE (L2) | `nn.MSELoss()` | 回归 | 均方误差 |
| Smooth L1 | `nn.SmoothL1Loss()` | 回归 | L1+L2结合，对异常值鲁棒 |
| 交叉熵 | `nn.CrossEntropyLoss()` | **多分类** | **内置Softmax**，输入为logits |
| 二元交叉熵 | `nn.BCELoss()` | 二分类 | 输入需先过Sigmoid |
| BCE+Logits | `nn.BCEWithLogitsLoss()` | 二分类 | 内置Sigmoid，数值更稳定 |

```python
# 多分类（最常用）：注意target是类别索引，不是one-hot
loss_fn = nn.CrossEntropyLoss()
output = torch.randn(8, 6)          # 8个样本，6个类别
target = torch.tensor([1, 0, 5, 4, 2, 3, 0, 5])  # 类别标签
loss = loss_fn(output, target)

# 二分类
loss_fn = nn.BCEWithLogitsLoss()     # 推荐：内置Sigmoid
output = torch.randn(5)
target = torch.tensor([0, 0, 1, 1, 0]).float()
loss = loss_fn(output, target)
```

> **关键**：`CrossEntropyLoss` 已内置 Softmax，训练时输出层**不要**加 Softmax！

---

### 3.2 反向传播与自动微分

#### 计算图与链式法则

反向传播 = 利用链式法则，从输出层到输入层**反向**计算梯度。

> 直觉：前向传播算"结果"，反向传播算"每个参数对结果的影响有多大"。

**为什么不正向计算梯度？** 正向需要对每个参数单独做一次完整前向传播，计算量 = 参数数 × 单次计算量，完全不可行。反向传播通过**结果复用**，一次反向遍历即可算出所有参数的梯度。

#### PyTorch 自动微分

```python
x = torch.tensor([10.0])
y = torch.tensor([3.0])
w = torch.randn(1, requires_grad=True)  # 关键：requires_grad=True
b = torch.randn(1, requires_grad=True)

z = w * x + b
loss_fn = nn.MSELoss()
loss = loss_fn(z, y)

loss.backward()      # 反向传播，自动计算梯度
print(w.grad)        # 查看w的梯度
print(b.grad)        # 查看b的梯度

optimizer.step()     # 更新参数
optimizer.zero_grad()# 梯度清零（必须！否则梯度会累加）
```

**重要规则**：
- `requires_grad=True`：PyTorch 追踪该张量的所有操作
- 叶子节点的梯度在 `backward()` 后**保留**（累加），需手动 `zero_grad()`
- 非叶子节点的梯度在 `backward()` 后**释放**（除非 `retain_grad=True`）

### 3.3 优化器

#### 训练四步循环

```python
for epoch in range(epochs):
    for x_batch, y_batch in dataloader:
        y_pred = model(x_batch)       # 1. 前向传播
        loss = loss_fn(y_pred, y_batch)  # 2. 计算损失
        loss.backward()               # 3. 反向传播（计算梯度）
        optimizer.step()              # 4. 更新参数
        optimizer.zero_grad()         # 梯度清零
```

#### 核心概念

| 概念 | 说明 |
|------|------|
| **Batch Size** | 每次训练的样本数。小batch噪声多→有助泛化；大batch梯度稳定 |
| **Iteration** | 一次完整前向+反向传播（一个batch） |
| **Epoch** | 模型完整遍历一次整个训练集 |
| **学习率 (lr)** | 参数更新步长。过大→震荡不收敛；过小→收敛极慢 |

> 例：2000样本，batch_size=64，10个epoch → 每epoch 32次iteration，共320次。

#### 优化器对比

| 优化器 | PyTorch API | 核心思想 | 直觉类比 |
|--------|-------------|---------|---------|
| **SGD** | `optim.SGD(lr=)` | 沿负梯度方向更新 | 下山只看脚下坡度 |
| **Momentum** | `optim.SGD(momentum=0.9)` | 累积历史梯度方向 | 球滚下山，有惯性 |
| **AdaGrad** | `optim.Adagrad()` | 自适应学习率（梯度大的参数学习率小） | 走得多的路步子变小 |
| **RMSProp** | `optim.RMSprop()` | 指数加权移动平均改进AdaGrad | 修复AdaGrad学习率单调递减 |
| **Adam** | `optim.Adam(betas=(0.9,0.999))` | Momentum + RMSProp | **最常用**，兼顾动量和自适应 |

```python
# SGD + 动量
optimizer = optim.SGD(model.parameters(), lr=0.01, momentum=0.9)

# Adam（默认推荐）
optimizer = optim.Adam(model.parameters(), lr=0.001, betas=(0.9, 0.999))
```

#### 学习率衰减

```python
from torch.optim.lr_scheduler import StepLR, MultiStepLR, ExponentialLR

# 等间隔衰减：每10个epoch，lr *= 0.7
scheduler = StepLR(optimizer, step_size=10, gamma=0.7)

# 指定间隔衰减：在epoch=10,20,70时衰减
scheduler = MultiStepLR(optimizer, milestones=[10, 20, 70], gamma=0.7)

# 指数衰减：lr *= 0.99^epoch
scheduler = ExponentialLR(optimizer, gamma=0.99)

# 在训练循环中调用
for epoch in range(epochs):
    train(...)
    scheduler.step()  # 每个epoch结束后更新学习率
```

### 3.5 正则化

#### L2 正则化（权值衰减）

对损失函数添加权重的L2范数惩罚项，抑制过拟合。PyTorch优化器默认支持：

```python
optimizer = optim.SGD(model.parameters(), lr=0.1, weight_decay=1e-2)
```

#### Dropout（随机失活）

训练时以概率 p 随机关闭神经元，测试时全部激活。本质是**隐式集成**（每次训练不同子网络）。

```python
dropout = nn.Dropout(p=0.5)  # 50%的神经元被关闭
x = dropout(x)               # 训练时生效，eval()时自动关闭
```

> 放置位置：激活函数之后，下一线性层之前。

#### Batch Normalization（批量标准化）

对每个mini-batch的激活值做标准化，再通过可学习参数 γ、β 进行缩放平移。

```python
bn = nn.BatchNorm1d(num_features=128)  # 全连接层用 BatchNorm1d
# nn.BatchNorm2d(num_features=64)      # CNN用 BatchNorm2d
```

**BN层的三大好处**：
1. 允许更高的学习率 → 训练更快
2. 不那么依赖参数初始值
3. 有一定正则化效果，可减少 Dropout 需求

> 直觉：BN = 每层的"交通信号灯"，让数据分布保持在合理范围，避免"堵车"（梯度消失/爆炸）。

**放置位置**：线性层/卷积层之后，激活函数**之前**。

---

### 3.6 参数初始化

权重不能全初始化为相同值（否则反向传播时所有权重同步更新，失去多神经元的意义），必须随机初始化。

| 初始化方法 | API | 适用激活函数 |
|-----------|-----|------------|
| 正态分布 | `nn.init.normal_(w, mean=0, std=1)` | 通用 |
| 均匀分布 | `nn.init.uniform_(w, a=0, b=1)` | 通用 |
| **Xavier** | `nn.init.xavier_normal_(w)` / `xavier_uniform_` | **Sigmoid, Tanh** |
| **He (Kaiming)** | `nn.init.kaiming_normal_(w)` / `kaiming_uniform_` | **ReLU及变体** |

```python
def init_weights(layer):
    if type(layer) == nn.Linear:
        nn.init.xavier_normal_(layer.weight)  # 或 kaiming_normal_ 配ReLU
model.apply(init_weights)
```

## 四、CNN（卷积神经网络）

### 4.1 卷积层

卷积操作：用小尺寸的**卷积核（filter）** 在输入上滑动，做局部加权求和，提取局部特征。

```python
torch.nn.Conv2d(
    in_channels,    # 输入通道数（如RGB=3）
    out_channels,   # 输出通道数 = 卷积核个数
    kernel_size,    # 卷积核大小
    stride=1,       # 步幅
    padding=0       # 填充
)
```

```python
# 对一张图片做卷积
input = torch.tensor(img).permute(2, 0, 1).float()  # HWC → CHW
conv = nn.Conv2d(in_channels=3, out_channels=3, kernel_size=3, padding=1)
out = conv(input)  # 输出形状: (out_channels, H', W')
```

> 直觉：卷积核就像"特征探测器"——有的探测边缘，有的探测纹理。多个卷积核 = 多种特征探测器并行工作。

### 4.2 池化层

池化 = 对局部区域做**下采样**，减小特征图尺寸，降低计算量，增强平移不变性。

```python
nn.MaxPool2d(kernel_size=2, stride=2)   # 最大池化：取局部最大值
nn.AvgPool2d(kernel_size=2, stride=2)   # 平均池化：取局部平均值
```

> 一般 kernel_size = stride，padding = 0。

### 4.3 深度CNN架构

| 架构 | 年份 | 核心创新 | 深度 | 直觉 |
|------|------|---------|------|------|
| **AlexNet** | 2012 | 首个深度CNN，ReLU + Dropout，ImageNet冠军 | 8层 | 深度学习复兴的标志 |
| **VGG** | 2014 | 统一使用3×3小卷积核堆叠 | 16/19层 | "小卷积核堆叠 > 大卷积核" |
| **GoogLeNet** | 2014 | Inception结构（多尺度并行卷积后合并） | 22层 | 横向也有深度，减少参数 |
| **ResNet** | 2015 | **残差连接**（跳跃连接），解决梯度消失 | 152层+ | 学习目标从H(x)变为F(x)=H(x)-x |

```python
# ResNet 的核心思想：残差块
class ResidualBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.conv1 = nn.Conv2d(channels, channels, 3, padding=1)
        self.bn1 = nn.BatchNorm2d(channels)
        self.conv2 = nn.Conv2d(channels, channels, 3, padding=1)
        self.bn2 = nn.BatchNorm2d(channels)

    def forward(self, x):
        residual = x                          # 保存输入
        out = torch.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out += residual                       # 残差连接：输出 = F(x) + x
        return torch.relu(out)
```

> 直觉：残差连接 = 给梯度修了一条"高速公路"，即使深层也能顺畅回传。

## 六、面试高频问题

### Q1: 梯度消失与梯度爆炸的原因和解决方案？

**梯度消失**：深层网络反向传播时，梯度经过多次乘积后趋近于0（Sigmoid/Tanh的导数<1）。**梯度爆炸**：梯度经过多次乘积后趋近于无穷大。

| 问题 | 解决方案 |
|------|---------|
| 梯度消失 | 使用ReLU激活函数；BatchNorm；残差连接（ResNet）；合理初始化（He/Xavier） |
| 梯度爆炸 | 梯度裁剪（`torch.nn.utils.clip_grad_norm_`）；BatchNorm；合理初始化；降低学习率 |

### Q2: ReLU相比Sigmoid的优势？

1. **缓解梯度消失**：ReLU正区间导数恒为1，不会衰减
2. **计算高效**：只需判断正负，无需指数运算
3. **稀疏激活**：负数输出为0，增加稀疏性
4. **缺点**：负数区梯度为0（神经元死亡），可用LeakyReLU缓解

### Q3: BatchNorm的原理和作用？

**原理**：对每个mini-batch做标准化（减均值除标准差），再通过可学习参数γ、β缩放平移。

**作用**：
1. 加速收敛（允许更大学习率）
2. 减少对初始值的敏感度
3. 有正则化效果
4. 缓解内部协变量偏移（Internal Covariate Shift）

**训练 vs 推理**：训练时用batch统计量，推理时用全局滑动平均统计量。

### Q4: CNN相比全连接网络的优势？

1. **参数共享**：同一个卷积核在全图滑动，大幅减少参数量
2. **局部连接**：每个输出只与输入的局部区域连接
3. **平移不变性**：特征检测器在任何位置都有效
4. **层次化特征提取**：浅层提取边缘纹理，深层提取高级语义

> 例：224×224×3图像，全连接到1000个神经元需要约1.5亿参数；CNN只需几百万。

### Q5: 过拟合的解决方案？

| 方案 | 说明 |
|------|------|
| **增加数据量** | 数据增强（旋转、翻转、裁剪等） |
| **L2正则化** | `weight_decay`，惩罚大权重 |
| **Dropout** | 随机关闭神经元，隐式集成 |
| **BatchNorm** | 有正则化副作用 |
| **早停（Early Stopping）** | 验证集loss不再下降时停止训练 |
| **减小模型复杂度** | 减少层数或神经元数 |
| **交叉验证** | 更可靠地评估泛化能力 |

### Q6: Adam优化器为什么常用？

Adam = Momentum（动量）+ RMSProp（自适应学习率）。一阶矩估计跟踪梯度方向，二阶矩估计跟踪梯度幅度。兼顾了收敛速度和稳定性，对超参数不敏感，通常是默认首选。

### Q7: 训练时 model.train() 和 model.eval() 的区别？

- `model.train()`：启用 Dropout 和 BatchNorm 的训练行为（Dropout随机丢弃，BN用batch统计量）
- `model.eval()`：关闭 Dropout，BN使用全局统计量

## 七、背诵版总结

PyTorch以张量（Tensor）为核心数据结构，支持GPU加速和自动微分。神经网络由全连接层（y=xW+b）和激活函数（隐藏层用ReLU，输出层按任务选Sigmoid/Softmax）堆叠而成，通过nn.Module或Sequential搭建。训练四步循环：前向传播→计算损失→反向传播（loss.backward）→优化器更新参数。常用优化器为Adam（动量+自适应学习率）。正则化手段包括L2权值衰减、Dropout随机失活、BatchNorm批量标准化。CNN通过卷积核提取局部特征、池化下采样、参数共享大幅减少参数量，深层架构（ResNet）用残差连接解决梯度消失。关键初始化：Sigmoid/Tanh用Xavier，ReLU用He初始化。
