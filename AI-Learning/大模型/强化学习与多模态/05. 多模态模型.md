# 多模态模型

## 一、Vision Transformer (ViT)

### 1.1 核心思想：将图像拆分为 patch

ViT 将 Transformer 用于图像分类任务，核心思路是**将图像视为 token 序列**而非网格结构。2021 年 Dosovitsky 等人在论文《An Image is Worth 16x16 Words》中提出，在同等资源下性能可比肩 CNN。

> **为什么用 patch 而不是直接把像素当 token？**
> 一张 224x224 的图像有 50176 个像素——如果每个像素作为一个 token，自注意力的 $O(n^2)$ 复杂度将使计算不可承受。切成 16x16 的 patch 后只有 196 个 token，计算量降低数百倍，同时每个 patch 仍包含足够的空间语义信息（边缘、纹理、局部形状）。
>
> **为什么不用 CNN 来做？** CNN 通过滑动窗口提取局部特征，天然具有平移不变性和局部性偏置，但难以建模长距离依赖。ViT 的自注意力可以一步建立任意两个 patch 之间的关系，天然具备全局感受野。在大规模数据集上，ViT 的表现可以超越 CNN。

### 1.2 Patch 嵌入 + 位置编码

**Patch Embedding**：用 Conv2d（kernel\_size=stride=patch\_size）将图像切分为无重叠 patch，同时做线性投影到 $d_{model}$ 维。

```
PatchEmbedding 类核心逻辑：
linear_project = nn.Conv2d(
    n_channels,   # 输入通道（如 3 或 1）
    d_model,      # 输出维度（嵌入宽度）
    kernel_size=patch_size,
    stride=patch_size   # kernel=stride → 无重叠切分
)
```

张量形状变化：

$$
(B, C, H, W) \xrightarrow{\text{Conv2d}} (B, d_{model}, P_{col}, P_{row}) \xrightarrow{\text{flatten}} (B, d_{model}, P) \xrightarrow{\text{transpose}} (B, P, d_{model})
$$

其中 $P = P_{col} \times P_{row}$ 是 patch 总数。例如 32x32 图像、patch\_size=16 时，$P = 2 \times 2 = 4$ 个 patch。

**CLS Token**：在序列前添加一个可学习的 `[CLS]` 向量（`nn.Parameter(torch.randn(1, 1, d_model))`）。

> **cls\_token 的作用**：Transformer 的自注意力机制使每个 token 都能聚合来自序列中所有其他 token 的信息。经过多层编码后，cls\_token 的输出汇集了整个图像的全局语义摘要，因此可以直接用于分类，而不需要对所有 patch 的输出做池化。这也是 BERT 在 NLP 中使用 `[CLS]` token 做分类的同一思路。

**位置编码**：Transformer 并行处理所有 token，不感知顺序。如果不加位置信息，打乱 patch 顺序不会改变输出，但图像的语义显然依赖空间排列（把 "0" 的上下颠倒可能变成 "8"）。位置编码为每个位置注入唯一的位置标识：

$$
PE(pos, 2i) = \sin(pos / 10000^{2i/d_{model}}), \quad PE(pos, 2i+1) = \cos(pos / 10000^{2i/d_{model}})
$$

位置编码通过 `register_buffer` 注册（不参与反向传播），在 forward 中直接加到 patch embedding 上。

### 1.3 Transformer Encoder + 分类头

**Scaled Dot-Product Attention**（核心公式）：

$$
Attention(Q, K, V) = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

- Q、K、V 分别是"要查找什么"、"包含什么"、"携带什么信息"，通过线性层从输入计算得到
- $QK^T$ 计算 token 之间的相关性，除以 $\sqrt{d_k}$ 缩放以控制方差
- Softmax 将相关性归一化为注意力权重，再与 V 做加权求和完成信息聚合

**多头注意力**：并行 h 个头（每个头维度 $d_k = d_{model} / h$），各自独立计算注意力后拼接，再经线性层 $W_o$ 合并。多头机制让模型能同时关注不同子空间的信息（如一个头关注纹理、另一个头关注形状）。

**Transformer Encoder 层结构**：

```
x → LayerNorm → MultiHeadAttention → (+Residual) → LayerNorm → MLP(GELU) → (+Residual) → output
```

- LayerNorm 在每个子层开头做归一化，稳定训练
- MLP（两层 Linear + GELU）让每个 token 独立"思考"注意力收集到的信息
- 残差连接（直接相加）帮助梯度直通，防止深层网络的梯度消失

**分类头**：取 cls\_token 的输出（`x[:, 0]`），经 `Linear(d_model, n_classes) + Softmax` 输出分类概率。

MNIST 上 5 epoch 即达约 92% 准确率，证明自注意力可替代 CNN。

## 二、CLIP：图文对比学习

### 2.1 核心思想：图文嵌入空间对齐

CLIP 突破固定类别限制，通过对比学习让图像和文本嵌入在统一空间中对齐——**匹配的图文对相似度最大化，不匹配的最小化**。属于 zero-shot 范式，无需微调即可处理未见类别。

### 2.2 架构：双编码器 + 对比损失

- **图像编码器**：ViT 结构（patch embedding + cls\_token + Transformer Encoder），取 cls\_token 输出作为图像特征
- **文本编码器**：Transformer Encoder（非解码器，不使用因果掩码），取 EOT token 输出作为文本特征
- **投影层**：两者各自经可学习矩阵 $W_{width \times emb\_dim}$ 映射到统一维度，再 L2 归一化

> **为什么点积代表相似度？** 两个 L2 归一化向量（模长为 1）的点积等于它们的余弦相似度：$a \cdot b = \|a\| \|b\| \cos\theta = \cos\theta$。值域 [-1, 1]，越接近 1 表示方向越一致（语义越相似）。这正是 CLIP 对齐图文空间的数学基础。

**正负样本对构建**：在一个 batch 中，有 N 张图像和 N 条文本（第 i 张图像对应第 i 条文本）。构造 $N \times N$ 的相似度矩阵：

$$
logits = (I_e @ T_e^T) \cdot \exp(\tau)
$$

- 对角线元素（第 i 张图 vs 第 i 条文本）是**正样本对**（应该匹配）
- 非对角线元素是**负样本对**（不应该匹配）
- 对角线的值应该最大

**可学习温度参数 $\tau$**：

$$
\tau = \exp(\log(1/0.07))
$$

> **温度的作用**：$\tau$ 缩放相似度矩阵。$\tau$ 越大，softmax 输出越尖锐，模型对正负样本的区分越"果断"；$\tau$ 越小，输出越平滑，区分越"温和"。用可学习参数让模型自动调整区分的严格程度：训练初期 $\tau$ 自然演化以适应当前梯度，避免手动调参。初始值 $1/0.07 \approx 14.3$ 是经验起始点。

**对称对比损失**（图像到文本 + 文本到图像）：

$$
loss = \frac{CE(logits^T, \text{labels}) + CE(logits, \text{labels})}{2}
$$

- `CE(logits, labels)`：以图像为锚，让每张图找对应的文本（行方向）
- `CE(logits^T, labels)`：以文本为锚，让每条文本找对应的图像（列方向）
- 对称设计确保两个模态都主动向对方对齐

### 2.3 Zero-shot 能力

训练后，将图像特征与**任意文本类别名称**的特征计算余弦相似度，Top-K 即为预测。本质是开放世界分类——类别不在训练标签中也能识别。

> **应用：文搜图**。1) 用 CLIP 图像编码器提取数据库所有图片特征，存入向量数据库；2) 用文本编码器提取搜索文本特征；3) 余弦相似度检索最近邻图片。

## 三、ClipCap：图生文

### 3.1 架构：CLIP + 映射网络 + GPT-2

```
输入图像 → CLIP图像编码器(512维) → MLP投影 → 10个token×768维 → 拼接到GPT-2前缀 → 生成文本描述
```

> **为什么 CLIP 和 GPT-2 的嵌入空间不同？**
> - CLIP 通过对比学习训练，目标是让**图像和文本在同一空间中相似**，其嵌入空间编码的是"模态无关的语义"
> - GPT-2 通过自回归语言建模训练，其嵌入空间编码的是"下一个 token 的概率分布"
> - 两者训练目标、数据来源、架构完全不同，维度也不同（512 vs 768），所以 CLIP 的图像嵌入无法直接喂给 GPT-2
>
> **Mapping Network 如何桥接？** 映射网络是一个 MLP（`Linear(512, 3840) → Tanh → Linear(3840, 7680)`），将 512 维的 CLIP 嵌入投影为 $768 \times 10 = 7680$ 维的张量，再 reshape 成 $(10, 768)$——相当于把一张图片"翻译"成 10 个 GPT-2 能理解的 token。

### 3.2 Prefix 概念

映射后的图像嵌入称为 **prefix embeddings**——作为 GPT-2 生成的上下文前缀，位于真实文本 token 之前。

> **Prefix 如何作为 GPT-2 的"视觉上下文"？** GPT-2 的输入是 token embedding 序列。在 ClipCap 中，序列前 10 个位置由图像 prefix 占据，后面的才是文本 token。GPT-2 的自注意力机制会自动让文本 token 关注到这些 prefix——就像用一段"视觉描述"引导生成。推理时，prefix 固定不动，GPT-2 逐个预测后续 token。

### 3.3 两种训练变体

| 变体 | GPT-2 | 映射网络 | 训练目标 |
|------|-------|----------|----------|
| 微调版 | 微调 | MLP | GPT-2 + MLP 都训 |
| 冻结版 | 冻结 | Transformer (Bert架构) | 只训映射网络 |

CLIP 生成的图像嵌入开箱即用已经足够好，所以**不训练 CLIP 模型**。教程采用第一种（GPT-2 微调 + MLP 映射）。

### 3.4 推理流程

```
t=1: [Prefix Embeddings] → GPT-2 → 预测 "A"
t=2: [Prefix + "A"]      → GPT-2 → 预测 "blue"
t=3: [Prefix + "A blue"]  → GPT-2 → 预测 "car"
t=4: [Prefix + "A blue car"] → GPT-2 → 预测 <EOS>（停止）
```

推理时使用 temperature 采样（如 0.7），将 logits 除以 temperature 后做多项式分布采样。遇到 `<EOS>` 或达到最大长度时停止。

### 3.5 训练目标

损失只在文本 token 上计算，prefix 部分不参与 loss：

$$
shift\_logits = logits[:, \text{IMAGE\_TOKEN\_LENGTH}-1 : -1, :]
$$

$$
loss = CE(shift\_logits, caption\_ids)
$$

这是因为 prefix 是"输入条件"而非"需要预测的目标"——模型的任务是根据 prefix 生成文本，所以只对文本部分的预测计算损失。

## 四、Qwen2.5-VL

### 4.1 架构总览

Qwen2.5-VL 的整体架构由三部分组成：

```
输入图像/视频 → 视觉编码器(ViT) → 视觉-语言合并器(MLP) → LLM解码器(Qwen2.5) → 文本输出
```

### 4.2 视觉编码器：重构 ViT

采用重构的 ViT 架构，引入了多项现代化改进：

- **SwiGLU 激活**：替代传统 GELU，在前馈网络中引入门控机制，提升表达能力
- **RMSNorm**：替代 LayerNorm，计算更高效（省去均值计算）
- **窗口注意力**：将全局注意力限制在局部窗口内，降低计算复杂度
- **二维 RoPE**：位置编码扩展到 2D，支持原生分辨率输入
- 输入图像在送入 ViT 前，高度和宽度被调整为 28 的倍数；以步幅 14 切分为 patch

### 4.3 MRoPE：多模态旋转位置编码

> **MRoPE 如何对齐时间维度？**
>
> 标准 RoPE 将位置信息编码为旋转角度，使模型感知 token 的绝对位置。但传统 1D RoPE 无法区分"空间位置"和"时间位置"——对于视频，帧与帧之间的时间关系和帧内像素的空间关系是不同维度。
>
> MRoPE 将位置编码扩展为三维：**时间维度 + 高度维度 + 宽度维度**。
> - 对于图像：时间维度固定为 1（单帧），只在空间维度上编码
> - 对于视频：时间 ID 与**绝对时间对齐**——例如第 0 秒的帧时间 ID 为 0，第 1 秒的帧时间 ID 为 1，无论帧率如何变化
>
> 这使得模型能够理解事件节奏（"人在跑步"vs"人在跳跃"的速度差异）和精确时刻定位（"在第几秒发生了什么"）。

### 4.4 视觉-语言合并器：压缩 token

> **为什么需要合并器？**
> 高分辨率图像经过 ViT 后可能产生数千个 patch token。如果直接送入 LLM，长序列的注意力计算成本极高。
>
> **如何压缩？**
> 1. 将空间上**相邻的 4 个 patch 嵌入**分为一组
> 2. 将 4 个嵌入 concatenate 成一个向量
> 3. 通过一个 **2 层 MLP** 投影到与 LLM 文本嵌入对齐的维度
>
> 效果：token 数量减少为原来的 1/4，同时 MLP 的投影保留了相邻 patch 之间的空间关系信息。这种方法灵活高效，可以根据输入图像的大小动态压缩不同长度的特征序列。

## 五、关键要点总结

| 模型 | 架构 | 核心机制 | 关键创新 |
|------|------|----------|----------|
| ViT | Conv2d切patch + Transformer Encoder | 自注意力替代卷积 | 纯注意力图像分类，全局感受野 |
| CLIP | ViT(image) + Transformer(text) | 对比学习对齐图文空间 | zero-shot 开放世界分类 |
| ClipCap | CLIP + MLP + GPT-2 | prefix embedding 驱动生成 | 图生文，跨嵌入空间映射 |
| Qwen2.5-VL | ViT + LLM + MLP合并器 | MRoPE多模态位置编码 | 统一理解图文视频，时间对齐 |
