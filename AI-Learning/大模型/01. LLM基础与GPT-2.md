# LLM 基础与 GPT-2 实践

## 一、LLM 要解决什么问题

大语言模型的核心任务是 **下一个 Token 预测（Next Token Prediction, NTP）**。

给定前面的文本，预测下一个最可能出现的 token。例如：

| 输入 | 预测目标 |
|------|----------|
| 君不见黄河 | 之 |
| 君不见黄河之 | 水 |
| 君不见黄河之水 | 天 |

训练数据通过程序自动处理原始文本生成，属于 **自监督学习** ——预测目标不是人类标注的，而是通过滑动窗口切分文本自动产生的。

为了利用 GPU 并行计算，训练数据组织为矩阵形式：

```
君 不 见 黄 河
不 见 黄 河 之
见 黄 河 之 天
...
```

## 二、如何训练 LLM

训练一个 LLM 需要回答以下关键问题：

1. **训练数据**：输入-目标对如何构造（见上文）
2. **模型结构**：采用 **Decoder-Only Transformer** 架构
3. **损失函数**：交叉熵损失（下一 token 分类）
4. **优化方法**：梯度下降及其变体（SGD、Adam、AdamW、Muon 等）
5. **反向传播**：`loss.backward()` 计算参数梯度

## 三、损失函数：LLM 本质是分类模型

从极大似然估计角度理解，给定训练数据 "abcd"，希望以下概率越大越好：

$$P_\theta(d|abc) \cdot P_\theta(c|ab) \cdot P_\theta(b|a)$$

最大化 log 似然等价于最小化：

$$- \log P_\theta(d|abc) - \log P_\theta(c|ab) - \log P_\theta(b|a)$$

这就是 **交叉熵损失**。LLM 本质上是一个分类模型 —— 给定前面的 token，预测下一个 token 属于词汇表中哪个类别。

## 四、GPT-2 模型架构

### 4.1 整体结构

```
文本 → 分词 → Token嵌入 → +位置嵌入 → Dropout
       → TransformerBlock × N → LayerNorm → Linear → logits → softmax → 下一个token
```

核心配置参数（GPT-2 124M）：

| 参数 | 值 | 说明 |
|------|-----|------|
| vocab_size | 50257 | 词汇表大小 |
| context_length | 1024 | 最大上下文长度 |
| emb_dim | 768 | 嵌入维度 |
| n_heads | 12 | 注意力头数 |
| n_layers | 12 | Transformer 层数 |
| drop_rate | 0.1 | Dropout 率 |

### 4.2 核心组件详解

#### 词嵌入（Token Embedding）

将 token ID 映射为高维向量（768 维）。底层等价于独热编码 + 线性变换，但 `nn.Embedding` 通过查表更高效。

```python
embedding = nn.Embedding(vocab_size, emb_dim)
# token ID [2, 3, 1] → 形状 [3, emb_dim] 的张量
```

#### 位置编码（Position Embedding）

GPT-2 使用 **可学习位置编码** —— 为每个位置（0, 1, ..., context_length-1）学习一个嵌入向量，与 token embedding 直接相加。

```python
pos_emb = nn.Embedding(context_length, emb_dim)
x = tok_embeddings + pos_embeddings
```

> **关键理解**：相同 token 在不同位置有相同的 token embedding 和不同的位置 embedding，语义由后续注意力机制区分。

#### TransformerBlock 结构

每个 TransformerBlock 包含：

```
输入 → LayerNorm → MultiHeadAttention → Dropout → +残差
     → LayerNorm → FeedForward       → Dropout → +残差 → 输出
```

注意 **Pre-LN** 结构（LayerNorm 在前），这是 GPT-2 与原始 Transformer 的重要区别。

#### 层归一化（LayerNorm）

在嵌入维度上做归一化，保持输入输出形状一致：

$$y = \frac{x - E[x]}{\sqrt{Var[x] + \epsilon}} \cdot \gamma + \beta$$

- $\gamma$（scale）和 $\beta$（shift）是可学习参数
- 输入输出形状相同：`[batch_size, seq_len, emb_dim]`

#### 多头注意力（Multi-Head Attention）

$$\text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1, ..., \text{head}_h)W^O$$

其中每个注意力头：

$$\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

关键设计：
- **因果掩码（Causal Mask）**：每个位置只能看到之前的 token，不能看到未来
- **Q/K/V 三个投影矩阵**：分别学习不同的查询、键、值空间
- **头维度 = emb_dim / num_heads**：将嵌入空间分割到多个头

#### 前馈网络（FeedForward）

```
x → Linear(emb_dim → 4*emb_dim) → GELU → Linear(4*emb_dim → emb_dim)
```

每个位置独立应用相同的 FFN，不跨位置交互。

### 4.3 推理生成

自回归生成：每次只生成一个 token，然后追加到输入中继续生成。

```python
for _ in range(max_new_tokens):
    # 取最近 context_size 个 token（截断超长上下文）
    idx_cond = idx[:, -context_size:]
    # 前向传播得到 logits
    logits = model(idx_cond)[:, -1, :]  # 只取最后一个位置
    # 取概率最高的 token
    idx_next = torch.argmax(logits, dim=-1, keepdim=True)
    # 追加到序列
    idx = torch.cat((idx, idx_next), dim=1)
```

## 五、数据集构造

使用 **滑动窗口** 将文本切分为重叠的训练样本：

```python
class GPTDatasetV1(Dataset):
    def __init__(self, txt, tokenizer, max_length, stride):
        token_ids = tokenizer.encode(txt)
        # 滑动窗口切割
        for i in range(0, len(token_ids) - max_length, stride):
            input_chunk  = token_ids[i : i + max_length]
            target_chunk = token_ids[i + 1 : i + max_length + 1]
            # input[i] → target[i] 的配对
```

- `stride < max_length` 时样本有重叠，充分利用数据
- 每个训练样本：`input` 是前 N 个 token，`target` 是每个位置的后继 token

## 六、关键要点总结

| 概念 | 要点 |
|------|------|
| 训练目标 | 下一 token 预测（自监督） |
| 架构 | Decoder-Only Transformer |
| 损失函数 | 交叉熵（token 级分类） |
| 位置编码 | 可学习，与 token embedding 相加 |
| 注意力 | 因果掩码，每个位置只看前面 |
| 归一化 | Pre-LN 结构（LayerNorm 在前） |
| 推理 | 自回归，一次生成一个 token |
