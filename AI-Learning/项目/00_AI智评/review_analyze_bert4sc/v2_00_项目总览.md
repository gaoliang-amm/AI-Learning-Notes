# 📦 review_analyze_bert4sc — 情感分析（HuggingFace 高阶版）

> 同样是基于 BERT 的电商评论情感分类，但这一版改用 HuggingFace **内置**的 `AutoModelForSequenceClassification`，代码更简洁，是实际工程中更常见的写法。

---

## 和第一版的本质区别

| 对比点 | 第一版（`review_analyze_bert`） | 本版（`review_analyze_bert4sc`） |
|--------|-------------------------------|-------------------------------|
| 模型定义 | 自己写 `ReviewAnalyzeModel` 类 | 直接用 `AutoModelForSequenceClassification` |
| 有无 `model.py` | ✅ 有 | ❌ 没有（不需要了） |
| 损失函数 | 手动定义 `BCEWithLogitsLoss` | **模型自动计算**（传入 labels 即可） |
| 模型输出 | 标量 `(batch_size,)` | `outputs.logits` 形状 `(batch_size, 2)` |
| 概率转换 | `torch.sigmoid(outputs)` | `torch.softmax(outputs.logits, dim=-1)[:, 1]` |
| 保存模型 | `torch.save(state_dict, ...)` | `model.save_pretrained(MODELS_DIR)` |
| 加载模型 | `model.load_state_dict(torch.load(...))` | `AutoModelForSequenceClassification.from_pretrained(MODELS_DIR)` |

## 🗂️ 项目结构

```
review_analyze_bert4sc/
├── src/
│   ├── config.py       # 配置（去掉了 BEST_MODEL）
│   ├── preprocess.py   # 数据预处理（与第一版完全相同）
│   ├── dataset.py      # 数据加载（与第一版完全相同）
│   ├── train.py        # 训练脚本（简化了损失计算）
│   ├── evaluate.py     # 评估脚本（改用新模型加载方式）
│   └── predict.py      # 预测脚本（改用 softmax + logits）
│   ⚠️ 没有 model.py！
├── data/
├── models/             # 保存 HuggingFace 格式的模型文件夹
└── logs/
```

---

## 🚀 运行流程（与第一版相同）

```bash
python src/preprocess.py   # 数据预处理（只需一次）
python src/train.py        # 训练
python src/evaluate.py     # 评估
python src/predict.py      # 交互预测
```
