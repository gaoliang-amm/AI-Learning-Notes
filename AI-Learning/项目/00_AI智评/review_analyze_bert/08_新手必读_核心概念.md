## 🤗 BERT 是什么？

**BERT**（Bidirectional Encoder Representations from Transformers）是 Google 在 2018 年发布的**预训练语言模型**。

**类比**：把 BERT 想象成一个"读过海量中文书"的人，他对语言有很强的理解能力。你只需要告诉他"这是情感分类任务"，他很快就能学会。

```
预训练（Google 已做好）→ 我们拿来直接用
        +
微调（在自己数据集上继续训练）→ 适配具体任务
```

本项目用的 BERT 位于 `../../mybert`（本地文件夹）。

---

## 🔤 Tokenizer（分词器）是什么？

计算机不认识文字，需要把文字转成数字。Tokenizer 做这件事：

```
"质量很好" 
  → ["质", "量", "很", "好"]          ← 分词
  → [6167, 6893, 2523, 1962]          ← 转 ID（input_ids）
  → 前后加 [CLS]=101, [SEP]=102
  → [101, 6167, 6893, 2523, 1962, 102]
  → 补零到 128 位（padding）
```

---

## 🎭 [CLS] 向量是什么？

`[CLS]` 是 BERT 强制加在句子开头的一个特殊 token，BERT 在预训练时让它**代表整句话的含义**，所以做分类时用它的输出向量最合适。

---

## 📦 DataLoader 是什么？

把整个数据集（可能有 10 万条）切成小批次喂给模型：

```
10000 条数据，batch_size=16
→ 625 个 batch，每 batch 16 条
→ 模型每次处理 16 条，重复 625 次 = 一个 epoch
```

**为什么不一次全放进去？** → 显存装不下，而且小批次训练效果更好。

---

## 📉 损失函数是什么？

衡量"模型预测有多错"的指标。损失越小 = 模型越准。

本项目用 `BCEWithLogitsLoss`（二元交叉熵）：
- 预测 0.9，真实是 1（正面）→ 损失小
- 预测 0.1，真实是 1（正面）→ 损失大

---

## 🎛️ 超参数是什么？

不是模型学出来的，而是**人工设定**的参数：

| 超参数 | 本项目值 | 说明 |
|--------|---------|------|
| `EPOCHS` | 50 | 数据集过几遍 |
| `BATCH_SIZE` | 16 | 每批多少条 |
| `LEARNING_RATE` | 1e-5 | 每次更新参数的步长 |
| `MAX_SEQ_LEN` | 128 | 文本最大长度 |

---

## 🔁 训练轮次（Epoch）

一个 epoch = 模型把训练集**完整看了一遍**。

```
Epoch 1: 损失 0.52 ← 刚开始，预测很差
Epoch 5: 损失 0.23 ← 有进步
Epoch 20: 损失 0.08 ← 很好了
Epoch 50: 损失 0.05 ← 可能过拟合了
```

---

## ⚡ 什么是过拟合？

模型把训练数据"背下来"了，对新数据反而不准：

```
训练集准确率 99% + 测试集准确率 70% = 过拟合
```

解决方法：减少 EPOCHS、增加数据量、加 Dropout 等。

## 🖥️ GPU vs CPU

| | GPU | CPU |
|--|-----|-----|
| 训练速度 | 快（10-100x） | 慢 |
| 代码写法 | `.to('cuda')` | `.to('cpu')` |

代码自动检测：
```python
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
```
