# 209. 长度最小的子数组

## 🎯 人话翻译

> 给一个全是正整数的数组和一个目标值 target，找出**最短的连续子数组**，它的和 >= target。如果不存在这样的子数组，返回 0。

## 💡 核心思想

> [[滑动窗口]]（可变窗口）。用两个指针 left 和 right 维护一个窗口，right 向右扩展窗口增加和，当窗口和 >= target 时，不断收缩左边界以找到最短长度。因为数组全是正整数，扩展必定增加和，收缩必定减少和，保证了正确性。

用到：[[滑动窗口]]

## 📝 解题步骤

1. **初始化** — `left = 0`, `window_sum = 0`, `min_len = 无穷大`
2. **扩展右边界** — `for right in range(n)`，将 `nums[right]` 加入窗口
3. **满足条件时收缩** — 当 `window_sum >= target` 时，用 `while` 不断缩小左边界：记录当前窗口长度，减去 `nums[left]`，`left += 1`
4. **返回结果** — 如果 `min_len` 还是无穷大，说明没有满足条件的子数组，返回 0；否则返回 `min_len`

## 🔢 手动走一遍

输入: `target = 7`, `nums = [2, 3, 1, 2, 4, 3]`

| 5 | - | 3 | 5 | 否 | [3] | 2 | 停止收缩 |

最终返回 `2`（子数组 `[4, 3]`）。

## 💻 解法

> 可变滑动窗口：right 不断扩展，满足条件时用 while 循环收缩 left 以最小化窗口长度。

```python
from typing import List


class Solution:
    def minSubArrayLen(self, target: int, nums: List[int]) -> int:
        """
        找出和 >= target 的最短连续子数组的长度。
        滑动窗口法：扩展右边界累加和，满足条件时收缩左边界并更新最小长度。
        """
        n = len(nums)
        left = 0
        window_sum = 0
        min_len = float('inf')

        for right in range(n):
            # 扩大窗口，加入 nums[right]
            window_sum += nums[right]

            # 当窗口和满足条件时，尝试缩小窗口
            while window_sum >= target:
                # 更新最小长度
                min_len = min(min_len, right - left + 1)
                # 缩小窗口，移除 nums[left]
                window_sum -= nums[left]
                left += 1

        return min_len if min_len != float('inf') else 0
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 每个元素最多被加入一次和移出一次，right 和 left 各走 n 步 |
| 空间 | O(1) | 只用常数个变量存储和、长度 |

## ⚠ 易错点

- [ ] 收缩时用 `while` 不是 `if`：因为可能收缩一次后窗口和仍然 >= target，需要继续收缩以找到更短的
- [ ] 窗口长度的计算：`right - left + 1`，容易写成 `right - left`（忘了 +1）
- [ ] 返回值需要判断 `min_len` 是否还是 `inf`：如果没找到满足条件的子数组，返回 0
- [ ] 窗口和 `window_sum` 必须先加再判断，还是先判断再加？应该是**先扩展（加）再判断**，顺序不能反
