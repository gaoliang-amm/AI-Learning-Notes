# 30. 串联所有单词的子串

## 🎯 人话翻译

> 给一个字符串 s 和一个单词列表 words（所有单词长度相同）。在 s 中找到所有"串联子串"的起始位置。"串联子串"是指 words 中所有单词按任意顺序拼接起来的子串。简单说就是找一个子串，正好能用 words 里的每个单词用一次拼出来。

## 💡 核心思想

> [[滑动窗口]] + 分组起点。因为所有单词长度相同（设为 L），我们可以把 s 按步长 L 切分成"单词"。然后枚举 0 到 L-1 作为起始偏移（此乃解决"定长滑动窗口"中单词跨越边界的技巧）。在每个偏移下，用滑动窗口统计单词频次：right 每次加一个单词，left 在窗口过长时缩一个单词，当窗口内所有单词频次都匹配时就找到一个答案。

用到：[[滑动窗口]]、[[哈希表]]

## 📝 解题步骤

1. **预处理** — 用 `Counter` 统计 words 中每个单词需要出现的次数（`need`），记录单词长度 L、总单词数 word_cnt、子串总长度 total_len
2. **枚举起始偏移** — 从 0 到 L-1，每个偏移下独立进行滑动窗口扫描（解决单词边界对齐问题）
3. **在偏移内滑动窗口** — 以步长 L 扩展 right，截取当前单词，如果单词不在 need 中则重置窗口；否则加入窗口统计，更新已匹配的单词种类数
4. **收缩窗口** — 当窗口长度超过 total_len 时，从左侧移出一个单词
5. **记录答案** — 当已匹配的单词种类数 == len(need)（所有单词频次达标），记录 `left` 作为答案

## 💻 解法

> 定长单词的滑动窗口：枚举起始偏移（0 到 L-1），在每组内以步长 L 滑动，用频次计数器判断是否匹配。

```python
from typing import List
from collections import Counter, defaultdict


class Solution:
    def findSubstring(self, s: str, words: List[str]) -> List[int]:
        """
        找出所有串联子串的起始索引。
        """
        if not s or not words:
            return []

        L = len(words[0])  # 单词长度
        word_cnt = len(words)  # 单词个数
        total_len = L * word_cnt  # 子串总长度

        if len(s) < total_len:
            return []

        need = Counter(words)  # 目标单词频次
        res = []

        # 枚举起始偏移 0 到 L-1
        for start in range(L):
            window = defaultdict(int)  # 当前窗口单词频次
            left = start
            count = 0  # 已匹配的单词种类数

            # 按步长 L 遍历
            for right in range(start, len(s) - L + 1, L):
                word = s[right:right + L]

                # 单词不在目标中，窗口重置
                if word not in need:
                    window.clear()
                    count = 0
                    left = right + L
                    continue

                # 加入窗口
                window[word] += 1
                if window[word] == need[word]:
                    count += 1

                # 窗口长度超过总长度，从左侧移出单词
                while right + L - left > total_len:
                    left_word = s[left:left + L]
                    if window[left_word] == need[left_word]:
                        count -= 1
                    window[left_word] -= 1
                    left += L

                # 所有单词匹配，记录答案
                if count == len(need):
                    res.append(left)

        return res
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(L * n/L) = O(n) | L 个起始偏移，每个偏移内遍历的步长是 L，总共 n 个字符扫描 |
| 空间 | O(L * word_cnt) | 哈希表存储单词频次，最坏情况 O(len(words) * L) |

## ⚠ 易错点

- [ ] 必须枚举起始偏移 0 到 L-1，因为单词在 s 中的起始位置可能不是 0。如果只从 0 开始以步长 L 滑动，会漏掉从偏移 1 或 2 开始的匹配
- [ ] 遇到不在 need 中的单词必须**重置整个窗口**：清空 window、count 归零、left 跳到 right+L
- [ ] 窗口长度判断：`right + L - left > total_len`，注意 `right+L` 是当前单词的右端，`left` 是窗口真正左端
- [ ] `count` 记录的是"频次达标的单词种类数"，不是单词总个数：一个单词可能出现多次但只贡献一次 count
- [ ] 收缩窗口时，先检查 `window[left_word] == need[left_word]`（说明移除前刚好达标），再 `count -= 1`，然后才 `window[left_word] -= 1`，顺序不能乱

## 🎤 面试怎么说

```text
这道题是滑动窗口的经典难题。核心技巧有两个：

第一，因为所有单词长度相同 L，我们把字符串按步长 L 切成"单词"。
第二，要枚举起始偏移 0 到 L-1，每个偏移都做一轮独立的滑动窗口。
这是因为子串的起始位置不一定在 0 位置对齐。

在每轮中，用一个哈希表统计窗口内单词频次，
当所有单词频次都达标时（count == len(need)），记录 left。

总体时间是 O(n)，因为总字符扫描次数 = L * (n/L) = n。

这道题和 438 题找字母异位词类似，区别是这里找的是"单词"而非"字符"。
```
