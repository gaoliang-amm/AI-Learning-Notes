# 3. 无重复字符的最长子串

## 🎯 人话翻译

> 找一个字符串里最长的子串，这个子串里不能有重复字符。比如 "abcabcbb"，答案是 3（子串 "abc"）。注意是子串（连续的），不是子序列。

## 💡 核心思想

> [[滑动窗口]] + [[哈希表]]。用一个哈希表记录每个字符最近一次出现的位置。right 不断右移扩展窗口，当遇到重复字符且它在窗口内时，left 直接跳到重复字符的下一个位置。每次更新最大长度。

用到：[[滑动窗口]]、[[哈希表]]

## 📝 解题步骤

1. **初始化** — `char_index = {}`（字符 -> 最近出现的位置），`left = 0`, `max_len = 0`
2. **扩展右边界** — `for right, ch in enumerate(s)`，遍历每个字符
3. **检测重复** — 如果 `ch` 已经在哈希表中且其位置 >= `left`（说明在窗口内），将 `left` 跳到 `char_index[ch] + 1`
4. **更新位置** — 无论是否重复，都更新 `char_index[ch] = right`
5. **更新最大长度** — `max_len = max(max_len, right - left + 1)`

## 🔢 手动走一遍

输入: `s = "abcabcbb"`

| right | ch | char_index 中 ch 的位置 | >= left? | left | 窗口 | max_len |
|-------|----|--------------------------|----------|------|------|---------|
| 0 | 'a' | 不存在 | - | 0 | "a" | 1 |
| 1 | 'b' | 不存在 | - | 0 | "ab" | 2 |
| 2 | 'c' | 不存在 | - | 0 | "abc" | 3 |
| 3 | 'a' | 0 | 0 >= 0 是 | 1 | "bca" | 3 |
| 4 | 'b' | 1 | 1 >= 1 是 | 2 | "cab" | 3 |
| 5 | 'c' | 2 | 2 >= 2 是 | 3 | "abc" | 3 |
| 6 | 'b' | 4 | 4 >= 3 是 | 5 | "cb" | 3 |
| 7 | 'b' | 6 | 6 >= 5 是 | 7 | "b" | 3 |

最终返回 `3`。

## 💻 解法

> 滑动窗口 + 哈希表记录字符最后出现位置，遇到重复直接将 left 跳到重复字符之后。

```python
class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        """
        找出不含重复字符的最长子串的长度。
        算法：滑动窗口 + 哈希表
        - 使用哈希表记录每个字符最后一次出现的位置
        - 遇到重复字符时，左指针直接跳到重复字符的下一个位置
        - 时间复杂度 O(n)，空间复杂度 O(min(n, 字符集大小))
        """
        # 哈希表：字符 -> 该字符最后一次出现的索引
        char_index = {}
        max_len = 0
        left = 0  # 窗口左边界

        for right, ch in enumerate(s):
            # 如果当前字符已出现过，且上一次位置在当前窗口内
            if ch in char_index and char_index[ch] >= left:
                # 左指针直接跳到重复字符的下一个位置
                left = char_index[ch] + 1

            # 更新当前字符的最新位置
            char_index[ch] = right

            # 更新最长无重复子串长度
            max_len = max(max_len, right - left + 1)

        return max_len
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 每个字符被访问一次 |
| 空间 | O(min(n, C)) | C 为字符集大小，最多存 128（ASCII）或 26（小写字母） |

## ⚠ 易错点

- [ ] `left` 只能往前跳，不能后退：所以检查条件是 `char_index[ch] >= left`（而非仅仅 `ch in char_index`），确保重复字符在**当前窗口内**
- [ ] 更新 `max_len` 在**每次迭代末尾**做，不是在移动 left 之前
- [ ] left 跳到 `char_index[ch] + 1`，不是跳到 `char_index[ch]`（后者把重复字符也包括进去了）
- [ ] 即使有重复也更新 `char_index[ch] = right`，这很重要——下次遇到同样的字符时就能正确判断了

## 🎤 面试怎么说

```text
这道题是滑动窗口 + 哈希表的经典组合。

用哈希表记录每个字符最后一次出现的位置，right 指针遍历字符串。
当遇到一个字符在当前窗口内出现过（哈希表中的位置 >= left），
就把 left 直接跳到那个重复字符的下一个位置。

这样每个字符最多进窗口一次、出窗口一次，O(n) 时间。
空间取决于字符集大小，ASCII 就是 O(128) = O(1)。

面试官可能会问用 set 也能做吗？用 set 需要内层 while 循环
不断移除 left 的字符直到没有重复，同样是 O(2n) = O(n)。
哈希表的版本跳得更快，但两种都正确。
```
