# 76. 最小覆盖子串

## 🎯 人话翻译

> 给两个字符串 s 和 t，在 s 中找一个最短的子串，要求这个子串包含 t 的所有字符（包括重复的）。如果不存在，返回空字符串。

## 💡 核心思想

> [[滑动窗口]] + 字符频次统计。用 `need` 记录 t 中每个字符需要的次数，用 `window` 记录当前窗口内字符的频次。`valid` 计数器表示"频次已达标"的字符种类数。right 不断扩展窗口，当 `valid == len(need)`（所有字符都够了），用 while 循环收缩 left 来缩小窗口，同时更新最短子串。

用到：[[滑动窗口]]、[[哈希表]]

## 📝 解题步骤

1. **统计需求** — 用 `defaultdict` 统计 t 中每个字符需要的频次，记录 `need_cnt = len(need)`（需要满足的字符种类数）
2. **初始化窗口** — `window` 为空字典，`left = 0`, `valid = 0`, `min_len = inf`, `start = 0`
3. **扩展右边界** — `for right, ch in enumerate(s)`，如果 ch 在 need 中，加入 window，检查是否达标（`window[ch] == need[ch]` 则 `valid += 1`）
4. **满足条件时收缩** — 当 `valid == need_cnt`，用 while 循环：记录当前最小窗口（如果更短），移出 left 字符，更新 valid 和 window，`left += 1`
5. **返回结果** — 如果 `min_len` 仍为 inf 返回空字符串，否则返回 `s[start:start+min_len]`

## 💻 解法

> 可变滑动窗口 + 字符频次计数：用 `valid` 计数"已达标字符种类"，收缩条件为 `valid == need_cnt`。

```python
from collections import defaultdict


class Solution:
    def minWindow(self, s: str, t: str) -> str:
        """
        返回 s 中包含 t 所有字符的最短子串。
        """
        if not s or not t:
            return ""

        # 统计目标字符串 t 的字符频次
        need = defaultdict(int)
        for ch in t:
            need[ch] += 1
        need_cnt = len(need)  # 需要满足的字符种类数

        window = defaultdict(int)  # 窗口内字符频次
        left = 0
        valid = 0  # 已满足需求的字符种类数
        min_len = float('inf')
        start = 0  # 记录最小子串起始位置

        for right, ch in enumerate(s):
            # 扩大窗口
            if ch in need:
                window[ch] += 1
                if window[ch] == need[ch]:
                    valid += 1

            # 窗口已包含所有所需字符，尝试缩小
            while valid == need_cnt:
                # 更新最小子串
                if right - left + 1 < min_len:
                    min_len = right - left + 1
                    start = left

                # 缩小窗口，移出左边界字符
                left_char = s[left]
                if left_char in need:
                    if window[left_char] == need[left_char]:
                        valid -= 1
                    window[left_char] -= 1
                left += 1

        return s[start:start + min_len] if min_len != float('inf') else ""
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(m + n) | m=s长度, n=t长度。每个字符最多被加入和移出各一次 |
| 空间 | O(C) | C 为字符集大小。need 和 window 最多存 52（大小写字母） |

## ⚠ 易错点

- [ ] `valid` 计数是"已达标字符的种类数"，不是"窗口内满足要求的字符总数"。只有 `window[ch]` 刚好等于 `need[ch]` 时才 `valid += 1`（恰好达标），避免重复计数
- [ ] 收缩时的判断顺序：先检查 `window[left_char] == need[left_char]`（移出前是否刚好达标），再 `valid -= 1`，最后 `window[left_char] -= 1`。顺序错了会导致 valid 错误
- [ ] `need_cnt = len(need)` 是关键——只要 valid 达到这个数，窗口就包含所有 t 的字符。如果用 `len(t)` 是不对的，因为可能有重复字符
- [ ] 最后返回 `s[start:start+min_len]` 而不是 `s[left:left+min_len]`，因为 left 在 while 循环后已经变了
- [ ] 边界检查：如果 t 比 s 还长，直接返回空字符串（虽然代码可以跑通，但提前返回更优）

## 🎤 面试怎么说

```text
这道题是滑动窗口的模板题。核心思路是用两个哈希表：
need 记录 t 需要多少字符，window 记录当前窗口有多少字符。
再加一个 valid 计数器，表示"频次已达标"的字符种类数。

当 valid == len(need) 说明窗口包含了 t 的所有字符，
此时用 while 循环收缩左边界以找到最短窗口。

收缩的时候注意：先判断 left_char 是否在 need 中，
判断移出前是否刚好达标（如果是 valid--），
再更新 window 计数。

时间复杂度 O(m+n)，空间 O(字符集大小)。
这个模板可以直接用于大多数"找满足条件的子串/子数组"问题。
```
