# 219. 存在重复元素 II

## 🎯 人话翻译

> 给一个数组和一个整数 k，判断数组里是否存在两个相同的数，且它们下标的距离 <= k。比如 nums = [1, 2, 3, 1], k = 3，两个 1 的距离是 3（下标 3-0=3），等于 k，返回 true。

## 💡 核心思想

> [[哈希表]]记录每个元素上一次出现的位置。遍历时，如果当前元素已经在哈希表中，检查 `i - 上次位置 <= k`，满足就返回 `True`；不满足就更新该元素的位置为当前下标（因为我们要找最近的重复）。

用到：[[哈希表]]

## 📝 解题步骤

1. **初始化哈希表** — `hash_map = {}`，key 为值，value 为最近一次出现的位置
2. **遍历数组** — 对于每个 `(i, num)`：
   - 如果 `num in hash_map` 且 `i - hash_map[num] <= k`，返回 `True`
   - 无论是否满足条件，都更新 `hash_map[num] = i`（保持最近位置）
3. **遍历结束** — 返回 `False`

## 🔢 手动走一遍

输入: `nums = [1, 2, 3, 1]`, `k = 3`

| i | num | 在hash_map中? | hash_map[num] | i - 上次距离 | <= k? | 操作 |
|---|-----|---------------|---------------|------------|-------|------|
| 0 | 1 | 否 | - | - | - | {1: 0} |
| 1 | 2 | 否 | - | - | - | {1: 0, 2: 1} |
| 2 | 3 | 否 | - | - | - | {1: 0, 2: 1, 3: 2} |
| 3 | 1 | 是 | 0 | 3 - 0 = 3 | 是(<=3) | 返回 True |

最终返回 `True`。

反例: `nums = [1, 2, 3, 1, 2, 3]`, `k = 2`
| i | num | 上次位置 | 距离 | <= 2? |
|---|-----|---------|------|-------|
| 0 | 1 | - | - | - |
| 1 | 2 | - | - | - |
| 2 | 3 | - | - | - |
| 3 | 1 | 0 | 3 | 否，更新为 {1:3} |
| 4 | 2 | 1 | 3 | 否，更新为 {2:4} |
| 5 | 3 | 2 | 3 | 否，结束 |

返回 `False`。

## 💻 解法

> 哈希表记录元素最近一次出现的下标：找到重复且距离 <= k 就返回 True，始终更新位置为最新下标。

```python
class Solution:
    def containsNearbyDuplicate(self, nums: list[int], k: int) -> bool:
        # 哈希表：key = 数字，value = 上次出现的下标
        hash_map = {}

        for idx, num in enumerate(nums):
            # 如果这个数之前出现过，并且下标差 <= k
            if num in hash_map and idx - hash_map[num] <= k:
                return True
            # 更新成当前最新下标
            hash_map[num] = idx

        # 遍历完都没找到
        return False
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 每个元素访问一次 |
| 空间 | O(n) | 哈希表最多存 n 个不同元素 |

## ⚠ 易错点

- [ ] 每次都要更新 `hash_map[num] = idx`，即使距离 > k 也要更新：因为要找"最近"的一对，保留最新的下标有助于后续匹配
- [ ] 如果距离 = k 也算满足条件，判断条件是 `<= k` 不是 `< k`
- [ ] 也可以用滑动窗口（窗口大小为 k+1，用 set 维护），但本题的哈希表记录位置更简洁
- [ ] `hash_map` 的 value 存的是下标而不是计数：因为我们要判断距离，不是频次
