# 242. 有效的字母异位词

## 🎯 人话翻译

> 判断两个字符串是否互为"字母异位词"——即两个字符串包含的字母种类和数量完全相同，只是顺序不同。比如 "anagram" 和 "nagaram"，字母一样多只是排列不同。

## 💡 核心思想

> [[哈希表]]做字母频次统计。对 s 中每个字母计数 +1，对 t 中每个字母计数 -1。最后如果所有计数都是 0，就是字母异位词。因为只有 26 个小写字母，直接用固定大小的数组而非字典。

用到：[[哈希表]]

## 📝 解题步骤

1. **长度检查** — 如果 `len(s) != len(t)`，直接返回 `False`
2. **初始化计数数组** — `count = [0] * 26`，索引 0~25 对应 'a'~'z'
3. **统计 s** — 遍历 s 的每个字符，`count[ord(ch) - ord('a')] += 1`
4. **抵消 t** — 遍历 t 的每个字符，`count[ord(ch) - ord('a')] -= 1`；如果减后 < 0，说明 t 中这个字符比 s 多，提前返回 `False`
5. **最终检查** — 由于长度相等且无负数，所有计数一定为 0，返回 `True`

## 🔢 手动走一遍

输入: `s = "anagram"`, `t = "nagaram"`

| 步骤 | 操作 | count 数组状态（只列非 0） |
|------|------|---------------------------|
| 初始化 | - | 全 0 |
| 统计 s | a+1, n+1, a+1(2), g+1, r+1, a+1(3), m+1 | a:3, n:1, g:1, r:1, m:1 |
| 抵消 t | n-1(0), a-1(2), g-1(0), a-1(1), r-1(0), a-1(0), m-1(0) | 全 0 |

全部为 0，返回 `True`。

反例: `s = "rat"`, `t = "car"`
| 步骤 | 操作 | 状态 |
|------|------|------|
| 统计 s | r:1, a:1, t:1 | - |
| 抵消 t | c:'c'不在s中，c-1=-1<0 | 返回 False |

## 💻 解法

> 用固定大小数组做频次统计：s 中加，t 中减，减到负数就提前返回。

```python
class Solution:
    def isAnagram(self, s: str, t: str) -> bool:
        """
        判断 s 和 t 是否为字母异位词。
        使用长度为 26 的数组统计小写字母出现次数。
        """
        if len(s) != len(t):
            return False

        # 初始化计数数组，记录 26 个小写字母的出现次数
        count = [0] * 26

        # 统计 s 中每个字符的出现次数
        for ch in s:
            count[ord(ch) - ord('a')] += 1

        # 遍历 t，减去对应字符的计数
        for ch in t:
            idx = ord(ch) - ord('a')
            count[idx] -= 1
            # 如果某个字符在 t 中出现次数多于 s，提前返回 False
            if count[idx] < 0:
                return False

        # 由于长度相等且无负数，所有计数必然为 0，可直接返回 True
        return True
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | n 为字符串长度，每个字符访问一次 |
| 空间 | O(1) | 固定 26 大小的数组 |

## ⚠ 易错点

- [ ] 如果 `len(s) != len(t)` 直接返回 False——长度都不等，字母异位词不可能成立
- [ ] 在 t 的循环中减到负数就提前返回，不需要最后再遍历一遍检查全 0（因为长度相等 + 无负数 = 必然全 0）
- [ ] 也可以最后检查 `all(c == 0 for c in count)`，但提前返回更高效
- [ ] 如果涉及 Unicode 字符（不只是小写字母），用 `collections.Counter` 或者字典更安全
