# 49. 字母异位词分组

## 🎯 人话翻译

> 给你一堆字符串，把"字母异位词"（字母组成完全相同、只是顺序不同的词）分到同一组。比如 ["eat", "tea", "tan", "ate", "nat", "bat"] -> [["eat","tea","ate"], ["tan","nat"], ["bat"]]。

## 💡 核心思想

> [[哈希表]]的 key 设计问题。字母异位词的共同特征是什么？**排序后的字符串完全相同**。所以用 `sorted(s)` 的结果作为 key，用 `defaultdict(list)` 把相同 key 的字符串归到一组。这就是用哈希表实现"分类"的经典模式。

用到：[[哈希表]]

## 📝 解题步骤

1. **初始化** — `hash_map = defaultdict(list)`，key 为排序后的字符串，value 为原字符串列表
2. **遍历并分类** — 对每个字符串 `s`，计算 `key = "".join(sorted(s))`，`hash_map[key].append(s)`
3. **返回结果** — `list(hash_map.values())` 即为所有分组

## 🔢 手动走一遍

输入: `strs = ["eat", "tea", "tan", "ate", "nat", "bat"]`

| 字符串 | 排序后的 key | hash_map |
|--------|-------------|----------|
| "eat" | "aet" | {"aet": ["eat"]} |
| "tea" | "aet" | {"aet": ["eat", "tea"]} |
| "tan" | "ant" | {"aet": ["eat", "tea"], "ant": ["tan"]} |
| "ate" | "aet" | {"aet": ["eat", "tea", "ate"], "ant": ["tan"]} |
| "nat" | "ant" | {"aet": ["eat", "tea", "ate"], "ant": ["tan", "nat"]} |
| "bat" | "abt" | {..., "abt": ["bat"]} |

最终结果: `[["eat", "tea", "ate"], ["tan", "nat"], ["bat"]]`

## 💻 解法

> 排序后的字符串作为 key，用 defaultdict 自动创建空列表来分组。

```python
from collections import defaultdict
from typing import List


class Solution:
    def groupAnagrams(self, strs: List[str]) -> List[List[str]]:
        #  key: 排序后的字符串; value: 属于同一异位词的原字符串列表
        hash_map = defaultdict(list)

        for s in strs:
            # 字母异位词排序后完全相同，用这个作为唯一 key
            key = "".join(sorted(s))
            hash_map[key].append(s)

        # 所有分组就是字典的值
        return list(hash_map.values())
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n * k log k) | n 个字符串，每个长 k，排序 O(k log k) |
| 空间 | O(n * k) | 存储所有字符串 |

## ⚠ 易错点

- [ ] `sorted(s)` 返回的是字符列表，需要用 `"".join()` 拼回字符串才能作为字典的 key（list 不可哈希）
- [ ] 用 `defaultdict(list)` 而不是普通 dict，否则第一次遇到新 key 需要手动 `hash_map[key] = []`
- [ ] 排序不是唯一方案：也可以用字符频次统计（26 位数组转成元组）作为 key，避免排序开销。当字符串很长时，频次统计（O(k)）优于排序（O(k log k)）
- [ ] 返回 `list(hash_map.values())` 即可，题目不要求组内或组间有序

## 🎤 面试怎么说

```text
这道题的核心是找到一个合适的 key 来识别字母异位词。

最简单的做法：对每个字符串排序，字母异位词排序后结果相同。
用排序后的字符串作为哈希表的 key，把原字符串加到对应的列表里。

时间复杂度 O(n * k log k)，空间 O(n * k)。

面试官可能会问能不能更快？可以用字符频次计数代替排序。
比如对每个字符串统计 26 个字母的出现次数，
把这个计数数组转成元组作为 key，这样时间复杂度降为 O(n * k)。
```
