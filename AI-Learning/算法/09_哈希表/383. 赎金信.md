# 383. 赎金信

## 🎯 人话翻译

> 你有一本杂志 magazine（里面每个字母只能用一次），想拼出一封赎金信 ransomNote。判断杂志里的字母够不够拼出这封信。比如 "aa" 要用 "ab" 拼就不够（少一个 a）。

## 💡 核心思想

> [[哈希表]]做字符频次统计。先遍历 magazine 统计每个字母的库存，再遍历 ransomNote 逐个消耗库存。库存不够（计数为 0）就返回 False。因为只有 26 个小写字母，直接用长度为 26 的数组代替字典，更省空间。

用到：[[哈希表]]

## 📝 解题步骤

1. **长度剪枝** — 如果 `len(ransomNote) > len(magazine)`，直接返回 `False`
2. **统计 magazine 库存** — 用长度为 26 的数组 `count` 记录 magazine 中每个字母的出现次数
3. **消耗库存** — 遍历 ransomNote 的每个字符，`count[idx] -= 1`
4. **检查是否够用** — 如果某次减后 `count[idx] < 0`，说明库存不够，返回 `False`
5. **全部通过** — 返回 `True`

## 🔢 手动走一遍

输入: `ransomNote = "aa"`, `magazine = "aab"`

统计 magazine: count = [a:2, b:1, 其余:0]

| 字符 | idx | count[idx] 减前 | 减后 | < 0? |
|------|------|-----------------|------|------|
| 'a' | 0 | 2 | 1 | 否 |
| 'a' | 0 | 1 | 0 | 否 |

全部通过，库存充足，返回 `True`。

反例: `ransomNote = "aa"`, `magazine = "ab"`
| 字符 | idx | count[idx] 减前 | 减后 | < 0? |
|------|------|-----------------|------|------|
| 'a' | 0 | 1 | 0 | 否 |
| 'a' | 0 | 0 | -1 | 是！返回 False |

## 💻 解法

> 用固定大小数组（26 个英文字母）做字符频次统计，先加后减，发现负数即库存不够。

```python
def canConstruct(ransomNote: str, magazine: str) -> bool:
    """
    判断 ransomNote 能否由 magazine 中的字符构成（每个字符只能用一次）
    参数：
        ransomNote: 目标字符串（赎金信）
        magazine:   来源字符串（杂志）
    返回：
        True 可以构成，False 不可以
    时间复杂度：O(len(ransomNote) + len(magazine))
    空间复杂度：O(1)  （固定大小的数组）
    """
    # 优化：如果赎金信比杂志还长，肯定不可能
    if len(ransomNote) > len(magazine):
        return False

    # 创建一个长度为 26 的列表，索引 0~25 分别对应字母 a~z
    # 初始值全为 0，用来统计 magazine 中每个字母出现的次数
    count = [0] * 26

    # 第一遍扫描：统计 magazine 里每个字母的频次
    for ch in magazine:
        # 将字符转换成对应的索引：'a' -> 0, 'b' -> 1, ..., 'z' -> 25
        idx = ord(ch) - ord('a')
        count[idx] += 1

    # 第二遍扫描：检查 ransomNote 的每个字母
    for ch in ransomNote:
        idx = ord(ch) - ord('a')
        # 如果该字母的剩余数量为 0，说明 magazine 不够用了
        if count[idx] == 0:
            return False
        # 用掉一个，数量减 1
        count[idx] -= 1

    # 所有字母都够用，返回 True
    return True
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(m + n) | m = len(ransomNote), n = len(magazine) |
| 空间 | O(1) | 固定大小的 26 数组，不随输入规模变化 |

## ⚠ 易错点

- [ ] 使用 `ord(ch) - ord('a')` 计算索引，要确保字符是小写字母（题目保证），否则索引会越界
- [ ] `count[idx] == 0` 检查放在减之前：判断的是"有没有存粮可以消耗"，如果为 0 说明库存空了
- [ ] 也可以先减再判断 `< 0`，两种写法等价，但逻辑上"先判断再消耗"更直观
- [ ] 别忘了前置剪枝：`len(ransomNote) > len(magazine)` 时直接返回 False

这道题是字符频次统计的模板题。因为全是小写字母，
直接用长度为 26 的整数数组代替字典。

先遍历 magazine，每个字符 count++；
