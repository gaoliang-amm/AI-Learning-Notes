# 1. 两数之和

## 🎯 人话翻译

> 给一个数组和一个目标值，找出数组中两个数的下标，使它们的和等于目标值。题目保证有且只有一个答案，同一个元素不能用两次。

## 💡 核心思想

> [[哈希表]]一次遍历。遍历时，对于当前数 `num`，计算它需要的补数 `complement = target - num`。如果补数已经在哈希表中，说明找到了答案；否则把当前的 `(num, index)` 存进哈希表供后面查找。核心是"边走边查，存入历史"，避免了 O(n^2) 的双重循环。

用到：[[哈希表]]

## 📝 解题步骤

1. **初始化哈希表** — `num_to_index = {}`，键为数组值，值为下标
2. **一次遍历** — 对于每个 `(i, num)`：
   - 计算补数 `complement = target - num`
   - 检查 `complement` 是否在哈希表中
   - 在：返回 `[num_to_index[complement], i]`
   - 不在：`num_to_index[num] = i`
3. **不会走到最后** — 题目保证有解

## 🔢 手动走一遍

输入: `nums = [2, 7, 11, 15]`, `target = 9`

| i | num | complement=9-num | 在哈希表中? | 操作 |
|---|-----|-------------------|------------|------|
| 0 | 2 | 7 | 否(表空) | {2: 0} |
| 1 | 7 | 2 | 是(2在表中) | 返回 [0, 1] |

最终返回 `[0, 1]`。

## 💻 解法

> 一次遍历 + 哈希表：边走边查补数，不在表中就存自己，O(n) 时间 O(n) 空间。

```python
from typing import List


class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        """
        使用哈希表一次遍历找出两数之和等于 target 的下标。

        时间复杂度: O(n)
        空间复杂度: O(n)
        """
        # 哈希表：键为数组元素的值，值为该元素的下标
        num_to_index = {}

        # 遍历数组
        for i, num in enumerate(nums):
            # 计算当前元素需要的补数
            complement = target - num

            # 如果补数已经在哈希表中，说明找到了答案
            if complement in num_to_index:
                # 返回补数的下标和当前元素的下标
                return [num_to_index[complement], i]

            # 否则将当前元素的值和下标存入哈希表
            num_to_index[num] = i

        # 根据题目假设，这里不会执行到，但为了完整性返回空列表
        return []
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 每个元素只访问一次 |
| 空间 | O(n) | 哈希表最多存 n 个元素 |

## ⚠ 易错点

- [ ] 先检查补数是否存在，**再**存入当前数：如果反过来，在 `nums = [3, 3]`, `target = 6` 时会出错——第一个 3 存进去、补数是 3、查到的是自己（需要不同的下标）
- [ ] 返回顺序：`[num_to_index[complement], i]`，补数的下标在前（因为补数先出现）
- [ ] 哈希表的 key 是数值，value 是下标——不要搞反，因为我们要用数值快速查找下标
- [ ] 题目保证有解，但代码里最后返回一个空列表或 `[-1, -1]` 是好习惯

LeetCode 第 1 题，哈希表的经典应用。

遍历数组，对每个数计算 target - num 作为补数。
如果补数已经在哈希表中，说明之前出现过一个数能和当前数凑成 target，直接返回。
