# 55. 跳跃游戏

## 🎯 人话翻译

> 每个位置上的数字表示最多能跳多远。从索引 0 出发，问能否跳到最后一个位置。

## 💡 核心思想

> **贪心：维护当前能到达的最远位置 `farthest`，一路更新。如果某个位置超出了最远可达范围，说明"过不去"，返回 False。**
>
> `farthest = max(farthest, i + nums[i])`：在每个位置，以该位置为跳板，看能否把最远可达范围往外推。
>
> 关键判断：`if i > farthest`，说明当前这个位置根本无法到达，游戏结束。

用到：[[贪心]]

## 📝 解题步骤

1. **初始化** — `farthest = 0`（当前能到达的最远下标）
2. **遍历数组** — 对于每个位置 i：
   - 若 `i > farthest`：当前位置不可达，返回 False
   - `farthest = max(farthest, i + nums[i])`：更新最远可达范围
   - 若 `farthest >= n - 1`：提前返回 True
3. **返回 True** — 遍历完成说明所有位置都可达，末尾自然也可达

## 🔢 手动走一遍

输入: `nums = [2, 3, 1, 1, 4]`

| i | nums[i] | i > farthest? | farthest 前 | i+nums[i] | farthest 后 | farthest >= 4? |
|---|---------|--------------|------------|-----------|------------|----------------|
| 0 | 2 | 0 > 0? 否 | 0 | 2 | 2 | 2 >= 4? 否 |
| 1 | 3 | 1 > 2? 否 | 2 | 4 | 4 | 4 >= 4? 是，返回 True |

结果: `True`

输入: `nums = [3, 2, 1, 0, 4]`

| i | nums[i] | i > farthest? | farthest 前 | i+nums[i] | farthest 后 | farthest >= 4? |
|---|---------|--------------|------------|-----------|------------|----------------|
| 0 | 3 | 0 > 0? 否 | 0 | 3 | 3 | 3 >= 4? 否 |
| 1 | 2 | 1 > 3? 否 | 3 | 3 | 3 | 3 >= 4? 否 |
| 2 | 1 | 2 > 3? 否 | 3 | 3 | 3 | 3 >= 4? 否 |
| 3 | 0 | 3 > 3? 否 | 3 | 3 | 3 | 3 >= 4? 否 |
| 4 | 4 | 4 > 3? **是** | - | - | - | 返回 False |

结果: `False`（被卡在位置 3，nums[3]=0 无法前进）

## 💻 解法

> 贪心维护最远可达位置，不可达时返回 False。

```python
from typing import List


class Solution:
    def canJump(self, nums: List[int]) -> bool:
        n = len(nums)
        farthest = 0  # 当前能到达的最远下标

        for i in range(n):
            # 如果当前位置不可达，则无法继续
            if i > farthest:
                return False
            # 更新能到达的最远位置
            farthest = max(farthest, i + nums[i])
            # 如果已经能到达末尾，提前返回
            if farthest >= n - 1:
                return True

        return True
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 遍历一次数组 |
| 空间 | O(1) | 只用了 farthest 变量 |

## ⚠ 易错点

- [ ] 判断 `i > farthest` 必须在更新 farthest 之前——如果当前位置不可达，后面的更新就没有意义了
- [ ] `farthest` 的更新公式是 `i + nums[i]` 而不是 `farthest + nums[i]`——是从当前位置跳，不是从最远位置跳
- [ ] 和 LeetCode 45 的区别：55 只需要判断 True/False，45 需要最小步数。55 不需要维护 end 和 steps
- [ ] 也可以从后往前 DP（dp[i] 表示从 i 能否到终点），但贪心 O(n) 更优
