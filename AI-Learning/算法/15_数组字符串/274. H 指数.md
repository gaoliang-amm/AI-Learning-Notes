# 274. H 指数

## 🎯 人话翻译

> 一个研究者有 n 篇论文，每篇有引用次数。H 指数 = 最大的那个数 h，使得至少有 h 篇论文被引用了至少 h 次。

## 💡 核心思想

> **降序排序后遍历：如果第 i 篇论文的引用数 >= i+1，说明至少有 i+1 篇论文的引用次数 >= i+1，H 指数至少为 i+1。**
>
> 排序后论文按引用次数从高到低排列。对于排在第 i 位的论文（i 从 0 开始），有 i+1 篇论文的引用次数 >= 当前论文的引用次数。如果当前引用次数 >= i+1，说明这 i+1 篇论文都满足"至少被引用 i+1 次"，所以 H 指数至少为 i+1。
>
> 一旦遇到 `citations[i] < i+1`，之后论文引用更少，H 指数不可能再增，直接退出。

用到：[[排序]] [[贪心]]

## 📝 解题步骤

1. **降序排序** — `citations.sort(reverse=True)`
2. **遍历统计** — 初始化 `h = 0`，对于每个位置 i：
   - 若 `citations[i] >= i + 1`：更新 `h = i + 1`
   - 否则：break（后面的引用数更小，不可能增大 h）
3. **返回 h**

## 🔢 手动走一遍

输入: `citations = [3, 0, 6, 1, 5]`

**降序排序后：** `[6, 5, 3, 1, 0]`

| i | i+1 | citations[i] | citations[i] >= i+1? | h |
|---|-----|-------------|---------------------|---|
| 0 | 1 | 6 | 是 | 1 |
| 1 | 2 | 5 | 是 | 2 |
| 2 | 3 | 3 | 是 | 3 |
| 3 | 4 | 1 | 否（1 < 4） | break |

结果: `3`（至少有 3 篇论文被引用了至少 3 次：6, 5, 3）

## 💻 解法

> 降序排序后遍历，检查 citations[i] >= i+1 的条件。

```python
from typing import List


class Solution:
    def hIndex(self, citations: List[int]) -> int:
        # 降序排序，引用高的在前面
        citations.sort(reverse=True)

        h = 0
        # 遍历，i 表示当前处理的论文索引（从0开始）
        for i in range(len(citations)):
            # 当前论文的引用数至少为 i+1 时，h 可以更新为 i+1
            if citations[i] >= i + 1:
                h = i + 1
            else:
                # 一旦不满足，后面的论文引用更少，不可能再增加 h，提前退出
                break
        return h
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n log n) | 排序占主导，遍历是 O(n) |
| 空间 | O(1) | 排序原地（Python sort 是原地排序） |

## ⚠ 易错点

- [ ] 必须降序排序——如果升序排，条件 `citations[i] >= i+1` 的逻辑就不对了
- [ ] 比较条件是 `citations[i] >= i + 1`（不是 `citations[i] >= h + 1`）：i+1 表示当前已审查的论文数
- [ ] `h = i + 1` 而不是 `h += 1`：h 是满足条件的论文数，直接等于 i+1（因为 i 从 0 开始）
- [ ] 如果所有论文引用次数都很大（如 [100, 200, 300]），H 指数最多为 n（论文总数）
- [ ] 进阶：可以用计数排序做到 O(n) 时间——因为 H 指数不可能超过 n，创建一个长度为 n+1 的计数数组

## 🎤 面试怎么说

```text
H 指数的定义是"最大 h，使得至少有 h 篇论文被引用了至少 h 次"。

解法是降序排序后遍历：排好序后，第 i 篇论文（i 从 0 开始）的引用次数至少为 citations[i]，
且有 i+1 篇论文引用次数 >= citations[i]。
当 citations[i] >= i+1 时，这 i+1 篇论文满足 H 指数条件。

遍历过程中 i+1 递增但 citations[i] 递减，一旦 citations[i] < i+1 就再也无法满足条件，
可以提前退出。

时间复杂度 O(n log n)（排序），还有一种 O(n) 的计数排序做法，但 O(n log n) 在面试中通常就够了。
```
