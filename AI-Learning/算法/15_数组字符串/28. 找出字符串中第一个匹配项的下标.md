# 28. 找出字符串中第一个匹配项的下标

## 🎯 人话翻译

> 实现 Java 的 indexOf()：在 haystack 里找 needle 第一次出现的位置，找不到返回 -1。

## 💡 核心思想

> **暴力匹配：枚举每个可能的起点，逐字符比较。进阶用 KMP 算法实现 O(n+m)。**
>
> 暴力法：外层循环遍历 haystack 的每个可能的起始位置（0 到 n-m），内层循环逐字符比较 needle。
>
> 进阶：KMP 算法通过构建前缀函数（next 数组），在失配时利用已知信息跳过部分匹配，避免回溯主串指针。

用到：[[字符串]] [[KMP]] [[双指针]]

## 📝 解题步骤

**暴力法：**

1. 处理边界：needle 为空返回 0，needle 比 haystack 长返回 -1
2. 外层循环：遍历所有可能的起点（0 到 n-m）
3. 内层循环：从起点开始逐个比较 needle 的每个字符
4. 匹配成功返回起点下标；全部失败返回 -1

**KMP（进阶）：**

1. 构建 next 数组（最长相等前后缀长度）
2. 双指针遍历主串和模式串
3. 失配时根据 next 数组回退模式串指针

## 🔢 手动走一遍

输入: `haystack = "sadbutsad", needle = "sad"`

**暴力法：**

|--------|----------|------|
| 0 | h[0]='s'==n[0], h[1]='a'==n[1], h[2]='d'==n[2] | 匹配！返回 0 |

输入: `haystack = "leetcode", needle = "leeto"`

| 起点 i | 内层比较 | 结果 |
|--------|----------|------|
| 0 | h[0]='l'==n[0], h[1]='e'==n[1], h[2]='e'==n[2], h[3]='t'=='t', h[4]='c'!='o' | 失配 |
| 1 | h[1]='e'!='l' | 失配 |
| 2 | h[2]='e'!='l' | 失配 |
| 3 | h[3]='t'!='l' | 失配 |
| 4 | 超出范围 n-m+1 = 4 | 停止 |

结果: `-1`

## 💻 解法一（暴力匹配）

> 枚举每个起点，逐字符匹配，简单直观。

```python
class Solution:
    def strStr(self, haystack: str, needle: str) -> int:
        n = len(haystack)
        m = len(needle)

        # 模式串为空 → 题目规定返回 0
        if m == 0:
            return 0

        # 模式串比主串长 → 不可能匹配
        if m > n:
            return -1

        # 遍历所有可能的起点
        for i in range(n - m + 1):
            match = True
            for j in range(m):
                if haystack[i + j] != needle[j]:
                    match = False
                    break
            if match:
                return i

        return -1
```

## 🚀 解法二（KMP 算法）

> 构建 next 前缀表，失配时利用已知信息跳转，避免主串指针回溯。

```python
class Solution:
    def strStr(self, haystack: str, needle: str) -> int:
        n, m = len(haystack), len(needle)
        if m == 0:
            return 0
        if m > n:
            return -1

        # 构建 next 数组：next[j] = needle[0..j-1] 的最长相等前后缀长度
        next_arr = [0] * m
        j = 0  # j 表示前缀末尾（也是最长相等前后缀长度）
        for i in range(1, m):
            # 失配时回退 j
            while j > 0 and needle[i] != needle[j]:
                j = next_arr[j - 1]
            if needle[i] == needle[j]:
                j += 1
            next_arr[i] = j

        # 搜索
        j = 0  # needle 指针
        for i in range(n):
            # 失配时根据 next 回退
            while j > 0 and haystack[i] != needle[j]:
                j = next_arr[j - 1]
            if haystack[i] == needle[j]:
                j += 1
            if j == m:
                return i - m + 1

        return -1
```

## ⏱ 复杂度

| 版本 | 时间 | 空间 | 说明 |
|------|------|------|------|
| 暴力法 | O(n*m) | O(1) | 最坏情况：haystack="aaaa...b", needle="aaa" |
| KMP | O(n+m) | O(m) | next 数组占 O(m) 空间 |

## ⚠ 易错点

- [ ] needle 为空字符串时题目规定返回 0，LeetCode 的测试用例可能不会出现，但代码要处理
- [ ] 暴力法外层循环范围是 `range(n - m + 1)`，不是 `range(n)`。如果 needle 长度大于剩余字符串长度，直接从该起点比较会越界
- [ ] KMP 的 next 数组构建中，`j` 是不匹配时回退到的位置，初始为 0。后缀指针 `i` 从 1 开始
- [ ] Python 有内置的 `haystack.find(needle)`，面试时一般不让用，但可以提一下作为另一种写法

## 🎤 面试怎么说

```text
这道题是实现字符串匹配，本质就是 indexOf。

最简单的是暴力匹配：枚举每个可能的起点，逐字符比较 needle。
时间复杂度 O(n*m)，空间 O(1)。

面试官可能要求优化。经典的字符串匹配算法是 KMP：
先构建 needle 的 next 数组（最长相等前后缀），
然后在匹配过程中，当字符不匹配时，根据 next 数组跳过已匹配的部分，
主串指针不回溯，实现 O(n+m) 的时间复杂度。

next 数组的核心：next[j] 表示 needle[0..j-1] 这段子串中，
最长的相同前缀和后缀的长度。
```
