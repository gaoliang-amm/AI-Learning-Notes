# 135. 分发糖果

## 🎯 人话翻译

> 一排小孩每人有评分。每人至少 1 颗糖，评分高的相邻小孩要比邻居糖多。问最少需要多少糖。

## 💡 核心思想

> **两次遍历贪心：先从左到右保证"右边的分高糖就多"，再从右到左保证"左边的分高糖就多"，每个位置取两次结果的最大值。**
>
> 问题拆成两个方向的约束：
> 1. 左到右：如果 `ratings[i] > ratings[i-1]`，则 `candies[i] = candies[i-1] + 1`
> 2. 右到左：如果 `ratings[i] > ratings[i+1]`，则 `candies[i] = max(candies[i], candies[i+1] + 1)`
>
> 取 max 的原因：两次遍历的结果分别满足了"左边约束"和"右边约束"，取较大值能同时满足两边。

用到：[[贪心]] [[前缀和]]

## 📝 解题步骤

1. **初始化** — 每人先给 1 颗糖，`candies = [1] * n`
2. **左到右遍历** — 若 `ratings[i] > ratings[i-1]`：`candies[i] = candies[i-1] + 1`
3. **右到左遍历** — 若 `ratings[i] > ratings[i+1]`：`candies[i] = max(candies[i], candies[i+1] + 1)`
4. **求和返回** — `sum(candies)`

## 🔢 手动走一遍

输入: `ratings = [1, 0, 2]`

**第一遍（左到右）：**

| i | ratings[i] | ratings[i] > ratings[i-1]? | candies[i] | candies |
|---|------------|---------------------------|------------|---------|
| 0 | 1 | - | 1 | [1, 1, 1] |
| 1 | 0 | 否 | 1（不变） | [1, 1, 1] |
| 2 | 2 | 是（2 > 0） | 1+1=2 | [1, 1, 2] |

**第二遍（右到左）：**

| i | ratings[i] | ratings[i] > ratings[i+1]? | 操作 | candies |
|---|------------|---------------------------|------|---------|
| 2 | 2 | - | 不变 | [1, 1, 2] |
| 1 | 0 | 否 | 不变 | [1, 1, 2] |
| 0 | 1 | 是（1 > 0） | max(1, 1+1)=2 | [2, 1, 2] |

结果: `sum([2, 1, 2]) = 5`

输入: `ratings = [1, 2, 2]`

**第一遍（左到右）：**

| i | ratings[i] | 比较 | candies |
|---|------------|------|---------|
| 0 | 1 | - | 1 |
| 1 | 2 | 2>1, 是 | 2 |
| 2 | 2 | 2>2=否 | 1 |

candies = [1, 2, 1]

**第二遍（右到左）：**

| i | ratings[i] | 比较 | candies |
|---|------------|------|---------|
| 2 | 2 | - | 1 |
| 1 | 2 | 2>2=否 | 2 |
| 0 | 1 | 1>2=否 | 1 |

candies = [1, 2, 1]

结果: `sum([1, 2, 1]) = 4`（注意：中间小孩评分为 2，左右都是 2，不比邻居高，糖果可以相等）

## 💻 解法

> 两次遍历贪心：左到右处理递增，右到左处理递减，取 max 保证两边约束。

```python
from typing import List


class Solution:
    def candy(self, ratings: List[int]) -> int:
        n = len(ratings)
        if n == 0:
            return 0

        # 初始化每个孩子至少 1 颗糖果
        candies = [1] * n

        # 第一次遍历：从左到右，保证右边评分高则糖果数更多
        for i in range(1, n):
            if ratings[i] > ratings[i - 1]:
                candies[i] = candies[i - 1] + 1

        # 第二次遍历：从右到左，保证左边评分高则糖果数更多
        for i in range(n - 2, -1, -1):
            if ratings[i] > ratings[i + 1]:
                candies[i] = max(candies[i], candies[i + 1] + 1)

        # 累加所有糖果数
        return sum(candies)
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 两次遍历数组 |
| 空间 | O(n) | 糖果数组占用 O(n) |

## ⚠ 易错点

- [ ] 第二遍遍历时条件是从**右到左**检查 `ratings[i] > ratings[i+1]`（i 比右边高），不是检查 i 比左边高（已经在第一遍做了）
- [ ] 第二遍要用 `max` 而不是直接赋值——因为第一遍的结果可能已经更大，直接赋值会覆盖掉。例如 `[1, 3, 2]`：第一遍得到 [1,2,1]，第二遍如果不取 max，i=1 时 `ratings[1]=3 > ratings[2]=2`，赋值 `candies[1]=candies[2]+1=2`，但应该取 max(2,2)=2（这个例子刚好相等，但若 ratings 是 [1,3,4,3,2,1] 就能看出区别）
- [ ] 相邻评分**相等**时没有约束——评分相同的小朋友不需要糖果数相同，可以给更少的。如 `[2,2]` 答案是 `[1,1]` 而非 `[2,2]`
- [ ] 也可以压缩到 O(1) 空间，用"上坡下坡"计数法，但两次遍历的实现更通用易写

## 🎤 面试怎么说

```text
这道题的核心是把"相邻约束"拆成两个独立方向：左边和右边。

先用一次从左到右的遍历，保证每个小孩如果评分比左边高，糖果就比左边多。
再用一次从右到左的遍历，保证每个小孩如果评分比右边高，糖果就比右边多。
每个位置取两次遍历的最大值，因为需要同时满足两边约束。

最后求和就是最少糖果总数。

时间 O(n)，空间 O(n)。还有一种 O(1) 空间的"上坡下坡"方法，
但两次遍历更好理解和书写。
```
