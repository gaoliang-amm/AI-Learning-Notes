# 12. 整数转罗马数字

## 🎯 人话翻译

> 给定一个整数（1-3999），转成罗马数字字符串。

## 💡 核心思想

> **贪心：从大到小列好所有面值（包括 4 和 9 的特殊组合），每次用最大可能的面值去减。**
>
> 罗马数字的核心规则是"大数在前，逐级递减"。把 13 个面值（1000、900、500、400、100、90、50、40、10、9、5、4、1）从大到小排列好，每次从数字里减去尽可能多的当前面值，拼接对应的罗马字符。
>
> 为什么列出 900、400 等？因为罗马数字用减法表示（CM=900，CD=400），预先列入可以避免单独处理特殊规则。

用到：[[贪心]]

## 📝 解题步骤

1. **建立面值映射** — 从大到小列出 13 组 (数值, 罗马字符) 对
2. **遍历映射** — 对于每个面值，用 while 循环尽可能多地减去当前面值
3. **拼接字符** — 每减一次，结果字符串追加对应的罗马字符
4. **提前退出** — num 归零时 break，节省遍历

## 💻 解法

> 贪心：从大到小匹配，每次尽可能多用大面值。

```python
class Solution:
    def intToRoman(self, num: int) -> str:
        # 从大到小排列，包含所有减法情况（如 900=CM, 400=CD 等）
        val_sym = [
            (1000, 'M'),
            (900, 'CM'),
            (500, 'D'),
            (400, 'CD'),
            (100, 'C'),
            (90, 'XC'),
            (50, 'L'),
            (40, 'XL'),
            (10, 'X'),
            (9, 'IX'),
            (5, 'V'),
            (4, 'IV'),
            (1, 'I')
        ]
        res = []

        # 贪心：每次用尽可能大的数值去匹配
        for val, sym in val_sym:
            while num >= val:
                res.append(sym)
                num -= val
            if num == 0:
                break

        return ''.join(res)
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(1) | 输入范围固定（1-3999），循环次数有上限（最多 3999/1 = 3999 次，实际上中途会提前退出） |
| 空间 | O(1) | 结果字符串长度与输入成正比，但输入范围有限 |

## ⚠ 易错点

- [ ] 面值顺序必须是**从大到小**，如果从小到大则会出错（如 4 会变成 IIII 而不是 IV）
- [ ] 必须包含 900(CM)、400(CD)、90(XC)、40(XL)、9(IX)、4(IV) 这六个减法组合，否则无法正确处理 4 和 9 的情况
- [ ] while 循环条件不要写成 `if`——因为同一个面值可能使用多次（如 3000 = MMM，需要用 3 次 M）
- [ ] 题目保证输入在 1-3999 之间，不需要处理 0 或负数
