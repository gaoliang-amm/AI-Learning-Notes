# 134. 加油站

## 🎯 人话翻译

> 一圈加油站，每站能加油 gas[i]，到下一站耗油 cost[i]。问从哪个站出发能绕一圈回来，找不到返回 -1。

## 💡 核心思想

> **贪心：总油量够就一定有解。当前累积剩余油变负，说明这一段都不可能作为起点，直接从下一站重新开始。**
>
> 两个关键结论：
> 1. 若 `sum(gas) - sum(cost) >= 0`，就一定有解（全局总油量 >= 总消耗）；否则无解返回 -1
> 2. 遍历过程中，若从 `start` 出发走到某站时 `cur_tank < 0`，说明 `start` 到当前位置之间的任何一站作为起点都无法到达当前位置。因此直接设 `start = i + 1`，重置 `cur_tank = 0`

用到：[[贪心]] [[前缀和]]

## 📝 解题步骤

1. **初始化** — `total_tank = 0`（全局总油量 - 总消耗），`cur_tank = 0`（当前起点开始累积的剩余油），`start = 0`（候选起点）
2. **遍历每站** — 计算 `diff = gas[i] - cost[i]`：
   - `total_tank += diff`：累加全局净油量
   - `cur_tank += diff`：累加当前段的净油量
   - 若 `cur_tank < 0`：`start = i + 1`，`cur_tank = 0`
3. **判断** — 若 `total_tank >= 0` 返回 `start`，否则返回 -1

## 🔢 手动走一遍

输入: `gas = [1, 2, 3, 4, 5], cost = [3, 4, 5, 1, 2]`

| i | gas[i] | cost[i] | diff | total_tank | cur_tank | cur_tank < 0? | start |
|---|--------|---------|------|------------|----------|---------------|-------|
| 0 | 1 | 3 | -2 | -2 | -2 | 是 | 1 |
| 1 | 2 | 4 | -2 | -4 | -2 | 是 | 2 |
| 2 | 3 | 5 | -2 | -6 | -2 | 是 | 3 |
| 3 | 4 | 1 | +3 | -3 | 3 | 否 | 3 |
| 4 | 5 | 2 | +3 | 0 | 6 | 否 | 3 |

最终 total_tank = 0 >= 0，返回 start = 3。

结果: `3`（从站 3 出发：加油 4，耗油 1 到站 4，加油 5 耗油 2 到站 0...可以绕一圈）

## 💻 解法

> 贪心：一旦当前段油不够就放弃，从下一站重新开始。

```python
from typing import List


class Solution:
    def canCompleteCircuit(self, gas: List[int], cost: List[int]) -> int:
        total_tank = 0  # 全局总盈亏
        cur_tank = 0  # 当前起点累计剩余油
        start = 0  # 答案起点

        for i in range(len(gas)):
            diff = gas[i] - cost[i]
            total_tank += diff
            cur_tank += diff

            # 油不够开不到下一站：起点作废，更新为下一个站点
            if cur_tank < 0:
                start = i + 1
                cur_tank = 0

        # 总油量不够，无解
        return start if total_tank >= 0 else -1
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 遍历一次数组 |
| 空间 | O(1) | 只用了三个变量 |

## ⚠ 易错点

- [ ] 必须先判断 `total_tank >= 0` 再返回 start——如果全局油量不够，即使 local 贪心找到了起点，也绕不了一圈
- [ ] 当 `cur_tank < 0` 时，start 跳到 `i+1` 而不是 `i`——因为站 i 已经无法到达 i+1，自然也不能作为起点
- [ ] 不需要循环检查或两遍遍历——贪心保证一次遍历即可。虽然"绕圈"暗示环形，但解题不需要回到起点再往前
- [ ] 贪心原理的正确性：假设从 start 出发到 i 时 cur_tank < 0，则对任意 k 在 `[start, i]` 之间，从 k 出发到 i 的剩余油为 `sum(diff[k..i]) = sum(diff[start..i]) - sum(diff[start..k-1])`，因为从 start 到 k-1 时 cur_tank >= 0，所以 `sum(diff[k..i]) <= sum(diff[start..i]) = cur_tank < 0`，k 也不能作为起点

## 🎤 面试怎么说

```text
这道题用贪心，一次遍历就能解决。

两个核心结论：
1. 如果所有加油量之和 >= 所有耗油量之和，则一定有解，否则返回 -1
2. 遍历过程中维护当前起点出发的剩余油量 cur_tank。一旦 cur_tank < 0（说明从当前起点到不了下一站），就放弃当前起点，把下一站设为新起点，cur_tank 重置为 0。

为什么可以跳过中间的所有站？因为如果从 start 能开到 i-1 但开不到 i，
那么 start 到 i 之间的任何一站都无法开到 i（它们从 start 出发时油量优势只会更小）。

最后如果全局总油量 >= 0，返回记录的最后起点即可。
时间 O(n)，空间 O(1)。
```
