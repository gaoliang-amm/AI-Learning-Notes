# 189. 轮转数组

## 🎯 人话翻译

> 数组向右循环移位 k 步，原地修改。

## 💡 核心思想

> **三次反转法：先反转整个数组，再反转前 k 个，最后反转剩余部分。**
>
> 向右轮转 k 步等价于：把后 k 个元素移到前面。
>
> 三次反转的操作逻辑：
> 1. `reverse(0, n-1)`：把"后 k 个"翻到了前面（但顺序反了），"前 n-k 个"翻到了后面（也反了）
> 2. `reverse(0, k-1)`：把前面的 k 个恢复正序
> 3. `reverse(k, n-1)`：把后面的 n-k 个恢复正序

用到：[[双指针]] [[原地算法]]

## 📝 解题步骤

1. **处理 k** — `k %= n`（k 可能大于 n，取余得到实际需要轮转的步数），若 k == 0 直接返回
2. **反转整个数组** — `reverse(0, n-1)`
3. **反转前 k 个** — `reverse(0, k-1)`
4. **反转剩余** — `reverse(k, n-1)`

## 🔢 手动走一遍

输入: `nums = [1, 2, 3, 4, 5, 6, 7], k = 3`

| 步骤 | 操作 | 数组状态 |
|------|------|----------|
| 初始 | - | [1, 2, 3, 4, 5, 6, 7] |
| 1 | reverse(0, 6) 反转全部 | [7, 6, 5, 4, 3, 2, 1] |
| 2 | reverse(0, 2) 反转前 3 个 | [5, 6, 7, 4, 3, 2, 1] |
| 3 | reverse(3, 6) 反转剩余 4 个 | [5, 6, 7, 1, 2, 3, 4] |

验证：原始后 3 个元素 [5,6,7] 现在在前面，原始前 4 个 [1,2,3,4] 现在在后面。正确。

## 💻 解法

> 三次反转：整体反转 + 前 k 反转 + 后 n-k 反转。

```python
from typing import List


class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        """
        将数组 nums 向右旋转 k 个位置（原地修改）。
        使用三次反转法，空间复杂度 O(1)。
        """
        n = len(nums)
        if n == 0:
            return
        k %= n  # 处理 k 大于 n 的情况
        if k == 0:
            return

        # 辅助函数：反转数组区间 [left, right]
        def reverse(left: int, right: int) -> None:
            while left < right:
                nums[left], nums[right] = nums[right], nums[left]
                left += 1
                right -= 1

        # 1. 反转整个数组
        reverse(0, n - 1)
        # 2. 反转前 k 个元素
        reverse(0, k - 1)
        # 3. 反转剩余元素
        reverse(k, n - 1)
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 每个元素被反转两次（最多），每次反转 O(1) |
| 空间 | O(1) | 原地反转，只用了几个变量 |

## ⚠ 易错点

- [ ] **k 要取模：`k %= n`**。如果 k > n，不取模会越界（比如 n=5, k=7，实际等效于 k=2）
- [ ] 反转的区间要正确：第二步是 `reverse(0, k-1)`，第三步是 `reverse(k, n-1)`。写错区间会得到错误结果
- [ ] 另一种方法：新建数组 `new_arr[(i+k)%n] = nums[i]` 然后拷贝回去，空间 O(n)。三次反转能优化到 O(1)
- [ ] 也可以利用 Python 切片：`nums[:] = nums[-k:] + nums[:-k]`，但切片会创建新列表，不是真正的 O(1) 空间

## 🎤 面试怎么说

```text
这道题的经典解法是三次反转，空间 O(1)。

向右轮转 k 步，可以理解为把后 k 个元素移到前面。

三次反转的操作是：
1. 先反转整个数组——后 k 个到了前面（但顺序反了）
2. 再反转前 k 个——恢复它们的正序
3. 最后反转剩余部分——恢复它们的正序

先对 k 取模（k %= n），然后执行三次反转。
时间复杂度 O(n)，空间 O(1)。

还有一个用额外空间的做法（开一个新数组下标映射），但三次反转更优雅。
```
