# 27. 移除元素

## 🎯 人话翻译

> 给你一个数组和一个值 val，原地移除所有等于 val 的元素，返回剩余元素个数。

## 💡 核心思想

> **双指针，快慢指针。快指针扫描每个元素，慢指针指向"下一个有效元素应该放的位置"。**
>
> 快指针遇到不等于 val 的元素时，将其复制到慢指针位置，慢指针右移。等于 val 的元素直接跳过。
>
> 本质是：用快指针"过滤"，用慢指针"原地重建有效部分"。

用到：[[双指针]] [[原地算法]]

## 📝 解题步骤

1. **初始化慢指针** — `slow = 0`，指向下一个有效元素应该放的位置
2. **快指针遍历** — `fast` 从 0 到 n-1，扫描每个元素
3. **判断有效性** — 若 `nums[fast] != val`，将 `nums[fast]` 复制到 `nums[slow]`，`slow++`
4. **跳过无效元素** — 若 `nums[fast] == val`，什么都不做，快指针继续
5. **返回有效长度** — `slow` 就是有效元素个数

## 🔢 手动走一遍

输入: `nums = [3, 2, 2, 3], val = 3`

| fast | nums[fast] | 是否等于 val | 操作 | slow 操作前 | slow 操作后 | 数组状态 |
|------|------------|-------------|------|------------|------------|----------|
| 0    | 3          | 是          | 跳过 | 0          | 0          | [3, 2, 2, 3] |
| 1    | 2          | 否          | num[0]=2, slow++ | 0 | 1 | [**2**, 2, 2, 3] |
| 2    | 2          | 否          | num[1]=2, slow++ | 1 | 2 | [2, **2**, 2, 3] |
| 3    | 3          | 是          | 跳过 | 2          | 2          | [2, 2, 2, 3] |

结果: 返回 `slow = 2`，前 2 个元素为 `[2, 2]`

## 💻 解法

> 双指针：快指针扫描+过滤，慢指针写入有效元素。

```python
from typing import List


class Solution:
    def removeElement(self, nums: List[int], val: int) -> int:
        # 慢指针，初始为0（第一个有效元素的位置）
        slow = 0
        # 快指针遍历整个数组
        for fast in range(len(nums)):
            # 如果当前元素不等于val，说明是有效元素
            if nums[fast] != val:
                # 将有效元素放到慢指针位置
                nums[slow] = nums[fast]
                # 慢指针右移，准备存下一个有效元素
                slow += 1
        # 最终slow就是有效元素的数量，nums前slow个元素已更新为有效元素
        return slow
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 快指针遍历一遍数组，每个元素访问一次 |
| 空间 | O(1) | 只用了两个指针变量，原地修改 |

## ⚠ 易错点

- [ ] 返回值是 `slow`（计数），不是 `nums`，题目要求返回长度而非数组
- [ ] 不需要处理 `slow` 之后的元素——题目只关心前 k 个元素，后面是什么无所谓
- [ ] 遍历时不要一边删除一边遍历（Python 的 for 循环修改列表会导致索引错乱），用双指针赋值法更安全
- [ ] `slow` 只有在赋值后才 +1，初始为 0 时 `slow == 有效元素计数` 天然成立
- [ ] 当数组全等于 val 时，slow 始终为 0，返回 0 是正确的

这道题用快慢指针，是原地删除类问题的模板题。

快指针 fast 负责扫描整个数组，慢指针 slow 指向下一个有效元素该放的位置。
