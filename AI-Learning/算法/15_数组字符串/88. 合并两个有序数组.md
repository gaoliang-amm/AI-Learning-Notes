# 88. 合并两个有序数组

## 🎯 人话翻译

> 两个升序数组 nums1 和 nums2，把 nums2 合并到 nums1 里（nums1 后面有空位），保持升序。

## 💡 核心思想

> **从后往前填充，谁大放谁。**
>
> 如果从前往后填，需要把 nums1 的元素往后移，O(n^2)。从后往前填则完全不需要移动——nums1 的末尾是空的，直接覆盖即可。
>
> 三个指针：i 指向 nums1 有效部分的末尾，j 指向 nums2 的末尾，k 指向最终放置位置（nums1 的末尾）。

用到：[[双指针]] [[原地算法]]

## 📝 解题步骤

1. **初始化三个指针** — `i = m-1`（nums1 有效末尾），`j = n-1`（nums2 末尾），`k = m+n-1`（nums1 末尾空位）
2. **从后往前比较** — 比较 `nums1[i]` 和 `nums2[j]`，较大的放到 `nums1[k]`，对应指针左移
3. **处理 nums2 剩余** — 若 j >= 0（nums2 还有元素没放完），直接拷贝到 nums1 前面
4. **nums1 剩余无需处理** — 若 i >= 0（nums1 还有元素），它们自然在正确位置

## 🔢 手动走一遍

输入: `nums1 = [1, 2, 3, 0, 0, 0], m = 3, nums2 = [2, 5, 6], n = 3`

| 步骤 | k | i | j | nums1[i] | nums2[j] | 比较 | nums1[k] | 操作后 i,j,k | nums1 状态 |
|------|---|----|---|----------|----------|------|----------|-------------|-----------|
| 初始 | 5 | 2 | 2 | 3 | 6 | - | - | 2,2,5 | [1, 2, 3, 0, 0, 0] |
| 1    | 5 | 2 | 2 | 3 | 6 | 6>3 | 6 | 2,1,4 | [1, 2, 3, 0, 0, **6**] |
| 2    | 4 | 2 | 1 | 3 | 5 | 5>3 | 5 | 2,0,3 | [1, 2, 3, 0, **5**, 6] |
| 3    | 3 | 2 | 0 | 3 | 2 | 3>2 | 3 | 1,0,2 | [1, 2, 3, **3**, 5, 6] |
| 4    | 2 | 1 | 0 | 2 | 2 | 2>=2 | 2 | 0,0,1 | [1, 2, **2**, 3, 5, 6] |
| 5    | 1 | 0 | 0 | 1 | 2 | 2>1 | 2 | 0,-1,0 | [1, **2**, 2, 3, 5, 6] |

j=-1 循环结束，nums2 全部合并完毕。

结果: `[1, 2, 2, 3, 5, 6]`

## 💻 解法

> 从后往前填充，三个指针，谁大谁往后放。

```python
from typing import List


class Solution:
    def merge(self, nums1: List[int], m: int, nums2: List[int], n: int) -> None:
        """
        合并两个有序数组到 nums1 中（原地修改）
        """
        # 定义三个指针：i指向nums1有效末尾，j指向nums2末尾，k指向nums1最终末尾
        i, j, k = m - 1, n - 1, m + n - 1

        # 从后往前遍历，比较两个数组的元素，将较大的放到nums1[k]
        while i >= 0 and j >= 0:
            if nums1[i] > nums2[j]:
                nums1[k] = nums1[i]
                i -= 1
            else:
                nums1[k] = nums2[j]
                j -= 1
            k -= 1

        # 处理nums2剩余的元素（nums1已遍历完但nums2还有的情况）
        # nums1剩余不用管，因为本来就在正确位置
        while j >= 0:
            nums1[k] = nums2[j]
            j -= 1
            k -= 1
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(m+n) | 每个元素最多被访问一次 |
| 空间 | O(1) | 原地修改，只用了三个指针变量 |

## ⚠ 易错点

- [ ] **从前往后**做会很难——每次插入需要移动后面所有元素，O(n^2)。必须从后往前
- [ ] nums1 剩余元素不需要处理——它们已经在 nums1 的前面，天然在正确位置上。只有 nums2 剩余时需要拷贝
- [ ] `k` 一定要从 `m+n-1` 开始，不是 `n-1`。nums1 的总长度是 `m+n`
- [ ] 当 nums1 为空（m=0）时要能正确处理，此时第一个 while 直接跳过，第二个 while 把 nums2 全部拷贝过去
- [ ] `nums1[i] > nums2[j]` 中的处理：不要把 `> 换成 >=` 导致结果顺序不稳定（虽然不影响正确性）

## 🎤 面试怎么说

```text
这道题的关键是反问自己的直觉——通常合并两个有序数组会从前往后，
但 nums1 后面有空位，从前往后会导致元素被覆盖或者需要频繁移动。

正确的做法是从后往前填充：用三个指针 i、j、k，
分别指向 nums1 有效部分末尾、nums2 末尾、nums1 最终末尾。

每次比较 nums1[i] 和 nums2[j]，把较大的放到 nums1[k]，对应指针左移。
最后如果 nums2 还有剩余，直接拷贝到 nums1 前面（nums1 剩余就不用管了）。

时间 O(m+n)，空间 O(1)。这是一个经典的逆向思维题。
```
