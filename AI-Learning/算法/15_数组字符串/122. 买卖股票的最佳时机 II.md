# 122. 买卖股票的最佳时机 II

## 🎯 人话翻译

> 一支股票，可以买卖多次（但手里最多同时持有一股），问最大总利润。

## 💡 核心思想

> **贪心：只要今天比昨天价格高，就把差价收入囊中。**
>
> 因为可以多次交易，且没有手续费，任何上涨区间我们都可以吃满。等价于：累加所有正差值（`prices[i] - prices[i-1] > 0` 的部分）。
>
> 为什么这样做是对的？假设价格序列是 [1, 3, 5]，方案 A：第 1 天买第 3 天卖，利润 4。方案 B：第 1 天买第 2 天卖（+2），第 2 天买第 3 天卖（+2），总利润还是 4。任何单次交易都可以拆成多个单日交易的和。

用到：[[贪心]]

## 📝 解题步骤

1. **初始化** — `profit = 0`
2. **遍历比较** — 从第 1 天开始（i 从 1 到 n-1），比较当天和前一天价格
3. **累加涨幅** — 若 `prices[i] > prices[i-1]`，`profit += prices[i] - prices[i-1]`
4. **返回总利润**

## 🔢 手动走一遍

输入: `prices = [7, 1, 5, 3, 6, 4]`

| i | prices[i] | prices[i-1] | prices[i] > prices[i-1]? | 利润 | 累计 profit |
|---|-----------|------------|-------------------------|------|-------------|
| 1 | 1 | 7 | 否（7 > 1） | 0 | 0 |
| 2 | 5 | 1 | 是 | +4 | 4 |
| 3 | 3 | 5 | 否（5 > 3） | 0 | 4 |
| 4 | 6 | 3 | 是 | +3 | 7 |
| 5 | 4 | 6 | 否（6 > 4） | 0 | 7 |

结果: `7`（交易：1买5卖赚4，3买6卖赚3，总利润7）

## 💻 解法

> 贪心：只要明天比今天贵，就把差价算进利润。

```python
from typing import List


class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        profit = 0
        for i in range(1, len(prices)):
            if prices[i] > prices[i - 1]:
                profit += prices[i] - prices[i - 1]
        return profit
```

## 🚀 解法二（更 Pythonic）

```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        return sum(max(prices[i] - prices[i - 1], 0) for i in range(1, len(prices)))
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 遍历数组一次 |
| 空间 | O(1) | 只用了 profit 变量 |

## ⚠ 易错点

- [ ] 和 LeetCode 121 的区别：121 只能买卖一次（找最低买入点+最高卖点），122 可以多次（累加所有涨幅）。混淆这两题是面试中最常见的错误
- [ ] 不需要记录"买入价格"——因为是在微观层面把上涨拆成了每天的交易，"今天买明天卖"等价于"低点买高点卖"
- [ ] 如果价格一直跌（如 [7, 6, 5, 4]），profit 始终为 0，正确——不交易
- [ ] 更复杂的变种：122 是"不限交易次数"，123 是"最多 2 次"，188 是"最多 k 次"，309 加了冷却期，714 加了手续费。但 122 是最简单的贪心版本

## 🎤 面试怎么说

```text
这道题用贪心算法，一句话总结：只要今天比昨天贵，就把差价算进利润。

为什么贪心是正确的？因为不限交易次数，且没有手续费，
上涨可以拆成每天的交易——从最低点买到最高点卖出的利润，
等于这期间每天涨价的利润之和。

所以只需要遍历一次，累加所有正差价即可。
时间 O(n)，空间 O(1)。

如果面试官问"怎么保证不持有多股"，解释：因为交易在当天可以同时完成
（卖了立刻买），所以每天的操作不冲突。
```
