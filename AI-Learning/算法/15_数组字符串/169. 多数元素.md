# 169. 多数元素

## 🎯 人话翻译

> 数组里有一个数出现次数超过一半（大于 n/2），找出这个数。

## 💡 核心思想

> **Boyer-Moore 摩尔投票算法。两派士兵打架，不同派别的同归于尽，最后剩下的一定是人多的那派。**
>
> 维护一个候选数 `candidate` 和计数器 `count`。遍历数组：
> - `count == 0` 时，选当前数当候选
> - 当前数等于候选，count++
> - 当前数不等于候选，count--（一对抵消）
>
> 因为多数元素的数量超过一半，它永远不会被完全抵消，最后 `candidate` 就是多数元素。

用到：[[摩尔投票]] [[贪心]]

## 📝 解题步骤

1. **初始化** — `candidate = None`，`count = 0`
2. **遍历数组** — 对每个 num：
   - 若 `count == 0`，设 `candidate = num`，`count = 1`
   - 若 `num == candidate`，`count++`
   - 若 `num != candidate`，`count--`
3. **验证（可选）** — 若题目**不保证**存在多数元素，需第二次遍历验证候选出现次数 > n/2
4. **返回候选** — 题目保证一定存在多数元素时，直接返回 candidate

## 🔢 手动走一遍

输入: `nums = [2, 2, 1, 1, 1, 2, 2]`, n=7

| i | nums[i] | count 操作前 | candidate 操作前 | 操作 | count 操作后 | candidate 操作后 |
|---|---------|-------------|-----------------|------|-------------|-----------------|
| 0 | 2       | 0           | None            | count=0, 选 2 | 1 | 2 |
| 1 | 2       | 1           | 2               | num==candidate, count++ | 2 | 2 |
| 2 | 1       | 2           | 2               | num!=candidate, count-- | 1 | 2 |
| 3 | 1       | 1           | 2               | num!=candidate, count-- | 0 | 2 |
| 4 | 1       | 0           | 2               | count=0, 选 1 | 1 | 1 |
| 5 | 2       | 1           | 1               | num!=candidate, count-- | 0 | 1 |
| 6 | 2       | 0           | 1               | count=0, 选 2 | 1 | 2 |

结果: `candidate = 2`（验证：2 出现 4 次 > 7/2 = 3.5）

## 💻 解法

> Boyer-Moore 摩尔投票算法：不同派别抵消，最后剩下多数派。

```python
from typing import List


class Solution:
    def majorityElement(self, nums: List[int]) -> int:
        candidate = None  # 候选多数元素
        count = 0  # 候选元素的"净票数"

        # 第一轮：找出可能的候选元素
        for num in nums:
            if count == 0:
                # 当前没有候选，将当前元素设为候选
                candidate = num
                count = 1
            elif num == candidate:
                # 当前元素等于候选，票数 +1
                count += 1
            else:
                # 当前元素不等于候选，票数 -1（相当于一对抵消）
                count -= 1

        # 第二轮：验证候选是否真的是多数元素（题目保证一定存在，此步可省略）
        # 如果题目不保证存在多数元素，则需要此验证
        if nums.count(candidate) > len(nums) // 2:
            return candidate
        else:
            # 根据题目约定，这里不会执行到
            return -1
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 遍历一次数组（+ 可选的一次验证遍历） |
| 空间 | O(1) | 只用了两个变量 |

## ⚠ 易错点

- [ ] 摩尔投票的前提是**多数元素一定存在**（出现次数 > n/2）。如果不保证存在，最终 candidate 可能不是真正的多数元素，需要第二次遍历验证
- [ ] `count == 0` 时重置 candidate 的时机要正确——必须在判断 `num == candidate` 之前，否则逻辑错误
- [ ] `count` 不是多数元素的总次数，而是"净优势票数"。比如数组 [1,1,1,2,2,2,1]，count 会在 1 和 0 之间摇摆
- [ ] 另一个简单思路是排序后取中间元素 `nums[n//2]`（因为多数元素一定占据中点），时间 O(nlogn)，空间 O(1)。但摩尔投票更优
