# 13. 罗马数字转整数

## 🎯 人话翻译

> 给定一个罗马数字字符串，转回整数。

## 💡 核心思想

> **从左到右遍历，遇到"小值在大值左边"的情况就减去小值，否则加上。**
>
> 罗马数字的减法规则：当一个小值符号出现在一个大值符号左边时（如 IV 中 I 在 V 左边），I 应该被减掉而不是加上。
>
> 判断方法：遍历到当前字符时，如果它的值小于下一个字符的值，说明这是减法情况，减去当前值；否则加上当前值。

用到：[[贪心]] [[字符串]]

## 📝 解题步骤

1. **建立字符映射** — 7 个罗马字符到整数的映射表
2. **遍历字符串** — 对每个位置 i，取当前字符值 `cur` 和下一个字符值 `next_val`
3. **判断加减** — 若 `cur < next_val`，减去 cur（减法规则）；否则加上 cur
4. **最后一位** — 没有下一个字符，直接加
5. **返回总和**

## 🔢 手动走一遍

输入: `s = "MCMXCIV"`

| i | 当前字符 | cur | 下一个字符 | next_val | cur < next? | 操作 | total |
|---|----------|-----|-----------|----------|-------------|------|-------|
| 0 | M | 1000 | C | 100 | 否 | +1000 | 1000 |
| 1 | C | 100 | M | 1000 | 是 | -100 | 900 |
| 2 | M | 1000 | X | 10 | 否 | +1000 | 1900 |
| 3 | X | 10 | C | 100 | 是 | -10 | 1890 |
| 4 | C | 100 | I | 1 | 否 | +100 | 1990 |
| 5 | I | 1 | V | 5 | 是 | -1 | 1989 |
| 6 | V | 5 | (无) | - | - | +5 | 1994 |

结果: `1994`

## 💻 解法

> 从左到右遍历，若当前值小于后一位则减，否则加。

```python
class Solution:
    def romanToInt(self, s: str) -> int:
        # 建立罗马字符与对应整数的映射表
        roman_map = {
            'I': 1,
            'V': 5,
            'X': 10,
            'L': 50,
            'C': 100,
            'D': 500,
            'M': 1000
        }
        total = 0
        n = len(s)

        # 遍历每个字符
        for i in range(n):
            cur = roman_map[s[i]]
            # 如果不是最后一位，且当前值 < 下一位值，则减去当前值
            if i < n - 1 and cur < roman_map[s[i + 1]]:
                total -= cur
            # 否则加上当前值
            else:
                total += cur

        return total
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 遍历字符串一次，每个字符 O(1) 处理 |
| 空间 | O(1) | 映射表大小固定（7 个条目），只用了几个变量 |

## ⚠ 易错点

- [ ] 判断 `i < n - 1` 防止越界——最后一位没有下一个字符，直接加
- [ ] 必须是严格小于 `cur < next_val` 时才减——等于的情况（如 II、XX）都是加
- [ ] 减法情况的典型模式：I 在 V/X 前，X 在 L/C 前，C 在 D/M 前。这些映射表已经能自然处理
- [ ] 不要从**右往左**做另一种思路，那样虽然也能做，但判断逻辑不同（右往左：若当前 >= 上一个则加，否则减）
