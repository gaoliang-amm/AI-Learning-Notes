# 45. 跳跃游戏 II

## 🎯 人话翻译

> 每个位置上的数字表示从该位置最多能跳多远。从索引 0 出发，问最少跳几次到最后一个位置。

## 💡 核心思想

> **贪心（BFS 式的贪心）：维护"当前步数能到达的最远边界 end"和"全局最远 max_pos"。**
>
> 遍历数组时，不断更新能到达的最远位置 `max_pos = max(max_pos, i + nums[i])`。当遍历到 `i == end`（当前步数的边界）时，说明必须再跳一步才能继续前进，`steps++`，`end = max_pos`。
>
> 本质是模拟"层次遍历"：每跳一次，检查这一跳能覆盖的范围，并计算下一跳能到达的最远边界。当边界覆盖到末尾时，当前步数就是答案。

用到：[[贪心]] [[BFS]]

## 📝 解题步骤

1. **边界处理** — n == 1 时返回 0，不需要跳
2. **初始化** — `steps = 0`（跳跃次数），`max_pos = 0`（当前能到达的最远位置），`end = 0`（当前步数覆盖范围的边界）
3. **遍历** — 遍历 `i` 从 0 到 n-2（不遍历最后一个，因为到达末尾就结束）：
   - 更新 `max_pos = max(max_pos, i + nums[i])`
   - 若 `i == end`：`steps++`，`end = max_pos`
   - 若 `end >= n-1`：提前结束
4. **返回 steps**

## 🔢 手动走一遍

输入: `nums = [2, 3, 1, 1, 4]`

| i | nums[i] | i+nums[i] | max_pos | i == end? | steps | end | 说明 |
|---|---------|-----------|---------|-----------|-------|-----|------|
| 0 | 2 | 2 | 2 | i(0)==end(0)? 是 | 1 | 2 | 第 1 跳，覆盖到 i=2 |
| 1 | 3 | 4 | 4 | i(1)!=end(2)? 否 | 1 | 2 | 更新最远到 4 |
| 2 | 1 | 3 | 4 | i(2)==end(2)? 是 | 2 | 4 | 第 2 跳，边界到 4>=n-1，结束 |

结果: `2`（跳法：0 -> 1 -> 4，2 步）

## 💻 解法

> 贪心 BFS：每步计算最远覆盖边界，到达边界时 step++。

```python
from typing import List


class Solution:
    def jump(self, nums: List[int]) -> int:
        n = len(nums)
        if n == 1:  # 只有一个元素，不需要跳跃
            return 0

        steps = 0  # 跳跃次数
        max_pos = 0  # 当前能到达的最远位置
        end = 0  # 当前步数下能到达的边界

        # 不遍历最后一个元素，避免在边界正好为最后一个位置时增加一次不必要的跳跃
        for i in range(n - 1):
            # 更新能到达的最远位置
            max_pos = max(max_pos, i + nums[i])
            # 如果到达了当前步数的边界，必须再跳一次
            if i == end:
                end = max_pos  # 更新边界
                steps += 1  # 跳跃次数加1
                # 如果已经可以到达末尾，提前退出
                if end >= n - 1:
                    break

        return steps
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 遍历一次数组 |
| 空间 | O(1) | 只用了几个变量 |

## ⚠ 易错点

- [ ] 遍历范围是 `range(n - 1)`，不包含最后一个元素——因为在 i == n-1 时如果 i == end，会多算一次跳跃（已经到末尾了不需要再跳）
- [ ] `end` 的更新要在 `steps++` 的时候一起做，顺序是：先更新 end，再 steps++（或反过来都行，但要保证逻辑正确）
- [ ] 如果 `end >= n-1` 提前退出可以少遍历一些元素，但不是必须的
- [ ] 和 LeetCode 55 的区别：55 是判断"能否到达"（只需维护 max_pos），45 是"最小步数"（需要维护 end 边界 + steps 计数）
- [ ] 贪心成立的前提：题目保证一定能到达末尾（所以不需要处理无法到达的情况）

```text
这道题是 LeetCode 55（能否到达）的升级版，需要求最小跳跃次数。

用贪心算法，核心是维护两个变量：
- max_pos：遍历过程中能到达的最远位置
- end：当前步数覆盖范围的右边界

遍历时，不断更新 max_pos。当遍历到 end（当前步数边界）时，
说明必须再跳一步，steps++，同时把 end 更新为 max_pos（下一跳的边界）。

这个过程中隐含了 BFS 的思想——每跳一步，就确定了这一步能覆盖的最远范围。

```
