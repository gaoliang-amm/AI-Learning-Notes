# 80. 删除有序数组中的重复项 II

## 🎯 人话翻译

> 有序数组，原地去重但每个元素最多保留两个，返回新长度。

## 💡 核心思想

> **双指针，比较的是 `nums[fast]` 与 `nums[slow-2]`。**
>
> 因为是升序数组，如果 `nums[fast] == nums[slow-2]`，说明 slow 前面已经有两个相同的元素了，当前元素是第三个重复项，应该被丢弃。
>
> 核心洞察：slow 指向"下一个可放置位置"，slow-2 就是"该位置往前两个"，如果 fast 的值等于 slow-2 的值，说明已经有两个相同值了，不能再放。

用到：[[双指针]] [[原地算法]]

## 📝 解题步骤

1. **边界处理** — 数组长度 <= 2 直接返回原长度，因为最多保留两个，无需删除
2. **初始化 slow** — `slow = 2`，前两个元素无论如何都会被保留
3. **fast 从 2 开始遍历** — 检查每个元素是否允许保留
4. **判断条件** — 若 `nums[fast] != nums[slow-2]`，将 `nums[fast]` 复制到 `nums[slow]`，`slow++`
5. **返回 slow** — slow 就是新数组的长度

## 🔢 手动走一遍

输入: `nums = [1, 1, 1, 2, 2, 3]`

| fast | nums[fast] | slow | nums[slow-2] | 比较 | 操作 | 数组状态 |
|------|------------|------|-------------|------|------|----------|
| -    | -          | 2    | -           | -    | 初始 | [1, 1, 1, 2, 2, 3] |
| 2    | 1          | 2    | nums[0]=1   | 相等 | 跳过 | [1, 1, 1, 2, 2, 3] |
| 3    | 2          | 2    | nums[0]=1   | 不等 | nums[2]=2, slow=3 | [1, 1, **2**, 2, 2, 3] |
| 4    | 2          | 3    | nums[1]=1   | 不等 | nums[3]=2, slow=4 | [1, 1, 2, **2**, 2, 3] |
| 5    | 3          | 4    | nums[2]=2   | 不等 | nums[4]=3, slow=5 | [1, 1, 2, 2, **3**, 3] |

结果: 返回 `slow = 5`，前 5 个元素为 `[1, 1, 2, 2, 3]`

## 💻 解法

> 双指针，比较 `nums[fast]` 与 `nums[slow-2]`，核心在于理解为什么是和 slow-2 比较。

```python
from typing import List


class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        # 特殊情况：数组长度小于等于2，无需删除，直接返回原长度
        if len(nums) <= 2:
            return len(nums)

        # 慢指针从索引2开始，因为前两个元素无论如何都保留
        slow = 2
        # 快指针从索引2开始遍历
        for fast in range(2, len(nums)):
            # 如果当前元素不等于它前面第2个元素（即slow-2位置的元素）
            # 说明当前元素可以保留（使得相同元素个数不超过2）
            if nums[fast] != nums[slow - 2]:
                # 将当前元素放到慢指针位置
                nums[slow] = nums[fast]
                # 慢指针前进
                slow += 1
            # 如果相等，说明已经有2个相同元素了，当前元素跳过，快指针继续
        # 慢指针最终指向的就是新数组的长度
        return slow
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | fast 遍历一遍数组 |
| 空间 | O(1) | 原地修改，只用了几个变量 |

## ⚠ 易错点

- [ ] 和 LeetCode 26 的区别：26 比较 `nums[slow]`，80 比较 `nums[slow-2]`。因为 26 只保留 1 个，检查"与上一个是否相同"；80 最多保留 2 个，检查"与上上个是否相同"
- [ ] slow 从 2 开始，若数组长度 <= 2 直接返回 n——前提是前两个元素保留的逻辑在长度大于 2 时才需要
- [ ] 注意 `nums[slow-2]` 的值可能已经被修改了——但因为 slow 是"已通过区域"的末尾，slow-2 始终代表"已通过区域中从后往前数第 2 个"，其值与原始值一致（slow 不会覆盖未检查的元素）
- [ ] 最后返回 `slow`（不用 +1），因为 slow 已经代表长度了（初始化 2 = 前两个元素的长度，每次赋值后 slow 递增，始终等于有效长度）

```text
这道题是 LeetCode 26 的升级版，26 要求每个元素最多一次，这里是最多两次。

核心思路还是双指针：
- slow 指向下一个可放置的位置
- fast 扫描整个数组

```
