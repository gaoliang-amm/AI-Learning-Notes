# 380. O(1) 时间插入、删除和获取随机元素

## 🎯 人话翻译

> 设计一个数据结构，支持 insert、remove、getRandom 三个操作，全部 O(1) 平均时间。

## 💡 核心思想

> **数组 + 哈希表。数组用于 O(1) 随机访问，哈希表用于 O(1) 查找下标。删除时把待删元素与数组末尾元素交换再弹出，保证 O(1) 删除。**
>
> 问题分解：
> - O(1) 插入：HashMap + 数组末尾追加
> - O(1) 删除：如果直接删数组中间的元素，后续元素需要前移，O(n)。解决方案：把末尾元素移到待删除位置（覆盖），然后删除末尾。HashMap 同步更新末尾元素的下标
> - O(1) 随机获取：数组支持通过下标 O(1) 随机访问，Python 的 `random.choice()` 实现

用到：[[哈希表]] [[数组]] [[设计]]

## 📝 解题步骤

**insert(val)：**
1. 若 val 已在 `val_to_idx` 中，返回 False
2. 在数组末尾添加 val，在哈希表中记录 `val -> len(nums)-1`
3. 返回 True

**remove(val)：**
1. 若 val 不在 `val_to_idx` 中，返回 False
2. 获取 val 的下标 idx。获取数组末尾元素 last_val
3. 将 last_val 放到 idx 位置（覆盖 val）: `nums[idx] = last_val`，更新 `val_to_idx[last_val] = idx`
4. 删除数组末尾 `nums.pop()`，删除哈希表项 `del val_to_idx[val]`
5. 返回 True

**getRandom()：**
1. `return random.choice(nums)`

| 步骤 | nums | val_to_idx | 说明 |
|------|------|-----------|------|
| 初始 | [1, 2, 3] | {1:0, 2:1, 3:2} | - |

| pop() + del 1 | [3, 2] | {2:1, 3:0} | 删除末尾和键 1 |

## 💻 解法

> 数组存值（O(1) 随机访问），哈希表存值到下标（O(1) 查找），删除时交换到末尾再弹出。

```python
import random


class RandomizedSet:

    def __init__(self):
        # nums：动态数组，用于 O(1) 随机取值
        # val_to_idx：哈希字典，记录值到数组下标的映射
        self.nums = []
        self.val_to_idx = dict()

    def insert(self, val: int) -> bool:
        """插入元素，已存在返回 False，不存在插入返回 True"""
        if val in self.val_to_idx:
            return False
        # 记录下标，末尾添加
        self.val_to_idx[val] = len(self.nums)
        self.nums.append(val)
        return True

    def remove(self, val: int) -> bool:
        """删除元素，不存在返回 False，存在删除返回 True"""
        if val not in self.val_to_idx:
            return False

        # 核心：交换待删元素与数组末尾元素，保证 O(1) 删除
        idx = self.val_to_idx[val]  # 待删元素的下标
        last_val = self.nums[-1]  # 末尾元素的值

        # 用末尾元素覆盖待删位置
        self.nums[idx] = last_val
        self.val_to_idx[last_val] = idx

        # 删除末尾元素和待删元素的哈希表项
        self.nums.pop()
        del self.val_to_idx[val]
        return True

    def getRandom(self) -> int:
        """随机返回一个元素"""
        return random.choice(self.nums)
```

## ⏱ 复杂度

| 操作 | 时间 | 说明 |
|------|------|------|
| insert | O(1) 平均 | 数组末尾追加 + 字典插入，均摊 O(1) |
| remove | O(1) 平均 | 交换 + pop + del，均摊 O(1) |
| getRandom | O(1) | `random.choice` 基于数组下标随机访问 |

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 空间 | O(n) | 数组和哈希表各存 n 个元素 |

## ⚠ 易错点

- [ ] 删除时顺序很重要：先覆盖待删位置，再更新 last_val 的下标，再 pop，再删除哈希表项。如果先 pop 再访问 `nums[-1]` 就取不到正确的 last_val
- [ ] 如果待删除元素恰好是数组末尾元素（`idx == len(nums) - 1`），覆盖操作是自己覆盖自己，不影响结果。但顺序仍需一致
- [ ] `getRandom` 必须等概率——用 `random.choice` 或 `random.randint(0, len-1)` 都可以，但不能加任何过滤
- [ ] Python 的 `dict` 插入和查找平均 O(1)，`list.append` 和 `list.pop()` 也是 O(1)。整个设计依赖于这些底层操作的均摊复杂度

## 🎤 面试怎么说

```text
这道题的设计难点在于 getRandom 必须是 O(1) 且等概率。
支持 O(1) 随机访问的数据结构是数组（通过下标）。
但数组的查找和删除是 O(n)。

所以结合哈希表：用数组存值用于随机访问，用哈希表存值到下标的映射用于 O(1) 查找。

删除的 O(1) 是关键：不能直接从数组中间删除（会导致 O(n) 的元素移动），
而是把待删元素与数组末尾元素交换，然后 pop 末尾。
同时更新哈希表中末尾元素的新下标。

这样一来三个操作都是 O(1) 均摊时间。
```
