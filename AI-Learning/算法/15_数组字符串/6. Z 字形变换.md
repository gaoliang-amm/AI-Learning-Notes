# 6. Z 字形变换

## 🎯 人话翻译

> 把字符串按 Z 字形（从上到下、然后斜向上）排列成 numRows 行，再按顺序逐行读出来。

## 💡 核心思想

> **模拟法：用 numRows 个 StringBuilder（Python 中为字符串列表），维护当前行号和方向，逐个字符填入对应行。**
>
> 遍历每个字符：把它追加到当前行的末尾，然后行号根据方向上下移动。到达第 0 行或第 numRows-1 行时反转方向（向下变为斜向上，斜向上变为向下）。
>
> 最后把所有行的字符串拼接起来。

用到：[[字符串]] [[模拟]]

## 📝 解题步骤

1. **处理特殊情况** — `numRows == 1` 或 `numRows >= len(s)` 时，直接返回 s（只有一行或行数比字符还多）
2. **初始化 rows** — 创建一个长度为 `numRows` 的字符串列表，每行初始为空串
3. **维护方向** — `direction = 1`（向下），`current_row = 0`
4. **遍历字符** — 每个字符追加到 `rows[current_row]`，然后 `current_row += direction`
5. **转向** — 当 `current_row == 0` 时 `direction = 1`（改为向下），当 `current_row == numRows - 1` 时 `direction = -1`（改为向上）
6. **拼接结果** — `''.join(rows)`

## 🔢 手动走一遍

输入: `s = "PAYPALISHIRING", numRows = 3`

| i | 字符 | current_row 前 | direction 前 | 追加后 rows | current_row 后 | 方向变化? |
|---|------|---------------|-------------|------------|---------------|----------|
| 0 | P | 0 | 1 | ["P", "", ""] | 1 | 否 |
| 1 | A | 1 | 1 | ["P", "A", ""] | 2 | 否 |
| 2 | Y | 2 | 1 | ["P", "A", "Y"] | 1 | row=2=numRows-1, 方向改为 -1 |
| 3 | P | 1 | -1 | ["P", "AP", "Y"] | 0 | 否 |
| 4 | A | 0 | -1 | ["PA", "AP", "Y"] | 1 | row=0, 方向改为 1 |
| ... | ... | ... | ... | ... | ... | ... |

最终 rows:
```
["PAHN", "APLSIIG", "YIR"]
```

结果: `"PAHNAPLSIIGYIR"`

## 💻 解法

> 模拟行遍历：维护行号和方向，逐个字符放入对应行。

```python
class Solution:
    def convert(self, s: str, numRows: int) -> str:
        # 特殊情况：只有一行，或行数大于等于字符串长度，直接返回原串
        if numRows == 1 or numRows >= len(s):
            return s

        # 初始化每行的字符串容器
        rows = [''] * numRows
        cur_row = 0
        # 方向：1 表示向下，-1 表示向上
        direction = 1

        for c in s:
            # 把字符加入当前行
            rows[cur_row] += c
            # 移动行号
            cur_row += direction

            # 到达边界时反转方向
            if cur_row == 0:
                direction = 1  # 改为向下
            elif cur_row == numRows - 1:
                direction = -1  # 改为向上

        # 拼接所有行
        return ''.join(rows)
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 每个字符处理一次 |
| 空间 | O(n) | rows 列表存储所有字符 |

## ⚠ 易错点

- [ ] `numRows == 1` 是特殊情况，因为此时行号始终为 0，`cur_row == numRows - 1` 也成立（0 == 0），方向会反复翻转导致死循环或错误
- [ ] 方向切换的**两个**条件都要检查（cur_row == 0 和 cur_row == numRows-1），只检查一个会导致方向无法正确反转
- [ ] 也可以直接数学计算：第 0 行字符的下标是 `0, 2*numRows-2, 4*numRows-2, ...`，中间行的字符有两个，但模拟法更直观不易错
- [ ] 字符追加到字符串在 Python 中每次是 O(k) 复制，可以用 `[''] * numRows` 初始化后当成列表来用——每次 `rows[cur_row] += c` 会创建新字符串，但因为字符串平均长度 = n/numRows，整体还是 O(n) 的总时间
