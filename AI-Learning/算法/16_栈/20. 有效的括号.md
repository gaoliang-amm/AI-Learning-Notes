# 20. 有效的括号

## 🎯 人话翻译

> 给你一串括号（`()`、`{}`、`[]`），判断括号的开闭是否正确匹配，不能交叉嵌套。

## 💡 核心思想

> **后遇到的左括号必须先闭合**——这正是栈的"后进先出"特性。
>
> 遍历时遇到左括号就入栈，遇到右括号就弹出栈顶检查是否匹配。遍历结束后栈为空说明全部匹配。

用到：[[栈]]

## 📝 解题步骤

1. **长度预判** — 若字符串长度为奇数，直接返回 `False`（因为一个右括号必须配一个左括号）
2. **建立映射表** — `{')': '(', ']': '[', '}': '{'}`，右括号为 key，左括号为 value，方便 O(1) 查找
3. **遍历字符串** — 遇到左括号就压入栈；遇到右括号就弹出栈顶，若栈为空或弹出的左括号不匹配当前右括号，返回 `False`
4. **最终检查** — 遍历结束若栈为空则全匹配返回 `True`，否则有未闭合的左括号返回 `False`

## 🔢 手动走一遍

输入: `s = "([]){}"`

| 步骤 | 当前字符 | 操作       | 栈内容       | 说明               |
|------|----------|------------|--------------|--------------------|
| 1    | `(`      | push       | `['(']`      | 左括号入栈         |
| 2    | `[`      | push       | `['(', '[']` | 左括号入栈         |
| 3    | `]`      | pop 栈顶   | `['(']`      | `[` 匹配 `]`，弹出 |
| 4    | `)`      | pop 栈顶   | `[]`         | `(` 匹配 `)`，弹出 |
| 5    | `{`      | push       | `['{']`      | 左括号入栈         |
| 6    | `}`      | pop 栈顶   | `[]`         | `{` 匹配 `}`，弹出 |

遍历结束，栈为空 -> 有效，返回 `True`

## 💻 解法

> 一次遍历 + 栈，右括号匹配时弹栈检查。

```python
class Solution:
    def isValid(self, s: str) -> bool:
        """
        判断一个只包含括号的字符串是否有效。

        有效条件：
        1. 左括号必须用相同类型的右括号闭合。
        2. 左括号必须以正确的顺序闭合。
        3. 每个右括号都有一个对应的相同类型的左括号。

        参数:
            s: 输入的括号字符串

        返回:
            bool: 字符串是否有效

        时间复杂度: O(n)，n 为字符串长度，遍历一次即可。
        空间复杂度: O(n)，最坏情况下栈会存储所有左括号。
        """
        # 小优化：如果字符串长度为奇数，肯定无效，直接返回 False
        if len(s) % 2 == 1:
            return False

        # 1. 建立一个哈希表，key 是右括号，value 是与之匹配的左括号。
        #    这样做可以方便地检查括号是否匹配，且易于扩展。
        mapping = {
            ')': '(',
            ']': '[',
            '}': '{'
        }

        # 2. 初始化一个空栈，用于存放遇到的左括号。
        stack = []

        # 3. 遍历字符串中的每一个字符
        for char in s:
            # 情况一：遇到的是右括号
            if char in mapping:
                # 如果栈不为空，则弹出栈顶元素，否则给一个占位符 '#'
                # 这样即使栈为空，代码也不会因为 `pop()` 操作而报错，并将匹配失败的判断交给下一步。
                top_element = stack.pop() if stack else '#'

                # 检查弹出的栈顶元素是否与当前右括号匹配
                if mapping[char] != top_element:
                    return False
            # 情况二：遇到的是左括号
            else:
                # 将左括号推入栈中，等待匹配
                stack.append(char)

        # 4. 遍历结束，如果栈为空，说明所有括号都成功匹配，返回 True；否则返回 False
        return not stack
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 遍历字符串一次，每次入栈/出栈 O(1) |
| 空间 | O(n) | 最坏情况全为左括号，栈存储所有字符 |

## ⚠ 易错点

- [ ] 奇数长度直接返回 `False` 可提前剪枝，避免无效遍历
- [ ] 遇到右括号时 `pop()` 前必须检查栈是否为空，否则会报 `IndexError`
- [ ] 映射表 key 是右括号、value 是左括号更方便——遇到右括号才需要查表匹配，遇到左括号直接入栈即可
- [ ] 遍历结束后必须检查 `not stack`，若栈中还有残留左括号则无效（如 `"((("`）
- [ ] 不要用多个 if-else 分别判断三种括号，用映射表统一处理更易扩展

```text
这道题是栈的经典应用题。核心思路是利用栈的"后进先出"特性——
后遇到的左括号必须先与对应的右括号闭合。

具体做法：用一个哈希表存右括号到左括号的映射。遍历字符串，
遇到左括号就压栈，遇到右括号就检查栈顶是否匹配。如果栈为空
或者不匹配就返回 False。最后判断栈是否为空。

时间复杂度 O(n)，每个字符只入栈出栈一次；空间复杂度最坏 O(n)，
全为左括号时栈存满。

可以把这种思路推广到任何需要"成对消除"的场景。
```
