# 224. 基本计算器

## 🎯 人话翻译

> 给一个只有 `+`、`-`、`(`、`)` 和空格的字符串表达式，算出它的值。比如 `"(1+(4+5+2)-3)+(6+8)"` = 23。

## 💡 核心思想

> **遇到括号就把当前状态（结果和符号）入栈暂存，等括号结束再合并回来。**
>
> 核心是用 `res` 累积当前层的结果，`sign` 记录下一个数字正负。遇到 `(` 就把当前 `res` 和 `sign` 压栈并重置状态；遇到 `)` 就把括号内的结果与栈中保存的外部结果合并。

用到：[[栈]]

## 📝 解题步骤

1. **初始化** — `res = 0` 存放当前结果，`sign = 1` 表示当前符号（1 为正，-1 为负），`i = 0` 遍历字符串
2. **逐字符处理**：
   - 数字：解析完整多位数 `num`，令 `res += sign * num`
   - `+`：设 `sign = 1`
   - `-`：设 `sign = -1`
   - `(`：将当前 `res` 和 `sign` 入栈，重置 `res = 0, sign = 1`
   - `)`：括号内结果算完，弹出栈顶的 `sign` 和 `res`，合并为 `res = prev_res + prev_sign * res`
   - 空格：跳过
3. **返回 `res`**

## 🔢 手动走一遍

输入: `s = "1-(2+3)"`

| 步骤 | 字符   | 操作                          | res | sign | stack      | 说明                     |
|------|--------|-------------------------------|-----|------|------------|--------------------------|
| 初始 |        |                               | 0   | 1    | `[]`       |                          |
| 1    | `1`    | num=1, res += 1*1             | 1   | 1    | `[]`       |                          |
| 2    | `-`    | sign = -1                     | 1   | -1   | `[]`       |                          |
| 3    | `(`    | 入栈 res=1, sign=-1, 重置     | 0   | 1    | `[1, -1]`  | 保存外层状态             |
| 4    | `2`    | num=2, res += 1*2             | 2   | 1    | `[1, -1]`  |                          |
| 5    | `+`    | sign = 1                      | 2   | 1    | `[1, -1]`  |                          |
| 6    | `3`    | num=3, res += 1*3             | 5   | 1    | `[1, -1]`  |                          |
| 7    | `)`    | prev_sign=-1, prev_res=1      |     |      | `[]`       | res=1+(-1)*5 = 1-5 = -4  |
|       |        | res = 1 + (-1)*5 = -4         | -4  | 1    |            |                          |

返回 `-4`

## 💻 解法

> 用栈保存括号前的上下文，线性遍历一次解决。

```python
class Solution:
    def calculate(self, s: str) -> int:
        """
        计算包含 +、-、括号和空格的表达式值。
        使用栈保存括号前的符号和结果。
        """
        stack = []
        res = 0  # 当前计算结果
        sign = 1  # 当前符号：1 正，-1 负
        i = 0
        n = len(s)

        while i < n:
            c = s[i]
            if c.isdigit():
                # 解析完整数字
                num = 0
                while i < n and s[i].isdigit():
                    num = num * 10 + int(s[i])
                    i += 1
                res += sign * num
                continue  # 已经处理完数字，跳过后续 i++
            elif c == '+':
                sign = 1
            elif c == '-':
                sign = -1
            elif c == '(':
                # 保存当前状态，开始计算括号内表达式
                stack.append(res)
                stack.append(sign)
                res = 0
                sign = 1
            elif c == ')':
                # 括号内表达式计算完毕，与外部合并
                # 弹出括号前的符号和结果
                prev_sign = stack.pop()
                prev_res = stack.pop()
                # 括号内结果乘以括号前的符号，再与括号前的结果相加
                res = prev_res + prev_sign * res
            # 空格忽略

            i += 1

        return res
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 遍历字符串一次，每个字符处理 O(1) |
| 空间 | O(n) | 最坏情况嵌套括号很多，栈存储所有层的上下文 |

## ⚠ 易错点

- [ ] 入栈顺序：先 `res` 再 `sign`；出栈时先弹出 `sign`（因为后入先出），再弹出 `res`
- [ ] 遇到 `(` 后必须重置 `res = 0` 和 `sign = 1`，否则会把外层的值带入括号内
- [ ] 数字可能是多位数（如 `123`），必须用 while 循环完整解析
- [ ] 遇到数字时用了 `continue` 跳过主循环的 `i += 1`，因为数字解析 while 已经递增了 `i`
- [ ] `res += sign * num`：`sign` 在数字之前遇到，所以当前符号是紧挨着 `num` 前面的那个符号

## 🎤 面试怎么说

```text
这道题的核心难点在于处理括号——括号会改变计算的层次。

解决方案是用栈保存"上下文"。每当遇到左括号，就把当前的结果
res 和符号 sign 压栈暂存，然后把 res 和 sign 重置为初始状态，
好比进入了一个新的"子计算器"来计算括号里的内容。

遇到右括号时，子计算已经完成，从栈中弹出之前保存的 sign 和
res，把括号内的结果合并：外面结果 + 外面符号 * 括号内结果。

这样不管括号嵌套多深，都能用单次遍历 + 栈来优雅处理。

时间复杂度 O(n)，空间 O(n) 用于栈。这种"栈保存上下文"的思
路也可以用于处理中缀表达式转后缀、带多种运算符的计算器等场景。
```
