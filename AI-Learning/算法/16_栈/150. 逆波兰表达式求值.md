# 150. 逆波兰表达式求值

## 🎯 人话翻译

> 给你一个逆波兰表达式（操作数在前，运算符在后），计算表达式的值。比如 `["2","1","+","3","*"]` 就是 `(2+1)*3 = 9`。

## 💡 核心思想

> **用栈存操作数，遇到运算符就弹出栈顶两个数计算，再把结果压回去。**
>
> 逆波兰表达式的设计就是为栈求值而生——运算符总是在它所需的两个操作数之后出现，所以用栈后进先出刚好匹配。

用到：[[栈]]

## 📝 解题步骤

1. **初始化空栈** — 用于存储操作数和中间结果
2. **遍历 token** — 若为数字则转为整数压入栈；若为运算符则从栈中弹出两个数（先弹出的是右操作数 `b`，后弹出的是左操作数 `a`）
3. **执行运算** — 根据运算符计算 `a op b`，将结果压回栈
4. **返回结果** — 遍历结束后栈中只剩一个数，即最终结果

## 🔢 手动走一遍

输入: `tokens = ["4","13","5","/","+"]` 即 `4 + (13/5) = 4 + 2 = 6`

| 步骤 | token | 操作                     | 栈内容        | 说明                              |
|------|-------|--------------------------|---------------|-----------------------------------|
| 1    | `4`   | push 4                   | `[4]`         | 数字入栈                          |
| 2    | `13`  | push 13                  | `[4, 13]`     | 数字入栈                          |
| 3    | `5`   | push 5                   | `[4, 13, 5]`  | 数字入栈                          |
| 4    | `/`   | pop b=5, a=13, 计算 13/5 | `[4, 2]`      | 13//5=2（向零截断）               |
| 5    | `+`   | pop b=2, a=4, 计算 4+2   | `[6]`         | 结果 6                            |

返回栈顶 `6`

## 💻 解法

> 遍历 tokens，数字入栈，运算符弹栈计算。

```python
from typing import List


class Solution:
    def evalRPN(self, tokens: List[str]) -> int:
        """
        计算逆波兰表达式的值。
        使用栈存储操作数，遇到运算符则弹出两个数计算后压回。
        """
        stack = []

        for token in tokens:
            if token in {"+", "-", "*", "/"}:
                # 弹出右操作数（b）和左操作数（a）
                b = stack.pop()
                a = stack.pop()

                if token == '+':
                    res = a + b
                elif token == '-':
                    res = a - b
                elif token == '*':
                    res = a * b
                elif token == '/':
                    # 向零截断：Python 的 // 是向下取整，需用 int(a / b)
                    res = int(a / b)
                else:
                    raise ValueError("Invalid operator")

                stack.append(res)
            else:
                # 数字直接入栈
                stack.append(int(token))

        return stack[0]
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 遍历所有 token，每次操作 O(1) |
| 空间 | O(n) | 最坏情况栈存储约 n/2 个操作数 |

## ⚠ 易错点

- [ ] 弹出顺序必须注意：先弹出的是右操作数 `b`，后弹出的是左操作数 `a`。减法和除法对顺序敏感
- [ ] Python 中 `//` 是向下取整（`-3 // 2 = -2`），而题目要求向零截断（`-3 / 2 = -1`）。必须用 `int(a / b)` 而非 `a // b`
- [ ] 除法要考虑除数为 0 的情况（虽然题目不会给，但面试时可以提一嘴）
- [ ] 需要确保栈中至少有两个数再进行弹栈操作（合法逆波兰表达式保证如此）
- [ ] 不要用 `eval()` 直接计算——既不够安全，也不展示对栈的理解

做法：遍历每个 token，是数字就压栈，是运算符就从栈顶连弹

时间复杂度 O(n)，每遍历一个 token 做常数时间操作；空间也是
O(n)，栈中最大存约一半 token。
