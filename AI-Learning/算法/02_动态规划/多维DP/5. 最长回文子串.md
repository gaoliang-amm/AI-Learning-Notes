# 5. 最长回文子串

## 🎯 人话翻译

> 给一个字符串，找出其中最长的回文子串（连续子串，正读反读一样）。如果有多个答案，返回任意一个。

## 💡 核心思想

> **中心扩展法：从每个可能的中心向两边扩展，找到以该中心为基础的最长回文子串。**
>
> 回文有两种形式：
> 1. 奇数长度：中心是一个字符，如 "bab" 以 'a' 为中心
> 2. 偶数长度：中心是两个字符之间，如 "baab" 以 "aa" 之间的间隙为中心
>
> 遍历字符串的每个位置 `i`，分别以 `i` 为中心（奇数）和以 `i, i+1` 为中心（偶数）向两边扩展，记录最长的结果。
>
> 也有 DP 解法（O(n²) 时间 + O(n²) 空间），但中心扩展法空间 O(1) 更优。

用到：[[双指针]]、[[中心扩展]]

## 📝 解题步骤

1. **写扩展函数** — `expand(l, r)` 从中心 `(l, r)` 向两边扩展，只要 `s[l] == s[r]` 就继续，返回回文长度 `r - l - 1`
2. **遍历每个中心** — 对每个位置 `i`（0 到 n-1），分别计算奇数中心 `expand(i, i)` 和偶数中心 `expand(i, i+1)` 的回文长度
3. **更新最长回文** — 取两种扩展的最大值 `cur_max = max(len1, len2)`，如果比当前 `max_len` 大，更新起始位置和最大长度
4. **计算起始位置** — `start = i - (cur_max - 1) // 2`（从中心向左移动到回文起点）
5. **返回子串** — `s[start : start + max_len]`

## 🔢 手动走一遍

输入: `s = "babad"`

| 中心 i | 奇数扩展 len1 | 偶数扩展 len2 | cur_max | max_len 更新 | start |
|--------|-------------|-------------|---------|-------------|-------|
| 0      | expand(0,0): "b" -> len=1 | expand(0,1): "b"!="a" -> len=0 | 1 | max_len=1 -> 1 | start=0-(1-1)//2=0 |
| 1      | expand(1,1): "a"->"bab": s[0]=s[2]="b" -> len=3 | expand(1,2): "a"!="b" -> len=0 | 3 | 3>1 -> max_len=3 | start=1-(3-1)//2=0 |
| 2      | expand(2,2): "b"->"aba": len=3 | expand(2,3): "b"!="a" -> len=0 | 3 | 3==3 不更新 | (保持 start=0) |
| 3      | expand(3,3): "a"->"bad": len=1 | expand(3,4): "a"!="d" -> len=0 | 1 | 1<3 不更新 | (保持 start=0) |
| 4      | expand(4,4): "d" -> len=1 | expand(4,5): 右侧越界 -> len=0 | 1 | 1<3 不更新 | (保持 start=0) |

结果: `s[0:3] = "bab"`，长度为 3。

再看看偶数中心的情况——没有展示出来。换个更合适的例子: `s = "cbbd"`

| 中心 i | 奇数扩展 len1 | 偶数扩展 len2 | cur_max | max_len | start |
|--------|-------------|-------------|---------|---------|-------|
| 0      | expand(0,0): "c" -> 1 | expand(0,1): "c"!="b" -> 0 | 1 | 1 -> 1 | 0 |
| 1      | expand(1,1): "b" -> 1 | expand(1,2): "b"=="b" -> "cbbd": s[0]!="d" -> 2 | 2 | 2 -> 2 | start=1-(2-1)//2=0 |
| 2      | expand(2,2): "b" -> 1 | expand(2,3): "b"!="d" -> 0 | 1 | 1<2 不更新 | (保持) |
| 3      | expand(3,3): "d" -> 1 | expand(3,4): 越界 -> 0 | 1 | 1<2 不更新 | (保持) |

结果: `s[0:2] = "cb"`... wait, that should be `s[1:3] = "bb"`. Let me recalculate start for i=1, cur_max=2:
start = 1 - (2-1)//2 = 1 - 0 = 1. `s[1:1+2] = s[1:3] = "bb"`. Correct!

## 💻 解法（中心扩展法 — O(1) 空间）

> 遍历每个位置作为中心，分别处理奇数和偶数回文。

```python
class Solution:
    def longestPalindrome(self, s: str) -> str:
        # 时间复杂度 O(n²)，空间复杂度 O(1)

        n = len(s)
        if n < 2:
            return s

        start = 0   # 最长回文起始下标
        max_len = 1 # 最长回文长度

        def expand(l: int, r: int) -> int:
            """中心扩展，返回回文长度"""
            while l >= 0 and r < n and s[l] == s[r]:
                l -= 1
                r += 1
            # 退出循环时 l, r 多走了一步，实际回文长度为 r - l - 1
            return r - l - 1

        for i in range(n):
            # 奇数长度回文：以 i 为中心
            len1 = expand(i, i)
            # 偶数长度回文：以 i, i+1 为中心
            len2 = expand(i, i + 1)
            curr_max = max(len1, len2)

            if curr_max > max_len:
                max_len = curr_max
                # 计算起始下标：中心 i 向左移动 (curr_max - 1) // 2 步
                start = i - (curr_max - 1) // 2

        return s[start:start + max_len]
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n²) | 每个中心最多向两边扩展 O(n)，共 n 个中心 |
| 空间 | O(1) | 只用了几个变量，没有额外数组 |

## ⚠ 易错点

- [ ] 偶数回文的中心是两个字符 `expand(i, i+1)`，不能只考虑奇数中心。漏掉偶数中心会导致 `"cbbd"` 这样的案例失败
- [ ] 扩展函数中退出循环时 `l` 和 `r` 都多走了一步（因为 `while` 在比较失败后才退出，但之前已经 `l-=1, r+=1` 了），所以回文长度是 `r - l - 1` 而不是 `r - l + 1`
- [ ] 起始位置的计算公式 `start = i - (cur_max - 1) // 2`，这里 `(cur_max - 1) // 2` 是从中心向左移动的步数。对于奇数回文，中心就是 i，向左 `(len-1)//2` 步到达起点；对于偶数回文（s[i]==s[i+1]），中心在 i 和 i+1 之间，`(len-1)//2` 也能得到正确起点
- [ ] 字符串长度为 0 或 1 时需要单独处理，返回原字符串即可
- [ ] 不要忽略 DP 解法。面试时如果面试官问其他方法，可以说有 Manacher 算法（O(n)），但中心扩展法在实际中最常用

## 🎤 面试怎么说

```text
这道题求最长回文子串，我采用中心扩展法。

核心思路：回文关于中心对称。遍历字符串每个位置作为中心，
用双指针向两边扩展，直到不相等或越界。

两种中心：
- 奇数回文：以 i 为中心，expand(i, i)
- 偶数回文：以 i, i+1 为双中心，expand(i, i+1)

每次扩展记录回文长度，维护最大值和起始位置。

时间复杂度 O(n²)（每个中心最坏扩展 n 次），空间复杂度 O(1)。

如果需要 O(n) 的解法，有 Manacher 算法，但一般面试不需要写。

另外也可以提 DP 解法：dp[i][j] 表示 s[i:j+1] 是否回文，
dp[i][j] = (s[i]==s[j] and dp[i+1][j-1])，
时间 O(n²)，空间 O(n²)，通常不作为首选。
```
