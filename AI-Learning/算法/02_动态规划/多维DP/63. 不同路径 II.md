# 63. 不同路径 II

## 🎯 人话翻译

> 从左上角走到右下角，只能向右或向下走。网格中有障碍物（值为 1 的格子不能走），问有多少条不同的路径能到达终点。

## 💡 核心思想

> **带障碍物的网格路径 DP。**
>
> 基本递推：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`（从上边来的路径数 + 从左边来的路径数）。如果当前格子是障碍物，`dp[i][j] = 0`。
>
> 空间优化：因为每行只依赖上一行，可以用一维数组 `dp[j] = dp[j] + dp[j-1]`（dp[j] 是上一行同列的值，dp[j-1] 是本行左边的值）。遇到障碍物，`dp[j] = 0`。

用到：[[动态规划]]、[[多维DP]]

## 📝 解题步骤

1. **特殊情况处理** — 起点或终点是障碍物，直接返回 0（无路可走）
2. **初始化一维 dp** — `dp = [0] * n`，`dp[0] = 1`（起点只有一种方式到达）
3. **逐行遍历** — 对于每个格子 `(i, j)`：
   - 如果是障碍物：`dp[j] = 0`（不可到达）
   - 如果 `j > 0`（不是第一列）：`dp[j] = dp[j] + dp[j-1]`（上 + 左）
   - 如果 `j == 0`（第一列）：`dp[0]` 保持上一行的值（只能从上面来，障碍物时会被清 0）
4. **返回结果** — `dp[-1]` 即到达右下角的路径数

## 🔢 手动走一遍

输入:
```
obstacleGrid = [
  [0, 0, 0],
  [0, 1, 0],
  [0, 0, 0]
]
```

dp 初始化为 `[1, 0, 0]`（n=3）

| 行 i | 列 j | grid[i][j] | 操作 | dp 更新后 | 说明 |
|------|------|------------|------|----------|------|
| 0    | 0    | 0 | j=0, 非障碍物, dp[0] 保持 | [1, 0, 0] | 起点 |
| 0    | 1    | 0 | j=1>0: dp[1]=dp[1]+dp[0]=0+1=1 | [1, 1, 0] | 可从左边(0,0)来 |
| 0    | 2    | 0 | j=2>0: dp[2]=dp[2]+dp[1]=0+1=1 | [1, 1, 1] | 可从左边(0,1)来 |
| 1    | 0    | 0 | j=0, 非障碍物, dp[0] 保持=1 | [1, 1, 1] | 可从上面(0,0)来 |
| 1    | 1    | **1** | 障碍物! dp[1]=0 | [1, **0**, 1] | 无法到达 |
| 1    | 2    | 0 | dp[2]=dp[2]+dp[1]=1+0=1 | [1, 0, 1] | 可从左边(1,1)来(=0)或上面(0,2)来(=1) |
| 2    | 0    | 0 | j=0, 非障碍物, dp[0] 保持=1 | [1, 0, 1] | 可从上面(1,0)来 |
| 2    | 1    | 0 | dp[1]=dp[1]+dp[0]=0+1=1 | [1, **1**, 1] | 可从左边(2,0)或上面(1,1)来 |
| 2    | 2    | 0 | dp[2]=dp[2]+dp[1]=1+1=2 | [1, 1, **2**] | 两路汇聚 |

结果: `dp[-1] = 2`。两条路径: (右右下下) 避开障碍物 或 (右下下右)。

## 💻 解法（一维 DP — O(n) 空间）

> 空间优化版：只用一行 dp，逐行更新。

```python
from typing import List


class Solution:
    def uniquePathsWithObstacles(self, obstacleGrid: List[List[int]]) -> int:
        # 时间复杂度 O(m × n)，空间复杂度 O(n)

        if not obstacleGrid or not obstacleGrid[0]:
            return 0

        m, n = len(obstacleGrid), len(obstacleGrid[0])

        # 起点或终点是障碍物，直接返回 0
        if obstacleGrid[0][0] == 1 or obstacleGrid[m - 1][n - 1] == 1:
            return 0

        # 一维 dp 数组
        dp = [0] * n
        dp[0] = 1  # 起点

        for i in range(m):
            for j in range(n):
                # 遇到障碍物，路径数为 0
                if obstacleGrid[i][j] == 1:
                    dp[j] = 0
                    continue

                # 不是第一列时，路径数 = 上方 + 左边
                if j > 0:
                    dp[j] = dp[j] + dp[j - 1]
                # 第一列时，dp[0] 保持原值（上一行的值）
                # 注意：第一行第一列已在循环外初始化

        return dp[-1]
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(m * n) | 遍历网格中每个格子一次 |
| 空间 | O(n) | 一维 dp 数组长度为列数 |

## ⚠ 易错点

- [ ] 必须检查起点和终点是否是障碍物。如果起点 `grid[0][0] == 1`，不能到达起点本身，直接返回 0。题目没有保证起点不是障碍物
- [ ] 遇到障碍物时 `dp[j] = 0`，然后必须 `continue` 跳过 `dp[j] = dp[j] + dp[j-1]` 的更新。否则 dp[j] 被加回去了
- [ ] 第一列的 dp[0] 只有上方来源（没有左边），所以 j=0 时不需要 `dp[j] + dp[j-1]`。但遇到障碍物时 dp[0] 也会被清 0，且后续行的 dp[0] 会保持 0（因为无法从上面来了）
- [ ] dp 数组的长度是 n（列数），不是 m（行数）。`dp[-1]` 访问的是最后一列
- [ ] 第 0 行第 0 列的初始化：第一行的 j > 0 位置通过 `dp[j] + dp[j-1]` 累积，dp[0] 始终为 1。但当第一行有障碍物时 dp[j] = 0 会阻断后续

```text
这道题是"不同路径"的变种，加了障碍物。

核心思路：dp[j] 表示到达当前行第 j 列的路径数。
递推公式：dp[j] = dp[j] + dp[j-1]
（dp[j] 是上一行同列的值"从上边来"，dp[j-1] 是本行左边的值"从左边来"）

遇到障碍物时，dp[j] = 0（该格子不可到达）。

- 起点是障碍物 -> 直接返回 0
- 终点是障碍物 -> 直接返回 0

时间复杂度 O(m*n)，空间复杂度 O(n)。

可以用二维 dp 增加可读性，但一维 dp 更省空间。
```
