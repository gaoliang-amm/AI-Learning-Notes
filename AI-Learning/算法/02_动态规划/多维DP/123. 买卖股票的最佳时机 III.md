# 123. 买卖股票的最佳时机 III

## 🎯 人话翻译

> 给一个数组表示每天股票的价格。你最多可以做两笔交易（买一次卖一次算一笔），交易不能重叠（卖了才能再买）。问最多能赚多少钱。

## 💡 核心思想

> **状态机 DP：每天都有四种状态，跟踪每种状态下的最大利润。**
>
> 两笔交易涉及四次状态变化：
> - `buy1`：完成第一次买入后的最大利润（通常是负数，因为花钱买入了）
> - `sell1`：完成第一次卖出后的最大利润
> - `buy2`：完成第二次买入后的最大利润
> - `sell2`：完成第二次卖出后的最大利润
>
> 状态转移方程（对每天的价格）：
> - `buy1 = max(buy1, -price)` —— 要么之前已买入，要么今天第一次买
> - `sell1 = max(sell1, buy1 + price)` —— 要么之前已卖出，要么今天第一次卖
> - `buy2 = max(buy2, sell1 - price)` —— 要么之前已第二次买入，要么用第一次赚的钱买
> - `sell2 = max(sell2, buy2 + price)` —— 要么之前已第二次卖出，要么今天第二次卖
>
> 最终答案 = `sell2`（也可能只做一笔交易，sell2 会等于 sell1）。

用到：[[动态规划]]、[[状态机DP]]、[[股票买卖]]

## 📝 解题步骤

1. **初始化四种状态** — `buy1 = -inf`, `sell1 = 0`, `buy2 = -inf`, `sell2 = 0`
2. **遍历每一天的价格** — 对每个 `price`，按顺序更新四个状态变量
3. **更新 buy1** — `buy1 = max(buy1, -price)`（第一次买入，花 price 的钱）
4. **更新 sell1** — `sell1 = max(sell1, buy1 + price)`（第一次卖出，赚 price 的钱）
5. **更新 buy2** — `buy2 = max(buy2, sell1 - price)`（用第一次赚的钱，买第二次）
6. **更新 sell2** — `sell2 = max(sell2, buy2 + price)`（第二次卖出）
7. **返回结果** — `sell2`

## 🔢 手动走一遍

输入: `prices = [3, 3, 5, 0, 0, 3, 1, 4]`

| 天 | price | buy1 (max(buy1, -price)) | sell1 (max(sell1, buy1+price)) | buy2 (max(buy2, sell1-price)) | sell2 (max(sell2, buy2+price)) |
|----|-------|--------------------------|-------------------------------|-------------------------------|--------------------------------|
| 初始 | - | -inf | 0 | -inf | 0 |
| 1 | 3 | max(-inf, -3) = -3 | max(0, -3+3=0) = 0 | max(-inf, 0-3=-3) = -3 | max(0, -3+3=0) = 0 |
| 2 | 3 | max(-3, -3) = -3 | max(0, -3+3=0) = 0 | max(-3, 0-3=-3) = -3 | max(0, -3+3=0) = 0 |
| 3 | 5 | max(-3, -5) = -3 | max(0, -3+5=2) = 2 | max(-3, 2-5=-3) = -3 | max(0, -3+5=2) = 2 |
| 4 | 0 | max(-3, 0) = 0 | max(2, 0+0=0) = 2 | max(-3, 2-0=2) = 2 | max(2, 2+0=2) = 2 |
| 5 | 0 | max(0, 0) = 0 | max(2, 0+0=0) = 2 | max(2, 2-0=2) = 2 | max(2, 2+0=2) = 2 |
| 6 | 3 | max(0, -3) = 0 | max(2, 0+3=3) = 3 | max(2, 3-3=0) = 2 | max(2, 2+3=5) = 5 |
| 7 | 1 | max(0, -1) = 0 | max(3, 0+1=1) = 3 | max(2, 3-1=2) = 2 | max(5, 2+1=3) = 5 |
| 8 | 4 | max(0, -4) = 0 | max(3, 0+4=4) = 4 | max(2, 4-4=0) = 2 | max(5, 2+4=6) = **6** |

结果: `sell2 = 6`。两笔交易：第 4 天 0 元买入，第 6 天 3 元卖出（赚 3）；第 7 天 1 元买入，第 8 天 4 元卖出（赚 3）。合计赚 6。

## 💻 解法（状态机 DP — O(1) 空间）

> 跟踪四笔关键交易状态，每个价格更新一次。最优雅的写法。

```python
from typing import List


class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        # 时间复杂度 O(n)，空间复杂度 O(1)

        # 四次状态：
        # buy1: 第一次买入后的最大利润（通常是负数，因为花钱买入）
        # sell1: 第一次卖出后的最大利润
        # buy2: 第二次买入后的最大利润
        # sell2: 第二次卖出后的最大利润

        # 初始化为负无穷（或极小值）
        buy1 = float('-inf')
        sell1 = 0
        buy2 = float('-inf')
        sell2 = 0

        for price in prices:
            # 第一次买入：要么之前已买（保持），要么现在买（花钱）
            buy1 = max(buy1, -price)
            # 第一次卖出：要么之前已卖（保持），要么现在卖（获利）
            sell1 = max(sell1, buy1 + price)
            # 第二次买入：要么之前已买（保持），要么用第一次卖出的钱买
            buy2 = max(buy2, sell1 - price)
            # 第二次卖出：要么之前已卖（保持），要么现在卖（获利）
            sell2 = max(sell2, buy2 + price)

        return sell2
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 一次遍历，每天做 4 次 O(1) 操作 |
| 空间 | O(1) | 只使用 4 个变量 |

## ⚠ 易错点

- [ ] `buy1` 和 `buy2` 初始化为 `-inf`（不是 0），因为在初始时你还没有持有任何股票。"有股票"的状态利润是负数或不存在，用 `-inf` 表示"不可能"
- [ ] 更新顺序不能打乱：必须先更新 `buy1` 再 `sell1`，再 `buy2` 再 `sell2`。因为 `sell1` 依赖当天的 `buy1`，`buy2` 依赖当天的 `sell1` 等
- [ ] `buy1 = max(buy1, -price)` 中的 `-price` 意味着"花钱买入"。注意不是 `price` 而是 `-price`，因为买入后你的利润是 `-price`
- [ ] 最终返回 `sell2`，不是 `sell1`。但即使最优解只做一笔交易，`sell2` 的值也会等于 `sell1`（因为 `buy2` 可以接相同的 `sell1` 后卖出不亏不赚）
- [ ] 注意不能写成 `buy1 = -price` 直接赋值，必须是 `max(buy1, -price)`。因为你需要保留"之前某天更低价买入"的可能性

这道题是股票买卖问题的升级版，限定最多两笔交易。

核心思路：把问题看成状态机，每天处于四种状态之一：
buy1、sell1、buy2、sell2，分别表示第一次买入后、卖出后、第二次买入后、卖出后的最大利润。
