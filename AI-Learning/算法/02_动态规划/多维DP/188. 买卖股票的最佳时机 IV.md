# 188. 买卖股票的最佳时机 IV

## 🎯 人话翻译

> 给一个数组表示每天股票的价格，你最多可以做 k 笔交易（一笔交易 = 买一次 + 卖一次），不能同时持有多笔交易（卖了才能再买）。问最多能赚多少钱。

## 💡 核心思想

> **123 题（k=2）的泛化：把状态机扩展到 k 对 (buy, sell) 状态。**
>
> 对每一笔交易 j（1 到 k），维护：
> - `buy[j]`：第 j 次买入后的最大利润
> - `sell[j]`：第 j 次卖出后的最大利润
>
> 状态转移：
> - `buy[j] = max(buy[j], sell[j-1] - price)`：要么保持之前的买入，要么用第 j-1 次卖出的钱买第 j 次
> - `sell[j] = max(sell[j], buy[j] + price)`：要么保持之前的卖出，要么在第 j 次买入后卖出
>
> 特殊情况：当 `k >= n/2` 时，相当于无限次交易（因为一天最多做一次买卖）。此时直接用贪心，累加所有正差价即可。

用到：[[动态规划]]、[[状态机DP]]、[[股票买卖]]

## 📝 解题步骤

1. **边界处理** — 如果 `n <= 1` 或 `k == 0`，返回 0（无法交易）
2. **k 过大处理** — 如果 `k >= n // 2`，等价于无限次交易。遍历数组，每次涨价就累加差价（贪心）
3. **初始化 buy/sell 数组** — `buy = [-inf] * (k+1)`, `sell = [0] * (k+1)`
4. **遍历每天的价格** — 对每个 `price`，内层 j 从 1 到 k：
   - `buy[j] = max(buy[j], sell[j-1] - price)`
   - `sell[j] = max(sell[j], buy[j] + price)`
5. **返回结果** — `sell[k]`

## 🔢 手动走一遍

输入: `k = 2`, `prices = [3, 2, 6, 5, 0, 3]`

这里 k=2 等价于 123 题，但用数组泛化写法。

初始化: `buy = [-inf, -inf, -inf]`, `sell = [0, 0, 0]`

| 天 | price | j=1: buy[1] | sell[1] | j=2: buy[2] | sell[2] |
|----|-------|-------------|---------|-------------|---------|
| 初始 | - | [-inf, -inf, -inf] | [0, 0, 0] | - | - |
| 1 | 3 | max(-inf, 0-3=-3) = -3 | max(0, -3+3=0) = 0 | max(-inf, 0-3=-3) = -3 | max(0, -3+3=0) = 0 |
| 2 | 2 | max(-3, 0-2=-2) = -2 | max(0, -2+2=0) = 0 | max(-3, 0-2=-2) = -2 | max(0, -2+2=0) = 0 |
| 3 | 6 | max(-2, 0-6=-6) = -2 | max(0, -2+6=4) = 4 | max(-2, sell[1]-6=4-6=-2) = -2 | max(0, -2+6=4) = 4 |
| 4 | 5 | max(-2, 0-5=-5) = -2 | max(4, -2+5=3) = 4 | max(-2, 4-5=-1) = -1 | max(4, -1+5=4) = 4 |
| 5 | 0 | max(-2, 0) = 0 | max(4, 0+0=0) = 4 | max(-1, 4-0=4) = 4 | max(4, 4+0=4) = 4 |
| 6 | 3 | max(0, -3) = 0 | max(4, 0+3=3) = 4 | max(4, 4-3=1) = 4 | max(4, 4+3=7) = **7** |

结果: `sell[2] = 7`。两笔交易：第 2 天 2 买入 -> 第 3 天 6 卖出（赚 4）；第 5 天 0 买入 -> 第 6 天 3 卖出（赚 3）。合计赚 7。

## 💻 解法

> 将 123 题泛化到 k 笔交易，用数组代替固定变量。

```python
from typing import List


class Solution:
    def maxProfit(self, k: int, prices: List[int]) -> int:
        # 时间复杂度 O(n × min(k, n))，空间复杂度 O(k)

        n = len(prices)
        if n <= 1 or k == 0:
            return 0

        # 当 k >= n/2 时，相当于无限次交易，贪心累加所有正差价
        if k >= n // 2:
            profit = 0
            for i in range(1, n):
                if prices[i] > prices[i - 1]:
                    profit += prices[i] - prices[i - 1]
            return profit

        # 一般情况：dp_buy[j] 表示第 j 次买入后的最大利润
        # dp_sell[j] 表示第 j 次卖出后的最大利润
        dp_buy = [float('-inf')] * (k + 1)
        dp_sell = [0] * (k + 1)

        for price in prices:
            for j in range(1, k + 1):
                # 第 j 次买入：要么之前已买，要么用第 j-1 次卖出的钱买入
                dp_buy[j] = max(dp_buy[j], dp_sell[j - 1] - price)
                # 第 j 次卖出：要么之前已卖，要么在第 j 次买入后卖出
                dp_sell[j] = max(dp_sell[j], dp_buy[j] + price)

        return dp_sell[k]
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n * min(k, n)) | n 天 * 最多 k 次交易；当 k >= n/2 时退化为 O(n) |
| 空间 | O(k) | 两个长度为 k+1 的数组 |

## ⚠ 易错点

- [ ] 当 `k >= n // 2` 时必须特殊处理，转化为无限次交易。如果不处理，内循环 `for j in range(1, k+1)` 会 O(n*k) 且 k 可能很大（k <= 100, n <= 1000），可能超时
- [ ] `buy` 数组初始化为 `-inf`，`sell` 数组初始化为 0。与 123 题的四变量初始化逻辑一致
- [ ] 内循环 `j` 从 1 到 k（1-indexed，索引 0 不使用），`buy[0]` 和 `sell[0]` 始终保持初始值 `-inf` 和 `0`
- [ ] 状态转移中 `buy[j]` 依赖 `sell[j-1]`（前一笔交易的卖出利润），而不是 `sell[j]` 本身。这意味着 j 的循环顺序无关紧要（因为 `buy[j]` 依赖的是 `sell[j-1]`，不是当天的 `sell[j]`）
- [ ] 遍历价格时，内循环更新 `buy` 和 `sell` 的顺序：先 `buy[j]` 后 `sell[j]`
