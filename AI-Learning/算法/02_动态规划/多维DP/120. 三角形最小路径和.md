# 120. 三角形最小路径和

## 🎯 人话翻译

> 一个三角形数组，从顶部开始，每次只能走到下一行相邻的数字。问走到底部，经过的数字之和最小是多少。

## 💡 核心思想

> **自底向上 DP，每个位置的最小路径和 = 当前值 + min(左下结果, 右下结果)。**
>
> 如果从上往下算，三角形不同行的列数不同，不太方便。反过来，从最后一行向上递推：对于第 i 行的第 j 个元素，它的最小路径和 = `triangle[i][j] + min(dp[j], dp[j+1])`，其中 `dp[j]` 和 `dp[j+1]` 分别是下一行左下方和右下方的结果。
>
> 空间可以优化到 O(n)：直接用一维数组，初始化为最后一行的值，从倒数第二行向上递推，就地更新。

用到：[[动态规划]]、[[多维DP]]

## 📝 解题步骤

1. **初始化 dp 数组** — 将最后一行作为初始 dp 值：`dp = triangle[-1][:]`
2. **从倒数第二行向上递推** — `for i in range(n-2, -1, -1)`，对每一行的每个元素 j
3. **状态转移** — `dp[j] = triangle[i][j] + min(dp[j], dp[j+1])`。dp[j] 是下一行同列（左下方），dp[j+1] 是下一行右边一列（右下方）
4. **返回结果** — `dp[0]` 即从顶点出发的最小路径和

## 🔢 手动走一遍

输入:
```
[
     [2],
    [3,4],
   [6,5,7],
  [4,1,8,3]
]
```

dp 初始化为最后一行: `dp = [4, 1, 8, 3]`

| 行 i | 当前行值 | dp 更新 | 更新后 dp |
|------|---------|--------|-----------|
| 初始 | - | - | [4, 1, 8, 3] |
| i=2 | [6,5,7] | j=0: dp[0]=6+min(dp[0],dp[1])=6+min(4,1)=6+1=7<br>j=1: dp[1]=5+min(dp[1],dp[2])=5+min(1,8)=5+1=6<br>j=2: dp[2]=7+min(dp[2],dp[3])=7+min(8,3)=7+3=10 | [7, 6, 10, 3] |
| i=1 | [3,4] | j=0: dp[0]=3+min(dp[0],dp[1])=3+min(7,6)=3+6=9<br>j=1: dp[1]=4+min(dp[1],dp[2])=4+min(6,10)=4+6=10 | [9, 10, 10, 3] |
| i=0 | [2] | j=0: dp[0]=2+min(dp[0],dp[1])=2+min(9,10)=2+9=11 | [11, 10, 10, 3] |

结果: `dp[0] = 11`。对应路径: 2 -> 3 -> 5 -> 1 = 11。

## 💻 解法（自底向上 — O(n) 空间）

> 从最后一层开始往上递推，每次决策走左下还是右下。

```python
from typing import List


class Solution:
    def minimumTotal(self, triangle: List[List[int]]) -> int:
        # 时间复杂度 O(n²)，空间复杂度 O(n)

        n = len(triangle)
        # dp 数组初始化为最后一行的值
        dp = triangle[-1][:]

        # 从倒数第二行向上递推
        for i in range(n - 2, -1, -1):
            for j in range(len(triangle[i])):
                # 当前节点的最小路径和 = 当前值 + min(左下, 右下)
                dp[j] = triangle[i][j] + min(dp[j], dp[j + 1])

        return dp[0]
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n²) | n 行，每行最多 n 个元素，每个 O(1) 计算 |
| 空间 | O(n) | 一维 dp 数组，长度为三角形的宽度 n |

## ⚠ 易错点

- [ ] 必须从倒数第二行（`n-2`）开始向上遍历，循环 `range(n-2, -1, -1)`。如果写成 `range(n-1, -1, -1)`（从最后一行开始），最后一行没有"下一行"可以比较，会出错
- [ ] 内层循环 `j` 遍历行内元素时，范围是 `len(triangle[i])`（当前行的列数），不是 `n`。因为三角形每行的元素个数不等
- [ ] `dp[j]` 和 `dp[j+1]` 分别代表下一行的左下方和右下方元素。三角形中第 i 行有 i+1 个元素，第 i+1 行有 i+2 个元素，所以 `dp[j+1]` 不会越界（因为 j 最大为 `len(triangle[i]) - 1`，而 dp 长度 >= `len(triangle[i]) + 1`）
- [ ] 如果从上往下 DP，空间也是 O(n²)。自底向上的好处是 dp 只需要 O(n) 空间
