# 64. 最小路径和

## 🎯 人话翻译

> 一个 m x n 的网格，每个格子有一个非负数。从左上角走到右下角，只能向右或向下走，问路径上的数字之和最小是多少。

## 💡 核心思想

> **网格 DP：当前位置的最小路径和 = 当前数字 + min(上方来的路径和, 左方来的路径和)。**
>
> 递推公式：`dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])`。
>
> 空间优化：因为每行只依赖上一行和本行左边的值，可以用一维数组：`dp[j] = grid[i][j] + min(dp[j], dp[j-1])`，其中 `dp[j]` 是上方路径和（上一行同列），`dp[j-1]` 是左方路径和（本行前一列）。
>
> 第一行只能从左边来，第一列只能从上面来，需要单独处理。

用到：[[动态规划]]、[[多维DP]]

## 📝 解题步骤

1. **初始化第一行** — 第一行每个格子只能从左边来：`dp[j] = dp[j-1] + grid[0][j]`
2. **逐行遍历** — 从第二行（i=1）开始，对每行：
   - 第一个元素（j=0）：只能从上面来，`dp[0] += grid[i][0]`
   - 其余元素（j>0）：`dp[j] = grid[i][j] + min(dp[j], dp[j-1])`
3. **返回结果** — `dp[-1]` 即到达右下角的最小路径和

## 💻 解法一（二维 DP — O(m*n) 空间）

> 标准的二维 DP 表，思路最直观。

```python
from typing import List


class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        # 时间复杂度 O(m × n)，空间复杂度 O(m × n)

        if not grid or not grid[0]:
            return 0

        m, n = len(grid), len(grid[0])
        dp = [[0] * n for _ in range(m)]
        dp[0][0] = grid[0][0]

        # 初始化第一行
        for j in range(1, n):
            dp[0][j] = dp[0][j - 1] + grid[0][j]

        # 初始化第一列
        for i in range(1, m):
            dp[i][0] = dp[i - 1][0] + grid[i][0]

        # 填充剩余位置
        for i in range(1, m):
            for j in range(1, n):
                dp[i][j] = grid[i][j] + min(dp[i - 1][j], dp[i][j - 1])

        return dp[m - 1][n - 1]
```

## 🚀 解法二（空间优化 — O(n) 空间）

> 一维数组滚动更新，面试推荐。

```python
from typing import List


class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        # 时间复杂度 O(m × n)，空间复杂度 O(n)

        if not grid or not grid[0]:
            return 0

        m, n = len(grid), len(grid[0])

        # 一维 dp 数组，dp[j] 表示到达当前行第 j 列的最小路径和
        dp = [0] * n
        dp[0] = grid[0][0]

        # 初始化第一行（只能从左边来）
        for j in range(1, n):
            dp[j] = dp[j - 1] + grid[0][j]

        # 从第二行开始
        for i in range(1, m):
            # 每行第一个格子只能从上方来
            dp[0] += grid[i][0]
            for j in range(1, n):
                # 当前格子 = 当前值 + min(上边, 左边)
                dp[j] = grid[i][j] + min(dp[j], dp[j - 1])

        return dp[-1]
```

## ⏱ 复杂度

| 版本 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 二维 DP | O(m*n) | O(m*n) | 需要记录路径或回溯时 |
| 一维 DP | O(m*n) | O(n) | 面试推荐，只需求最值 |

## ⚠ 易错点

- [ ] 第一行的初始化：`dp[j] = dp[j-1] + grid[0][j]`，不是 `grid[0][j]` 直接放进去。第一行只能从左边来，必须累加
- [ ] 第一列不能简单用 `dp[0] = dp[0] + grid[i][0]` 但忘了每次进入新行都要更新。写成 `dp[0] += grid[i][0]` 更简洁
- [ ] 一维优化版中，更新顺序很重要：每行必须从左到右更新。如果从右到左，`dp[j-1]` 可能已经被本行更新过了，不再是上一行的值
- [ ] 当 `j > 0` 时，`dp[j]` 当前存储的是上一行同列的值（上方来源），`dp[j-1]` 是本行刚更新的值（左方来源）。理解这一点有助于避免初始化错误
- [ ] 如果要用原数组做 in-place 修改（空间 O(1)），需要同样处理第一行和第一列的累加逻辑

## 🎤 面试怎么说

```text
这道题是经典的网格 DP 问题，类似"不同路径"但求的是最小和。

核心思路：dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])
当前位置的最小路径和 = 当前值 + min(从上方来的和, 从左边来的和)

可以用二维 DP，但空间可以优化到 O(n)：
维护一维数组 dp，dp[j] 表示到达当前行第 j 列的最小路径和。
初始化第一行累加，然后从第二行开始：
- 第一列 dp[0] += grid[i][0]（只能从上方来）
- 其余列 dp[j] = grid[i][j] + min(dp[j], dp[j-1])

时间复杂度 O(m*n)，空间复杂度 O(n)。

如果不允许修改输入和 dp 数组，可以在原数组上 in-place 累加，空间 O(1)。
```
