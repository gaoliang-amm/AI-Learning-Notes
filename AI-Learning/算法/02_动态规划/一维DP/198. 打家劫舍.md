# 198. 打家劫舍

## 🎯 人话翻译

> 一排房屋挨着，每间房有钱。你不能偷相邻的两间（会触发警报），问最多能偷多少钱。

## 💡 核心思想

> **每个房间只有两种选择：偷或不偷。**
>
> 如果偷第 i 间，则不能偷第 i-1 间，最大金额 = 第 i 间的钱 + 前 i-2 间的最大金额。
> 如果不偷第 i 间，最大金额 = 前 i-1 间的最大金额。
>
> 递推公式：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`
>
> 由于每次只需要前两个状态，可以将空间优化到 O(1)，用两个变量 `prev2`（dp[i-2]）和 `prev1`（dp[i-1]）滚动更新。

用到：[[动态规划]]、[[一维DP]]

## 📝 解题步骤

1. **处理边界** — 空数组返回 0，只有一个元素直接返回 `nums[0]`
2. **初始化两个变量** — `prev2 = nums[0]`（只有第一间），`prev1 = max(nums[0], nums[1])`（前两间的最优解）
3. **滚动递推** — 从索引 2 开始：`current = max(prev1, prev2 + nums[i])`，然后 `prev2, prev1 = prev1, current`
4. **返回结果** — `prev1` 保存了最终结果

## 🔢 手动走一遍

输入: `nums = [2, 7, 9, 3, 1]`

| 索引 i | nums[i] | prev2 (dp[i-2]) | prev1 (dp[i-1]) | current = max(prev1, prev2+nums[i]) | 选择 |
|--------|---------|------|------|-------------------------------------|------|
| 0      | 2       | 2 (初始) | - | - | 偷 2 |
| 1      | 7       | 2 | max(2,7)=7 (初始) | - | 不偷 2，偷 7 |
| 2      | 9       | 2 | 7 | max(7, 2+9=11) = 11 | 偷 2 和 9，不偷 7 |
| 3      | 3       | 7 | 11 | max(11, 7+3=10) = 11 | 不偷 3，沿用前值 |
| 4      | 1       | 11 | 11 | max(11, 11+1=12) = 12 | 偷 1 和之前的最优解 |

最终 `prev1 = 12`。偷的房屋: 2 + 9 + 1 = 12（索引 0, 2, 4）。

## 💻 解法（空间优化）

> 用两个变量代替整个 dp 数组，滚动更新。

```python
from typing import List


class Solution:
    def rob(self, nums: List[int]) -> int:
        # 时间复杂度 O(n)，空间复杂度 O(1)

        # 边界情况：空数组
        if not nums:
            return 0
        # 边界情况：只有一个房屋，只能偷它
        if len(nums) == 1:
            return nums[0]

        # 动态规划，用两个变量代替整个 dp 数组
        # prev2 对应 dp[i-2]：到前前一个房屋为止能偷到的最大金额
        # prev1 对应 dp[i-1]：到前一个房屋为止能偷到的最大金额

        # 初始化：只有第一个房屋时，最大金额就是 nums[0]
        prev2 = nums[0]
        # 只有前两个房屋时，最大金额是两者中的较大值（不能同时偷）
        prev1 = max(nums[0], nums[1])

        # 从第 3 个房屋开始遍历（索引 2）
        for i in range(2, len(nums)):
            # 对于当前房屋 i，有两个选择：
            # 1. 不偷当前房屋：金额 = prev1（前一个房屋的最大金额）
            # 2. 偷当前房屋：金额 = prev2 + nums[i]（前前一个房屋的金额 + 当前房屋）
            # 取两者中的较大值
            current = max(prev1, prev2 + nums[i])

            # 滚动更新：为下一轮循环做准备
            # 下一轮的 prev2 应该是这一轮的 prev1
            # 下一轮的 prev1 应该是这一轮的 current
            prev2, prev1 = prev1, current

        # 最终结果就是 prev1
        return prev1
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 一次遍历，每个房屋 O(1) 决策 |
| 空间 | O(1) | 只使用 prev2 和 prev1 两个变量 |

## ⚠ 易错点

- [ ] 空数组和小数组的边界处理：`not nums` 返回 0，`len(nums) == 1` 返回 `nums[0]`。漏掉任何一个都会 IndexError
- [ ] `prev2` 初始化为 `nums[0]`（不是 0），`prev1` 初始化为 `max(nums[0], nums[1])`（不是 `nums[1]`）。从索引 2 开始遍历
- [ ] 滚动更新时，`prev2, prev1 = prev1, current` 是同时赋值的（Python 元组赋值），这没问题。但如果分开写，必须先用临时变量保存旧值再更新
- [ ] 递推公式是 `dp[i] = max(dp[i-1], dp[i-2] + nums[i])`，不要写成 `max(dp[i-2], dp[i-1] + nums[i])`
- [ ] 环形版本（213. 打家劫舍 II）要额外处理首尾相邻的情况，分两次调用此逻辑

## 🎤 面试怎么说

```text
这道题是动态规划的经典入门题。

核心思路：每个房子有两个选择——偷或不偷。
- 偷当前房子：金额 = 当前房子的钱 + 前前个的最优解（因为不能偷相邻的）
- 不偷当前房子：金额 = 前一个的最优解

dp[i] = max(dp[i-1], dp[i-2] + nums[i])

由于只依赖前两个状态，可以用两个变量滚动更新：
prev2 = dp[i-2], prev1 = dp[i-1]

每次更新：current = max(prev1, prev2 + nums[i])
然后滚动：prev2, prev1 = prev1, current

时间复杂度 O(n)，空间复杂度 O(1)。

如果面试官问环形版本（213 题），可以分两种情况：
1. 偷第一间不偷最后一间
2. 不偷第一间偷最后一间
取两者的最大值。
```
