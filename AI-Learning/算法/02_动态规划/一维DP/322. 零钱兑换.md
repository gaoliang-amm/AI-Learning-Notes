# 322. 零钱兑换

## 🎯 人话翻译

> 给你几种不同面额的硬币和一个总金额 amount，问最少用多少个硬币能凑出这个金额。如果凑不出来，返回 -1。每种硬币可以无限使用。

## 💡 核心思想

> **完全背包问题：每种硬币无限用，求凑出目标金额的最少硬币数。**
>
> `dp[i]` 表示凑出金额 i 最少需要多少个硬币。对于金额 i，枚举每一种硬币 coin，如果 `i >= coin`，则 `dp[i] = min(dp[i], dp[i - coin] + 1)`。即：凑出 i 的最少硬币数 = min（不选当前硬币，选当前硬币：凑出 i-coin 的最少硬币数 + 1）。
>
> 初始化时 `dp[0] = 0`（凑 0 元需要 0 个硬币），其余位置初始化为一个大数（如 `amount + 1`，表示不可能凑成）。

用到：[[动态规划]]、[[一维DP]]、[[背包问题]]

## 📝 解题步骤

1. **初始化 dp 数组** — `dp = [amount + 1] * (amount + 1)`，长度 amount+1，`dp[0] = 0`
2. **外循环遍历金额** — `i` 从 1 到 amount，表示当前要凑的目标金额
3. **内循环遍历硬币** — 对每种硬币，如果 `i >= coin`，尝试 `dp[i] = min(dp[i], dp[i - coin] + 1)`
4. **返回结果** — 若 `dp[amount]` 仍是初始值 `amount + 1`，说明凑不出来，返回 -1；否则返回 `dp[amount]`

## 🔢 手动走一遍

输入: `coins = [1, 2, 5]`, `amount = 11`

dp 数组初始化为 `[amount+1] * 12 = [12, 12, ..., 12]`，`dp[0] = 0`

| 金额 i | coin=1 | coin=2 | coin=5 | dp[i] |
|--------|--------|--------|--------|-------|
| 1      | dp[0]+1=0+1=1 | i<2 跳过 | i<5 跳过 | 1 (用 1 个 1 元) |
| 2      | dp[1]+1=2 | dp[0]+1=1 | i<5 跳过 | 1 (用 1 个 2 元) |
| 3      | dp[2]+1=2 | dp[1]+1=2 | i<5 跳过 | 2 (1+2 或 2+1) |
| 4      | dp[3]+1=3 | dp[2]+1=2 | i<5 跳过 | 2 (2+2) |
| 5      | dp[4]+1=3 | dp[3]+1=3 | dp[0]+1=1 | 1 (用 1 个 5 元) |
| 6      | dp[5]+1=2 | dp[4]+1=3 | dp[1]+1=2 | 2 (5+1 或 1+5) |
| 7      | dp[6]+1=3 | dp[5]+1=2 | dp[2]+1=2 | 2 (5+2) |
| 8      | dp[7]+1=3 | dp[6]+1=3 | dp[3]+1=3 | 3 (5+2+1) |
| 9      | dp[8]+1=4 | dp[7]+1=3 | dp[4]+1=3 | 3 (5+2+2) |
| 10     | dp[9]+1=4 | dp[8]+1=4 | dp[5]+1=2 | 2 (5+5) |
| 11     | dp[10]+1=3 | dp[9]+1=4 | dp[6]+1=3 | 3 (5+5+1) |

结果: `dp[11] = 3`，用 3 个硬币（5 + 5 + 1 或 5 + 2 + 2 + 1 + 1... 最少是 5+5+1=3）。

## 💻 解法

> 标准的自底向上 DP，外循环金额内循环硬币。

```python
from typing import List


class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        # 时间复杂度 O(n × amount)，空间复杂度 O(amount)

        # dp[i] 表示凑成金额 i 所需的最少硬币数
        # 初始化为一个大数（amount + 1 表示不可能）
        dp = [amount + 1] * (amount + 1)
        dp[0] = 0  # 金额 0 需要 0 个硬币

        for i in range(1, amount + 1):
            for coin in coins:
                if i - coin >= 0:
                    # 当前金额 i 可以由 i-coin 加上一个 coin 得到
                    dp[i] = min(dp[i], dp[i - coin] + 1)

        # 如果 dp[amount] 还是初始值，说明无法凑成
        return dp[amount] if dp[amount] != amount + 1 else -1
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n * amount) | 外循环 amount 次，内循环 n = len(coins) 次 |
| 空间 | O(amount) | dp 数组长度为 amount + 1 |

## ⚠ 易错点

- [ ] dp 数组初始化为 `amount + 1`，不能初始化为 `float('inf')`——除了影响后续的比较操作外，用 `amount+1` 的好处是：任何合法解最多需要 amount 个硬币（全用面额 1），所以 `amount+1` 严格大于所有合法解
- [ ] 返回时需要判断 `dp[amount] != amount + 1`（或 `dp[amount] == amount + 1` 则返回 -1），不能直接返回 `dp[amount]`，因为可能凑不出来
- [ ] `dp[0]` 必须设为 0。如果忘记初始化，`dp[1] = dp[0] + 1 = amount+1 + 1 = amount+2`，会导致所有 dp 值都错误
- [ ] 循环顺序不能随意交换：外循环金额、内循环硬币——这是"组合"问题的顺序。如果反过来（外循环硬币、内循环金额），结果也是正确的，但这是"排列"的写法。对于最少硬币数问题，两种顺序都正确
- [ ] 内循环检查 `if i >= coin` 或 `if i - coin >= 0`，不能少这个保护条件否则会数组越界
