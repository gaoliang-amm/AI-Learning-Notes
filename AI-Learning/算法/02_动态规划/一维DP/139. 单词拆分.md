# 139. 单词拆分

## 🎯 人话翻译

> 给你一个字符串 s 和一个单词列表 wordDict，判断字符串 s 是否可以被拆分成若干个单词，每个单词都必须在 wordDict 中出现过。单词可以重复使用。

## 💡 核心思想

> **用动态规划判断字符串前缀是否可拆分。**
>
> `dp[i]` 表示字符串 s 的前 i 个字符（即 `s[0:i]`）能否被 wordDict 中的单词拼出来。对于每个位置 i，尝试在所有可能的分割点 j 处切开：如果 `s[0:j]` 可拆分，且 `s[j:i]` 在字典中，那么 `dp[i] = True`。即 `dp[i] = any(dp[j] and s[j:i] in wordSet for j in range(i))`。
>
> 将 wordDict 转成 Set 以 O(1) 时间判断单词是否存在。

用到：[[动态规划]]、[[一维DP]]

## 📝 解题步骤

1. **转集合** — 将 `wordDict` 转为 `set` 以便 O(1) 查找
2. **初始化 dp 数组** — `dp[0] = True`（空字符串可以被拆分），其余为 `False`
3. **双重循环填表** — 外层 `i` 从 1 到 n（前缀长度），内层 `j` 从 0 到 i-1（分割点位置）。若 `dp[j] == True` 且 `s[j:i]` 在集合中，则 `dp[i] = True`，`break`（找到一个就行，无需继续尝试）
4. **返回结果** — `dp[n]` 就是答案

## 🔢 手动走一遍

输入: `s = "leetcode"`, `wordDict = ["leet", "code"]`

dp 数组（长度 n+1 = 9）：

| i | s[0:i]  | 遍历 j | dp[i] | 原因 |
|---|---------|--------|-------|------|
| 0 | ""      | -      | True  | 空字符串可拆分 |
| 1 | "l"     | j=0: s[0:1]="l" 不在字典 | False | 无匹配 |
| 2 | "le"    | j=0: "le" 不在; j=1: dp[1]=False 跳过 | False | 无匹配 |
| 3 | "lee"   | j=0..2: 都不匹配 | False | 无匹配 |
| 4 | "leet"  | j=0: dp[0]=True, s[0:4]="leet" 在字典 | True | dp[0] + "leet" |
| 5 | "leetc" | j=0..4: s[j:5] 无一在字典 | False | 无匹配 |
| 6 | "leetco"| j=0..5: 无一在字典 | False | 无匹配 |
| 7 | "leetcod"| j=0..6: 无一在字典 | False | 无匹配 |
| 8 | "leetcode" | j=4: dp[4]=True, s[4:8]="code" 在字典 | True | dp[4] + "code" |

结果: `dp[8] = True`，可以拆分。

## 💻 解法一（基础 DP）

> 标准的二维遍历写法，思路清晰。

```python
from typing import List


class Solution:
    def wordBreak(self, s: str, wordDict: List[str]) -> bool:
        # 时间复杂度 O(n²)，空间复杂度 O(n)

        word_set = set(wordDict)
        n = len(s)

        # dp[i] 表示 s[0:i]（前 i 个字符）能否被拆分
        dp = [False] * (n + 1)
        dp[0] = True  # 空字符串可以被拆分

        for i in range(1, n + 1):
            # 尝试所有可能的分割点 j
            for j in range(i):
                # 如果 s[0:j] 可拆分，且 s[j:i] 在字典中
                if dp[j] and s[j:i] in word_set:
                    dp[i] = True
                    break

        return dp[n]
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n²) | 外层 n 次，内层最多 n 次，每次子串切片和查找 O(1) |
| 空间 | O(n) | dp 数组长度 n+1，wordDict 集合大小 O(k) |

## ⚠ 易错点

- [ ] dp 数组的长度是 `n + 1`（不是 n），因为 `dp[0]` 表示空字符串。循环时 `i` 从 1 到 n（包括 n）
- [ ] `dp[0]` 必须初始化为 `True`，这是递归的基础：空字符串总是可以被拆分
- [ ] 内层 `j` 的范围是 `range(i)`（即 0 到 i-1），不能写成 `range(i+1)`，因为 `s[j:i]` 中 j 是分割点，i 是前缀长度
- [ ] 子串 `s[j:i]` 的 Python 切片是半开区间 `[j, i)`，对于长度为 i 的前缀，子串到索引 i-1
- [ ] 找到一种拆分方式后要用 `break` 跳出，否则继续遍历浪费时间和可能改变结果

## 🎤 面试怎么说

```text
这道题是动态规划判断字符串是否可拆分的经典题。

核心思路：dp[i] 表示 s 的前 i 个字符能否用 wordDict 拼出来。
对于位置 i，枚举分割点 j（0 <= j < i）：
如果 s[0:j] 可拆分（dp[j]=True）且 s[j:i] 在字典中，那么 dp[i]=True。

具体做法：
- 把 wordDict 转成 Set 方便 O(1) 查找
- dp[0] = True（空字符串）
- 双重循环：i 从 1 到 n，j 从 0 到 i-1
- 找到一种可行的拆分就 break，继续下一个 i

时间复杂度 O(n²)，空间复杂度 O(n)。

如果是面试，还可以提到：
- 可以用 Trie 优化字典查找的常数时间（但在 Python 中 Set 足够）
- 可以用记忆化递归（DFS + memo）作为另一种解法
```
