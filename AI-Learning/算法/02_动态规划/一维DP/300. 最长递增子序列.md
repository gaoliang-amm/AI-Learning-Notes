# 300. 最长递增子序列

## 🎯 人话翻译

> 给一个整数数组，找出最长的严格递增子序列（不要求连续，但顺序不能变），返回其长度。

## 💡 核心思想

> **定义以每个元素结尾的最长递增子序列，然后取最大值。**
>
> `dp[i]` 表示以 `nums[i]` 结尾的最长递增子序列的长度。遍历 i 之前的所有位置 j，如果 `nums[j] < nums[i]`，说明 `nums[i]` 可以接在 `nums[j]` 后面，形成更长的递增子序列。所以 `dp[i] = max(dp[i], dp[j] + 1)` 对所有 `j < i` 且 `nums[j] < nums[i]`。
>
> 最终答案为 `max(dp)`，因为最长子序列不一定以最后一个元素结尾。
>
> 此解法为 O(n²)。进阶可以用二分查找优化到 O(n log n)。

用到：[[动态规划]]、[[一维DP]]

## 📝 解题步骤

1. **初始化** — `dp = [1] * n`，每个元素自身就是一个长度为 1 的递增子序列
2. **双重循环** — 外层 `i` 从 0 到 n-1，内层 `j` 从 0 到 i-1
3. **状态转移** — 如果 `nums[j] < nums[i]`，说明 `nums[i]` 可以接在 `nums[j]` 后面，`dp[i] = max(dp[i], dp[j] + 1)`
4. **返回结果** — `max(dp)`，因为最长子序列可能以任意元素结尾

## 🔢 手动走一遍

输入: `nums = [10, 9, 2, 5, 3, 7, 101, 18]`

dp 初始化为 `[1, 1, 1, 1, 1, 1, 1, 1]`

| i | nums[i] | 检查 j | dp[i] 更新过程 | 最终 dp[i] | 以 i 结尾的 LIS |
|---|---------|--------|---------------|------------|----------------|
| 0 | 10      | - | 初始 1 | 1 | [10] |
| 1 | 9       | j=0: 10<9? No | - | 1 | [9] |
| 2 | 2       | j=0: 10<2? No; j=1: 9<2? No | - | 1 | [2] |
| 3 | 5       | j=0: 10<5? No; j=1: 9<5? No; j=2: 2<5? Yes -> dp[3]=max(1,dp[2]+1)=2 | dp[2]+1=2 | 2 | [2, 5] |
| 4 | 3       | j=0,1 No; j=2: 2<3? Yes -> 1+1=2; j=3: 5>3? No | dp[2]+1=2 | 2 | [2, 3] |
| 5 | 7       | j=2: 2<7 -> 2; j=3: 5<7 -> 3; j=4: 3<7 -> 3 | max(2,3,3)=3 | 3 | [2, 5, 7] 或 [2, 3, 7] |
| 6 | 101     | j=0..5: 取 max(dp[j]+1) for all nums[j]<101 | dp[5]+1=4 | 4 | [2, 5, 7, 101] 或 [2, 3, 7, 101] |
| 7 | 18      | j=0..6: nums[j]<18 的有 j=2,3,4,5 -> max(dp[j]+1) | dp[5]+1=4 | 4 | [2, 5, 7, 18] |

`max(dp) = 4`，最长递增子序列长度为 4。

## 💻 解法一（基础 DP — O(n²)）

> 标准解法，思路清晰，适合面试首选。每个位置往前找所有更小的数。

```python
from typing import List


class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        # 时间复杂度 O(n²)，空间复杂度 O(n)

        if not nums:
            return 0

        n = len(nums)
        # dp[i] 表示以 nums[i] 结尾的最长递增子序列长度
        dp = [1] * n

        for i in range(n):
            # 遍历 i 之前的所有元素
            for j in range(i):
                if nums[j] < nums[i]:
                    # 如果 nums[j] 可以接在 nums[i] 前面
                    dp[i] = max(dp[i], dp[j] + 1)

        return max(dp)
```

## 🚀 解法二（二分优化 — O(n log n)）

> 维护一个 `tails` 数组，`tails[k]` 表示长度为 k+1 的递增子序列的最小结尾元素。贪心地更新 tails。

```python
from typing import List
import bisect


class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        # 时间复杂度 O(n log n)，空间复杂度 O(n)

        tails = []  # tails[k] = 长度为 k+1 的 IS 的最小结尾元素

        for num in nums:
            # 在 tails 中二分查找第一个 >= num 的位置
            idx = bisect.bisect_left(tails, num)

            if idx == len(tails):
                # num 比所有 tails 都大，可以扩展 LIS
                tails.append(num)
            else:
                # 用 num 替换 tails[idx]（贪心：结尾越小，后面越容易接）
                tails[idx] = num

        return len(tails)
```

## ⏱ 复杂度

| 版本 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 基础 DP | O(n²) | O(n) | 面试首选，简单直观；n <= 2500 |
| 二分优化 | O(n log n) | O(n) | n 很大时；需要理解 tails 数组的含义 |

## ⚠ 易错点

- [ ] `dp[i]` 初始化为 1 而不是 0，因为每个元素自身就是一个长度为 1 的递增子序列
- [ ] 最终返回 `max(dp)` 而不是 `dp[-1]`。因为最长递增子序列不一定以最后一个元素结尾
- [ ] 内层循环 `j` 的范围是 `0` 到 `i-1`，写成 `range(i)` 即可（不包含 i），不要写成 `range(i+1)` 或 `range(n)`
- [ ] 判断条件是 `nums[j] < nums[i]`（严格递增），不能写成 `<=`，因为题目要求严格递增
- [ ] 二分优化解法中，`tails` 数组不一定是实际的 LIS，只是长度等于 LIS 的长度。面试时记得解释这一点
