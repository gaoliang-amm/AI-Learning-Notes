# 70. 爬楼梯

## 🎯 人话翻译

> 走楼梯，每次走 1 步或 2 步。问到第 n 级有多少种走法。

## 💡 核心思想

> **本质是斐波那契数列问题。**
>
> 到第 n 级的方法数 = 到第 n-1 级的方法数 + 到第 n-2 级的方法数
>
> 因为最后一步只能从 n-1（走 1 步）或 n-2（走 2 步）过来。

用到：[[动态规划]]

## 📝 解题步骤

1. **定义状态** — `f(n)` = 走到第 n 级的方法数
2. **找递推关系** — `f(n) = f(n-1) + f(n-2)`，因为最后一步只有两种可能
3. **初始化** — `f(1) = 1`（走 1 步），`f(2) = 2`（1+1 或 2）
4. **递推到 n** — 用两个变量滚动，避免 O(n) 空间

## 🔢 手动走一遍

输入: n = 5

| 台阶 n | 走法数 | 计算过程      | 具体走法                         |
|--------|--------|---------------|----------------------------------|
| 1      | 1      | 初始          | 1                                |
| 2      | 2      | 初始          | 1+1, 2                           |
| 3      | 3      | 1+2           | 1+1+1, 1+2, 2+1                  |
| 4      | 5      | 2+3           | 1+1+1+1, 1+1+2, 1+2+1, 2+1+1, 2+2 |
| 5      | 8      | 3+5           | ...8 种                          |

## 💻 解法一（基础 DP — O(n) 空间）

> 用数组存下每一步的结果，思路直观。

```python
def climbStairs(n: int) -> int:
    # 边界
    if n <= 2:
        return n

    # dp[i] = 走到第 i 级的方法数
    dp = [0] * (n + 1)
    dp[1] = 1
    dp[2] = 2

    for i in range(3, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]

    return dp[n]
```

## 🚀 解法二（空间优化 — O(1) 空间）

> 观察递推公式：每次只需要前两个值。用两个变量滚动即可。

```python
def climbStairs(n: int) -> int:
    if n <= 2:
        return n

    # a = f(n-2), b = f(n-1)
    a, b = 1, 2

    for _ in range(3, n + 1):
        a, b = b, a + b

    return b
```

## ⏱ 复杂度对比

| 版本 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 基础 DP | O(n) | O(n) | 需要打印具体走法时 |
| 空间优化 | O(n) | O(1) | 面试推荐，只需求数量 |

## ⚠ 易错点

- [ ] 递归解法会超时（指数级），面试时不要用
- [ ] n=1 和 n=2 的初始值容易写错，`a, b = 1, 2` 不是 `0, 1`
- [ ] 数组版 `dp` 的下标：数组长度为 `n+1`，`dp[0]` 不使用

## 🎤 面试怎么说

```text
这道题是动态规划的入门题，本质是斐波那契数列。

核心思路：走到第 n 级，最后一步要么从 n-1 跨 1 步上来，
要么从 n-2 跨 2 步上来，所以 f(n) = f(n-1) + f(n-2)。

用两个变量 a、b 滚动递推，时间复杂度 O(n)，空间 O(1)。

如果面试官问能不能更快，可以用矩阵快速幂优化到 O(log n)，
但一般题不需要。
```
