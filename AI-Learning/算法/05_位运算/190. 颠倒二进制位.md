# 190. 颠倒二进制位

## 🎯 人话翻译

> 把一个 32 位无符号整数的二进制位前后颠倒，返回颠倒后的整数。

## 💡 核心思想

> **逐位反转法：** 每次取出 n 的最低位（`n & 1`），把它放到结果 result 的最低位，然后 result 左移一位（为下一次腾位置），n 右移一位（准备取下一个位）。循环 32 次即可完成全部反转。
>
> **本质是位操作的「先进后出」——每轮从 n 的末尾取一位，塞到 result 的末尾，经过 32 轮后 result 完全颠倒。**

用到：[[位运算]]

## 📝 解题步骤

1. **初始化结果** — `res = 0`，用于存放颠倒后的结果
2. **循环 32 次** — 处理 32 位中的每一位
3. **每次循环四步操作：**
   - `res <<= 1` — 结果左移一位，空出最低位给新位
   - `bit = n & 1` — 取出 n 当前的最低位
   - `res |= bit` — 把取出的位放入 res 的最低位
   - `n >>= 1` — n 右移一位，丢弃已处理的最低位
4. **返回 res** — 32 次循环后，所有位都已反转

## 🔢 手动走一遍

输入: `n = 43261596`，二进制（只展示后 8 位作为示例）：

为简化展示，取 n 的低 8 位 `10101100`（实际 n 有 32 位，但原理相同）。

| 轮次 | 操作前 n (低8位) | bit (n&1) | res<<=1 | res|=bit | n>>=1 后 |

| 1 | 10101100 | 0 | 00000000 | 00000000 | 01010110 |
| 2 | 01010110 | 0 | 00000000 | 00000000 | 00101011 |
| 3 | 00101011 | 1 | 00000000 | 00000001 | 00010101 |
| 4 | 00010101 | 1 | 00000010 | 00000011 | 00001010 |
| 5 | 00001010 | 0 | 00000110 | 00000110 | 00000101 |

| 8 | 00000001 | 1 | 00110100 | 00110101 | 00000000 |

原始低 8 位 `10101100` 反转后变为 `00110101`。实际对 32 位处理后即为答案。

## 💻 解法

> 每次取 n 的最低位，放入结果的高位（通过不断左移实现），循环 32 次。

```python
class Solution:
    def reverseBits(self, n: int) -> int:
        """
        颠倒 32 位无符号整数的二进制位。
        使用循环法，每次取 n 的最低位，放入结果的高位。
        """
        res = 0
        # 处理 32 个位
        for _ in range(32):
            # 1. 结果左移一位，空出最低位
            res <<= 1
            # 2. 取出 n 当前的最低位
            bit = n & 1
            # 3. 将该位放入 res 的最低位
            res |= bit
            # 4. n 右移一位，准备处理下一个位
            n >>= 1
        return res
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(1) | 固定循环 32 次，与输入大小无关 |
| 空间 | O(1) | 只使用了常数个变量 |

## ⚠ 易错点

- [ ] 必须循环完整的 32 次，不能提前终止。因为前导 0 也需要反转到高位
- [ ] 操作顺序不能错：先 `res <<= 1` 再 `res |= bit`，如果顺序反了最高位会丢失
- [ ] `n >>= 1` 在 Python 中是算术右移（保留符号位），但题目给定的是无符号整数，所以对正数没问题
- [ ] 不要使用 `bin(n)[2:].zfill(32)[::-1]` 之类的字符串操作，虽然能过但面试官期望的是位运算法
- [ ] 分治法也可以：交换奇偶位、交换半字节、交换字节等，但循环法最直观

## 🎤 面试怎么说

```text
这道题要求颠倒 32 位整数的二进制位。

核心思路是逐位反转：用一个循环处理 32 次。
每次迭代做四件事：
1. 结果 res 左移一位，空出最低位
2. 用 n & 1 取出 n 的当前最低位
3. 用 res |= bit 将这一位放到 res 的最低位
4. n 右移一位，准备处理下一位

经过 32 轮之后，原数的最低位被放到了结果的最高位，
最高位被放到了结果的最低位，完成反转。

由于固定循环 32 次，时间和空间复杂度都是 O(1)。

如果面试官要求优化，还可以提到分治法：
先交换相邻的奇偶位，再交换相邻的 2 位组，再交换 4 位组，
最后交换 16 位组，可以在 O(log 32) = 5 步内完成，不需要循环 32 次。
但面试中循环法已经足够了。
```
