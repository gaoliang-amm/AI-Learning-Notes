# 137. 只出现一次的数字 II

## 🎯 人话翻译

> 数组中其他数都出现了三次，只有一个数出现一次。找出这个唯一出现一次的数。

## 💡 核心思想

> **逐位统计法（更容易理解）：**
> 统计每个二进制位上 1 出现的总次数。如果某个位上 1 出现的次数是 3 的倍数，说明出现一次的数在该位是 0；如果次数 mod 3 = 1，说明该位是 1。
>
> **有限状态自动机法（更优，O(1) 空间）：**
> 用两个变量 `ones` 和 `twos` 追踪每一位上 1 出现的次数（对 3 取模）。`ones` 记录出现 1 次的位，`twos` 记录出现 2 次的位。当某位出现 3 次时，自动清零回到 0。最终 `ones` 就是只出现一次的数。

用到：[[位运算]]、[[有限状态自动机]]

## 📝 解题步骤

### 方法一：逐位统计（清晰易懂）

1. **遍历 32 个位** — 对 int 类型的每一位（0 到 31）分别处理
2. **统计当前位上 1 的总数** — 遍历数组每个数，用 `(num >> i) & 1` 检查该位是否为 1
3. **对 3 取模** — 如果 `count % 3 == 1`，说明结果在该位是 1，用 `|` 设置该位
4. **返回结果** — 32 位统计完毕后，结果就是答案

### 方法二：状态机（面试亮点）

1. **初始化** — `ones = twos = 0`
2. **遍历每个数** — 对每个 `num`，更新状态：
   - `ones = (ones ^ num) & ~twos` — 只在还没有出现两次的位上更新 ones
   - `twos = (twos ^ num) & ~ones` — 只在还没有被 ones 标记的位上更新 twos
3. **返回 ones** — 最终 ones 即为只出现一次的数

## 🔢 手动走一遍

输入: `nums = [2, 2, 3, 2]`，二进制：2 = `010`, 3 = `011`

### 逐位统计法

| 位 | num[0]=2 | num[1]=2 | num[2]=3 | num[3]=2 | 总计 | mod 3 | 结果该位 |
|----|----------|----------|----------|----------|------|-------|---------|
| 0 | 0 | 0 | 1 | 0 | 1 | 1 | 1 |
| 1 | 1 | 1 | 1 | 1 | 4 | 1 | 1 |
| 2 | 0 | 0 | 0 | 0 | 0 | 0 | 0 |

结果 = `011` = 3。正确。

### 状态机法

| 步骤 | num | ones 变化 | twos 变化 | ones | twos | 含义 |
|------|-----|-----------|-----------|------|------|------|
| 初始 | - | - | - | 0 (000) | 0 (000) | 还没开始 |
| 1 | 2 (010) | ones = (0^2)&~0 = 2 | twos = (0^2)&~2 = 0 | 2 (010) | 0 (000) | 最低位出现 1 次 |
| 2 | 2 (010) | ones = (2^2)&~0 = 0 | twos = (0^2)&~0 = 2 | 0 (000) | 2 (010) | 最低位出现 2 次 |
| 3 | 3 (011) | ones = (0^3)&~2 = 1 | twos = (2^3)&~1 = 0 | 1 (001) | 0 (000) | 3 的第 0,1 位由不同频率组成 |
| 4 | 2 (010) | ones = (1^2)&~0 = 3 | twos = (0^2)&~3 = 0 | 3 (011) | 0 (000) | ones=3，即结果 |

最终 `ones = 3`，正确。

## 💻 解法

### 解法一：逐位统计

> 对每一位分别统计 1 的出现次数，对 3 取模后设置结果位。

```python
from typing import List


class Solution:
    def singleNumber(self, nums: List[int]) -> int:
        """
        逐位统计法：统计每个位上 1 的出现次数，对 3 取模。
        时间复杂度: O(32n) = O(n)
        空间复杂度: O(1)
        """
        result = 0
        for i in range(32):
            # 统计第 i 位上 1 的个数
            count = 0
            for num in nums:
                count += (num >> i) & 1
            # 如果该位 1 的个数不是 3 的倍数，说明结果在该位是 1
            if count % 3 != 0:
                # Python 处理负数：第 31 位是符号位
                if i == 31:
                    result -= (1 << 31)  # 处理负数的符号位
                else:
                    result |= (1 << i)
        return result
```

### 解法二：有限状态自动机

> 用 ones 和 twos 两个变量追踪每个位上出现 1 次和 2 次的状态。出现 3 次时自动清零。

```python
from typing import List


class Solution:
    def singleNumber(self, nums: List[int]) -> int:
        """
        找出数组中只出现一次的元素（其余出现三次）。
        使用有限状态自动机：ones 和 twos 记录每一位出现 1 次和 2 次的情况。
        时间复杂度: O(n)
        空间复杂度: O(1)
        """
        ones = 0
        twos = 0
        for num in nums:
            # 更新 ones：如果该位在 twos 中已经出现，则不计入；否则按位异或
            ones = (ones ^ num) & ~twos
            # 更新 twos：如果该位在 ones 中已经出现（即新的 ones 中该位为1），则计入 twos
            twos = (twos ^ num) & ~ones
        return ones
```

## ⏱ 复杂度

| 版本 | 时间 | 空间 | 说明 |
|------|------|------|------|
| 逐位统计 | O(32n) = O(n) | O(1) | 更容易理解，推荐面试先讲这个 |
| 状态机 | O(n) | O(1) | 单次遍历，更高效，面试加分项 |

## ⚠ 易错点

- [ ] 状态机法中 `ones` 和 `twos` 的更新顺序不能颠倒：先更新 ones，再更新 twos
- [ ] 状态机更新公式 `ones = (ones ^ num) & ~twos` 中的 `~twos` 不能漏，否则出现三次的位不会清零
- [ ] 逐位统计法需要注意 Python 中负数的处理，第 31 位是符号位，不能直接 `|=`
- [ ] 不要试图用哈希表（虽然也能做出来），面试官期望的是位运算的 O(1) 空间解法
- [ ] 区分本题（出现三次）和 [[136. 只出现一次的数字]]（出现两次），前者无法直接用简单异或
