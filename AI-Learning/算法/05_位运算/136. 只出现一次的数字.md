# 136. 只出现一次的数字

## 🎯 人话翻译

> 数组中所有数都出现了两次，只有一个数出现了一次。找出这个落单的数。

## 💡 核心思想

> **利用异或运算（XOR）的三个性质：**
>
> 1. `a ^ a = 0` — 相同数字异或抵消
> 2. `a ^ 0 = a` — 任何数与 0 异或等于自身
> 3. 交换律和结合律 — 异或顺序不影响结果
>
> 把所有数字全部异或起来，成对的数字互相抵消为 0，最终只剩下那个出现一次的数字。

用到：[[位运算]]

## 📝 解题步骤

1. **初始化结果** — `result = 0`，作为异或的起始值（0 异或任何数等于该数本身）
2. **遍历数组** — 对每个元素执行 `result ^= num`
3. **利用性质** — 相同的数字成对出现，`a ^ a = 0`，它们互相抵消
4. **返回结果** — 遍历结束后，`result` 就是只出现一次的那个数

## 🔢 手动走一遍

输入: `nums = [4, 1, 2, 1, 2]`

| 步骤 | 当前数字 | 异或运算 | result 的二进制 | result |
|------|---------|---------|-----------------|--------|
| 初始 | - | - | 000 | 0 |
| 1 | 4 | 0 ^ 4 | 100 | 4 |
| 2 | 1 | 4 ^ 1 | 101 | 5 |
| 3 | 2 | 5 ^ 2 | 111 | 7 |
| 4 | 1 | 7 ^ 1 | 110 | 6 |
| 5 | 2 | 6 ^ 2 | 100 | 4 |

最终 `result = 4`，即落单的数字。成对的 1 和 2 都在异或过程中抵消了。

## 💻 解法

> 遍历数组，对每个元素执行异或。成对出现的数字互相抵消，落单的数字最终留在 result 中。

```python
from typing import List


class Solution:
    def singleNumber(self, nums: List[int]) -> int:
        """
        找出数组中只出现一次的元素。
        使用异或运算：a ^ a = 0, a ^ 0 = a，且异或满足交换结合律。
        所有数字异或后，成对出现的抵消，剩下的即为所求。

        时间复杂度: O(n)
        空间复杂度: O(1)
        """
        result = 0
        for num in nums:
            result ^= num
        return result
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 遍历一次数组，每个元素执行一次异或 |
| 空间 | O(1) | 只使用了一个变量 result，没有额外空间 |

## ⚠ 易错点

- [ ] 初始化 `result = 0` 而非 `result = nums[0]`，因为 0 是异或的单位元
- [ ] 不要用哈希表统计频率再遍历，虽然也能 O(n) 但空间是 O(n)，面试官期望 O(1) 空间的位运算解法
- [ ] 异或满足交换律和结合律，不需要先排序，直接遍历即可
- [ ] 如果题目变为其他数出现奇数次（非 2 次），需要考虑其他方法
- [ ] 注意 Python 中 `^` 是异或运算符，不是乘方（乘方是 `**`）

## 🎤 面试怎么说

```text
这道题考察的是对异或运算性质的理解。

核心思路：异或运算有三个关键性质——a ^ a = 0，a ^ 0 = a，以及交换律和结合律。

因为除了一个数出现一次外，其他所有数都恰好出现两次，
所以我把所有数字全部异或起来，成对的数字会互相抵消变成 0，
最终剩下的就是那个只出现一次的数字。

只需要遍历一次数组，用一个变量累计异或结果即可。
时间复杂度 O(n)，空间复杂度 O(1)。

这个解法很巧妙，不需要额外的哈希表空间。
如果面试官问扩展，比如其他数出现三次怎么办，
那就需要用状态机的方法，或者统计每个二进制位上 1 的个数对 3 取模。
```
