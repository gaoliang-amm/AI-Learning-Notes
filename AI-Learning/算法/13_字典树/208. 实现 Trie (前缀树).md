# 208. 实现 Trie (前缀树)

## 🎯 人话翻译

> 实现一个前缀树（Trie），支持三个操作：插入单词、搜索单词是否完整存在、搜索是否有单词以某前缀开头。

## 💡 核心思想

> **Trie（前缀树）是一种树形数据结构，每个节点代表一个字符，从根到某节点的路径就是一个前缀。**
>
> - 每个节点有一个长度为 26 的 children 数组（对应 a-z），值是指向子节点的指针
> - 每个节点有一个 `is_end` 布尔标记，表示从根到这里是否构成一个完整单词
>
> **核心操作：**
> - **insert**：逐字符往下走，没有对应子节点就创建，最后标记 `is_end = True`
> - **search**：逐字符往下走，走完所有字符后检查 `is_end` 是否为 True
> - **startsWith**：逐字符往下走，能在树中走完就说明有这个前缀

用到：[[字典树]]、[[Trie]]

## 📝 解题步骤

### 数据结构

1. **TrieNode 节点**：
   - `children[26]`：26 个字母的子节点指针
   - `is_end`：标记是否为单词结尾

2. **Trie 根节点**：一个空的 TrieNode，不存储任何字符

### insert(word)

1. 从根节点出发，遍历 word 的每个字符
2. 计算字符索引 `idx = ord(ch) - ord('a')`
3. 如果 `node.children[idx]` 为空，创建新节点
4. 移动到子节点 `node = node.children[idx]`
5. 遍历结束后，`node.is_end = True` 标记单词结尾

### search(word)

1. 调用 `_search_prefix(word)` 找到 word 对应的节点
2. 如果节点存在且 `is_end == True`，返回 True
3. 否则返回 False

### startsWith(prefix)

1. 调用 `_search_prefix(prefix)` 找到 prefix 对应的节点
2. 如果节点存在，返回 True（有任意单词以此开头）
3. 否则返回 False


### 搜索 "app"

- root -> a -> p -> p
- 到达 p 节点，`is_end = True`，返回 True

### 搜索 "apl"

- root -> a -> p -> l
- 到达 l 节点，`is_end = True`（因为 "apple" 以 "apl" 中间的 l 结尾... 不对）
- 实际上搜索 "apl"：root -> a -> p -> p -> 然后想找 l，但 p 节点下面没有 l 子节点
- 所以 `search("apl")` 返回 False

### startsWith("ap")

- root -> a -> p
- 找到 p 节点，存在，返回 True

## 💻 解法

> 用 TrieNode 类存储节点，Trie 类实现 insert/search/startsWith。

```python
class TrieNode:
    """Trie 节点类"""

    def __init__(self):
        self.children = [None] * 26  # 26 个小写字母
        self.is_end = False  # 是否为单词结尾


class Trie:
    def __init__(self):
        """初始化 Trie 树，创建根节点"""
        self.root = TrieNode()

    def insert(self, word: str) -> None:
        """
        插入单词 word
        时间复杂度 O(L)，L 为单词长度
        """
        node = self.root
        for ch in word:
            idx = ord(ch) - ord('a')  # 计算字符索引
            if not node.children[idx]:
                node.children[idx] = TrieNode()
            node = node.children[idx]
        node.is_end = True  # 标记单词结尾

    def search(self, word: str) -> bool:
        """
        查找单词 word 是否完整存在
        时间复杂度 O(L)
        """
        node = self._search_prefix(word)
        return node is not None and node.is_end

    def startsWith(self, prefix: str) -> bool:
        """
        查找是否存在以 prefix 为前缀的单词
        时间复杂度 O(L)
        """
        node = self._search_prefix(prefix)
        return node is not None

    def _search_prefix(self, prefix: str) -> TrieNode:
        """
        内部方法：查找 prefix 对应的节点
        如果路径存在返回最后一个节点，否则返回 None
        """
        node = self.root
        for ch in prefix:
            idx = ord(ch) - ord('a')
            if not node.children[idx]:
                return None
            node = node.children[idx]
        return node
```

## ⏱ 复杂度

| 操作 | 时间 | 空间（每次插入）| 说明 |
|------|------|---------------|------|
| insert | O(L) | O(L) | L 是单词长度，每个字符可能创建一个新节点 |
| search | O(L) | O(1) | 只遍历不创建新节点 |
| startsWith | O(L) | O(1) | 与 search 逻辑相同但不检查 is_end |

总空间：O(总字符数 x 26)，最坏情况下每个单词的每个字符都创建一个节点。

## ⚠ 易错点

- [ ] **search 和 startsWith 的区别**：search 需要在路径结束时检查 `is_end` 是否为 True；startsWith 只需要路径存在即可
- [ ] **is_end 标记的时机**：只有完整单词的最后一个字符才标记 `is_end = True`，中间节点的 `is_end` 默认为 False
- [ ] **search 中的空字符串**：如果搜索空字符串，`_search_prefix` 返回 root，`root.is_end` 默认为 False（除非手动设置）
- [ ] **字符索引计算**：`ord(ch) - ord('a')` 必须在 0-25 范围内。如果不是纯小写字母，需要额外处理
- [ ] **children 数组大小**：如果是 ASCII 字符集，children 应该是 128 或 256；如果是小写字母，26 就够了

## 🎤 面试怎么说

```text
Trie（前缀树）是一种专门为字符串搜索设计的数据结构。

数据结构：
- 每个节点包含 children[26] 数组和 is_end 布尔标记
- children[i] 存储对应字母的子节点
- is_end 标记从根到此节点路径是否构成一个完整单词

三个核心操作：
1. insert：逐字符向下遍历，如果子节点不存在就创建，
   遍历完后在当前节点标记 is_end = True
2. search：同 insert 的逻辑向下走，走完所有字符后
   判断 is_end 是否为 True
3. startsWith：和 search 一样走，但只需要路径存在就返回 True，
   不关心 is_end

每个操作的时间复杂度都是 O(L)，L 是单词长度。
空间复杂度取决于插入的单词数量和长度，每个新字符创建一个新节点。

Trie 的优势在于：当有大量字符串且需要频繁做前缀匹配时，
Trie 比哈希表更高效，因为它天然支持前缀搜索。
常见的应用场景包括：自动补全、拼写检查、IP 路由匹配。
```
