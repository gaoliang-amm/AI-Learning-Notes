# 211. 添加与搜索单词 - 数据结构设计

## 🎯 人话翻译

> 设计一个数据结构支持添加单词和搜索单词。搜索时可以包含通配符 `.`，`.` 可以匹配任意一个字母。

## 💡 核心思想

> **Trie（前缀树）+ DFS 处理通配符。**
>
> 添加单词（`addWord`）与标准 Trie 的 insert 完全相同。
>
> 搜索单词（`search`）因为有通配符 `.`，不能简单地逐字符走下去。遇到 `.` 时，需要尝试当前节点的**所有子节点**（最多 26 个），用 DFS/回溯逐一尝试。
>
> 本质是 [[208. 实现 Trie (前缀树)]] 的扩展：search 从 O(L) 的简单遍历变成了可能 O(26^L) 的 DFS 搜索（当 word 全是 `.` 时最坏）。

用到：[[字典树]]、[[Trie]]、[[DFS]]、[[回溯]]

## 📝 解题步骤

### addWord(word)

1. 从根节点出发，逐字符向下走
2. 如果对应子节点不存在，创建新节点
3. 遍历完后，标记 `is_end = True`
4. 与普通 Trie 的 insert 完全一致

### search(word)

1. 调用 `_dfs(node, word, idx)`，idx 表示当前处理到 word 的第几个字符
2. **终止条件**：`idx == len(word)`，返回 `node.is_end`
3. **当前字符 ch = word[idx]**：
   - **如果是 '.'**：遍历 node 的所有子节点（26 个），如果有任何一个子节点存在且 DFS 返回 True，则返回 True
   - **如果是普通字母**：计算索引，去对应的子节点继续 DFS
4. 如果所有分支都返回 False，则返回 False

## 🔢 手动走一遍

### 插入 "bad", "dad", "mad"

Trie 结构：
```
       root
      /  |  \
     b   d   m
     |   |   |
     a   a   a
     |   |   |
     d*  d*  d*   (* 表示 is_end=True)
```

### 搜索 "pad"

- root -> p 不存在，直接返回 False

### 搜索 "bad"

- root -> b -> a -> d，d 节点 `is_end=True`，返回 True

### 搜索 ".ad"

- idx=0，ch='.'，遍历 root 的所有子节点：
  - 尝试 b：b.a.d -> d 节点 is_end=True -> 返回 True（不再继续尝试）
  - （不会继续试 d 和 m，因为已经找到了）

### 搜索 "b.."

- idx=0: ch='b'，进入 b 节点
- idx=1: ch='.'，遍历 b 的所有子节点：
  - 只有 a
  - idx=2: ch='.'，遍历 a 的所有子节点：
    - 只有 d
    - idx=3 == len(word)（3），d.is_end = True -> 返回 True

## 💻 解法

> Trie + DFS：添加与标准 Trie 一致，搜索时遇到 '.' 遍历所有子节点。

```python
class TrieNode:
    """Trie 节点类"""

    def __init__(self):
        self.children = [None] * 26  # 26 个小写字母
        self.is_end = False  # 是否为单词结尾


class WordDictionary:
    def __init__(self):
        """初始化数据结构"""
        self.root = TrieNode()

    def addWord(self, word: str) -> None:
        """
        添加单词到数据结构中
        时间复杂度 O(L)，L 为单词长度
        """
        node = self.root
        for ch in word:
            idx = ord(ch) - ord('a')
            if not node.children[idx]:
                node.children[idx] = TrieNode()
            node = node.children[idx]
        node.is_end = True

    def search(self, word: str) -> bool:
        """
        搜索单词，支持通配符 '.' 匹配任意字母
        时间复杂度 O(26^L) 最坏情况，L 为单词长度
        """
        return self._dfs(self.root, word, 0)

    def _dfs(self, node: TrieNode, word: str, idx: int) -> bool:
        """
        深度优先搜索辅助函数
        :param node: 当前节点
        :param word: 搜索的单词
        :param idx: 当前处理的字符索引
        :return: 是否存在匹配的单词
        """
        # 递归终止条件：处理完所有字符
        if idx == len(word):
            return node.is_end

        ch = word[idx]

        # 情况1：当前字符是通配符 '.'，尝试所有子节点
        if ch == '.':
            for child in node.children:
                if child and self._dfs(child, word, idx + 1):
                    return True
            return False

        # 情况2：当前字符是普通字母，只走对应的子节点
        else:
            idx_child = ord(ch) - ord('a')
            child = node.children[idx_child]
            if not child:
                return False
            return self._dfs(child, word, idx + 1)
```

## ⏱ 复杂度

| 操作 | 时间 | 空间 | 说明 |
|------|------|------|------|
| addWord | O(L) | O(L) | L 为单词长度，插入时可能创建新节点 |
| search（无 '.'）| O(L) | O(L) | 沿着一条路径直走 |
| search（有 '.'）| O(26^K) | O(K) | K 是 '.' 的数量，最坏需遍历所有分支 |

## ⚠ 易错点

- [ ] **DFS 的终止条件**：`idx == len(word)` 时返回 `node.is_end`，不是返回 True。只走到路径终点还不够，必须是某个单词的结尾
- [ ] ** '.' 的处理**：遍历所有 26 个子节点，跳过 None 的。找到一个匹配就返回 True
- [ ] **递归 vs 迭代**：因为有 '.'，search 必须用递归（DFS），不能用简单的迭代
- [ ] **addWord 和 search 用同一个 Trie 结构**：addWord 用标准 Trie insert，search 用 DFS。不要分别建两个数据结构
- [ ] 如果 word 全是 '.'，如 "..."，最坏情况会遍历 26^3 条路径，但因为有 is_end 提前剪枝，实际不会全遍历

## 🎤 面试怎么说

```text
这道题是 Trie（前缀树）的扩展，在标准 Trie 基础上加入了通配符 '.'。

addWord 和标准 Trie 的 insert 完全一样。

关键在 search：当遇到 '.' 时，需要尝试当前节点所有非空子节点。
我用 DFS 来处理：遇到普通字母就走对应子节点，遇到 '.' 就遍历 26 个子节点，
只要有任意一条路径能匹配成功，就返回 True。

最坏情况下，如果单词全由 '.' 组成，时间会是 O(26^L)，其中 L 是单词长度。
但实际中 '.' 通常很少，而且 Trie 的自然剪枝（不存在的路径提前返回 False）
会让搜索很快。

这个设计的优雅之处在于：Trie 天然地限制了搜索范围——
你只能沿着树中实际存在的字符路径走，不会盲目遍历整个字母表。
```
