# 212. 单词搜索 II

## 🎯 人话翻译

> 给你一个二维字母网格和一个单词列表。找出所有在网格中能连出来的单词。每个字母只能用一次，只能上下左右连着走。

## 💡 核心思想

> **Trie（前缀树） + 二维网格 DFS 回溯。**
>
> 朴素思路：对每个单词在网格中做一次 DFS 搜索，但这样效率太低（每个单词都要遍历整个网格）。
>
> **优化：用 Trie 剪枝。** 先把所有单词建成一棵 Trie。然后从网格的每个格子出发做 DFS，同时沿着 Trie 的节点往下走。如果当前路径上的字符在 Trie 中根本不存在对应节点（即没有任何单词以此路径作为前缀），直接剪枝返回。这样一次遍历网格就能同时搜索所有单词。
>
> **找到单词后的处理**：将 `node.word` 置为 None，避免同一个单词被重复加入结果。

用到：[[字典树]]、[[Trie]]、[[DFS]]、[[回溯]]

## 📝 解题步骤

1. **建 Trie** — 将所有单词插入 Trie。和标准 Trie 不同的是，在叶子节点存 `node.word = word`（完整单词），而不是简单的 `is_end = True`
2. **遍历网格每个格子** — 从每个格子出发，调用 `_dfs(i, j, root)` 开始搜索
3. **DFS 搜索函数**：
   - **越界/已访问检查**：`i < 0 or i >= m or j < 0 or j >= n`，`board[i][j] == '#'`
   - **Trie 匹配检查**：`node.children[idx]` 是否存在，不存在则剪枝
   - **找到单词**：如果 `node.word` 不为空，加入结果，并将 `node.word = None`（去重）
   - **标记已访问**：`board[i][j] = '#'`
   - **四方向递归**：上下左右
   - **回溯恢复**：`board[i][j] = ch`
4. **返回结果**

## 🔢 手动走一遍

`board = [["o","a","a","n"],["e","t","a","e"],["i","h","k","r"],["i","f","l","v"]]`, `words = ["oath","pea","eat","rain"]`

### 建 Trie

```
         root
        /  |  \
       o   p   e   r
       |   |   |   |
       a   e   a   a
       |   |   |   |
       t   a*  t   i
       |       |   |
       h*      |   n*
               (pea不在网格中，但Trie里存了)
```

`*` 标记处 `node.word = 对应单词`。

### DFS 从 (0,0)='o' 出发

```
board:
o a a n
e t a e
i h k r
i f l v
```

从 (0,0)='o'：
- root.children['o'-'a'] 存在 -> node 移动到 'o'
- (0,0) 标记 '#'：
  - 右 (0,1)='a'：node.children['a'-'a'] 存在 -> 继续
    - (0,1) 标记 '#'：
      - 右 (0,2)='a'：node.children['a'-'a'] 存在（但 "oaa" 不是前缀...等一下，Trie 中只有 "oath"，"oa" 是有效前缀，"oaa" 不是）-> node 是 'a'，children['a'-'a'] 不存在 -> 剪枝
      - 下 (1,1)='t'：node.children['t'-'a'] 存在 -> 继续
        - (1,1) 标记 '#'：
          - 右 (1,2)='a'：检查 'h' 路径...
          - 下 (2,1)='h'：node.children['h'-'a'] 存在 -> node 此时 `node.word = "oath"`！加入结果
          - node.word = None（避免重复）
        - 回溯：(1,1) 恢复 't'
    - 回溯：(0,1) 恢复 'a'
  - ...继续其他方向

最终找到的单词：`["oath", "eat"]`。

## 💻 解法

> 先用所有单词建 Trie，然后 DFS 遍历网格同时沿 Trie 走，Trie 做剪枝。

```python
from typing import List


class TrieNode:
    """
    Trie（前缀树）节点类

    属性说明：
        children: 长度为 26 的数组，存储当前节点的子节点
                 索引 0 对应字母 'a'，索引 25 对应字母 'z'
                 值为 None 表示该字母对应的子节点不存在
                 值为 TrieNode 对象表示存在该子节点
        word: 存储单词本身，只有从根节点到当前节点组成的单词在 words 中时才有值
              例如：插入 "oath" 后，在 'h' 节点上 word = "oath"
    """

    def __init__(self):
        self.children = [None] * 26
        self.word = None


class Solution:
    def findWords(self, board: List[List[str]], words: List[str]) -> List[str]:
        """
        在网格中查找所有出现在 words 中的单词

        参数：
            board: m x n 的字符网格
            words: 待查找的单词列表

        返回：
            在网格中能找到的单词列表
        """
        # 边界条件：网格或单词列表为空，直接返回空列表
        if not board or not board[0] or not words:
            return []

        # ========== 第一步：构建 Trie 树 ==========
        root = TrieNode()
        for word in words:
            node = root
            # 遍历单词的每个字符
            for ch in word:
                # 将字符转换为数组索引（'a' -> 0, 'b' -> 1, ..., 'z' -> 25）
                idx = ord(ch) - ord('a')
                # 如果当前字符对应的子节点不存在，则创建新节点
                if not node.children[idx]:
                    node.children[idx] = TrieNode()
                # 移动到子节点，继续处理下一个字符
                node = node.children[idx]
            # 单词所有字符处理完毕，在当前节点记录完整的单词
            node.word = word

        # 获取网格的尺寸
        m, n = len(board), len(board[0])
        # 存储最终结果
        result = []

        # ========== 第二步：从每个单元格开始 DFS 搜索 ==========
        for i in range(m):
            for j in range(n):
                self._dfs(board, i, j, root, result)

        return result

    def _dfs(self, board: List[List[str]], i: int, j: int, node: TrieNode, result: List[str]) -> None:
        """
        深度优先搜索，在网格中查找单词

        参数：
            board: 字符网格
            i: 当前行索引
            j: 当前列索引
            node: 当前 Trie 节点（表示已经匹配的前缀）
            result: 结果列表，用于收集找到的单词
        """
        # ========== 边界检查 ==========
        # 如果当前坐标超出网格范围，直接返回
        if i < 0 or i >= len(board) or j < 0 or j >= len(board[0]):
            return

        # 获取当前单元格的字符
        ch = board[i][j]
        # 将字符转换为数组索引
        idx = ord(ch) - ord('a')

        # ========== 剪枝条件 ==========
        # 条件1：当前单元格已被访问过（标记为 '#'）
        # 条件2：索引超出有效范围（理论上不会发生，因为网格只有小写字母）
        # 条件3：当前字符不在 Trie 的当前节点的子节点中（前缀不匹配）
        if ch == '#' or idx < 0 or idx >= 26 or not node.children[idx]:
            return

        # 移动到下一个 Trie 节点（匹配当前字符）
        node = node.children[idx]

        # ========== 检查是否找到完整单词 ==========
        # 如果当前节点有 word 字段，说明从根节点到当前节点的路径构成了一个完整单词
        if node.word:
            result.append(node.word)
            node.word = None  # 将 word 置空，防止重复添加同一个单词

        # ========== 标记已访问并继续搜索四个方向 ==========
        # 将当前单元格标记为 '#'，表示已访问，防止在同一个单词中重复使用
        board[i][j] = '#'

        # 四个方向：下、上、右、左
        self._dfs(board, i + 1, j, node, result)  # 向下
        self._dfs(board, i - 1, j, node, result)  # 向上
        self._dfs(board, i, j + 1, node, result)  # 向右
        self._dfs(board, i, j - 1, node, result)  # 向左

        # ========== 回溯 ==========
        # 恢复当前单元格为原始字符，以便其他路径使用
        board[i][j] = ch
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(M x N x 4^L) | M x N 个起点，每个起点 DFS 最坏 4^L 分支（L 是最长单词长度）。Trie 的剪枝极大减少实际搜索 |
| 空间 | O(W x L) | W 个单词建 Trie 的空间，加上递归栈 O(L) |

## ⚠ 易错点

- [ ] **Trie 的 word 字段不能漏**：用 `node.word = word` 而不是简单的 `is_end = True`，这样找到单词时直接获取，不需要维护 path 字符串
- [ ] **找到单词后必须置 None**：`node.word = None`，否则同一个单词可能通过不同路径被多次找到，或者在对称网格中重复加入
- [ ] **DFS 中的剪枝顺序**：先检查越界，再检查 visited（'#'），最后检查 Trie 子节点。顺序不能乱，否则可能越界后还访问 children
- [ ] **回溯恢复**：DFS 结束后必须 `board[i][j] = ch` 恢复原字符，否则会影响其他起始点的搜索
- [ ] **与 79. 单词搜索的区别**：79 题只搜一个单词（直接 DFS + 回溯），本题搜多个单词必须用 Trie 剪枝，否则对每个单词分别 DFS 会超时

```text
这道题是 Trie + DFS 回溯的经典组合，也是 79 题的进阶版。

朴素做法是对每个单词在网格中做一次 DFS，但如果单词很多会超时。
优化的关键是：先用所有单词建一棵 Trie，然后一次遍历网格，
DFS 的同时沿着 Trie 走，遇到不匹配的字符就立即剪枝。

具体流程：
1. 建 Trie：把每个单词的字符逐层插入，在叶子节点存储完整的 word 字符串
   （而不是简单的 is_end 标记，这样找到单词时直接取出）
2. 遍历网格每个格子，从 root 开始做 DFS
3. DFS 过程中：
   - 检查越界和 visited 标记
   - 检查当前字符是否在 Trie 的当前节点的子节点中（核心剪枝）
   - 如果当前 Trie 节点有 word 字段，加入结果并置 None（去重）

空间是 Trie 的 O(W x L)。

一个关键技巧：找到单词后要把 node.word = None，
避免同一个单词被重复加入结果。
```
