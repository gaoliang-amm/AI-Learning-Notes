# 21. 合并两个有序链表

## 🎯 人话翻译

> 给你两条已经排好序的单链表，把它们合并成一条新的升序链表，节点必须来自原来的两条链表。

## 💡 核心思想

> **双指针逐个比较 + 虚拟头节点。** 同时遍历两条链表，每次取 `l1` 和 `l2` 当前节点中值较小的那个，接到结果链表末尾。一条用完后再把另一条剩余部分直接接上。使用 [[虚拟头节点]]（dummy）避免处理空链表和头节点选择的边界。

用到：[[链表]] [[虚拟头节点]]

## 📝 解题步骤

1. **创建虚拟头节点** — `dummy = ListNode(0)`，`cur = dummy` 作为尾指针
2. **双指针遍历** — 当 `l1` 和 `l2` 都不为空时，比较 `l1.val` 和 `l2.val`，`cur.next` 指向较小者，移动对应指针和 `cur`
3. **接上剩余部分** — 跳出循环后，将 `cur.next` 指向 `l1` 或 `l2` 中非空的那条
4. **返回** — `dummy.next` 即合并后的链表头

## 🔢 手动走一遍

输入: `l1 = 1 -> 2 -> 4`, `l2 = 1 -> 3 -> 4`

```
初始: dummy -> (空), cur 在 dummy
l1 = 1 -> 2 -> 4
l2 = 1 -> 3 -> 4

步骤1: l1.val=1 <= l2.val=1, cur.next = l1(1)
       cur 移动到 1, l1 移动到 2
       dummy -> 1 -> (待续)

步骤2: l1.val=2 > l2.val=1, cur.next = l2(1)
       cur 移动到 l2的1, l2 移动到 3
       dummy -> 1(原l1) -> 1(原l2) -> (待续)

步骤3: l1.val=2 <= l2.val=3, cur.next = l1(2)
       cur 移动到 2, l1 移动到 4
       dummy -> 1 -> 1 -> 2 -> (待续)

步骤4: l1.val=4 > l2.val=3, cur.next = l2(3)
       cur 移动到 3, l2 移动到 4
       dummy -> 1 -> 1 -> 2 -> 3 -> (待续)

步骤5: l1.val=4 <= l2.val=4, cur.next = l1(4)
       cur 移动到 4, l1 移动到 None
       dummy -> 1 -> 1 -> 2 -> 3 -> 4 -> (待续)

步骤6: l1为空，cur.next = l2 (4)
       dummy -> 1 -> 1 -> 2 -> 3 -> 4 -> 4
```

结果: `1 -> 1 -> 2 -> 3 -> 4 -> 4`

## 💻 解法

> 双指针迭代，每次取较小者接到尾部。

```python
# 定义链表节点
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def mergeTwoLists(self, l1: ListNode, l2: ListNode) -> ListNode:
        """
        合并两个升序链表，返回新链表的头节点。
        使用迭代法（双指针），空间复杂度 O(1)。
        """
        # 虚拟头节点，简化边界处理
        dummy = ListNode(0)
        cur = dummy

        # 同时遍历两个链表，直到其中一个为空
        while l1 and l2:
            if l1.val <= l2.val:
                cur.next = l1
                l1 = l1.next
            else:
                cur.next = l2
                l2 = l2.next
            cur = cur.next

        # 将剩余的非空链表直接接上
        cur.next = l1 if l1 else l2

        return dummy.next
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(m+n) | 每条链表的每个节点访问一次 |
| 空间 | O(1) | 只用常数个指针，原地拼接（复用原节点） |

## ⚠ 易错点

- [ ] 每次比较后要移动 `cur = cur.next`，否则 `cur` 始终指向 dummy，后面的节点会被覆盖
- [ ] 跳出 while 循环后必须接上剩余链表（`cur.next = l1 if l1 else l2`），不然会丢失后半段
- [ ] 返回的是 `dummy.next` 而非 `dummy`，因为 dummy 是虚拟节点，值为 0
- [ ] 比较时用 `<=` 而非 `<`，当值相等时取 `l1` 还是 `l2` 不影响正确性，但保持稳定
- [ ] 如果两条链表都为空，`dummy.next` 就是 `None`，代码不需要特判

```text
这道题是合并两个有序数组在链表上的版本。核心做法是双指针
迭代：同时遍历两条链表，每次比较当前两个节点的值，把较小
的那个节点接到结果链表的尾部。

关键技巧是使用虚拟头节点（dummy node），这样不需要为"第一
个节点该选谁"写额外判断。维护一个 cur 指针指向结果链表的
尾部，每次接上新节点后 cur 就前移。

当其中一条链表遍历完，直接把另一条剩余的接在尾部即可，
因为剩余部分本身已经是排好序的。

时间 O(m+n)，空间 O(1)（在原地操作，复用原节点）。

进阶问法：如果要合并 K 个有序链表，可以用最小堆或分治法。
```
