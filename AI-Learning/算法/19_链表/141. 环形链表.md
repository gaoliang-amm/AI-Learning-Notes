# 141. 环形链表

## 🎯 人话翻译

> 判断一个链表里有没有环（某个节点的 next 指回了前面的节点，形成闭环）。

## 💡 核心思想

> **快慢指针（Floyd 判圈算法）。** 快指针每次走两步，慢指针每次走一步。如果链表中存在环，快指针最终会追上慢指针（两者在环内相遇）；如果无环，快指针会先到达 `null`。

用到：[[链表]] [[快慢指针]]

## 📝 解题步骤

1. **边界处理** — 空链表或只有一个节点（无环可能），直接返回 `False`
2. **初始化快慢指针** — `slow = head`, `fast = head`
3. **循环移动** — 每次 `slow` 走一步，`fast` 走两步（需检查 `fast` 和 `fast.next` 都不为空）
4. **判断相遇** — 若 `slow == fast`，说明有环，返回 `True`；若 `fast` 走到 `None`，说明无环，返回 `False`

## 🔢 手动走一遍

输入: 链表 = `3 -> 2 -> 0 -> -4 -> (回到2)`，有环

```
初始:  slow=3, fast=3

步骤1: slow=2, fast=0       (slow走1步, fast走2步)
       3 -> 2 -> 0 -> -4
       s    s    f          (slow在2, fast在0)

步骤2: slow=0, fast=2       (slow从2走到0, fast从0走到2)
       3 -> 2 -> 0 -> -4
            f    s          (slow在0, fast在2)

步骤3: slow=-4, fast=-4     (slow从0走到-4, fast从2走到-4)
       3 -> 2 -> 0 -> -4
                      s,f   相遇! 返回 True
```

## 💻 解法

> 快慢指针追及：有环则必相遇。

```python
class Solution:
    def hasCycle(self, head: ListNode) -> bool:
        """
        判断链表是否存在环
        思路：快慢指针，快指针走2步、慢指针走1步，若相遇则有环
        时间复杂度：O(n)，空间复杂度：O(1)
        """
        # 空链表 或 只有一个节点，不可能有环
        if not head or not head.next:
            return False

        # 定义快慢指针
        slow = head
        fast = head

        # 快指针能走到头，说明无环
        while fast and fast.next:
            slow = slow.next  # 慢指针走1步
            fast = fast.next.next  # 快指针走2步

            # 相遇说明存在环
            if slow == fast:
                return True

        # 走出循环，无环
        return False
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 无环时 fast 走 n/2 步到头；有环时 slow 最多走一圈就会被追上 |
| 空间 | O(1) | 只用了两个指针变量 |

## ⚠ 易错点

- [ ] 循环条件必须同时检查 `fast` 和 `fast.next` 都不为 `None`，否则 `fast.next.next` 会空指针异常
- [ ] 不要把判断 `slow == fast` 放在循环开头——初始时两者都指向 head，会立即误判为有环
- [ ] `slow` 和 `fast` 都从 `head` 开始，不能把 `fast` 初始设为 `head.next`（会导致在特定长度的环中错过相遇）
- [ ] 边界处理：只有一个节点的链表且 `head.next == None` 时，应返回 `False` 而非进入循环
- [ ] 不要尝试用哈希表记录访问过的节点——虽然时间也是 O(n)，但空间 O(n) 不是最优

## 🎤 面试怎么说

```text
这道题是快慢指针的经典应用，也叫 Floyd 判圈算法。

核心思想：设置一快一慢两个指针，快指针每次走两步，慢指针
每次走一步。如果链表有环，就像在操场上跑步，跑得快的人最
终会追上跑得慢的人——两者会在环内相遇。如果无环，快指针
会先到达链表末尾。

时间复杂度 O(n)，空间复杂度 O(1)。这是最优解法。

如果面试官追问"能否找到环的入口"，可以扩展成 LeetCode 142。
思路：相遇后把慢指针移回 head，然后两个指针都以步长 1 前进，
再次相遇点就是环的入口。
```
