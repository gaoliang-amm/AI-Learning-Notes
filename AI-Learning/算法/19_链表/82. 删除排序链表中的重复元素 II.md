# 82. 删除排序链表中的重复元素 II

## 🎯 人话翻译

> 给一条升序链表，把所有出现重复的节点全部删干净（一个不留），只保留那些值只出现一次的节点。

## 💡 核心思想

> **双指针 + 跳过重复段。** 维护 `prev`（已确认不重复部分的尾部）和 `curr`（当前扫描位置）。当发现 `curr.val == curr.next.val` 时，记录重复值，内层 while 跳过所有等于该值的节点，然后 `prev.next = curr` 一次性跳过整个重复段。无重复时 `prev` 正常前移。使用 [[虚拟头节点]] 应对头节点被删的情况。

用到：[[链表]] [[虚拟头节点]]

## 📝 解题步骤

1. **边界处理** — 空链表或只有一个节点，直接返回 `head`
2. **创建 dummy** — `dummy.next = head`，`prev = dummy`，`curr = head`
3. **遍历扫描** — 当 `curr` 和 `curr.next` 都存在时：
   - 若 `curr.val == curr.next.val`：记录 `dup_val`，内层 while 跳过所有值为 `dup_val` 的节点，`prev.next = curr`
   - 若不等：`prev = curr`，`curr = curr.next`
4. **返回** — `dummy.next`

## 🔢 手动走一遍

输入: `head = 1 -> 2 -> 3 -> 3 -> 4 -> 4 -> 5`

```
初始: dummy -> 1 -> 2 -> 3 -> 3 -> 4 -> 4 -> 5
       prev = dummy, curr = 1

步骤1: curr.val=1 != 2 -> 无重复, prev移到1, curr移到2
       prev=1, curr=2

步骤2: curr.val=2 != 3 -> 无重复, prev移到2, curr移到3
       prev=2, curr=3

步骤3: curr.val=3 == 3 -> 发现重复! dup_val=3
       内层while: curr走到3, 再走到4 (cur.val != 3, 停止)
       prev.next = curr (2 直接指向 4)
       dummy -> 1 -> 2 -> 4 -> 4 -> 5

步骤4: curr.val=4 == 4 -> 发现重复! dup_val=4
       内层while: curr走到4, 再走到5 (cur.val != 4, 停止)
       prev.next = curr (2 直接指向 5)
       dummy -> 1 -> 2 -> 5

步骤5: curr=5, curr.next=None, 循环结束
```

结果: `1 -> 2 -> 5`

## 💻 解法

> 双指针扫描，发现重复就用内层循环全部跳过。

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def deleteDuplicates(self, head: ListNode) -> ListNode:
        # 空链表或只有一个节点，没有重复可删
        if not head or not head.next:
            return head

        # 虚拟头节点：方便处理头节点被删除的情况
        dummy = ListNode(0)
        dummy.next = head

        # prev：指向已处理好的不重复链表的尾部
        # curr：当前遍历的节点
        prev = dummy
        curr = head

        while curr and curr.next:
            # 情况1：当前节点与下一个节点值相同 → 发现重复段
            if curr.val == curr.next.val:
                # 记录重复的值
                dup_val = curr.val
                # 跳过所有值为 dup_val 的节点
                while curr and curr.val == dup_val:
                    curr = curr.next
                # 将 prev 直接指向跳过重复段后的节点（删除整个重复段）
                prev.next = curr
            else:
                # 情况2：当前节点无重复 → prev 向前移动，curr 继续
                prev = curr
                curr = curr.next

        return dummy.next
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 每个节点最多被访问一次（内层 while 跳过的不再回头访问） |
| 空间 | O(1) | 只用几个指针变量，原地操作 |

## ⚠ 易错点

- [ ] 发现重复时 `prev` 不移动，直接 `prev.next = curr` 跳过整个重复段。等下次无重复节点出现时 `prev` 才后移
- [ ] 内层 while 跳过重复节点后，`curr` 落在下一个（可能不重复的）节点上，不要急于移动 `curr`
- [ ] 区分此题（82，重复的全删）与 83 题（保留一个重复节点）——82 要用 `prev` 直接跳过，83 是 `prev = curr` 然后 `prev.next` 修改
- [ ] 必须使用 `dummy` 节点，因为第一个节点如果有重复也会被删
- [ ] `prev` 在没有发现重复时才前移；发现重复时 `prev` 不动但 `prev.next` 被重定向

```text
这道题和 83 题的区别在于：重复的节点一个不留。所以我们需要
一个 prev 指针指向已确认不重复的尾部，curr 指针扫描后续。

当发现 curr.val 和 curr.next.val 相等时，记录这个重复值，
然后用一个内层循环跳过所有等于该值的节点。然后让 prev.next
直接指向跳过后的第一个节点，这样就"跳过"了整个重复段。

注意 prev 在跳过重复后不移动——因为跳过去后的新节点本身
还可能是重复的，要等下一次比较确认不重复后才能让 prev 移动。

时间 O(n)，空间 O(1)。
```
