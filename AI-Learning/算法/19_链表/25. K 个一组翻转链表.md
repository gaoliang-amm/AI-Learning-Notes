# 25. K 个一组翻转链表

## 🎯 人话翻译

> 把链表每 k 个节点分为一组，每组内部翻转。如果最后一组不足 k 个节点，保持原样不翻转。

## 💡 核心思想

> **分组 + 区间翻转。** 用一个 `pre` 指针指向每组的前驱节点，每次先检查后面是否还有 k 个节点。如果有，取出这一组 `[start, tail]`，调用区间翻转函数，翻转后重新连接：`pre.next = new_head`，`new_tail.next = next_group`。然后将 `pre` 移动到 `new_tail`，准备处理下一组。

用到：[[链表]] [[虚拟头节点]] [[反转链表]]

## 📝 解题步骤

1. **边界处理** — 空链表或 `k=1` 直接返回 `head`。创建 `dummy` 虚拟头节点
2. **检查剩余节点** — `pre` 指向当前组的前驱。`tail` 从 `pre` 走 k 步。若中途遇到 `None` 说明不足 k 个，结束
3. **记录关键节点** — `start = pre.next`（组起点），`next_group = tail.next`（下一组起点）
4. **翻转区间** — 调用 `reverse(start, tail)` 返回 `(new_head, new_tail)`
5. **重新连接** — `pre.next = new_head`，`new_tail.next = next_group`
6. **移动 pre** — `pre = new_tail`，继续循环

## 🔢 手动走一遍

输入: `head = 1 -> 2 -> 3 -> 4 -> 5`, `k = 3`

```
初始: dummy -> 1 -> 2 -> 3 -> 4 -> 5
       pre = dummy

=== 第1组 ===
检查 k=3 个: tail 走3步到 3, 够!
  start = 1, tail = 3, next_group = 4
  翻转 [1,2,3] -> [3,2,1]
  连接: dummy -> 3 -> 2 -> 1 -> 4 -> 5
  pre = 1 (new_tail)

=== 第2组 ===
检查 k=3 个: tail 从 pre(1) 走... 到 4(1步), 到 5(2步), 下一步 None!
  不足 3 个, 结束!
```

结果: `3 -> 2 -> 1 -> 4 -> 5`

### reverse 子过程详解

翻转区间 `[1, 2, 3]`，tail.next = 4:

```
prev = tail.next = 4
cur = 1

步骤1: nxt = cur.next = 2, cur.next = prev(4), prev = 1, cur = 2
       1 -> 4, 链表片断: 1->4   (4->5->...)

步骤2: nxt = cur.next = 3, cur.next = prev(1), prev = 2, cur = 3
       2 -> 1 -> 4

步骤3: nxt = cur.next = 4, cur.next = prev(2), prev = 3, cur = 4
       3 -> 2 -> 1 -> 4
       prev == tail(3), 循环结束

返回 (tail=3, head=1) 即 (new_head=3, new_tail=1)
```

## 💻 解法

> 分组检查 + 区间翻转，每次翻转后重新连接。

```python
from typing import Optional


class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        # 时间复杂度 O(n)，空间复杂度 O(1)

        # 边界情况：空链表或 k=1（无需翻转）直接返回原链表
        if not head or k == 1:
            return head

        # 创建虚拟头节点，方便处理头节点可能被改变的情况
        dummy = ListNode(0)
        dummy.next = head

        # pre 指向当前待翻转区间的前一个节点（即上一组的尾节点）
        pre = dummy

        while True:
            # ===== 1. 检查是否还有 k 个节点可供翻转 =====
            # tail 初始指向 pre，向后移动 k 步
            # 如果移动过程中遇到 None，说明剩余节点不足 k 个，直接返回结果
            tail = pre
            for _ in range(k):
                tail = tail.next
                if not tail:
                    return dummy.next

            # 记录当前区间的起点（即 pre 的下一个节点）
            start = pre.next
            # 记录下一组的起点（即 tail 的下一个节点）
            next_group = tail.next

            # ===== 2. 翻转区间 [start, tail] =====
            # 调用辅助函数，返回 (new_head, new_tail)
            new_head, new_tail = self.reverse(start, tail)

            # ===== 3. 将翻转后的区间重新连接到链表中 =====
            # pre 指向新的区间头节点
            pre.next = new_head
            # 新区间的尾节点指向下一组的起点
            new_tail.next = next_group

            # ===== 4. 移动 pre 到下一组的前驱（即当前区间的尾节点） =====
            pre = new_tail

        # 理论上不会执行到这里，因为循环内遇到不足 k 个节点时会提前返回
        return dummy.next

    def reverse(self, head: ListNode, tail: ListNode):
        """
        翻转链表区间 [head, tail]（包含两端）
        参数：
            head: 待翻转区间的第一个节点
            tail: 待翻转区间的最后一个节点
        返回：
            (new_head, new_tail) - 翻转后的新头节点和新尾节点
        """
        # prev 初始化为 tail 的下一个节点（即翻转后的尾节点应指向的节点）
        prev = tail.next
        # cur 指向当前正在处理的节点，从 head 开始
        cur = head

        # 循环直到 prev 等于 tail（即 cur 已经处理完 tail 节点）
        # 循环条件也可以写成 while cur != tail.next，但 prev 更直观
        while prev != tail:
            # 保存 cur 的下一个节点，防止丢失
            nxt = cur.next
            # 反转指针：cur 指向 prev
            cur.next = prev
            # prev 和 cur 同时向前移动
            prev = cur
            cur = nxt

        # 循环结束时，prev 指向原 tail 节点，cur 指向原 tail.next
        # 此时原 tail 变成了新头，原 head 变成了新尾
        return tail, head
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 每个节点被遍历常数次（检查 k 时一次，翻转时一次） |
| 空间 | O(1) | 原地翻转，只用常数指针 |

## ⚠ 易错点

- [ ] 检查 k 个节点时，`tail` 从 `pre`（而非 `pre.next`）开始走 k 步。如果走不到说明不够 k 个
- [ ] `reverse` 函数中 `prev` 初始化为 `tail.next`，这保证了翻转后尾部正确连到下一组
- [ ] `reverse` 的循环条件是 `while prev != tail`，不是 `while cur`——后者会翻转掉整条剩余链表
- [ ] 连接时顺序：先 `pre.next = new_head`，再 `new_tail.next = next_group`，顺序不可反
- [ ] 翻转后 `pre` 要移动到 `new_tail`（即原 `head`），而不是 `new_head`——因为 pre 需要指向下一组的前驱
