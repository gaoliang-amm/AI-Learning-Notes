# 61. 旋转链表

## 🎯 人话翻译

> 把链表每个节点向右移 k 个位置。比如 `1->2->3->4->5` 右移 2 位变成 `4->5->1->2->3`（最后 k 个节点被搬到开头）。

## 💡 核心思想

> **连成环再断开。** 先遍历一遍得到链表长度 n 和尾节点 tail。将 k 对 n 取模（避免无效旋转）。然后把链表连成环（`tail.next = head`），再找到新的断点（第 `n - k` 个节点作为新尾），在此断开即可。新头 = 新尾的 next。

用到：[[链表]] [[环形链表]]

## 📝 解题步骤

1. **边界处理** — 空链表、单节点或 `k=0`，直接返回 `head`
2. **计算长度和尾节点** — 遍历得到 `n` 和 `tail`
3. **取模** — `k = k % n`，若为 0 直接返回 `head`
4. **找新尾节点** — 从头走 `n - k - 1` 步，到达新尾节点 `new_tail`
5. **重连** — `new_head = new_tail.next`，`tail.next = head`（成环），`new_tail.next = None`（断环）
6. **返回** — `new_head`

## 🔢 手动走一遍

输入: `head = 1 -> 2 -> 3 -> 4 -> 5`, `k = 2`

```
步骤1: 计算长度
  n=1: tail到1, n=2: tail到2, n=3: tail到3, n=4: tail到4, n=5: tail到5
  tail.next = None (n=5)

步骤2: k = 2 % 5 = 2

步骤3: 找新尾节点
  需要走 n-k-1 = 5-2-1 = 2 步
  从 head(1) 走1步到 2, 走2步到 3
  new_tail = 3

步骤4: new_head = new_tail.next = 4

步骤5: 连成环再断开
  tail.next = head  ->  5 -> 1 (成环)
  环: 1 -> 2 -> 3 -> 4 -> 5 -> (回到1)
  new_tail.next = None -> 3.next = None (断开)
  结果: 4 -> 5 -> 1 -> 2 -> 3
```

## 💻 解法

> 连成环 + 找断点：先成环，在正确位置断开。

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def rotateRight(self, head: ListNode, k: int) -> ListNode:
        # 空链表或只有一个节点，旋转后不变
        if not head or not head.next or k == 0:
            return head

        # 1. 遍历链表，计算长度，并找到尾节点
        n = 1  # 链表长度，从1开始
        tail = head
        while tail.next:
            tail = tail.next
            n += 1

        # 2. k 取模，避免无效旋转
        k = k % n
        if k == 0:
            return head

        # 3. 找到新链表的尾节点（原链表的第 n-k 个节点）
        #   新链表的头节点就是这个节点的下一个节点
        new_tail = head
        for _ in range(n - k - 1):
            new_tail = new_tail.next

        new_head = new_tail.next

        # 4. 断开并连接：将原链表变成环，再在 new_tail 处断开
        tail.next = head  # 尾节点指向头节点，形成环
        new_tail.next = None  # 新尾节点断开，新头节点变成 new_head

        return new_head
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 计算长度遍历一次 O(n)，找断点再走 n-k 步 O(n) |
| 空间 | O(1) | 原地旋转，只用了几个指针变量 |

## ⚠ 易错点

- [ ] 必须先 `k = k % n`，否则可能做大量无效旋转（k 可能远大于 n）
- [ ] 找新尾时要走 `n - k - 1` 步而非 `n - k` 步——因为要从第 1 个节点走到第 n-k 个节点
- [ ] 连接顺序：先 `tail.next = head`（成环），再 `new_tail.next = None`（断环）。反过来会丢失后半段
- [ ] `k % n == 0` 时（即 k 是 n 的整数倍），直接返回 `head`，无需做任何操作
- [ ] 不要忘记处理 `n == 1` 的情况（已经在开头的边界判断中处理）

## 🎤 面试怎么说

```text
这道题的关键发现是：向右旋转 k 步等价于把最后 k 个节点移到
链表开头。而链表只能单向遍历，倒着数不方便。

所以思路是：先把链表尾部和头部连在一起形成环，然后在正确
的位置断开。具体做法：
1. 遍历一遍得到链表长度 n 和尾节点
2. k 对 n 取模，去掉整圈旋转
3. 从头走 n-k-1 步找到新尾巴
4. 新尾巴的下一个就是新头
5. 把原尾连到头（成环），在新尾巴处断开

时间 O(n)，空间 O(1)。

技巧是"成环再断开"这个思路，避免了从尾部倒数定位的困难。
```
