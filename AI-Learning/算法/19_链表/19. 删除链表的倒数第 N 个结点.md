# 19. 删除链表的倒数第 N 个结点

## 🎯 人话翻译

> 给你一条链表和一个整数 n，删掉从末尾往前数的第 n 个节点，返回新链表的头。

## 💡 核心思想

> **快慢指针。** 让快指针先走 n 步，然后快慢指针同步走。当快指针走到链表末尾（`fast.next == None`）时，慢指针恰好停在要删除节点的前驱位置。此时 `slow.next = slow.next.next` 即可完成删除。使用 [[虚拟头节点]] 避免删除头节点时的特殊情况。

用到：[[链表]] [[快慢指针]] [[虚拟头节点]]

## 📝 解题步骤

1. **创建 dummy** — `dummy.next = head`，快慢指针都从 `dummy` 开始
2. **快指针先走 n 步** — `fast = fast.next` 执行 n 次
3. **同步移动** — 当 `fast.next` 不为空时，`fast` 和 `slow` 各走一步
4. **删除** — `slow.next = slow.next.next`（跳过倒数第 n 个节点）
5. **返回** — `dummy.next`

## 🔢 手动走一遍

输入: `head = 1 -> 2 -> 3 -> 4 -> 5`, `n = 2`（删除倒数第2个即 4）

```
初始: dummy -> 1 -> 2 -> 3 -> 4 -> 5
       fast, slow 都在 dummy

快指针先走 2 步:
  fast 走1步到 1, 走2步到 2

当前状态: slow=dummy, fast=2
dummy -> 1 -> 2 -> 3 -> 4 -> 5
 s        f

同步移动 (条件: fast.next 不为空):
  第1次: slow 走到 1, fast 走到 3
  dummy -> 1 -> 2 -> 3 -> 4 -> 5
           s         f
  
  第2次: slow 走到 2, fast 走到 4
  dummy -> 1 -> 2 -> 3 -> 4 -> 5
                s         f
  
  第3次: slow 走到 3, fast 走到 5
  dummy -> 1 -> 2 -> 3 -> 4 -> 5
                     s         f
  
  fast.next == None, 停止!

删除: slow.next = slow.next.next
  即 3 跳过 4 直接指向 5
```

结果: `1 -> 2 -> 3 -> 5`

## 💻 解法

> 快慢指针间隔 n 步，找到倒数第 n+1 个节点后删除其下一个。

```python
from typing import Optional


# 链表节点定义
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def removeNthFromEnd(self, head: Optional[ListNode], n: int) -> Optional[ListNode]:
        # 虚拟头节点，统一处理删除头节点的边界情况
        dummy = ListNode(0)
        dummy.next = head

        # 快慢指针初始都指向虚拟头节点
        fast = slow = dummy

        # 快指针先走 n 步
        for _ in range(n):
            fast = fast.next

        # 快慢指针同步后移，直到快指针到达链表末尾
        while fast.next:
            fast = fast.next
            slow = slow.next

        # 跳过下一个节点，完成删除倒数第n个节点
        slow.next = slow.next.next

        # 返回新链表的头节点
        return dummy.next
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(L) | 遍历链表一次，L 为链表长度 |
| 空间 | O(1) | 只用了几个指针变量 |

## ⚠ 易错点

- [ ] 快慢指针必须从 `dummy`（而非 `head`）开始——如果从 `head` 开始且要删的是头节点，慢指针永远无法指向头节点的前驱
- [ ] `fast` 先走 n 步后，同步移动的循环条件是 `while fast.next` 而非 `while fast`——最终慢指针要停在待删节点的前驱，而不是待删节点本身
- [ ] 删除操作 `slow.next = slow.next.next` 前不需要判空，因为快指针已经保证了待删除节点存在
- [ ] 返回的是 `dummy.next` 而非 `head`，因为 `head` 可能已经被删除（当 n = 链表长度时）
- [ ] 如果 n 大于链表长度，此题假设不会出现，但可以提一嘴边界

## 🎤 面试怎么说

```text
这道题用快慢指针一次遍历解决。核心是制造 n 的间隔：快指针
先走 n 步，然后快慢指针同步走，快指针走到最后一个节点时，
慢指针恰好指向倒数第 n+1 个节点（就是要删节点的前一个）。

关键细节：要从 dummy 节点出发而不是 head。因为要删除的节点
可能就是头节点，慢指针需要能指到头节点的前一个位置。

删除只需要把慢指针的 next 指向 next.next 即可，因为快指针
的存在保证了慢指针后面至少还有 n 个节点。

时间 O(n)，空间 O(1)。
```
