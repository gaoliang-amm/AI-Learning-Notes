# 86. 分隔链表

## 🎯 人话翻译

> 给你一条链表和一个值 x，把链表中小于 x 的所有节点放在前面，大于等于 x 的所有节点放在后面，保持节点原有相对顺序不变。

## 💡 核心思想

> **拆分 + 合并。** 用两个虚拟头节点分别维护"小于 x"和"大于等于 x"的两条子链表。遍历原链表，每个节点根据其值与 x 的关系接入对应的子链表尾部。最后把两条子链表首尾相连，注意要把 large 链表的尾节点指向 `None`（防止成环）。

用到：[[链表]] [[虚拟头节点]]

## 📝 解题步骤

1. **边界处理** — 空链表直接返回 `head`
2. **创建两个虚拟头节点** — `small_dummy` 和 `large_dummy`，以及各自的尾指针 `small_tail` 和 `large_tail`
3. **遍历拆分** — 当前节点值 < x 则接到 small 链表尾部，否则接到 large 链表尾部；各自 tail 前移；原链表 cur 前移
4. **防止成环** — `large_tail.next = None`（关键！被接到 large 链表的最后一个节点可能还有残留的 next 指针）
5. **连接** — `small_tail.next = large_dummy.next`
6. **返回** — `small_dummy.next`

## 🔢 手动走一遍

输入: `head = 1 -> 4 -> 3 -> 2 -> 5 -> 2`, `x = 3`

```
初始: small_dummy -> (空), large_dummy -> (空)
      small_tail = small_dummy, large_tail = large_dummy
      cur = 1

步骤1: cur=1, 1<3 -> small: 1
       small_dummy -> 1, small_tail=1
       cur = 4

步骤2: cur=4, 4>=3 -> large: 4
       large_dummy -> 4, large_tail=4
       cur = 3

步骤3: cur=3, 3>=3 -> large: 4->3
       large_dummy -> 4 -> 3, large_tail=3
       cur = 2

步骤4: cur=2, 2<3 -> small: 1->2
       small_dummy -> 1 -> 2, small_tail=2
       cur = 5

步骤5: cur=5, 5>=3 -> large: 4->3->5
       large_dummy -> 4 -> 3 -> 5, large_tail=5
       cur = 2

步骤6: cur=2, 2<3 -> small: 1->2->2
       small_dummy -> 1 -> 2 -> 2, small_tail=最后那个2
       cur = None

断开 large 尾部: large_tail(5).next = None
连接: small_tail.next = large_dummy.next
结果: 1 -> 2 -> 2 -> 4 -> 3 -> 5
```

## 💻 解法

> 两条子链表分别收集，最后首尾相连。

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def partition(self, head: ListNode, x: int) -> ListNode:
        # 空链表直接返回
        if not head:
            return head

        # 创建两个虚拟头节点，分别存放小于 x 和大于等于 x 的节点
        small_dummy = ListNode(0)
        large_dummy = ListNode(0)

        # 两个链表的尾指针，方便追加新节点
        small_tail = small_dummy
        large_tail = large_dummy

        # 遍历原链表
        curr = head
        while curr:
            if curr.val < x:
                # 小于 x：接到 small 链表
                small_tail.next = curr
                small_tail = small_tail.next
            else:
                # 大于等于 x：接到 large 链表
                large_tail.next = curr
                large_tail = large_tail.next
            curr = curr.next

        # 关键：将 large 链表的尾节点指向 None，防止成环
        large_tail.next = None

        # 连接两个链表：small 的尾指向 large 的头
        small_tail.next = large_dummy.next

        # 返回 small 链表的头节点
        return small_dummy.next
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 遍历原链表一次 |
| 空间 | O(1) | 原地拆分拼接，只用了几个指针变量 |

## ⚠ 易错点

- [ ] `large_tail.next = None` 这一行极其关键！被接入 large 链表的最后一个节点如果原本的 next 指向一个小于 x 的节点，会导致链表出现环。例如 `1->2->3`，x=2：尾部节点 3 原本指向 None，但如果不显式置 None，某些场景可能产生环
- [ ] 连接时是 `small_tail.next = large_dummy.next`，不是 `large_dummy`。`large_dummy` 是值为 0 的虚拟节点
- [ ] `small_dummy` 或 `large_dummy` 对应的子链表可能为空，但以上逻辑不需要特殊处理——`small_tail.next = large_dummy.next` 在 small 为空时就是 `dummy_small.next = large_dummy.next`，正确
- [ ] 不需要创建新节点，直接复用原链表的节点（题目要求保持相对顺序时，这种解法自动保留了顺序）
- [ ] 注意区分此题（86，保持原有顺序）和"以 x 为基准小于的放左边大于的放右边但不要求顺序"——86 要求保持相对顺序，所以用双链表拆分法

```text
这道题需要将链表按值 x 重新分隔，且要保持原有相对顺序。

思路很简单：维护两条子链表——一条放小于 x 的节点，另一条
放大于等于 x 的节点。遍历原链表时，把每个节点接到对应的
子链表尾部。最后把小链表的尾和大链表的头连起来。

有两个关键细节：
1. 遍历结束后必须把大链表尾节点的 next 置为 None，否则可能
   出现环——因为原件中尾节点原本可能指向某个被分到小链表的
   节点。
2. 使用两个 dummy 节点简化边界处理，无需判断链表是否为空。

时间 O(n)，空间 O(1)。
```
