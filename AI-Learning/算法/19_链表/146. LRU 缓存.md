# 146. LRU 缓存

## 🎯 人话翻译

> 设计一个 LRU（Least Recently Used）缓存：`get(key)` 获取值并把该项标记为最近使用；`put(key, value)` 写入或更新值，当容量满时淘汰最久未使用的那个。两个操作都必须是 O(1)。

## 💡 核心思想

> **[[哈希表]] + [[双向链表]]。** 哈希表提供 O(1) 的键值查找；双向链表维护访问顺序（头部 = 最近访问，尾部 = 最久未访问）。每次 get/put 都把操作的节点移到链表头部；容量满时删除链表尾部节点。使用虚拟头尾节点简化边界操作。

用到：[[链表]] [[双向链表]] [[哈希表]]

## 📝 解题步骤

1. **初始化** — `cache = {}`（key -> Node），`capacity` 容量，虚拟头尾 `head` 和 `tail` 互指
2. **get(key)**：
   - key 不在 `cache` -> 返回 -1
   - key 存在 -> 把节点移到链表头部（先删再头插），返回 value
3. **put(key, value)**：
   - key 存在 -> 更新 value，移到头部
   - key 不存在 -> 若容量满了则删除尾部节点（同时从哈希表删除），创建新节点，加入哈希表，头插到链表
4. **辅助方法**：
   - `_add_to_head(node)`：插入到 head 之后
   - `_remove_node(node)`：从链表中移除
   - `_move_to_head(node)`：先移除再头插
   - `_remove_tail()`：删除 tail 之前的节点（真正的尾部）

## 🔢 手动走一遍

操作序列: `LRUCache(2) -> put(1,1) -> put(2,2) -> get(1) -> put(3,3) -> get(2) -> put(4,4) -> get(1) -> get(3) -> get(4)`

```
初始: head <-> tail, cache={}, cap=2

put(1,1):
  链表: head <-> [1,1] <-> tail
  cache: {1: Node(1,1)}

put(2,2):
  链表: head <-> [2,2] <-> [1,1] <-> tail
  cache: {1: Node(1,1), 2: Node(2,2)}

get(1): 命中!
  移动 [1,1] 到头部
  链表: head <-> [1,1] <-> [2,2] <-> tail
  返回 1

put(3,3): 容量满(2), 淘汰尾部 [2,2]
  删除尾部 -> 链表: head <-> [1,1] <-> tail
  从 cache 删除 key=2
  插入 [3,3] 到头部
  链表: head <-> [3,3] <-> [1,1] <-> tail
  cache: {1: Node(1,1), 3: Node(3,3)}

get(2): 未命中 -> 返回 -1

put(4,4): 容量满(2), 淘汰尾部 [1,1]
  删除尾部 -> 链表: head <-> [3,3] <-> tail
  从 cache 删除 key=1
  插入 [4,4] 到头部
  链表: head <-> [4,4] <-> [3,3] <-> tail
  cache: {3: Node(3,3), 4: Node(4,4)}

get(1): 未命中 -> 返回 -1
get(3): 命中 -> 返回 3
get(4): 命中 -> 返回 4
```

## 💻 解法

> 哈希表（O(1)查找）+ 双向链表（O(1)插入/删除），虚拟头尾简化边界。

```python
class ListNode:
    """双向链表节点"""

    def __init__(self, key=0, val=0):
        self.key = key  # 存放 key，用于删除哈希表
        self.val = val
        self.prev = None
        self.next = None


class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}  # key -> node

        # 创建虚拟头尾节点，简化边界处理
        self.head = ListNode()
        self.tail = ListNode()
        self.head.next = self.tail
        self.tail.prev = self.head

    def get(self, key: int) -> int:
        # 1. key 不存在，返回 -1
        if key not in self.cache:
            return -1

        # 2. key 存在，将节点移到头部，返回 value
        node = self.cache[key]
        self._move_to_head(node)
        return node.val

    def put(self, key: int, value: int) -> None:
        # 1. key 已存在：更新 value，移到头部
        if key in self.cache:
            node = self.cache[key]
            node.val = value
            self._move_to_head(node)
            return

        # 2. key 不存在：创建新节点
        # 2.1 如果容量满了，删除尾部节点
        if len(self.cache) >= self.capacity:
            self._remove_tail()

        # 2.2 插入新节点到头部
        new_node = ListNode(key, value)
        self.cache[key] = new_node
        self._add_to_head(new_node)

    # ========== 双向链表操作 ==========
    def _add_to_head(self, node):
        """将节点插入头部（head 之后）"""
        node.prev = self.head
        node.next = self.head.next
        self.head.next.prev = node
        self.head.next = node

    def _remove_node(self, node):
        """删除节点（只修改指针，不返回节点值）"""
        node.prev.next = node.next
        node.next.prev = node.prev

    def _move_to_head(self, node):
        """将节点移动到头部"""
        self._remove_node(node)
        self._add_to_head(node)

    def _remove_tail(self):
        """删除尾部节点（tail 之前）"""
        # 获取真正的尾部节点（tail.prev）
        tail_node = self.tail.prev
        # 删除它
        self._remove_node(tail_node)
        # 从哈希表中删除
        del self.cache[tail_node.key]
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(1) | get 和 put 均为常数时间：哈希表查找 O(1)，双向链表插入/删除 O(1) |
| 空间 | O(capacity) | 哈希表和链表最多存储 capacity 个节点 |

## ⚠ 易错点

- [ ] `_add_to_head` 中四行指针赋值顺序不能乱。必须先设置新节点的 prev 和 next（它们依赖旧状态），再修改旧节点的指针。顺序是：`node.prev` -> `node.next` -> `head.next.prev` -> `head.next`
- [ ] `_remove_node` 只需两行：`node.prev.next = node.next` 和 `node.next.prev = node.prev`。不需要判空因为有虚拟头尾保护
- [ ] `put` 更新已存在的 key 时，先更新 value 再 `_move_to_head`，不要忘记两步都做
- [ ] `_remove_tail` 时要同时删除哈希表中的 key，否则哈希表残留无效引用；`ListNode` 的 `key` 属性就是为这一步服务的
- [ ] 虚拟头尾节点永远不会被删除，不要把它们算入 capacity 的计数中

## 🎤 面试怎么说

```text
LRU 缓存是面试高频题，要求 get 和 put 都是 O(1)。

关键组合：哈希表 + 双向链表。哈希表负责 O(1) 查找（key -> Node），
双向链表负责 O(1) 的访问顺序维护（头部是最近访问，尾部是最
久未访问）。

每次 get 或 put 命中时，把节点移到链表头部（先删再头插）。
当 put 新 key 且容量满时，删除尾部节点（同时从哈希表清除）。

使用虚拟头尾节点可以统一处理边界——头部插入不需要判断空链表，
尾部删除不需要判断只剩一个节点。链表节点存了 key，删除尾部
时才能从哈希表中找到对应的 key 来清除。

所有操作 O(1)，空间 O(capacity)。
这道题在实际工程中非常常用，比如操作系统页面置换、数据库
缓存淘汰等。
```
