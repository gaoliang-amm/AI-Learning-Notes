# 138. 随机链表的复制

## 🎯 人话翻译

> 有一个链表，每个节点除了 `next` 指针还有一个 `random` 指针，随机指向链表中任意节点或 `null`。要求深拷贝一份全新的链表，不能修改原链表。

## 💡 核心思想

> **哈希表建映射。** 第一次遍历创建所有新节点，用哈希表建立"原节点 -> 新节点"的映射。第二次遍历利用映射设置每个新节点的 `next` 和 `random` 指针（新节点的 random = 原节点的 random 在映射中对应的新节点）。

用到：[[链表]] [[哈希表]]

## 📝 解题步骤

1. **边界处理** — 若 `head` 为空，返回 `None`
2. **第一次遍历（创建节点）** — 遍历原链表，为每个节点创建一个只有 `val` 的新节点，存入 `node_map[原节点] = 新节点`
3. **第二次遍历（设置指针）** — 再次遍历原链表，通过 `node_map` 查询：`new_node.next = node_map.get(cur.next)`，`new_node.random = node_map.get(cur.random)`
4. **返回** — `node_map[head]` 即为新链表的头节点

## 🔢 手动走一遍

输入: `head = [[7,null],[13,0],[11,4],[10,2],[1,0]]`
即 7->13->11->10->1，random: 13->7, 11->1, 10->11, 1->7

**第一遍（创建新节点）：**

| 原节点 | 新节点    | node_map                      |
|--------|-----------|-------------------------------|
| 7      | 7'(空)    | {7: 7'}                       |
| 13     | 13'(空)   | {7: 7', 13: 13'}              |
| 11     | 11'(空)   | {7: 7', 13: 13', 11: 11'}     |
| 10     | 10'(空)   | {... 10: 10'}                 |
| 1      | 1'(空)    | {... 1: 1'}                   |

**第二遍（设置指针）：**

| 原节点 | 操作                              | 新链表状态                     |
|--------|-----------------------------------|-------------------------------|
| 7      | 7'.next = 13', 7'.random = None   | 7'->13', r=None                |
| 13     | 13'.next = 11', 13'.random = 7'   | 13'->11', r=7'                 |
| 11     | 11'.next = 10', 11'.random = 1'   | 11'->10', r=1'                 |
| 10     | 10'.next = 1', 10'.random = 11'   | 10'->1', r=11'                 |
| 1      | 1'.next = None, 1'.random = 7'    | 1'->None, r=7'                 |

新链表完成: `7' -> 13' -> 11' -> 10' -> 1'`，各 random 指针正确

## 💻 解法

> 哈希表法：两次遍历，第一次建节点，第二次设指针。

```python
from typing import Optional


# 定义 Node 类（题目已给出，这里重新定义用于测试）
class Node:
    def __init__(self, x: int, next: 'Node' = None, random: 'Node' = None):
        self.val = int(x)
        self.next = next
        self.random = random


class Solution:
    def copyRandomList(self, head: 'Optional[Node]') -> 'Optional[Node]':
        """
        复制带随机指针的链表。
        哈希表法：第一次遍历创建新节点，第二次遍历设置 next 和 random。
        """
        if not head:
            return None

        # 第一遍：创建原节点到新节点的映射
        node_map = {}
        cur = head
        while cur:
            node_map[cur] = Node(cur.val)
            cur = cur.next

        # 第二遍：设置新节点的 next 和 random
        cur = head
        while cur:
            new_node = node_map[cur]
            new_node.next = node_map.get(cur.next)  # 如果 cur.next 为 None，get 返回 None
            new_node.random = node_map.get(cur.random)
            cur = cur.next

        return node_map[head]
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 两次遍历链表，每次 O(n) |
| 空间 | O(n) | 哈希表存储 n 个原节点到新节点的映射，另外新链表占用 n 个节点 |

## ⚠ 易错点

- [ ] 空链表需要单独处理，直接返回 `None`
- [ ] `node_map.get(cur.next)` 和 `node_map.get(cur.random)` 必须用 `.get()` 而非方括号——`cur.next` 或 `cur.random` 可能是 `None`，`None` 不在映射中，用 `[]` 会 KeyError，用 `.get()` 返回 `None`
- [ ] 在第二遍遍历中，不能直接用 `cur = cur.next` 的重新赋值——`cur` 必须始终指向原链表的节点，而 `cur` 变量移动是安全的（我们用的就是原链表的 `cur`）
- [ ] 不要在第一遍遍历中同时设置 `next` 和 `random`——此时目标节点可能还没创建（因为 `next` 指向后面的节点）
- [ ] 别忘记头节点也可能有 random 指针，不需要特殊处理

## 🎤 面试怎么说

```text
这道题的核心难点是 random 指针——它可能指向链表中任意位置，
如果按常规方式逐个复制，可能在设置指针时目标节点还不存在。

因此需要用哈希表来解耦：第一次遍历先把所有新节点创建好，
用哈希表建立"原节点 -> 新节点"的映射。第二次遍历再根据
映射设置每个新节点的 next 和 random 指针。

使用 dict.get() 而不是 dict[] 的原因是：random 可能指向
None，而 None 不在映射中，get() 会友好地返回 None。

时间 O(n)，空间 O(n)。也有空间 O(1) 的解法（把新节点穿插
在原节点之间），但面试中哈希表法已经足够清晰好写。
```
