# 48. 旋转图像

## 🎯 人话翻译

> 把一个 n x n 的矩阵原地顺时针旋转 90 度。

## 💡 核心思想

> **两步法：先沿主对角线转置（行列互换），再每一行左右翻转。**
>
> 顺时针旋转 90 度等价于：`matrix[i][j]` 移动到 `matrix[j][n-1-i]`。
>
> 拆分为两步：
> 1. **转置**：`matrix[i][j] <-> matrix[j][i]`（沿左上-右下对角线翻转）
> 2. **行翻转**：每行左右反转
>
> 这两步组合起来，效果正好是顺时针旋转 90 度。

用到：[[矩阵]] [[原地算法]]

## 📝 解题步骤

1. **转置** — 遍历 `i` 从 0 到 n-1，`j` 从 i+1 到 n-1：交换 `matrix[i][j]` 和 `matrix[j][i]`
2. **行翻转** — 遍历每一行：`matrix[i].reverse()`
3. 完成，原地修改

## 🔢 手动走一遍

输入: `matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]`

**原始矩阵：**
```
1 2 3
4 5 6
7 8 9
```

**步骤 1：对角线转置**

| i | j | 交换 | 矩阵状态 |
|---|---|------|----------|
| 0 | 1 | (0,1)<->(1,0): 2<->4 | [1,4,3] / [2,5,6] / [7,8,9] |
| 0 | 2 | (0,2)<->(2,0): 3<->7 | [1,4,7] / [2,5,6] / [3,8,9] |
| 1 | 2 | (1,2)<->(2,1): 6<->8 | [1,4,7] / [2,5,8] / [3,6,9] |

**步骤 2：每行翻转**

| 行号 | 翻转前 | 翻转后 |
|------|--------|--------|
| 0 | [1,4,7] | [7,4,1] |
| 1 | [2,5,8] | [8,5,2] |
| 2 | [3,6,9] | [9,6,3] |

结果:
```
7 4 1
8 5 2
9 6 3
```

验证：原始矩阵顺时针旋转 90 度后，(0,0)=1 移动到 (0,2)=1，(0,2)=3 移动到 (2,2)=3，(2,2)=9 移动到 (2,0)=9。

## 💻 解法

> 两步原地操作：先转置，再每行翻转。

```python
from typing import List


class Solution:
    def rotate(self, matrix: List[List[int]]) -> None:
        """
        原地顺时针旋转图像 90 度。
        思路：先对角线转置，再每行反转。
        """
        n = len(matrix)

        # 第一步：主对角线转置（行列互换）
        for i in range(n):
            for j in range(i + 1, n):
                matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]

        # 第二步：每一行左右翻转
        for i in range(n):
            matrix[i].reverse()
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n^2) | 两个双重循环，每个元素访问常数次 |
| 空间 | O(1) | 原地修改，只用了交换时的临时变量 |

## ⚠ 易错点

- [ ] 转置的内层循环 `j` 从 `i+1` 开始——如果从 0 开始会把已经交换过的再换回来（等于白做）
- [ ] 逆时针旋转 90 度是另一种操作：先左右翻转，再转置。不要记混
- [ ] 旋转 180 度 = 先上下翻转 + 再左右翻转；旋转 270 度 = 逆时针 90 度 = 先左右翻转再转置。面试官可能会问这些变种
- [ ] 如果不用原地操作，最简单的做法是 `matrix[:] = list(zip(*matrix[::-1]))`，一行代码但创建新矩阵。面试时面试官通常会要求 O(1) 空间

第二步：每一行左右翻转
