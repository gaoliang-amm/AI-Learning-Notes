# 918. 环形子数组的最大和

## 🎯 人话翻译

> 数组是首尾相连的环形数组，找出连续子数组的最大和。子数组可以跨越数组末尾回到开头（但不能重复使用同一个元素）。

## 💡 核心思想

> **环形子数组只可能有两种情况：中间一段（不跨越边界），或者去掉中间一段（跨越边界）。**
>
> 情况一：普通最大子数组和，直接用 Kadane 算法求 `max_sum`。
>
> 情况二：跨越边界的子数组 = 整个数组去掉中间最小的一段。因为环形中，跨越边界的子数组等价于"首段 + 尾段"，也就是总和减去中间最小子数组。所以还需要用 Kadane 求最小子数组和 `min_sum`，然后 `total - min_sum`。
>
> 特殊情况：如果所有元素都是负数，`total - min_sum` 会得到 0（相当于取了空数组），但题目要求至少选一个元素，此时应返回 `max_sum`（即最大的那个负数）。

用到：[[Kadane算法]]

## 📝 解题步骤

1. **初始化变量** — `cur_max = 0`, `cur_min = 0`, `max_sum = -inf`, `min_sum = +inf`, `total = 0`
2. **同时计算最大和与最小和** — 遍历数组，`cur_max = max(cur_max + num, num)` 追踪以当前元素结尾的最大子数组和；`cur_min = min(cur_min + num, num)` 追踪以当前元素结尾的最小子数组和；累加 `total`
3. **检查全为负数的情况** — 如果 `max_sum < 0`，说明没有正数，直接返回 `max_sum`（最大的负数）
4. **取两种情况的最大值** — `max(max_sum, total - min_sum)`。前者是不跨越边界的最大和，后者是跨越边界的最大和

## 💻 解法

> 一次遍历同时计算最大和与最小和，优雅且高效。

```python
from typing import List


class Solution:
    def maxSubarraySumCircular(self, nums: List[int]) -> int:
        # 时间复杂度 O(n)，空间复杂度 O(1)

        # 初始化
        total = 0  # 数组总和
        cur_max = 0  # 当前子数组最大和
        cur_min = 0  # 当前子数组最小和
        max_sum = float('-inf')  # 全局最大子数组和
        min_sum = float('inf')  # 全局最小子数组和

        for num in nums:
            total += num

            # 计算当前最大子数组和（Kadane 算法）
            cur_max = max(cur_max + num, num)
            max_sum = max(max_sum, cur_max)

            # 计算当前最小子数组和
            cur_min = min(cur_min + num, num)
            min_sum = min(min_sum, cur_min)

        # 特殊情况：所有数都是负数
        # 此时 min_sum == total，总和 - min_sum == 0
        # 但实际应该取 max_sum（最大的那个负数）
        if max_sum < 0:
            return max_sum

        # 比较两种情况：普通最大和 与 跨越首尾的最大和
        return max(max_sum, total - min_sum)
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 一次遍历，每个元素做 O(1) 操作 |
| 空间 | O(1) | 只使用固定数量的变量 |

## ⚠ 易错点

- [ ] 必须处理全为负数的情况。如果所有元素都是负数，`total - min_sum` 会等于 `total - total = 0`，但空子数组不被允许，应返回 `max_sum`（即数组中最大的那个负数）
- [ ] `cur_max` 和 `cur_min` 初始化为 0 而不是 `nums[0]`，`max_sum` 初始化为 `-inf` 而不是 `nums[0]`，这是因为需要从 0 开始计算才能正确处理 total-min_sum 的情况
- [ ] `min_sum` 初始化为 `+inf`（正无穷），`max_sum` 初始化为 `-inf`（负无穷），写反了会导致初始化错误
- [ ] 跨越边界的子数组和 = `total - min_sum`，这个等式的直观理解：环形中，取首尾两端 = 总和 - 中间不要的那段（最小子数组）

```text
这道题是 Kadane 算法的变种，处理环形数组。

核心思路：环形子数组的最大和只有两种可能性：
1. 不跨越边界 —— 就是普通的 Kadane 求最大子数组和
2. 跨越边界 —— 等于"总和减去最小子数组和"
   （因为跨越边界的子数组 = 首段 + 尾段 = 总和 - 中间最小段）

但我们必须至少选一个元素，所以此时返回 max_sum（最大的负数）。

时间复杂度 O(n)，空间复杂度 O(1)。
```
