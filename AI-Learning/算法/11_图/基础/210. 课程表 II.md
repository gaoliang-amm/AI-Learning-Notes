# 210. 课程表 II

## 🎯 人话翻译

> 和 207 题一样，给定课程依赖关系。但这次不只判断能否学完，还要返回一个可行的学习顺序（任意一个拓扑序即可）。如果有环（无法学完），返回空数组。

## 💡 核心思想

> **拓扑排序：与 207 题完全相同，只是在 BFS/DFS 遍历的过程中记录节点的出队/完成顺序。**
>
> - **BFS (Kahn 算法)**：出队顺序就是拓扑序，直接记录即可。
> - **DFS**：节点完成 DFS（状态变为 2）时，将节点加入结果栈。DFS 结束后，结果需要反转（因为后完成的在栈顶，即先依赖的排在后面）。
>
> 如果有环，返回空数组。

用到：[[图]]、[[拓扑排序]]、[[BFS]]、[[DFS]]

## 📝 解题步骤

### BFS (Kahn 算法) — 推荐

1. **建图 + 入度统计** — `graph[b].append(a)`，`indegree[a] += 1`
2. **初始化队列** — 入度为 0 的节点入队
3. **BFS** — 出队节点加入 `result`，其邻居入度减一，入度变 0 的入队
4. **判断** — 如果 `len(result) == numCourses`，返回 result（即拓扑序）；否则返回 `[]`

### DFS

1. **建图** — 同 BFS
2. **DFS** — 三色标记，如果发现环返回 False。完成一个节点（state=2）时 `result.append(node)`
3. **最终** — 如果无环，返回 `result[::-1]`（反转）；否则返回 `[]`

## 🔢 手动走一遍

输入: `numCourses = 4`, `prerequisites = [[1,0],[2,0],[3,1],[3,2]]`

图结构：
```
    0
   / \
  v   v
  1   2
   \ /
    v
    3
```

### BFS 执行过程

| 步 | 队列 | 出队 | result | 入度变化 |
|----|------|------|--------|---------|
| 初 | [0] | - | [] | 1:1, 2:1, 3:2 |
| 1 | [1, 2] | 0 | [0] | 1:0, 2:0 |
| 2 | [2] | 1 | [0, 1] | 3:1 |
| 3 | [3] | 2 | [0, 1, 2] | 3:0 |
| 4 | [] | 3 | [0, 1, 2, 3] | - |

`result = [0, 1, 2, 3]`，长度 = 4 = numCourses，有效顺序。

### DFS 执行过程

| 调用 | 状态变化 | result (逆序) |
|------|---------|--------------|
| dfs(0): state[0]=1 | [1,0,0,0] | [] |
| dfs(1): state[1]=1 | [1,1,0,0] | [] |
| dfs(3): state[3]=1 | [1,1,0,1] | [] |
| dfs(3) 完成: state[3]=2 | [1,1,0,2] | [3] |
| dfs(1) 完成: state[1]=2 | [1,2,0,2] | [3,1] |
| dfs(2): state[2]=1 | [1,2,1,2] | [3,1] |
| dfs(2) 完成: state[2]=2 | [1,2,2,2] | [3,1,2] |
| dfs(0) 完成: state[0]=2 | [2,2,2,2] | [3,1,2,0] |

反转 result: `[0, 2, 1, 3]`，也是一个有效的拓扑序。

## 💻 解法

> BFS (Kahn 算法) 直接记录出队顺序即拓扑序。DFS 需要反转后完成节点的收集顺序。

```python
from typing import List
from collections import deque


class Solution:
    def findOrder(self, numCourses: int, prerequisites: List[List[int]]) -> List[int]:
        # 时间复杂度 O(V+E)，空间复杂度 O(V+E)

        # 方法一：BFS（Kahn 算法）
        def method_bfs():
            # 建图和入度统计
            graph = [[] for _ in range(numCourses)]
            indegree = [0] * numCourses

            for a, b in prerequisites:
                graph[b].append(a)  # b → a
                indegree[a] += 1

            # 队列：存储所有入度为 0 的节点
            queue = deque([i for i in range(numCourses) if indegree[i] == 0])
            result = []

            while queue:
                node = queue.popleft()
                result.append(node)

                for neighbor in graph[node]:
                    indegree[neighbor] -= 1
                    if indegree[neighbor] == 0:
                        queue.append(neighbor)

            # 如果结果数量等于课程数，返回结果；否则存在环
            return result if len(result) == numCourses else []

        # 方法二：DFS
        def method_dfs():
            graph = [[] for _ in range(numCourses)]
            for a, b in prerequisites:
                graph[b].append(a)

            # 状态：0=未访问，1=访问中，2=已访问
            state = [0] * numCourses
            result = []  # 栈，存储拓扑排序（逆序）

            def dfs(u):
                if state[u] == 1:
                    return False  # 发现环
                if state[u] == 2:
                    return True  # 已处理过

                state[u] = 1
                for v in graph[u]:
                    if not dfs(v):
                        return False
                state[u] = 2
                result.append(u)  # 递归结束后加入结果
                return True

            for i in range(numCourses):
                if state[i] == 0:
                    if not dfs(i):
                        return []

            # DFS 的结果是逆序，需要反转
            return result[::-1]

        # 两种方法任选，这里用 BFS（更直观）
        return method_bfs()
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(V + E) | V 是课程数，E 是前置关系数 |
| 空间 | O(V + E) | 邻接表 + 入度数组/状态数组 + 队列/递归栈 |

## ⚠ 易错点

- [ ] **BFS 返回判断**：必须用 `len(result) == numCourses` 判断而非直接返回 result，有环时需返回 `[]`
- [ ] **DFS 反转结果**：DFS 的 result 收集的是「后序」（节点完成时的顺序），必须 `result[::-1]` 反转才是拓扑序
- [ ] **边的方向**：`prerequisites[i] = [a, b]` 表示 b 是 a 的前置，边是 `b → a`
- [ ] **可能会有多个起点**：入度为 0 的节点可能有多个，要全部入队，不能只从某一个开始
- [ ] DFS 和 BFS 得到的拓扑序可能不同（都是合法的），不要纠结于唯一性

## 🎤 面试怎么说

```text
这道题是 207 课程表的延伸，不只要判断能否学完，还要输出一个可行的学习顺序。

本质上就是求拓扑排序。我推荐用 BFS 的 Kahn 算法：

1. 建图 + 统计每个节点的入度
2. 把所有入度为 0 的节点（没有前置课程）放入队列
3. 每次出队一个节点，加入结果列表
4. 把这个节点的所有后继课程入度减一，如果变成 0 就入队
5. 最后如果结果包含所有课程，返回结果；否则说明有环，返回空数组

出队的顺序就是一种合法的拓扑序，不需要额外处理。

如果面试官想看 DFS 解法：用三色标记法检测环，
在节点完成访问（状态从 1 变 2）时把它加入结果栈，
最后反转栈就得到了拓扑序。

时间 O(V+E)，空间 O(V+E)。

这道题的变种就是 207 课程表（只判断）和 269 火星词典（通过字符关系推导顺序）。
```
