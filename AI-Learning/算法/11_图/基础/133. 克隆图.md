# 133. 克隆图

## 🎯 人话翻译

> 给你一个无向连通图的某个节点，把这个图完整复制一份出来（深拷贝）。新旧图节点之间不能共享引用。

## 💡 核心思想

> **DFS/BFS + 哈希表做映射：`visited = {原节点: 克隆节点}`。**
>
> 遍历原图的过程中：
> 1. 如果原节点已经克隆过，直接从哈希表返回克隆节点
> 2. 如果没克隆过，创建新节点，存入哈希表
> 3. 递归/迭代克隆它的所有邻居，把克隆邻居加入克隆节点的 neighbors 列表
>
> 哈希表同时扮演两个角色：**记录已访问的节点（避免死循环）**和**建立原节点到克隆节点的映射**。

用到：[[图]]、[[DFS]]、[[BFS]]、[[哈希表]]

## 📝 解题步骤

1. **空节点处理** — 如果 `node` 为 None，直接返回 None
2. **创建哈希表** — `visited = {}`，key 是原节点，value 是克隆节点
3. **DFS 克隆函数**：
   - 如果原节点已在 `visited` 中，直接返回 `visited[original]`
   - 否则创建新的克隆节点 `clone = Node(original.val)`，存入 `visited`
   - 遍历原节点的所有邻居，递归克隆每个邻居，加入 `clone.neighbors`
   - 返回 clone
4. **返回结果** — `dfs(node)`

## 🔢 手动走一遍

输入: 一个简单图
```
    1
   / \
  2---3
```

邻接表: `1: [2,3]`, `2: [1,3]`, `3: [1,2]`

### DFS 克隆过程

| 步骤 | 当前原节点 | 操作 | visited 状态 |
|------|-----------|------|-------------|
| 1 | 1 | 创建 clone1，visited={1:clone1} | {1: clone1} |
| 2 | 1 的邻居 2 | clone1.neighbors 递归处理 2 | - |
| 3 | 2 | 创建 clone2，visited={1:clone1, 2:clone2} | {1: clone1, 2: clone2} |
| 4 | 2 的邻居 1 | 1 已在 visited，直接返回 clone1 | 将 clone1 加入 clone2.neighbors |
| 5 | 2 的邻居 3 | clone2.neighbors 递归处理 3 | - |
| 6 | 3 | 创建 clone3，visited={1:clone1, 2:clone2, 3:clone3} | 全部创建完成 |
| 7 | 3 的邻居 1 | 1 已在 visited，返回 clone1 | 加入 clone3.neighbors |
| 8 | 3 的邻居 2 | 2 已在 visited，返回 clone2 | 加入 clone3.neighbors |
| 9 | 1 的邻居 3 | 3 已在 visited，返回 clone3 | 加入 clone1.neighbors |

最终克隆图结构：
```
  clone1
   / \
clone2--clone3
```

与原图结构完全一致，但没有任何节点共享引用。

## 💻 解法

> DFS + 哈希表：遍历原图，对每个节点创建克隆，用哈希表避免重复克隆和死循环。

```python
from typing import Optional


class Node:
    def __init__(self, val=0, neighbors=None):
        self.val = val
        self.neighbors = neighbors if neighbors is not None else []


class Solution:
    def cloneGraph(self, node: Optional['Node']) -> Optional['Node']:
        # 时间复杂度 O(n+e)，空间复杂度 O(n)

        if not node:
            return None

        # 哈希表：原节点 -> 克隆节点
        visited = {}

        def dfs(original):
            # 如果节点已经克隆过，直接返回克隆节点
            if original in visited:
                return visited[original]

            # 创建克隆节点（暂时没有邻居）
            clone = Node(original.val)
            visited[original] = clone

            # 递归克隆所有邻居
            for neighbor in original.neighbors:
                clone.neighbors.append(dfs(neighbor))

            return clone

        return dfs(node)
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(V + E) | V 是节点数，E 是边数，每个节点和每条边访问一次 |
| 空间 | O(V) | visited 哈希表存储 V 个节点映射，递归栈最深 V 层 |

## ⚠ 易错点

- [ ] **哈希表必须同时做「去重」和「映射」**：不能只用 set 记录已访问，因为还需要从原节点找到对应的克隆节点来连接邻居
- [ ] **先存哈希表再递归邻居**：必须先把 clone 放入 `visited[original]`，再递归处理邻居，否则遇到环会无限递归
- [ ] **不要直接赋值 neighbors**：`clone.neighbors = original.neighbors` 是浅拷贝，会共享引用，不符合深拷贝要求
- [ ] **边界检查**：如果输入 node 是 None，直接返回 None
- [ ] BFS 也可以做：用队列替代递归，同样需要哈希表映射。面试中可以提一下两种实现方式

## 🎤 面试怎么说

```text
这道题要求对无向连通图做深拷贝。

核心思路是用 DFS（或 BFS）遍历原图，同时用哈希表建立原节点到克隆节点的映射。

具体过程：
1. 每遇到一个新节点，创建一个克隆节点，存入哈希表 visited[原节点] = 克隆节点
2. 遍历原节点的所有邻居，递归克隆每个邻居
3. 如果某个邻居已经在哈希表中（已经克隆过），直接复用，避免死循环和重复创建

哈希表在这里有两个作用：
- 防止环导致无限递归（标记已访问）
- 快速找到已创建的克隆节点来连接邻居

时间 O(V+E)，空间 O(V)，其中 V 是节点数，E 是边数。

关键细节：一定要先存哈希表再递归邻居，否则遇到环就无限递归了。
另外不能直接把 original.neighbors 赋值给 clone.neighbors，那是浅拷贝。
```
