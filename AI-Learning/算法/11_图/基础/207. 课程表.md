# 207. 课程表

## 🎯 人话翻译

> 有些课有前置课，比如想学 B 必须先学 A。给一个课程依赖关系列表，判断能不能把所有课都修完（即是否存在环，有环就死锁了，返回 false）。

## 💡 核心思想

> **拓扑排序 / 环检测问题：把课程依赖关系建模为有向图。**
>
> 如果 `prerequisites[i] = [a, b]`，表示 `b → a`（学完 b 才能学 a）。目标是判断这个有向图是否**有环**。
>
> **两种方法：**
> 1. **BFS (Kahn 算法)**：统计每个节点的入度，入度为 0 的节点可以选课。依次移除入度为 0 的节点及其出边。如果最后所有的节点都被移除了，说明无环。
> 2. **DFS 三色标记法**：0=未访问，1=访问中（在当前递归路径上），2=已访问完。如果在 DFS 中遇到状态为 1 的节点，说明发现了环。

用到：[[图]]、[[拓扑排序]]、[[DFS]]、[[BFS]]、[[环检测]]

## 📝 解题步骤

### BFS (Kahn 算法)

1. **建图** — `graph[b].append(a)` 表示 b 是 a 的前置，同时 `indegree[a] += 1`
2. **入队** — 将所有入度为 0 的节点加入队列（这些课没有前置要求，可以直接学）
3. **BFS** — 逐个取出队首节点，计数加一。遍历它的所有邻居，邻居入度减一；如果入度变 0，入队
4. **判断** — 如果 `count == numCourses`，说明学完了所有课，无环；否则有环

### DFS (三色标记)

1. **建图** — 同 BFS
2. **DFS `has_cycle(node)`**：
   - `state[node] == 1`：正在访问中，发现环，返回 True
   - `state[node] == 2`：已访问完，返回 False
   - 标记为 1，遍历邻居递归
   - 标记为 2，返回 False
3. **遍历所有节点** — 对每个状态为 0 的节点启动 DFS

## 🔢 手动走一遍

输入: `numCourses = 4`, `prerequisites = [[1,0],[2,0],[3,1],[3,2]]`

### 图的直观理解

```
    0
   / \
  v   v
  1   2
   \ /
    v
    3
```

依赖：学 1 要先学 0，学 2 要先学 0，学 3 要先学 1 和 2。

### BFS (Kahn 算法) 执行过程

| 轮次 | 队列 | 出队 | count | 受影响的邻居 | 入队的新节点 |
|------|------|------|-------|-------------|-------------|
| 初始 | [0] | - | 0 | - | - |
| 1 | [] | 0 | 1 | 1 入度 1->0, 2 入度 1->0 | 1, 2 |
| 2 | [1, 2] | 1 | 2 | 3 入度 2->1 | - |
| 3 | [2] | 2 | 3 | 3 入度 1->0 | 3 |
| 4 | [3] | 3 | 4 | - | - |

`count = 4 = numCourses`，无环，返回 True。学习顺序：`0 -> 1 -> 2 -> 3`（或 `0 -> 2 -> 1 -> 3`）。

### 有环的例子

输入: `numCourses = 2`, `prerequisites = [[1,0],[0,1]]`

图：`0 → 1 → 0`，有环。

DFS 检测：从 0 开始，state[0]=1，递归到 1，state[1]=1，递归到 0，state[0]==1，发现环！

BFS 检测：入度都是 1，没有入度为 0 的节点，队列为空，count=0!=2，有环。

## 💻 解法

> BFS (Kahn 算法) 和 DFS (三色标记) 两种方法，推荐 BFS 更直观。

```python
from typing import List
from collections import deque


class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        # 时间复杂度 O(V+E)，空间复杂度 O(V+E)

        # 方法一：DFS 检测环
        def method_dfs():
            # 建图
            graph = [[] for _ in range(numCourses)]
            for a, b in prerequisites:
                graph[b].append(a)  # b → a，学了 b 才能学 a

            # 访问状态：0=未访问，1=访问中（在当前DFS路径上），2=已访问完
            state = [0] * numCourses

            def has_cycle(node):
                # 如果当前节点正在访问中，说明发现环
                if state[node] == 1:
                    return True
                # 如果已经访问完，直接返回 False
                if state[node] == 2:
                    return False

                # 标记为访问中
                state[node] = 1

                # 递归访问所有邻居
                for neighbor in graph[node]:
                    if has_cycle(neighbor):
                        return True

                # 标记为已访问完
                state[node] = 2
                return False

            # 对每个未访问节点进行 DFS
            for i in range(numCourses):
                if state[i] == 0:
                    if has_cycle(i):
                        return False
            return True

        # 方法二：BFS（Kahn 算法）
        def method_bfs():
            # 建图并统计入度
            graph = [[] for _ in range(numCourses)]
            indegree = [0] * numCourses

            for a, b in prerequisites:
                graph[b].append(a)  # b → a
                indegree[a] += 1  # a 的入度增加

            # 将所有入度为 0 的节点加入队列
            queue = deque([i for i in range(numCourses) if indegree[i] == 0])
            count = 0  # 已处理的节点数

            while queue:
                node = queue.popleft()
                count += 1

                for neighbor in graph[node]:
                    indegree[neighbor] -= 1
                    if indegree[neighbor] == 0:
                        queue.append(neighbor)

            # 如果处理了所有节点，则无环
            return count == numCourses

        # 两种方法任选一种，这里用 BFS（更直观）
        return method_bfs()
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(V + E) | V 是课程数，E 是前置关系数。每个节点和边最多访问一次 |
| 空间 | O(V + E) | 邻接表 O(E)，入度数组/状态数组 O(V)，队列/递归栈 O(V) |

## ⚠ 易错点

- [ ] **边的方向**：`prerequisites[i] = [a, b]` 表示「学 a 前必须先学 b」，图中应该是 `b → a` 而不是 `a → b`
- [ ] **DFS 三色标记**：state 的三种状态要区分清楚。只有当 state==1 时才说明有环（遇到了还在递归路径上的节点）
- [ ] **BFS 的计数比较**：`count == numCourses` 判断无环。如果 `count < numCourses`，说明有环（某些节点入度永远不为 0）
- [ ] **可能有多个连通分量**：图不一定连通（有些课没有前置），所以需要遍历所有节点，不能只从 0 号节点开始
- [ ] BFS 中入度数组的更新：减少邻居入度后，要检查是否为 0，为 0 才入队

## 🎤 面试怎么说

```text
这道题本质是判断有向图是否有环，我可以用两种方法。

方法一是 BFS 的拓扑排序（Kahn 算法）：
1. 建图，统计每个节点的入度
2. 把所有入度为 0 的节点放入队列（没有前置课程）
3. 每次从队列取出一个节点，修完它，把它所有后置课程的入度减一
4. 如果某个后置课程入度变成 0，入队
5. 修完的课程数等于总课程数，说明无环

方法二是 DFS 的三色标记法：
用 0/1/2 分别表示未访问、访问中、已访问完。
在 DFS 递归过程中，如果遇到状态为 1 的节点，说明遇到了环。

两种方法都是 O(V+E) 时间，O(V+E) 空间。
我推荐面试中用 BFS 的 Kahn 算法，逻辑更直观。

注意一个细节：prerequisites[i] = [a, b] 表示学 a 前必须先学 b，
所以边是 b -> a，方向不要弄反。
```
