# 399. 除法求值

## 🎯 人话翻译

> 给一堆除法等式（如 a/b = 2.0, b/c = 3.0），再给一堆查询（如 a/c = ?），求每个查询的结果。如果推不出来，返回 -1.0。

## 💡 核心思想

> **带权有向图 + DFS/BFS 路径搜索。**
>
> 把每个变量看作图的节点，等式 `a / b = val` 表示两条有权边：
> - `a → b`，权重 `val`（a / b = val，所以 b = a * 1/val）
> - `b → a`，权重 `1/val`
>
> 查询 `x / y` 等价于：从节点 x 出发，沿着边走到 y，路径上所有边的权重乘积就是结果。
>
> **或者用并查集（Union-Find）带权合并**：在 Union 时维护节点到父节点的比值关系，查询时路径压缩计算比值。

用到：[[图]]、[[DFS]]、[[BFS]]、[[并查集]]

## 📝 解题步骤

### DFS 搜索法

1. **建图** — 用邻接表，`graph[u] = [(v, weight)]`。对每个等式 `(a, b, val)`，添加两条边：`a → b (val)` 和 `b → a (1/val)`
2. **对每个查询 (x, y) 做 DFS**：
   - 如果 x 或 y 不在图中，返回 -1.0
   - 如果 x == y，返回 1.0
   - 用 `visited` 集合避免重复访问
   - 对 x 的每个邻居 (neighbor, weight)，递归搜索 neighbor 到 y 的路径
   - 如果找到路径（返回值 != -1.0），路径的乘积 = weight * 子路径结果
3. **返回所有查询的结果**

### 并查集法（更优）

1. 为每个变量分配一个 parent，初始 parent 是自己，weight = 1.0（表示自己/自己）
2. 对每个等式 `(a, b, val)`：Union(a, b, val)，把 a 连接到 b 上，`weight[a] = val * weight[b] / weight[a]`
3. 查询：如果 a 和 b 在同一集合，`a / b = weight[a] / weight[b]`；否则 -1.0

## 🔢 手动走一遍

输入: `equations = [["a","b"],["b","c"]]`, `values = [2.0, 3.0]`, `queries = [["a","c"],["b","a"],["a","e"]]`

### 建图

```
    a ---2.0---> b ---3.0---> c
    a <--0.5--- b <--0.333-- c
```

邻接表：
```
graph["a"] = [("b", 2.0)]
graph["b"] = [("a", 0.5), ("c", 3.0)]
graph["c"] = [("b", 1/3)]
```

### 查询 "a/c"

DFS 从 a 出发：
- a -> b (weight=2.0)，visited={a}
  - b -> c (weight=3.0)，visited={a, b}，c == end，返回 3.0
  - 当前路径乘积 = 2.0 * 3.0 = 6.0

### 查询 "b/a"

DFS 从 b 出发：
- b -> a (weight=0.5)，visited={b}，a == end，返回 0.5

### 查询 "a/e"

e 不在图中，直接返回 -1.0。

结果：`[6.0, 0.5, -1.0]`

## 💻 解法

> DFS 搜索法：建带权有向图，查询就是找两点间路径的权重乘积。

```python
from typing import List
from collections import defaultdict


class Solution:
    def calcEquation(self, equations: List[List[str]], values: List[float], queries: List[List[str]]) -> List[float]:
        # 时间复杂度 O(V+E)，空间复杂度 O(V+E)

        # 1. 构建图：邻接表，graph[u] = [(v, weight), ...]
        graph = defaultdict(list)

        # 构建有向图，存储 a/b 和 b/a 两条边
        for (a, b), val in zip(equations, values):
            graph[a].append((b, val))  # a -> b，权重 val
            graph[b].append((a, 1.0 / val))  # b -> a，权重 1/val

        # 2. DFS 搜索路径
        def dfs(start: str, end: str, visited: set) -> float:
            # 如果起点或终点不在图中，返回 -1
            if start not in graph or end not in graph:
                return -1.0

            # 起点等于终点，结果为 1
            if start == end:
                return 1.0

            visited.add(start)

            # 遍历所有邻居
            for neighbor, weight in graph[start]:
                if neighbor in visited:
                    continue

                # 递归查找 neighbor 到 end 的路径
                res = dfs(neighbor, end, visited)
                if res != -1.0:
                    # 当前路径的乘积 = 当前边权重 * 子路径乘积
                    return weight * res

            return -1.0

        # 3. 处理所有查询
        result = []
        for a, b in queries:
            result.append(dfs(a, b, set()))

        return result
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(Q x (V + E)) | Q 个查询，每个查询最多遍历整张图。V 是变量数，E 是等式数 |
| 空间 | O(V + E) | 邻接表存储，递归栈/visited 集合 |

## ⚠ 易错点

- [ ] **建双向边**：等式 `a/b = val` 既要加 `a→b (val)` 也要加 `b→a (1/val)`。单边会导致某些查询无法求解
- [ ] **路径权重是乘积不是求和**：从 a 到 b 到 c，`a/c = (a/b) * (b/c)`，不是相加
- [ ] **DFS 的 visited 不能忘记**：图的边可能形成环，不加 visited 会无限递归
- [ ] **不存在的变量返回 -1.0**：如果查询中的变量从未在任何等式中出现，直接返回 -1.0
- [ ] 可以用 `defaultdict(list)` 简化邻接表初始化，但也要在 DFS 中检查 key 是否存在

这道题把除法关系建模成带权有向图。

核心思路：
1. 每个变量是图的节点

   a → b 权重为 val，b → a 权重为 1/val
