# 433. 最小基因变化

## 🎯 人话翻译

> 基因序列是由 A、C、G、T 组成的 8 字符字符串。每次可以改一个字符，改完的序列必须在基因库中。问从起始序列到目标序列最少需要多少次变化。

## 💡 核心思想

> **与 [[127. 单词接龙]] 几乎完全相同，是简化版的最短路径 BFS 问题。**
>
> 每个基因序列看作图的节点，一次变化（改一个字符）产生的新序列如果是基因库中的有效序列，则有一条边。问题等价于求 start 到 end 的最短路径。
>
> 关键区别：字母集只有 4 个（A/C/G/T），所以不需要预处理通用状态模式，直接对每个位置的每个可能字符枚举即可（8 x 4 = 32 种尝试）。

用到：[[图]]、[[BFS]]、[[最短路径]]

## 📝 解题步骤

1. **边界判断** — start == end 返回 0；end 不在 bank 中返回 -1
2. **转为集合** — bank 转为 set，O(1) 查找
3. **BFS 初始化** — 队列 `(当前序列, 变化次数)`，visited 集合
4. **BFS 循环** — 出队一个序列，对每个位置尝试替换为 A/C/G/T：
   - 如果新序列等于 end，返回 `steps + 1`
   - 如果新序列在 bank 中且未访问，入队
5. **未找到** — 返回 -1

## 🔢 手动走一遍

输入: `start = "AACCGGTT"`, `end = "AACCGGTA"`, `bank = ["AACCGGTA"]`

### 基因字符集：A, C, G, T

| 步 | 当前序列 | 尝试位置 | 改变 | 新序列 | 在 bank 中? | 说明 |
|----|---------|---------|------|--------|------------|------|
| 1 | AACCGGTT | 0-6 | 各种 | ... | No | 前 7 个位置改完后都不在 bank |
| 1 | AACCGGTT | 7 | T->A | AACCGGTA | Yes | 找到了 end！ |

结果：steps = 1。

---

另一个例子: `start = "AAAAACCC"`, `end = "AACCCCCC"`, `bank = ["AAAACCCC","AAACCCCC","AACCCCCC"]`

### BFS 搜索图

```
     AAAAACCC (level 0)
         |
     AAAACCCC (level 1) [改第 4 位 A->C]
         |
     AAACCCCC (level 2) [改第 3 位 A->C]
         |
     AACCCCCC (level 3) [改第 2 位 A->C] = end!
```

最少变化次数 = 3。

## 💻 解法

> BFS：枚举每个位置替换为 A/C/G/T，检查是否在基因库中。

```python
from typing import List
from collections import deque


class Solution:
    def minMutation(self, start: str, end: str, bank: List[str]) -> int:
        # 时间复杂度 O(L × 4 × N)，空间复杂度 O(N)

        # 特殊情况：起点等于终点，不需要变化
        if start == end:
            return 0

        # 将 bank 转为集合，便于快速查找
        bank_set = set(bank)

        # 如果 end 不在 bank 中，直接返回 -1
        if end not in bank_set:
            return -1

        # 字符选项
        genes = ['A', 'C', 'G', 'T']

        # BFS 队列：(当前序列, 变化次数)
        queue = deque([(start, 0)])
        visited = set([start])

        while queue:
            curr, steps = queue.popleft()

            # 尝试改变每个位置的字符
            for i in range(len(curr)):
                for g in genes:
                    if curr[i] == g:
                        continue
                    # 生成新序列
                    new = curr[:i] + g + curr[i + 1:]

                    # 到达目标
                    if new == end:
                        return steps + 1

                    # 检查是否在 bank 中且未访问
                    if new in bank_set and new not in visited:
                        visited.add(new)
                        queue.append((new, steps + 1))

        return -1
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(L x 4 x N) | L=8（序列长度），4 种碱基，N 是 bank 大小。每个序列最多被访问一次 |
| 空间 | O(N) | visited 集合和队列最多存储 N 个序列 |

## ⚠ 易错点

- [ ] **end 不在 bank 中直接返回 -1**：即使 start 能变成 end，如果 end 不在 bank 中，也是非法的
- [ ] **跳过相同字符**：`if curr[i] == g: continue`，不跳过会导致当前序列被标记为 neighbor，浪费一次操作但 BFS 正确性不受影响
- [ ] **visited 在入队时标记**：不是出队时标记，否则同一个序列可能多次入队
- [ ] **与 127 的区别**：本题基因库小（最多 10 个序列），字母集只有 4 个，所以直接枚举 8x4=32 种变化比用通用状态预处理更简单
- [ ] BFS 的返回值：找到 end 时返回 `steps + 1`（因为当前是 steps 次变化，到 end 还需要一步）

## 🎤 面试怎么说

```text
这道题和 127 单词接龙几乎是同一道题，都是求无权图的最短路径，用 BFS。

区别在于：
1. 字母集只有 A/C/G/T 四种，不是 26 个字母
2. 序列长度固定为 8
3. 基因库最多 10 个序列，规模很小

所以不需要像 127 那样做通用状态预处理。
直接枚举每个位置的 4 种可能替换（共 8x4=32 种尝试）即可。

BFS 流程：
- 队列放 (当前序列, 变化次数)
- 出队，枚举所有一次变化可达的序列
- 如果在 bank 中且未访问，入队
- 找到 end 返回步数+1，队列空返回 -1

时间 O(L x 4 x N)，空间 O(N)。
```
