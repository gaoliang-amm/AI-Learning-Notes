# 127. 单词接龙

## 🎯 人话翻译

> 给你一个起始单词、一个目标单词、一个单词字典。每次只能改变单词中的一个字母，且改完的单词必须在字典里。问最少要几步能从起始变到目标（包括起始和目标），变不了就返回 0。

## 💡 核心思想

> **把每个单词看作图的节点，两个单词之间如果只差一个字母，则有一条边。问题变成：求图中 start 到 end 的最短路径长度。**
>
> BFS 天然适合求无权图的最短路径。关键在于如何快速找到当前单词的所有邻居（即字典中只差一个字母的单词）。
>
> **优化：预处理「通用状态」映射。** 将每个单词的每个位置替换为通配符 `*`（如 "hot" -> `["*ot", "h*t", "ho*"]`），把拥有相同通用状态的单词归为一组。这样找邻居时，只需要生成当前单词的所有通用状态，就能 O(1) 找到所有邻居。

用到：[[图]]、[[BFS]]、[[最短路径]]、[[哈希表]]

## 📝 解题步骤

1. **边界判断** — start == end 直接返回 1；end 不在字典中返回 0
2. **构建通用状态映射** — 对字典中每个单词，生成所有带通配符的模式（如 "hot" -> `*ot, h*t, ho*`），用哈希表将同一模式下的单词归类
3. **BFS 初始化** — 队列存放 `(当前单词, 步数)`，visited 集合记录已访问单词
4. **BFS 循环** — 出队一个单词，生成它的所有通用状态模式，每个模式对应一组邻居单词：
   - 如果邻居是 endWord，返回 `steps + 1`
   - 如果邻居未访问过，标记访问，入队
5. **BFS 结束未找到** — 返回 0

## 🔢 手动走一遍

输入: `beginWord = "hit"`, `endWord = "cog"`, `wordList = ["hot","dot","dog","lot","log","cog"]`

### 通用状态映射

```
*ot: hot, dot, lot
h*t: hot
ho*: hot
d*t: dot
do*: dot, dog
...
```

### BFS 遍历

```
       hit (level 1)
        |
       hot (level 2) [h*t: hot, *ot: hot/dot/lot, ho*: hot]
      /     \
    dot     lot (level 3) [来自 *ot]
    |        |
    dog     log (level 4) [do*: dog, d*t没有]
    |
    cog (level 5) [do*: dog]
```

| 层级 | 单词 | 步数 | 说明 |
|------|------|------|------|
| 1 | hit | 1 | 起点 |
| 2 | hot | 2 | hit -> hot（改 i 为 o） |
| 3 | dot, lot | 3 | hot -> dot（改 h 为 d）/ lot（改 h 为 l） |
| 4 | dog, log | 4 | dot -> dog / lot -> log |
| 5 | cog | 5 | dog -> cog（改 d 为 c），找到 target！ |

最短路径长度 = 5。路径：`hit -> hot -> dot -> dog -> cog`。

## 💻 解法

> BFS + 通用状态预处理：用通配符模式快速找邻居，标准 BFS 求最短路径。

```python
from typing import List
from collections import deque, defaultdict


class Solution:
    def ladderLength(self, beginWord: str, endWord: str, wordList: List[str]) -> int:
        # 时间复杂度 O(L × N × 26)，空间复杂度 O(N × L)

        # 1. 特殊情况：起点等于终点
        if beginWord == endWord:
            return 1

        # 2. 将 wordList 转为集合，便于 O(1) 查找
        word_set = set(wordList)

        # 3. 如果终点不在单词列表中，无法转换
        if endWord not in word_set:
            return 0

        # 4. 预处理：生成所有通用状态模式
        #    例如 "hot" -> ["*ot", "h*t", "ho*"]
        pattern_map = defaultdict(list)
        L = len(beginWord)

        for word in word_set:
            for i in range(L):
                pattern = word[:i] + '*' + word[i + 1:]
                pattern_map[pattern].append(word)

        # 5. BFS 初始化
        queue = deque([(beginWord, 1)])  # (当前单词, 路径长度，包含 beginWord)
        visited = {beginWord}  # 已访问的单词集合

        # 6. BFS 主循环
        while queue:
            curr, steps = queue.popleft()

            # 生成当前单词的所有通用状态
            for i in range(L):
                pattern = curr[:i] + '*' + curr[i + 1:]

                # 遍历所有匹配该通用状态的单词
                for neighbor in pattern_map[pattern]:
                    # 找到终点，直接返回
                    if neighbor == endWord:
                        return steps + 1

                    # 未访问过的单词，加入队列
                    if neighbor not in visited:
                        visited.add(neighbor)
                        queue.append((neighbor, steps + 1))

        # 7. BFS 结束未找到，返回 0
        return 0
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(L x N x 26) | L 是单词长度，N 是字典大小。每个单词生成 L 个模式，每个模式对应最多 26 个字母的变化 |
| 空间 | O(N x L) | 通用状态映射存储 N 个单词 x L 个模式 |

## ⚠ 易错点

- [ ] **start 不在 wordList 中**：起始单词可能不在字典中，但这不影响 BFS。但需要把 start 加入 visited
- [ ] **endWord 不在字典中直接返回 0**：这是最早要做的判断，否则 BFS 会白跑
- [ ] **通用状态的构建**：`pattern = word[:i] + '*' + word[i+1:]`，这样同一模式下的单词互为邻居
- [ ] **visited 的时机**：需要在入队时标记 visited，不能等到出队时再标记，否则大量重复入队
- [ ] 可以用双向 BFS 优化：同时从 start 和 end 出发，两个搜索相遇时停止，显著减少搜索空间

## 🎤 面试怎么说

```text
这道题本质是求无权图的最短路径，所以用 BFS。

关键优化在于如何高效找到邻居。如果每步都遍历整个字典 O(N*L)、
一个一个检查是否差一个字母，会超时。

我采用预处理通用状态的技巧：
把每个单词的每个位置依次替换成通配符 *，得到一个模式。
比如 hot 可以生成 *ot、h*t、ho*。
把字典中拥有相同模式的单词归为一组（用哈希表）。

这样在 BFS 时，对当前单词生成它的所有模式，
就能在 O(1) 时间内找到所有邻居。

BFS 的标准流程：
队列中存放 (单词, 步数)，visited 集合去重。
遇到 endWord 就返回步数+1。
队列空了还没找到就返回 0。

时间 O(L * N * 26)，空间 O(N * L)。
如果面试官要优化，可以提双向 BFS，从两端同时搜索。
```
