# 数据结构与算法基础

> 参考：《尚硅谷大模型技术之数据结构与算法》
> 刷题地址：https://leetcode.cn/studyplan/top-interview-150/

## 一、知识体系总览

整个知识体系分为三层：

```
第一层：基础理论 — 什么是数据结构和算法，如何衡量优劣（复杂度）
第二层：数据结构 — 数据怎么存：线性结构（数组/链表/栈/队列/哈希表）+ 非线性结构（树/图）
第三层：算法 — 数据怎么用：查找/排序/分治/动态规划/回溯/贪心
```

每一层都建立在前一层之上。数据结构是算法的载体，算法是数据结构的灵魂。

### 主线一：从存储到访问

```
连续存储（数组） → 随机访问 O(1)，插入删除 O(n)
分散存储（链表） → 插入删除 O(1)，随机访问 O(n)
映射存储（哈希表）→ 通过哈希函数直接定位 O(1) 均摊
```

**直觉**：数组像电影院座位——编号连续，找座位快，但插队要 everyone 挪动；链表像寻宝游戏——每个线索指向下一个，插队只需改线索，但找第100个要一步步走；哈希表像图书馆索书号——直接定位，但要处理"索书号相同"的冲突。

### 主线二：从暴力到优化

```
暴力枚举 O(n²) → 二分查找 O(logn) → 哈希查找 O(1)
冒泡/选择/插入 O(n²) → 归并/快排/堆排 O(nlogn)
递归重复计算 O(2ⁿ) → 记忆化 → DP O(n)
```

### 主线三：从独立子问题到重叠子问题

```
分治算法 → 子问题独立（归并排序）
动态规划 → 子问题重叠（爬楼梯、背包问题）
回溯算法 → 搜索所有可能解（全排列、N皇后）
贪心算法 → 每步局部最优（期望全局最优）
```

> [!tip] 关键区别
> - 分治：子问题**相互独立**，各自求解后合并
> - DP：子问题**有重叠**，保存中间结果避免重复计算
> - 回溯：子问题构成**搜索树**，走到死胡同回退
> - 贪心：**不回溯**，每步只看眼前最优

### 核心时间复杂度速查

| 复杂度 | 名称 | 典型场景 |
|--------|------|---------|
| O(1) | 常数 | 数组索引访问、哈希表 get/put |
| O(logn) | 对数 | 二分查找、平衡树操作 |
| O(n) | 线性 | 遍历数组、单遍扫描 |
| O(nlogn) | 线性对数 | 归并排序、快速排序（平均）、堆排序 |
| O(n²) | 平方 | 冒泡/插入/选择排序、双层循环 |
| O(2ⁿ) | 指数 | 子集枚举、递归斐波那契 |
| O(n*n!) | 阶乘 | 全排列 |

> [!note] 面试一句话
> 时间复杂度统计的是**基本指令数随输入规模 n 增大时的变化趋势**，不关注绝对运行时间。我们主要关注**最坏时间复杂度**。空间复杂度描述的是**内存空间随输入规模的变化趋势**。

## 二、数据结构核心

### 2.1 数组（Array）

连续存储的线性结构，通过索引 O(1) 访问。

**核心特性**：

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 随机访问 | O(1) | 已知索引直接计算地址 |
| 查找（无序） | O(n) | 需遍历 |
| 查找（有序） | O(logn) | 二分查找 |
| 插入 | O(n) | 可能需要搬移元素 |
| 删除 | O(n) | 可能需要搬移元素 |

**关键概念**：

- **连续存储**：内存中相邻，CPU 缓存友好
- **扩容机制**：容量满时创建 2 倍新数组 + 拷贝，摊还分析下 append 均摊 O(1)
- **Python 的 list**：不是纯数组，内部存储的是对象引用（指针），真正的数组用 `array` 模块或 `numpy.ndarray`

> **直觉**：数组是**空间换时间**的典型——预先分配连续内存，换取 O(1) 随机访问。

### 2.2 链表（Linked List）

分散存储的线性结构，每个节点包含数据和指向下一个节点的指针。

**三种类型**：

| 类型 | 特点 | 适用场景 |
|------|------|---------|
| 单向链表 | 只指向下一个 | 基础实现，栈/队列 |
| 双向链表 | 指向前后两个 | 需要反向遍历（Python dict 内部） |
| 环形链表 | 尾节点指向头节点 | 约瑟夫环、循环调度 |

**核心特性**：

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 随机访问 | O(n) | 必须从头遍历 |
| 查找 | O(n) | 必须遍历 |
| 插入（已知位置） | O(1) | 改指针即可 |
| 删除（已知位置） | O(1) | 改指针即可 |

**所有数据结构都是基于数组、链表或二者的组合实现的**。栈和队列可用数组或链表实现；哈希表的实现同时包含数组和链表。

> [!note] 面试必手撕
> 链表反转、快慢指针找中环、合并两个有序链表——面试最常见链表题。核心就是**指针操作**，关键是画图搞清楚每个步骤的指针指向。

> **直觉**：链表是**时间换空间**——用指针换连续内存，换取 O(1) 插入删除。

### 2.3 栈（Stack）

**后进先出（LIFO / FILO）**的线性结构。本质是**限制了操作的数组或链表**——只能在栈顶操作。

| 操作 | 时间复杂度 |
|------|-----------|
| push（入栈） | O(1) |
| pop（出栈） | O(1) |
| peek（查看栈顶） | O(1) |

**经典应用**：

- 括号匹配（LeetCode 20）——左括号入栈，右括号出栈匹配
- 函数调用栈——递归调用的底层实现
- **DFS 的迭代实现**——用栈模拟递归
- 表达式求值（中缀转后缀）

> **直觉**：栈像羽毛球筒——最后放的最先取。算法中栈是**"记住路径"**的工具。

### 2.4 队列（Queue）

**先进先出（FIFO）**的线性结构。

| 操作 | 时间复杂度 |
|------|-----------|
| enqueue（入队） | O(1) |
| dequeue（出队） | O(1) |
| peek（查看队首） | O(1) |

**变体**：

| 类型 | 特点 |
|------|------|
| 单向队列 | 队尾入、队首出 |
| 双向队列（Deque） | 两端都可入/出（Python `collections.deque`） |
| 优先队列 | 出队顺序由优先级决定（底层通常是堆） |
| 循环队列 | 用取模实现环形，避免假溢出 |

**经典应用**：

- **BFS 的迭代实现**——用队列存储待访问节点
- 消息队列、缓冲区管理

> **直觉**：队列像排队买票——先来先服务。算法中队列是**"按层探索"**的工具。

### 2.5 哈希表（Hash Table）

通过**哈希函数**将 key 映射到数组索引，实现 O(1) 均摊查找。

**核心机制**：`索引 = hash(key) % 数组长度`

**冲突解决**：

| 方法 | 思路 | 特点 |
|------|------|------|
| 链式法（Chaining） | 每个位置挂一个链表 | 实现简单，负载因子可 > 1 |
| 开放寻址法 | 冲突时按策略找空位（线性探测/二次探测） | 缓存友好，负载因子必须 < 1 |

**关键概念**：

- **负载因子** = 元素个数 / 表大小，通常 **0.75** 时扩容
- **扩容**：创建 2 倍新表，所有元素重新哈希（O(n)，但不频繁）
- **均摊 O(1)**：大多数操作 O(1)，扩容代价分摊到每次操作仍是 O(1)

> [!note] 面试要点
> Python 的 `dict` 和 `set` 底层就是哈希表。Python 3.7+ 的 dict 用开放寻址法，比早期的链式法更快且内存更紧凑，且保证插入顺序。

> **直觉**：哈希表是**空间换时间**的极致——用额外数组 + 哈希函数，换取 O(1) 直接定位。

### 2.6 树（Tree）

层次关系的非线性结构。

**核心术语**：

| 术语 | 定义 |
|------|------|
| 根节点 | 没有父节点的顶层节点 |
| 叶节点 | 没有子节点的底层节点 |
| 深度 | 从根到该节点的边数 |
| 高度 | 从该节点到最远叶节点的边数 |

**二叉树遍历**：

| 遍历方式 | 顺序 | 适用场景 |
|---------|------|---------|
| 前序（DLR） | 根 → 左 → 右 | 复制树、表达式前缀 |
| 中序（LDR） | 左 → 根 → 右 | BST 中序 = 升序序列 |
| 后序（LRD） | 左 → 右 → 根 | 删除树、表达式后缀 |
| 层序（BFS） | 逐层从左到右 | 最短路径、宽度优先 |

**为什么堆用数组实现**：完全二叉树可以紧凑地用数组存储，节点 i 的左子 = 2i+1，右子 = 2i+2，无需指针，空间连续、缓存友好。

### 2.7 图（Graph）

表示**多对多**关系的非线性结构。图是比树更普遍的结构，树是一种特殊的图。

**术语**：结点（顶点）、边、权重、邻接、度（入度/出度）、路径、环。

**两种存储方式**：

| 方式 | 空间 | 判断是否有边 | 适用场景 |
|------|------|-------------|---------|
| 邻接矩阵 | O(V²) | O(1) | 稠密图 |
| 邻接表 | O(V+E) | O(度) | 稀疏图（更常用） |

**两种遍历**：

| 方式 | 数据结构 | 时间复杂度（邻接表） | 本质 |
|------|---------|---------------------|------|
| BFS（广度优先） | 队列 | O(V+E) | 逐层扩展：先访问所有邻居，再访问邻居的邻居 |
| DFS（深度优先） | 栈/递归 | O(V+E) | 深入到底：一条路走到黑，走不通再回溯 |

## 三、算法核心

### 3.1 查找算法

#### 3.1.1 二分查找（Binary Search）

**前提**：数组必须有序。

**算法原理**：利用数据的有序性，每轮将搜索范围缩小一半，直到找到目标或搜索区间为空。

**基本步骤**：
1. 初始化：`left = 0, right = len(arr) - 1`
2. 循环条件：`left <= right`
3. 计算中点：`mid = left + (right - left) // 2`（防止整数溢出）
4. 比较判断：`arr[mid] == target` 返回；`arr[mid] < target` 则 `left = mid+1`；否则 `right = mid-1`
5. 循环结束未找到，返回 -1

```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = left + (right - left) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

**复杂度**：时间 O(logn)，空间 O(1)。

**易错点**：

- `mid` 用 `left + (right-left)//2` 而非 `(left+right)//2`，防止溢出
- `while left <= right`（闭区间）vs `while left < right`（左闭右开），取决于区间定义
- 找左/右边界时，相等时不能返回，要继续收缩

**变体**：

| 变体 | 调整方式 |
|------|---------|
| 找第一个 == target（左边界） | 相等时 `right = mid - 1`，最终检查 `left` |
| 找最后一个 == target（右边界） | 相等时 `left = mid + 1`，最终检查 `right` |
| 找第一个 >= target | `arr[mid] < target` 时 `left = mid+1` |
| 旋转数组查找 | 先判断哪边有序，再决定收缩方向 |

> [!note] 面试一句话
> 二分的精髓不是模板，而是理解**搜索区间的定义**（开/闭）和**收缩方向**（相等时往哪走取决于找左边界还是右边界）。

#### 3.1.2 多数元素（摩尔投票法）

**问题**：找出现次数超过一半的元素，要求 O(n) 时间、O(1) 空间。

**核心思想**：对立抵消。维护候选元素 `candidate` 和计数 `count`——遇到相同 +1，不同 -1，count 归零时换候选。活下来的即为多数元素。

```python
def majority_element(nums):
    count, candidate = 0, None
    for num in nums:
        if count == 0:
            candidate = num
        count += (1 if num == candidate else -1)
    return candidate
```

> **直觉**：好比投票，不同阵营两两抵消，人数超过半数的阵营最终会有人"活下来"。

### 3.2 排序算法

#### 八大排序对比

| 排序 | 最好 | 平均 | 最坏 | 空间 | 稳定性 | 适用场景 |
|------|------|------|------|------|--------|---------|
| 冒泡 | O(n) | O(n²) | O(n²) | O(1) | ✅稳定 | 基本有序的小数据 |
| 选择 | O(n²) | O(n²) | O(n²) | O(1) | ❌不稳定 | 数据量小 |
| 插入 | O(n) | O(n²) | O(n²) | O(1) | ✅稳定 | 基本有序/小数据 |
| 希尔 | O(nlogn) | O(nlogn) | O(n²) | O(1) | ❌不稳定 | 中等数据 |
| 归并 | O(nlogn) | O(nlogn) | O(nlogn) | O(n) | ✅稳定 | 需要稳定排序/外部排序 |
| 快排 | O(nlogn) | O(nlogn) | O(n²) | O(logn) | ❌不稳定 | **通用首选** |
| 堆排 | O(nlogn) | O(nlogn) | O(nlogn) | O(1) | ❌不稳定 | top-K 问题 |
| 计数/桶/基数 | O(n+k) | O(n+k) | O(n+k) | O(n+k) | ✅稳定 | 特定范围整数 |

**稳定性含义**：相等元素排序后是否保持原有相对顺序。稳定排序可以用来做多关键字排序（先按次要关键字排，再按主要关键字排）。

> [!tip] 面试高频
> **快排和归并是面试最爱考的**。快排核心是 partition，归并核心是 merge。

#### 3.2.1 冒泡排序（Bubble Sort）

**原理**：将数组分为无序区间（前）和有序区间（后）。每轮从无序区间第一个元素开始，两两比较相邻元素，若前者 > 后者则交换。一轮下来，无序区间的最大值"冒泡"到末尾进入有序区间。重复直到无序区间为空。

**过程示意**（`[5, 1, 3, 4, 2]`）：
```
第1轮: [1,3,4,2,5]  ← 5 冒到最后
第2轮: [1,3,2,4,5]  ← 4 冒到倒数第二
第3轮: [1,2,3,4,5]  ← 3 冒到倒数第三
第4轮: [1,2,3,4,5]  ← 无交换，提前结束
```

```python
def bubble_sort(arr):
    for i in range(len(arr)):
        swapped = False
        for j in range(0, len(arr) - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:  # 优化：本轮无交换说明已有序
            break
```

**复杂度**：时间 O(n²)（最好 O(n)——加 swapped 优化后），空间 O(1)，稳定。

**优化**：加 `swapped` 标志，若某轮无交换则提前结束。基本有序时可优化到 O(n)。

#### 3.2.2 选择排序（Selection Sort）

**原理**：将数组分为有序区间（前）和无序区间（后）。每轮从无序区间中找到最小元素，与无序区间第一个元素交换，使其进入有序区间。

```python
def selection_sort(arr):
    for i in range(len(arr)):
        min_idx = i
        for j in range(i + 1, len(arr)):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
```

**复杂度**：时间 O(n²)（最好/最坏/平均都是），空间 O(1)，**不稳定**。

**为什么不稳定**：例如 `[5, 5*, 3]`，第1轮选出 3 与位置 0 的 5 交换，两个 5 的相对顺序改变了。

#### 3.2.3 插入排序（Insertion Sort）

**原理**：将数组分为有序区间（前）和无序区间（后）。起初第一个元素视为有序。每轮从无序区间取第一个元素作为 key，从有序区间末尾向前比较，大于 key 的元素后移一位，找到正确位置插入 key。

```python
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i
        while j > 0 and arr[j - 1] > key:
            arr[j] = arr[j - 1]  # 后移
            j -= 1
        arr[j] = key  # 插入
```

**复杂度**：时间 O(n²)（最好 O(n)——已有序时），空间 O(1)，稳定。

**直觉**：像打扑克牌——每次摸一张牌，插到手里已排好的牌的正确位置。**基本有序时最优**（接近 O(n)），是小数据量的首选排序。

#### 3.2.4 希尔排序（Shell Sort）

**原理**：插入排序的改进版。先按较大间隔（gap）分组做插入排序，逐步缩小 gap 直到 1。最后一轮 gap=1 时就是标准插入排序，但此时数组已基本有序，效率很高。

```python
def shell_sort(arr):
    n = len(arr)
    gap = n // 2
    while gap > 0:
        for i in range(gap, n):
            key = arr[i]
            j = i
            while j >= gap and arr[j - gap] > key:
                arr[j] = arr[j - gap]
                j -= gap
            arr[j] = key
        gap //= 2
```

**复杂度**：时间取决于 gap 序列，一般 O(nlogn)~O(n²)，空间 O(1)，**不稳定**。

**直觉**：先让"大致在正确位置"的元素排好（大间隔），再做精细排序（小间隔）。就像先把大石头搬到位，再调整小石头。

#### 3.2.5 归并排序（Merge Sort）

**原理**：分治思想。分解阶段：将数组不断对半拆分，直到每个子数组只有 1 个元素（天然有序）。合并阶段：从最小的有序数组开始，两两合并为更大的有序数组。

**核心操作——归并（merge）**：两个有序数组合并为一个。比较两个数组的首元素，取较小者放入结果，直到一个数组为空，再把另一个数组的剩余元素全部追加。

```python
def merge(left, right):
    result, i, j = [], 0, 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)
```

**复杂度**：时间 O(nlogn)（最好/最坏/平均都是），空间 O(n)（临时数组），稳定。

**递归树分析**：每次对半分，共 log₂n 层，每层合并操作 O(n)，总计 O(nlogn)。

> [!tip] 适用场景
> 需要**稳定排序**（如按多关键字排序）、**外部排序**（数据太大无法全放内存）时首选归并。缺点是需要 O(n) 额外空间。

#### 3.2.6 快速排序（Quick Sort）

**原理**：分治思想。每次从数组中选一个元素作为**基准（pivot）**，将小于 pivot 的放左边，大于 pivot 的放右边（partition 操作）。完成后 pivot 就在最终正确位置。再对左右两个子数组递归执行同样操作。

**核心操作——partition**：选第一个元素为 pivot，从右找小于 pivot 的元素移到左边，从左找大于 pivot 的元素移到右边，左右指针重合时放入 pivot。

```python
def partition(nums, left, right):
    pivot = nums[left]  # 选第一个为基准，nums[left] 位置可被覆盖
    while left < right:
        while left < right and nums[right] >= pivot:
            right -= 1
        nums[left] = nums[right]  # 小的放左边，空出 nums[right]
        while left < right and nums[left] <= pivot:
            left += 1
        nums[right] = nums[left]  # 大的放右边，空出 nums[left]
    nums[left] = pivot  # pivot 放到重合处
    return left

def quick_sort(nums, left, right):
    if left < right:
        mid = partition(nums, left, right)
        quick_sort(nums, left, mid - 1)
        quick_sort(nums, mid + 1, right)
```

**复杂度**：

| 情况 | 时间 | 递归深度 | 空间 |
|------|------|---------|------|
| 最好（pivot 恰好中位数） | O(nlogn) | logn | O(logn) |
| 平均 | O(nlogn) | logn | O(logn) |
| 最坏（pivot 总是最大/最小） | O(n²) | n | O(n) |

**不稳定**，原地排序。

**优化**：pivot 选随机数或三数取中（first/mid/last 的中位数），避免最坏情况。

> [!note] 为什么快排最常用
> 虽然最坏 O(n²)，但通过随机 pivot 几乎不触发。平均 O(nlogn) 且**常数因子最小**（内循环短、缓存友好、原地排序）。实际测试中通常快于归并和堆排。

#### 3.2.7 堆排序（Heap Sort）

**原理**：利用堆（完全二叉树）的性质。步骤：
1. **建大顶堆**：从最后一个非叶节点 `(n//2 - 1)` 开始，自底向上堆化
2. **排序**：将堆顶（最大值）与堆末尾交换，堆大小减 1，然后自顶向下堆化恢复大顶堆性质
3. 重复步骤 2 直到堆大小为 1

**堆化（heapify）**：以节点 i 为根的子树，比较 i 与左右子节点，若子节点更大则交换，然后递归处理被交换的子树。

```python
def heapify(arr, length, parent):
    largest = parent
    left, right = 2 * parent + 1, 2 * parent + 2
    if left < length and arr[left] > arr[largest]:
        largest = left
    if right < length and arr[right] > arr[largest]:
        largest = right
    if largest != parent:
        arr[parent], arr[largest] = arr[largest], arr[parent]
        heapify(arr, length, largest)

def heap_sort(arr):
    n = len(arr)
    # 1. 建大顶堆（自底向上）
    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)
    # 2. 依次将堆顶放到末尾
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]
        heapify(arr, i, 0)  # 堆大小缩小为 i
```

**复杂度**：时间 O(nlogn)（建堆 O(n) + n 次堆化 O(logn)），空间 O(1)，**不稳定**。

**特点**：不受输入数据分布影响，最好/最坏/平均都是 O(nlogn)。但常数因子较大，实际效率不如快排。适合 **top-K** 问题（取前 K 大/小）。

#### 3.2.8 计数排序 / 桶排序 / 基数排序

**计数排序**：用额外数组统计每个值出现的次数，再按顺序输出。适合**值域小**的整数（如 0~100 的成绩排序）。时间 O(n+k)，空间 O(n+k)，稳定。

**桶排序**：将元素分配到若干个桶中，每个桶内单独排序（通常用插入排序），最后合并。适合**数据均匀分布**的场景。时间 O(n+k)（理想），最坏 O(n²)。

**基数排序**：按位排序，从最低位到最高位，每轮用稳定排序（如计数排序）。适合**位数固定**的整数/字符串。时间 O(d×(n+k))，d 为位数。

> [!tip] 面试要点
> 非比较排序（计数/桶/基数）可以突破 O(nlogn) 的下界（比较排序的理论下界），但有适用限制。面试重点还是**快排和归并**。

### 3.3 分治算法

**核心思想**：将原问题拆成多个独立子问题，递归求解，合并结果。

**三个条件**：
1. **可分解**：问题能拆成小问题
2. **有基线**：小到能直接求解
3. **可合并**：子问题的解能合并为原问题解

**典型应用**：归并排序、快速排序、汉诺塔、Karatsuba 大整数乘法。

> **与 DP 的区别**：分治的子问题**相互独立**（归并排序左右半部分互不影响），DP 的子问题**有重叠**（爬楼梯 f(n) 依赖 f(n-1) 和 f(n-2)，而 f(n-1) 又依赖 f(n-2)）。

#### 3.3.1 汉诺塔

将 n 个盘子从 A 移到 C（借助 B）：
1. 先将 n-1 个盘子从 A 移到 B（借助 C）
2. 将第 n 个盘子从 A 移到 C
3. 将 n-1 个盘子从 B 移到 C（借助 A）

```python
def hanota(n, source, target, buffer):
    if n == 1:
        target.append(source.pop())
        return
    hanota(n - 1, source, buffer, target)   # n-1 个从 source→buffer
    hanota(1, source, target, buffer)        # 第 n 个从 source→target
    hanota(n - 1, buffer, target, source)    # n-1 个从 buffer→target
```

#### 3.3.2 Karatsuba 大整数乘法

**传统乘法**：两个 n 位数相乘需要 O(n²) 次基本操作。

**Karatsuba 思想**：将 A、B 拆为高低位：`A = 10^m × A₁ + A₀`，`B = 10^m × B₁ + B₀`。

展开得 `A×B = 10^2m × A₁B₁ + 10^m × (A₁B₀ + A₀B₁) + A₀B₀`

关键技巧：令 `z₀ = A₀×B₀`，`z₂ = A₁×B₁`，`z₁ = (A₁+A₀)×(B₁+B₀)`，则：

```
A×B = 10^2m × z₂ + 10^m × (z₁ - z₂ - z₀) + z₀
```

**4次乘法变3次**，复杂度从 O(n²) 降到 O(n^1.585)。

### 3.4 动态规划（DP）

**核心思想**：把问题拆成子问题，**每个子问题只算一次**，保存中间结果。

**与分治的区别**：分治的子问题独立，DP 的子问题重叠（不同子问题有相同的子子问题）。

**两种实现方式**：
- **自上而下（记忆化递归）**：只需考虑递归公式，无需考虑子问题执行顺序
- **自下而上（迭代）**：需考虑递推关系 + 确保子问题从小到大依次执行

#### DP 解题四步法

1. **定义状态**：`dp[i]` 表示什么？
2. **状态转移方程**：`dp[i] = ...`
3. **初始化**：`dp[0]`, `dp[1]` 等基本值
4. **遍历顺序**：从下到上，确保子问题先算

#### 3.4.1 爬楼梯（LeetCode 70）

每次爬 1 或 2 阶，求到第 n 阶有多少种方法。

**状态转移**：`f(n) = f(n-1) + f(n-2)`（第 n 阶可能是从 n-1 爬 1 阶，或从 n-2 爬 2 阶）

```python
def climb_stairs(n):
    if n <= 2:
        return n
    prev, curr = 1, 2
    for _ in range(3, n + 1):
        prev, curr = curr, prev + curr
    return curr
```

时间 O(n)，空间 O(1)——只用两个变量。

#### 3.4.2 最大子数组和（LeetCode 53）

找数组中和最大的连续子数组。

**状态定义**：f(i) = 以 nums[i] 结尾的最大子数组和

**转移方程**：`f(i) = max(nums[i], f(i-1) + nums[i])`

- 若 `f(i-1) >= 0`：接上前面的连续子数组 → `f(i) = nums[i] + f(i-1)`
- 若 `f(i-1) < 0`：中断连续，从头开始 → `f(i) = nums[i]`

```python
def max_sub_array(nums):
    max_result = f = nums[0]
    for i in range(1, len(nums)):
        f = max(nums[i], f + nums[i])  # 连续子数组和 < 0 就中断
        max_result = max(max_result, f)
    return max_result
```

时间 O(n)，空间 O(1)。

#### 3.4.3 0-1 背包

n 个物品各有重量 weight[i] 和价值 value[i]，背包容量 W，每个物品只能选一次，求最大总价值。

**二维 DP**：`dp[i][j]` = 前 i 个物品、容量 j 时的最大价值

```
dp[i][j] = max(dp[i-1][j], value[i] + dp[i-1][j-weight[i]])  if weight[i] <= j
dp[i][j] = dp[i-1][j]  if weight[i] > j
```

**一维优化**：只保留上一轮结果，**从后往前遍历**（防止本轮覆盖上轮）：

```python
def knapsack_01(weights, values, W):
    dp = [0] * (W + 1)
    for i in range(len(weights)):
        for j in range(W, weights[i] - 1, -1):  # 从后往前！
            dp[j] = max(dp[j], values[i] + dp[j - weights[i]])
    return dp[W]
```

时间 O(n×W)，空间 O(W)。

#### 3.4.4 完全背包

与 0-1 背包的区别：每个物品可选**无限次**。

**状态转移唯一区别**：`dp[i][j] = max(dp[i-1][j], value[i] + dp[i][j-weight[i]])`（注意是 `dp[i]` 不是 `dp[i-1]`，因为可以重复选当前物品）

**一维优化**：**从前往后遍历**（允许使用本轮已更新的状态）：

```python
def knapsack_complete(weights, values, W):
    dp = [0] * (W + 1)
    for i in range(len(weights)):
        for j in range(weights[i], W + 1):  # 从前往后！
            dp[j] = max(dp[j], values[i] + dp[j - weights[i]])
    return dp[W]
```

> [!warning] 0-1 vs 完全背包的核心区别
> 一维优化时，**0-1 背包从后往前**（防止覆盖上轮状态），**完全背包从前往后**（允许使用本轮状态 = 重复选）。记忆口诀：**0-1 反向，完全正向**。

### 3.5 回溯算法

**核心思想**：尝试所有可能解，走不通就回退。本质是**DFS 搜索树**——每条从根到叶的路径是一个候选解。

**基本步骤**：
1. **选择**：在每个决策点选一个候选解
2. **探索**：递归地继续下一个决策点
3. **验证**：检查当前路径是否满足条件
4. **回溯**：不满足则撤销上一步，尝试其他选择

**模板**：

```python
def backtrack(路径, 选择列表):
    if 满足终止条件:
        结果.append(路径[:])
        return
    for 选择 in 选择列表:
        做选择
        backtrack(路径, 选择列表)
        撤销选择  # 核心：回溯
```

#### 3.5.1 全排列（LeetCode 46）

不含重复数字的数组，返回所有排列。

**思路**：每层固定一个位置，依次尝试所有未使用的元素。所有位置固定时记录结果。

```python
def permute(nums):
    result = []
    def backtrack(start):
        if start == len(nums):
            result.append(nums[:])
            return
        for i in range(start, len(nums)):
            nums[start], nums[i] = nums[i], nums[start]  # 选择
            backtrack(start + 1)                            # 探索
            nums[start], nums[i] = nums[i], nums[start]  # 回溯
    backtrack(0)
    return result
```

#### 3.5.2 N 皇后（LeetCode 51）

n×n 棋盘放 n 个皇后，互不攻击（不同行、不同列、不同对角线）。

**优化**：用 `set` 记录冲突，O(1) 判断合法性：
- 列冲突：`col` set
- 主对角线（左上→右下）：`row - col` 相同 → `diag1` set
- 副对角线（右上→左下）：`row + col` 相同 → `diag2` set

```python
def n_queens(n):
    result = []
    cols, diag1, diag2 = set(), set(), set()
    board = [["." for _ in range(n)] for _ in range(n)]

    def backtrack(row):
        if row == n:
            result.append(["".join(r) for r in board])
            return
        for col in range(n):
            if col in cols or (row - col) in diag1 or (row + col) in diag2:
                continue
            board[row][col] = "Q"
            cols.add(col); diag1.add(row - col); diag2.add(row + col)
            backtrack(row + 1)
            board[row][col] = "."
            cols.remove(col); diag1.remove(row - col); diag2.remove(row + col)

    backtrack(0)
    return result
```

> **直觉**：回溯的关键是搞清楚"选择列表是什么"和"什么时候终止"。N 皇后每行是一个决策层，列是选择列表，对角线冲突是剪枝条件。

### 3.6 贪心算法

**核心思想**：每步选当前最优，不回溯。

**适用条件**：
1. **贪心选择性质**：局部最优能导出全局最优
2. **最优子结构**：最优解包含子问题最优解

> [!warning] 贪心的陷阱
> 贪心**不保证全局最优**，只有在满足贪心选择性质时才正确。面试时如果追问"为什么贪心是对的"，你需要证明或说明问题性质。

#### 3.6.1 分发糖果（LeetCode 135）

n 个孩子站一排，评分高的必须比相邻的多分糖果，每人至少 1 个，求最少糖果数。

**思路——左右两次扫描**：
1. 从左到右：右边评分比左边高 → 右边糖果 = 左边 + 1
2. 从右到左：左边评分比右边高 → 左边糖果 = max(当前值, 右边 + 1)

```python
def candy(ratings):
    n = len(ratings)
    candies = [1] * n
    # 从左到右
    for i in range(1, n):
        if ratings[i] > ratings[i - 1]:
            candies[i] = candies[i - 1] + 1
    # 从右到左（取 max 保证两个方向都满足）
    for i in range(n - 2, -1, -1):
        if ratings[i] > ratings[i + 1]:
            candies[i] = max(candies[i], candies[i + 1] + 1)
    return sum(candies)
```

> **直觉**：为什么要两次扫描？一次扫描只能保证一个方向的约束。从左到右保证"右边比左边高则右边多"，从右到左保证"左边比右边高则左边多"，取 max 两个方向都满足。

#### 3.6.2 最大交换（LeetCode 670）

至多交换一次数字中的任意两位，返回能得到的最大值。

**思路——从右向左维护最大值索引**：
- 当前值 > 最大值 → 更新最大值索引
- 当前值 < 最大值 → 交换，记录结果，交换回来（恢复原样继续遍历）

```python
def maximum_swap(num):
    result = num
    digits = list(str(num))
    max_idx = -1  # 最大值索引（从右往左看）
    for i in range(len(digits) - 2, -1, -1):
        if digits[i] > digits[max_idx]:
            max_idx = i
        else:
            digits[i], digits[max_idx] = digits[max_idx], digits[i]
            result = max(result, int("".join(digits)))
            digits[i], digits[max_idx] = digits[max_idx], digits[i]
    return result
```

## 四、面试高频

### Q1：数组 vs 链表怎么选？

> 频繁查找 → 数组（O(1) 随机访问）。频繁插入删除 → 链表（O(1) 改指针）。Python 的 list 底层是变长数组，大多数场景直接用 list 即可。

### Q2：哈希表为什么是 O(1)？

> 哈希函数直接算出索引位置，不需要遍历。冲突时需要遍历链表，但好的哈希函数 + 合适的负载因子（0.75）保证冲突很少，均摊 O(1)。

### Q3：八大排序怎么选？

> - 数据量小（n < 50）→ 插入排序（常数小、稳定、接近有序时 O(n)）
> - 通用场景 → 快速排序（平均最快、原地、缓存友好）
> - 需要稳定 → 归并排序
> - 需要保证最坏 O(nlogn) → 堆排序
> - top-K → 堆排序/快速选择
> - 值域小的整数 → 计数排序

### Q4：快速排序为什么快？

> 虽然最坏 O(n²)，但随机 pivot 几乎不触发。平均 O(nlogn) 且常数因子最小（内循环短、缓存友好、原地排序）。实际测试中通常快于归并和堆排。

### Q5：二分查找模板和边界处理？

> 模板：`left=0, right=len-1, while left<=right, mid=left+(right-left)//2`。关键看搜索区间定义（开/闭）和收缩方向（找左边界还是右边界）。

### Q6：DP 解题四步法？

> 1. 定义状态 dp[i] 含义；2. 写转移方程；3. 初始化；4. 确定遍历顺序。最难的是第 1 步——状态定义。

### Q7：0-1 背包 vs 完全背包？

> 一维优化时，**0-1 背包内循环从后往前**（防覆盖），**完全背包从前往后**（允许重复选）。口诀：**0-1 反向，完全正向**。

### Q8：BFS vs DFS 怎么选？

> 求最短路径/最少步骤 → BFS（队列，逐层）。枚举所有解/搜索深度确定 → DFS（栈/递归）。解在浅层 → BFS 更快找到。内存有限 → DFS 更省空间。

### Q9：分治 vs DP vs 回溯 vs 贪心？

> - 分治：子问题**独立**，求解后合并（归并排序）
> - DP：子问题**重叠**，保存中间结果（背包、爬楼梯）
> - 回溯：搜索**所有可能解**，走不通回退（全排列、N 皇后）
> - 贪心：每步**局部最优**，不回溯（分发糖果）
> - 不确定时先想 DP（保证正确），再看能否贪心优化

### Q10：如何分析时间复杂度？

> - 单层循环 → O(n)
> - 双层嵌套 → O(n²)
> - 每轮减半 → O(logn)
> - 递归 T(n) = 2T(n/2) + O(n) → O(nlogn)（主定理）
> - 递归 T(n) = T(n-1) + O(n) → O(n²)

## 五、背诵版总结

数据结构解决"如何组织数据"——数组连续存储 O(1) 访问，链表分散存储 O(1) 插删，哈希表通过哈希函数实现 O(1) 均摊查找，栈和队列是受限的线性结构分别用于 DFS 和 BFS，树和图是非线性结构用于层次和网络关系。算法解决"如何操作数据"——八大排序中快排是通用首选（平均 O(nlogn)、常数最小），归并是稳定排序首选；查找用二分（O(logn)）或哈希（O(1)）；DP 解决重叠子问题（四步法：定义状态→转移方程→初始化→遍历顺序），回溯搜索所有可能解，贪心每步局部最优。面试高频：快排手写、二分变体、DP 背包、BFS/DFS 模板、链表操作、哈希表原理。
