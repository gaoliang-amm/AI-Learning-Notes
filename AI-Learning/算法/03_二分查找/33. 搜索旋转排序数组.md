# 33. 搜索旋转排序数组

## 🎯 人话翻译

> 一个原本升序的数组被拦腰砍了一刀，前后调了个位置（旋转了一次）。里面没有重复元素。给你一个目标值，找到它的下标，找不到就返回 -1。

## 💡 核心思想

> 旋转数组的关键性质：从中间切开后，**至少有一半是有序的**。先判断哪一半有序，再判断 `target` 是否落在有序那一半里，从而每次都能将搜索范围缩小一半。本质还是在用[[二分查找]]，只是多了一层"判断有序区间"的逻辑。

用到：[[二分查找]]、[[边界搜索]]、[[旋转数组]]

## 📝 解题步骤

1. **初始化** — `left = 0`, `right = len(nums) - 1`
2. **二分查找** — 循环条件 `left <= right`，计算 `mid = (left + right) // 2`
3. **命中检查** — 如果 `nums[mid] == target`，直接返回 `mid`
4. **判断有序区间** — 如果 `nums[left] <= nums[mid]`，说明左半有序；否则右半有序
5. **缩小范围** — 如果左半有序且 `target` 在 `[nums[left], nums[mid])` 之间，去左边；否则去右边。右半有序同理

## 🔢 手动走一遍

输入: `nums = [4, 5, 6, 7, 0, 1, 2]`, `target = 0`

| 步骤 | left | right | mid | nums[mid] | 有序半 | 判断逻辑 | 操作 |
|------|------|-------|------|-----------|--------|----------|------|
| 初始 | 0 | 6 | - | - | - | - | - |
| 1 | 0 | 6 | 3 | 7 | 左半 [4,5,6,7] 有序 | target=0 不在 [4,7) 中 | left = 4 |
| 2 | 4 | 6 | 5 | 1 | 左半 [0,1] 有序 | target=0 在 [0,1) 中 | right = 4 |
| 3 | 4 | 4 | 4 | 0 | - | nums[4]=0 == target | 返回 4 |

最终返回 `4`，`nums[4] = 0`。

## 💻 解法

> 标准旋转数组二分查找：先判断哪半边有序，再看 target 是否在有序半中，每次排除一半。

```python
from typing import List


class Solution:
    def search(self, nums: List[int], target: int) -> int:
        # 时间复杂度 O(log n)，空间复杂度 O(1)

        left, right = 0, len(nums) - 1

        while left <= right:
            mid = (left + right) // 2

            if nums[mid] == target:
                return mid

            # 判断左半部分是否有序
            if nums[left] <= nums[mid]:
                # 左半有序，检查 target 是否在左半区间内
                if nums[left] <= target < nums[mid]:
                    right = mid - 1
                else:
                    left = mid + 1
            else:
                # 右半有序，检查 target 是否在右半区间内
                if nums[mid] < target <= nums[right]:
                    left = mid + 1
                else:
                    right = mid - 1

        return -1
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(log n) | 每次排除一半，标准的二分查找复杂度 |
| 空间 | O(1) | 只使用常数个变量 |

## ⚠ 易错点

- [ ] 判断左半有序时用 `nums[left] <= nums[mid]`（带等号），因为当 `left == mid` 时只有两个元素，此时左半就是单个元素、天然有序
- [ ] `target` 区间判断的边界是严格/非严格：`nums[left] <= target < nums[mid]` 中，target 可以等于 left 但不能等于 mid（因为 mid 已经检查过了）
- [ ] 注意区分"判断哪半有序"用的 `nums[left] <= nums[mid]` 和"检查 target 是否在有序半"用的 `target < nums[mid]`，这是两个不同的场景
- [ ] 在有重复元素的版本（LeetCode 81）中，当 `nums[left] == nums[mid] == nums[right]` 时无法判断哪半有序，需要额外处理

## 🎤 面试怎么说

```text
旋转数组的二分查找核心就是一句话：从中间切开，至少有一半是有序的。

我们先判断哪半有序——如果 nums[left] <= nums[mid]，左半有序；
否则右半有序。然后判断 target 是否在有序那一半的区间内。
如果在，就去那半继续找；如果不在，就去另一半找。

这样每次都能排除一半元素，O(log n) 解决。

如果面试官问重复元素怎么办（81 题），需要处理
nums[left] == nums[mid] == nums[right] 的场景，
此时两边各缩一格就行。
```
