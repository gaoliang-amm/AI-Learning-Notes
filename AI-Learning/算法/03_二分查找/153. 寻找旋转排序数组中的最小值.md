# 153. 寻找旋转排序数组中的最小值

## 🎯 人话翻译

> 一个升序无重复的数组被旋转了一次（比如 [1,2,3,4,5] 变成 [3,4,5,1,2]），找到这个数组的最小值。要求在 O(log n) 时间内完成。

## 💡 核心思想

> 旋转点的特征：最小值一定在"断层"的位置。比较 `nums[mid]` 和 `nums[right]` 可以判断 mid 在哪一半：
> - 如果 `nums[mid] < nums[right]`，说明从 mid 到 right 是递增的，断层在左边（包含 mid），所以 `right = mid`
> - 如果 `nums[mid] > nums[right]`，说明断层在右边，`left = mid + 1`
>
> 不断缩小区间，最终 `left == right` 就是最小值的位置。这是[[二分查找]]找"旋转点"的经典范式。

用到：[[二分查找]]、[[旋转数组]]、[[边界搜索]]

## 📝 解题步骤

1. **初始化** — `left = 0`, `right = len(nums) - 1`
2. **二分搜索** — 循环条件 `left < right`（不是 `<=`，因为我们要缩到只剩一个元素）
3. **判断 mid 位置** — 比较 `nums[mid]` 和 `nums[right]`：如果 `nums[mid] < nums[right]`，mid 在右半有序段，最小值在左边，`right = mid`；否则 mid 在左半，最小值在右边，`left = mid + 1`
4. **返回最小值** — 循环退出时 `left == right`，`nums[left]` 就是最小值

## 🔢 手动走一遍

输入: `nums = [4, 5, 6, 7, 0, 1, 2]`

| 步骤 | left | right | mid | nums[mid] | nums[right] | 比较 | 操作 |
|------|------|-------|------|-----------|-------------|------|------|
| 初始 | 0 | 6 | - | - | - | - | - |
| 1 | 0 | 6 | 3 | 7 | 2 | 7 > 2（断层在右） | left = 4 |
| 2 | 4 | 6 | 5 | 1 | 2 | 1 < 2（有序，断层在左含 mid） | right = 5 |
| 3 | 4 | 5 | 4 | 0 | 1 | 0 < 1（有序，断层在左含 mid） | right = 4 |
| 退出 | 4 | 4 | - | - | - | left == right | 返回 nums[4] = 0 |

最终返回 `0`。

## 💻 解法

> 比较 `nums[mid]` 和 `nums[right]` 判断断层位置，用 `left < right` 循环逐渐缩小区间到只剩最小值。

```python
from typing import List


class Solution:
    def findMin(self, nums: List[int]) -> int:
        # 时间复杂度 O(log n)，空间复杂度 O(1)

        left, right = 0, len(nums) - 1

        while left < right:
            mid = (left + right) // 2

            # 如果中间值小于右边界，说明右半部分有序，最小值在左半（包括 mid）
            if nums[mid] < nums[right]:
                right = mid
            else:
                # 否则中间值大于右边界，说明最小值在右半（排除 mid）
                left = mid + 1

        return nums[left]
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(log n) | 每次缩小一半搜索区间 |
| 空间 | O(1) | 只使用常数个变量 |

## ⚠ 易错点

- [ ] 比较对象选 `nums[right]` 还是 `nums[left]`？**选 `right`**。如果数组没旋转（`nums[left] < nums[right]`），用 `left` 比较会出错，用 `right` 比较则正确缩小区间
- [ ] 循环条件 `left < right` 而不是 `left <= right`：因为我们在缩到 `left == right` 时即为答案，用 `<=` 会导致死循环（`right = mid` 不缩小规模）
- [ ] `right = mid` 而不是 `right = mid - 1`：因为当 `nums[mid] < nums[right]` 时，`mid` 自己可能就是最小值
- [ ] 特殊情况：数组没有旋转（完全有序）时，`left` 一直不动，`right` 不断往左缩，最终返回 `nums[0]`，正确
