# 162. 寻找峰值

## 🎯 人话翻译

> 在一个相邻元素互不相等的数组中，找到一个"峰顶"——比左右邻居都大的那个元素，返回它的下标。数组边界之外的元素视为负无穷。

## 💡 核心思想

> 利用"上坡必有峰"的直觉：如果 `nums[mid] < nums[mid+1]`，说明右侧是上坡，峰值一定在右边；否则峰值在左边（包括 mid 自己）。每次比较都能排除一半，本质是[[二分查找]]在"非有序"数组上的应用。

用到：[[二分查找]]、[[边界搜索]]

## 📝 解题步骤

1. **初始化** — `left = 0`, `right = len(nums) - 1`
2. **二分收缩** — 循环条件 `left < right`（注意没有等号，因为我们要比较 `mid` 和 `mid+1`）
3. **判断上坡方向** — 如果 `nums[mid] < nums[mid + 1]`，则右侧更高，峰值在右边，`left = mid + 1`；否则左侧有峰值（mid 自己可能就是峰），`right = mid`
4. **返回峰值** — 当 `left == right` 时退出，`left` 就是峰值下标

## 🔢 手动走一遍

输入: `nums = [1, 2, 3, 1]`

| 步骤 | left | right | mid | nums[mid] | nums[mid+1] | 比较 | 操作 |
|------|------|-------|------|-----------|-------------|------|------|
| 初始 | 0 | 3 | - | - | - | - | - |
| 1 | 0 | 3 | 1 | 2 | 3 | 2 < 3（上坡） | left = 2 |
| 2 | 2 | 3 | 2 | 3 | 1 | 3 > 1（下坡） | right = 2 |
| 退出 | 2 | 2 | - | - | - | left == right | 返回 2 |

最终返回 `2`，`nums[2] = 3`，确实是峰值（`3 > nums[1]=2` 且 `3 > nums[3]=1`）。

## 💻 解法

> 利用"上坡必有峰"的二分查找，循环条件 `left < right`，每次比较 mid 和 mid+1 决定方向。

```python
from typing import List


class Solution:
    def findPeakElement(self, nums: List[int]) -> int:
        # 时间复杂度 O(log n)，空间复杂度 O(1)

        left, right = 0, len(nums) - 1

        while left < right:
            mid = (left + right) // 2

            # 如果中间值小于右边相邻值，峰值在右侧
            if nums[mid] < nums[mid + 1]:
                left = mid + 1
            else:
                # 否则峰值在左侧（包括 mid）
                right = mid

        return left
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(log n) | 每次缩小一半搜索区间 |
| 空间 | O(1) | 只使用常数个变量 |

## ⚠ 易错点

- [ ] 循环条件必须用 `left < right` 不能用 `left <= right`，因为比较的是 `mid` 和 `mid + 1`，`left <= right` 会导致 `mid + 1` 越界
- [ ] `right = mid` 而不是 `right = mid - 1`：因为当 `nums[mid] >= nums[mid + 1]` 时，`mid` 自己可能就是峰值，不能排除
- [ ] 返回 `left` 或 `right` 都可以，因为退出时两者相等；但注意不能返回 `nums[left]`，题目要的是下标
- [ ] 边界外的"负无穷"条件天然满足：如果一直往左走，最终在数组第一个元素停下，它比左边更低（负无穷），`left` 就是答案
