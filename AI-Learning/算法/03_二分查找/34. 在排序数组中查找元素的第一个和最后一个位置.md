# 34. 在排序数组中查找元素的第一个和最后一个位置

## 🎯 人话翻译

> 在一个升序数组里找到目标值出现的第一个位置和最后一个位置。如果目标值不存在，返回 `[-1, -1]`。

## 💡 核心思想

> 做两次[[二分查找]]：一次找**左边界**（第一个 >= target 的位置），一次找**右边界**（最后一个 <= target 的位置）。关键在于修改收缩条件：
> - 找左边界时，当 `nums[mid] >= target` 时继续往左探，`right = mid - 1`
> - 找右边界时，当 `nums[mid] <= target` 时继续往右探，`left = mid + 1`

用到：[[二分查找]]、[[边界搜索]]

## 📝 解题步骤

1. **找左边界** — 标准二分，当 `nums[mid] >= target` 时 `right = mid - 1`（往左缩），最终 `left` 停在第一个 `>= target` 的位置
2. **找右边界** — 标准二分，当 `nums[mid] <= target` 时 `left = mid + 1`（往右扩），最终 `right` 停在最后一个 `<= target` 的位置
3. **验证结果** — 检查 `left <= right`（说明在数组中找到了）且 `nums[left] == target`，满足则返回 `[left, right]`，否则返回 `[-1, -1]`

## 🔢 手动走一遍

输入: `nums = [5, 7, 7, 8, 8, 10]`, `target = 8`

**找左边界（第一个 >= 8）：**

| 步骤 | left | right | mid | nums[mid] | nums[mid] >= 8? | 操作 |
|------|------|-------|------|-----------|-----------------|------|
| 初始 | 0 | 5 | - | - | - | - |
| 1 | 0 | 5 | 2 | 7 | 否 | left = 3 |
| 2 | 3 | 5 | 4 | 8 | 是 | right = 3 |
| 3 | 3 | 3 | 3 | 8 | 是 | right = 2 |
| 退出 | 3 | 2 | - | - | left > right | 返回 left=3 |

**找右边界（最后一个 <= 8）：**

| 步骤 | left | right | mid | nums[mid] | nums[mid] <= 8? | 操作 |
|------|------|-------|------|-----------|-----------------|------|
| 初始 | 0 | 5 | - | - | - | - |
| 1 | 0 | 5 | 2 | 7 | 是 | left = 3 |
| 2 | 3 | 5 | 4 | 8 | 是 | left = 5 |
| 3 | 5 | 5 | 5 | 10 | 否 | right = 4 |
| 退出 | 5 | 4 | - | - | left > right | 返回 right=4 |

最终返回 `[3, 4]`。

## 💻 解法

> 两次二分查找：找左边界时"遇到相等继续往左"，找右边界时"遇到相等继续往右"。

```python
from typing import List


class Solution:
    def searchRange(self, nums: List[int], target: int) -> List[int]:
        # 时间复杂度 O(log n)，空间复杂度 O(1)

        def find_left(nums: List[int], target: int) -> int:
            """返回第一个 >= target 的索引"""
            left, right = 0, len(nums) - 1
            while left <= right:
                mid = (left + right) // 2
                if nums[mid] >= target:
                    right = mid - 1
                else:
                    left = mid + 1
            return left

        def find_right(nums: List[int], target: int) -> int:
            """返回最后一个 <= target 的索引"""
            left, right = 0, len(nums) - 1
            while left <= right:
                mid = (left + right) // 2
                if nums[mid] <= target:
                    left = mid + 1
                else:
                    right = mid - 1
            return right

        # 边界情况：空数组
        if not nums:
            return [-1, -1]

        left = find_left(nums, target)
        right = find_right(nums, target)

        # 检查是否找到目标值
        if left <= right and nums[left] == target:
            return [left, right]
        return [-1, -1]
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(log n) | 两次二分查找，各 O(log n) |
| 空间 | O(1) | 只使用常数个变量 |

## ⚠ 易错点

- [ ] 左边界函数 `find_left` 返回的是 `left`，右边界函数 `find_right` 返回的是 `right`，两个返回值恰好互补
- [ ] 最终验证必须检查 `left <= right` 和 `nums[left] == target`，因为 target 可能不在数组中（这时 `left > right` 或 `nums[left] != target`）
- [ ] 找左边界时 `nums[mid] >= target` 带了等号：遇到相等也继续往左找，这就是边界搜索的核心
- [ ] 空数组单独处理，否则 `find_left` 内的 `len(nums) - 1` 会得到 -1 导致行为错误
- [ ] 找左右边界不能用同一个二分写完（比如先找到某个 target 再左右扩散），因为最坏情况下整个数组全是 target，扩散退化为 O(n)

## 🎤 面试怎么说

```text
这道题是二分查找找边界的经典题。核心是用两次二分查找。

找左边界时，当 nums[mid] >= target，我们继续往左找（right = mid - 1），
最终 left 停在第一个 >= target 的位置。

找右边界同理，当 nums[mid] <= target，继续往右找（left = mid + 1），
最终 right 停在最后一个 <= target 的位置。

最后验证 left <= right 且 nums[left] == target 确保真的找到了。

时间复杂度 O(log n)，空间 O(1)。这个模板可以直接用于找"大于等于某值的第一个位置"这类问题。
```
