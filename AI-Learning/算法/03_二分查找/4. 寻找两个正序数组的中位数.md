# 4. 寻找两个正序数组的中位数

## 🎯 人话翻译

> 给两个排好序的数组，把它们"合并"成一个有序数组，找这个虚拟合并数组的中位数。要求时间复杂度 O(log(m+n))，不能真的合并（那样是 O(m+n)）。

## 💡 核心思想

> 把问题转化为：在两个有序数组中找到一条**分割线**，使得：
> - 分割线左侧的总元素个数 = 右侧（或左侧多 1，对应奇数的情况）
> - 左侧的最大值 <= 右侧的最小值
>
> 然后对较短的数组做[[二分查找]]来确定分割线位置。因为一旦 `nums1` 的分割线位置定了，`nums2` 的分割线位置就自动确定了（左侧总数固定）。这就是"分割线法"的[[二分查找]]。

用到：[[二分查找]]、[[分割线法]]、[[中位数]]

## 📝 解题步骤

1. **保证短数组在前** — 让 `nums1` 是较短的数组，这样在它上面二分效率更高
2. **确定左侧总元素数** — `total_left = (m + n + 1) // 2`，保证奇数时中位数在左侧
3. **在 nums1 上二分** — 对 `nums1` 从 `0` 到 `m` 二分，确定 `partition1`，则 `partition2 = total_left - partition1`
4. **获取分割线四角的元素** — `left_max1`（nums1 左侧最大）、`right_min1`（nums1 右侧最小）、`left_max2`、`right_min2`，用正负无穷处理边界
5. **判断分割线是否正确** — 如果 `left_max1 <= right_min2` 且 `left_max2 <= right_min1`，则找到正确分割线，根据奇偶性计算中位数；否则调整二分方向

## 🔢 手动走一遍

输入: `nums1 = [1, 3]`, `nums2 = [2]`, m=2, n=1

左侧总数 `total_left = (2+1+1)//2 = 2`

二分 `nums1` 的 partition（0 到 2）：

| 步骤 | left | right | p1 | p2 | left_max1 | right_min1 | left_max2 | right_min2 | 条件满足? | 操作 |
|------|------|-------|----|----|-----------|------------|-----------|------------|-----------|------|
| 初始 | 0 | 2 | - | - | - | - | - | - | - | - |
| 1 | 0 | 2 | 1 | 1 | nums1[0]=1 | nums1[1]=3 | nums2[0]=2 | nums2[1]=+inf | 1<=+inf 且 2<=3 | 找到！|

找到分割线：`left_max1=1, left_max2=2`，总长度 3 是奇数，中位数 = `max(1, 2) = 2`。

## 💻 解法

> 分割线法：在较短数组上二分确定分割线位置，检查左右四角的大小关系是否满足条件。

```python
from typing import List


class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        # 时间复杂度 O(log(min(m, n)))，空间复杂度 O(1)

        # 确保 nums1 是较短的数组，减少二分查找的时间复杂度
        if len(nums1) > len(nums2):
            nums1, nums2 = nums2, nums1

        m, n = len(nums1), len(nums2)
        total_left = (m + n + 1) // 2  # 左侧元素的总个数

        # 在 nums1 上进行二分查找，确定分割线位置
        left, right = 0, m

        while left <= right:
            # partition1 表示 nums1 左侧的元素个数（0 到 m）
            partition1 = (left + right) // 2
            # partition2 表示 nums2 左侧的元素个数
            partition2 = total_left - partition1

            # 获取分割线两侧的值（使用负无穷和正无穷处理边界）
            left_max1 = nums1[partition1 - 1] if partition1 > 0 else float('-inf')
            right_min1 = nums1[partition1] if partition1 < m else float('inf')
            left_max2 = nums2[partition2 - 1] if partition2 > 0 else float('-inf')
            right_min2 = nums2[partition2] if partition2 < n else float('inf')

            # 检查是否满足条件：左侧最大值 <= 右侧最小值
            if left_max1 <= right_min2 and left_max2 <= right_min1:
                # 找到了正确的分割线
                if (m + n) % 2 == 1:
                    # 奇数个元素，中位数是左侧的最大值
                    return max(left_max1, left_max2)
                else:
                    # 偶数个元素，中位数是（左侧最大值 + 右侧最小值）/ 2
                    return (max(left_max1, left_max2) + min(right_min1, right_min2)) / 2
            elif left_max1 > right_min2:
                # 分割线太靠右，需要左移
                right = partition1 - 1
            else:
                # 分割线太靠左，需要右移
                left = partition1 + 1

        # 理论上不会执行到这里
        return 0.0
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(log(min(m, n))) | 在较短的数组上二分，每次排除一半 |
| 空间 | O(1) | 只使用常数个变量 |

## ⚠ 易错点

- [ ] 必须确保 nums1 是较短数组：`partition2` 可能变成负数，虽然可以用边界条件兜底，但保证短数组在前让二分范围更小更安全
- [ ] 边界处理用 `float('inf')` 和 `float('-inf')`：当 partition 在数组的最左或最右时，不存在的那侧用正/负无穷代替，让比较逻辑统一
- [ ] `total_left = (m + n + 1) // 2` 的 `+1` 是关键：保证了奇数个元素时左侧比右侧多一个，中位数直接取左侧最大值
- [ ] 二分搜索的区间是 `[0, m]` 而不是 `[0, m-1]`：因为分割线可以放在数组的最右端（全部元素在左侧）
- [ ] 调整二分方向时 `right = partition1 - 1` 和 `left = partition1 + 1`，别混淆方向
