# 167. 两数之和 II - 输入有序数组

## 🎯 人话翻译

> 给一个**已经排好序**的数组和一个目标值，找出两个数，它们的和等于目标值，返回这两个数的下标（从 1 开始计数）。题目保证有且只有一个解。

## 💡 核心思想

> 利用数组的有序性，使用对撞[[双指针]]从两端逼近。如果当前和太小，就把左指针右移（增加和）；如果当前和太大，就把右指针左移（减小和）。相比哈希表的 O(n) 空间方案，这种做法的空间是 O(1)。

用到：[[双指针]]

## 📝 解题步骤

1. **初始化双指针** — `left = 0`, `right = len(numbers) - 1`
2. **计算当前和** — `current_sum = numbers[left] + numbers[right]`
3. **三种情况** — 等于 target：返回 `[left+1, right+1]`（下标从 1 开始）；小于 target：`left += 1`（需要更大的和）；大于 target：`right -= 1`（需要更小的和）
4. **循环至相遇** — 题目保证有解，一定会返回

## 🔢 手动走一遍

输入: `numbers = [2, 7, 11, 15]`, `target = 9`

| 步骤 | left | right | numbers[left] | numbers[right] | 和 | 比较 | 操作 |
|------|------|-------|--------------|---------------|-----|------|------|
| 初始 | 0 | 3 | 2 | 15 | 17 | 17 > 9 | right = 2 |
| 2 | 0 | 2 | 2 | 11 | 13 | 13 > 9 | right = 1 |
| 3 | 0 | 1 | 2 | 7 | 9 | 9 == 9 | 返回 [1, 2] |

最终返回 `[1, 2]`。

## 💻 解法

> 对撞双指针：利用数组有序性，和太小移左，和太大移右，每次 O(1) 调整。

```python
from typing import List


class Solution:
    def twoSum(self, numbers: List[int], target: int) -> List[int]:
        """
        在有序数组中找出两数之和等于 target 的下标（从1开始）。
        使用双指针法，利用数组有序的特性。

        时间复杂度: O(n)
        空间复杂度: O(1)
        """
        left, right = 0, len(numbers) - 1

        while left < right:
            current_sum = numbers[left] + numbers[right]
            if current_sum == target:
                # 返回下标+1（因为题目要求从1开始计数）
                return [left + 1, right + 1]
            elif current_sum < target:
                left += 1
            else:
                right -= 1

        # 根据题意，一定存在解，这里不会执行到
        return [-1, -1]
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 每个元素最多被访问一次 |
| 空间 | O(1) | 只使用两个指针 |

## ⚠ 易错点

- [ ] 返回的下标要 +1（题目要求从 1 开始计数，不是 0）
- [ ] 如果和等于 target，直接返回不需要再遍历，题目保证只有一个解
- [ ] `while left < right` 不能用 `<=`，因为需要两个不同的数（不能同一个元素用两次）
- [ ] 这道题不能用哈希表（虽然哈希表也能过），因为数组已经有序，双指针是更优解（空间 O(1) vs O(n)）
