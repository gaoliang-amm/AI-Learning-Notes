# 15. 三数之和

## 🎯 人话翻译

> 给一个可能包含重复元素的数组，找出所有不同的三元组 `(a, b, c)`，使得 `a + b + c = 0`。要求三元组不能重复（不是下标不重复，是值不重复）。

## 💡 核心思想

> 排序 + 固定一个数 + 对撞[[双指针]]。先把数组排序，然后固定第一个数 `nums[i]`，在它右边的子数组上用双指针找两数之和等于 `-nums[i]`。去重是关键：固定的数和双指针移动时都要跳过重复值。

用到：[[双指针]]、[[排序]]

## 📝 解题步骤

1. **排序** — 先对数组排序（O(n log n)），便于去重和双指针
2. **固定第一个数** — 枚举 `i` 从 0 到 `n-3`，`target = -nums[i]`
3. **剪枝优化** — 如果 `nums[i] > 0` 直接结束（因为后面都是正数，三数和不可能为 0）
4. **跳过重复的 i** — 如果 `nums[i] == nums[i-1]`，跳过，避免重复三元组
5. **双指针找两数和** — `left = i + 1`, `right = n - 1`，计算 `nums[left] + nums[right]`，等于 target 则记录并跳过重复的 left 和 right，小于则移左，大于则移右

## 🔢 手动走一遍

输入: `nums = [-1, 0, 1, 2, -1, -4]`，排序后 = `[-4, -1, -1, 0, 1, 2]`

| i | nums[i] | 去重? | target | left | right | nums[L]+nums[R] | 操作 |

| 0 | -4 | i=0不跳过 | 4 | 1(-1) | 5(2) | -1+2=1 < 4 | left=2 |
| 0 | -4 | - | 4 | 2(-1) | 5(2) | -1+2=1 < 4 | left=3 |
| 0 | -4 | - | 4 | 3(0) | 5(2) | 0+2=2 < 4 | left=4 |
| 0 | -4 | - | 4 | 4(1) | 5(2) | 1+2=3 < 4 | left=5, 结束 |
| 1 | -1 | 跳过(i=1, nums[1]==nums[0]?) | - | - | - | - | - |

i=1 时 nums[1]=-1 不等于 nums[0]=-4，不跳过。

| i | nums[i] | 去重? | target | left | right | nums[L]+nums[R] | 操作 |
|---|---------|-------|--------|------|-------|-----------------|------|
| 1 | -1 | 否 | 1 | 2(-1) | 5(2) | -1+2=1 | 命中! [-1,-1,2], L=3, R=4 |
| 1 | -1 | - | 1 | 3(0) | 4(1) | 0+1=1 | 命中! [-1,0,1], L=4, R=3 退出 |

i=2 时 nums[2]=-1 == nums[1]=-1，跳过。

| i | nums[i] | 去重? | target | left | right | nums[L]+nums[R] | 操作 |
|---|---------|-------|--------|------|-------|-----------------|------|
| 3 | 0 | 否但 0>0? (nums[i] > 0?) | - | - | - | - | break |

结果：`[[-1, -1, 2], [-1, 0, 1]]`

## 💻 解法

> 排序后固定一个数，剩余用对撞双指针找两数和。三重去重：i 去重、left 去重、right 去重。

```python
from typing import List


class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        """
        找出所有和为 0 且不重复的三元组。
        算法：排序 + 双指针。
        时间复杂度: O(n^2)
        空间复杂度: O(1) 或 O(log n)（排序所需栈空间）
        """
        n = len(nums)
        if n < 3:
            return []

        # 排序
        nums.sort()
        result = []

        for i in range(n - 2):
            # 如果当前元素已经大于0，则后面不可能有三数和为0（因为排序后递增）
            if nums[i] > 0:
                break
            # 跳过重复的第一个数
            if i > 0 and nums[i] == nums[i - 1]:
                continue

            left, right = i + 1, n - 1
            target = -nums[i]  # 寻找两数和为 target

            while left < right:
                current_sum = nums[left] + nums[right]
                if current_sum == target:
                    result.append([nums[i], nums[left], nums[right]])
                    # 跳过重复的 left
                    while left < right and nums[left] == nums[left + 1]:
                        left += 1
                    # 跳过重复的 right
                    while left < right and nums[right] == nums[right - 1]:
                        right -= 1
                    left += 1
                    right -= 1
                elif current_sum < target:
                    left += 1
                else:
                    right -= 1

        return result
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n^2) | 排序 O(n log n)，外层 O(n) 内层双指针 O(n) |
| 空间 | O(1) | 除结果数组外只用常数变量（排序的 O(log n) 栈空间不计） |

## ⚠ 易错点

- [ ] 去重要做三处：固定数去重（`nums[i] == nums[i-1]`）、left 去重、right 去重，缺一不可
- [ ] `nums[i] > 0` 时 `break`（不是 `continue`），因为数组已排序，后面的数更大，不会再找到和为 0 的三元组
- [ ] 找到答案后必须同时移动 left 和 right（`left += 1; right -= 1`），否则会死循环
- [ ] 去重 left 时要注意 `left < right` 守卫条件，防止越界
- [ ] `target = -nums[i]`：这是固定一个数后，需要的两数和，容易算反成 `target = nums[i]`
