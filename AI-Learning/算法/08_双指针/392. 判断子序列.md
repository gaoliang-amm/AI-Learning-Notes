# 392. 判断子序列

## 🎯 人话翻译

> 给两个字符串 s 和 t，判断 s 是不是 t 的"子序列"——即你可以从 t 中删除一些字符（或不删）得到 s，但不能改顺序。比如 "ace" 是 "abcde" 的子序列（删除 b 和 d），但 "aec" 不是。

## 💡 核心思想

> [[双指针]]一个指向 s，一个指向 t。在 t 中按顺序寻找 s 的每个字符。t 的指针始终往前走（相当于扫描原串），s 的指针只在匹配上时才往前走。如果 s 走完了，说明全匹配上了。

用到：[[双指针]]

## 📝 解题步骤

1. **初始化双指针** — `i = 0`（指向 s），`j = 0`（指向 t）
2. **遍历 t** — `while i < len(s) and j < len(t)`，比较 `s[i]` 和 `t[j]`
3. **匹配则移 i** — 如果 `s[i] == t[j]`，说明 s 的当前字符在 t 中找到了，`i += 1`
4. **始终移 j** — 无论是否匹配，t 的指针 `j` 都要前进一步
5. **检查结果** — `i == len(s)` 说明 s 的所有字符都在 t 中按顺序找到了

## 🔢 手动走一遍

输入: `s = "abc"`, `t = "ahbgdc"`

| 步骤 | i | j | s[i] | t[j] | 匹配? | 操作 |
|------|---|---|------|------|-------|------|
| 初始 | 0 | 0 | - | - | - | - |
| 1 | 0 | 0 | 'a' | 'a' | 是 | i=1, j=1 |
| 2 | 1 | 1 | 'b' | 'h' | 否 | j=2 |
| 3 | 1 | 2 | 'b' | 'b' | 是 | i=2, j=3 |
| 4 | 2 | 3 | 'c' | 'g' | 否 | j=4 |
| 5 | 2 | 4 | 'c' | 'd' | 否 | j=5 |
| 6 | 2 | 5 | 'c' | 'c' | 是 | i=3, j=6 |
| 退出 | 3 | 6 | - | - | i == len(s) | 返回 True |

## 💻 解法

> 双指针同步遍历：s 指针只在匹配上时前进，t 指针始终前进。最终 s 指针到末尾即成功。

```python
def isSubsequence(s: str, t: str) -> bool:
    i = j = 0
    len_s, len_t = len(s), len(t)

    # 只要有一个字符串遍历完就退出
    while i < len_s and j < len_t:
        if s[i] == t[j]:
            i += 1  # 匹配成功，移动s指针
        j += 1  # 每次都移动t指针

    # 若s全部匹配完，则返回True
    return i == len_s
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(len(t)) | 最坏情况需要完整遍历 t |
| 空间 | O(1) | 只用两个指针变量 |

## ⚠ 易错点

- [ ] `while` 条件必须是 `i < len_s and j < len_t`，只看 `j < len_t` 的话可能在 s 没匹配完时 t 先走完，需要正确判断
- [ ] 匹配后 `i += 1` 和 `j += 1` 都要执行，不能只移一个：s 匹配上了这一位，两指针都要往前
- [ ] 返回值是 `i == len_s`，不是 `j == len_t`：判断是否把 s 全部匹配完了
- [ ] 子序列（subsequence）和子串（substring）不同：子序列不要求连续，子串必须是连续的，别搞混

```text
这道题用双指针。指针 i 遍历 s，指针 j 遍历 t。j 一直往前走，
只有当 s[i] == t[j] 时 i 才往前走。最后如果 i 走到了 s 的末尾，

```
