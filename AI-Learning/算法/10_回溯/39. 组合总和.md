# 39. 组合总和

## 🎯 人话翻译

> 给一个无重复元素的数组和一个目标值，找出所有加起来等于目标值的组合。每个数字可以重复使用无限次。

## 💡 核心思想

> **回溯法 + 剪枝：**
>
> 组合问题，用 `start` 索引保证不产生重复组合（`[2, 3]` 和 `[3, 2]` 算同一个）。
>
> 与普通组合问题的区别：**每个数字可以重复选**，所以递归时 `start` 不 +1（还是传 `i`，允许再次选同一个数）。
>
> **剪枝策略：** 如果当前数字已经大于剩余目标值，直接跳过（因为数组已排序，后面的数字更大，更不可能）。

用到：[[回溯]]、[[剪枝]]

## 📝 解题步骤

1. **排序** — 将 `candidates` 升序排列，方便剪枝（一旦当前数字 > 剩余目标，后面的全跳过）
2. **定义回溯函数 `backtrack(start, path, remaining)`**：
   - `start`：当前可以选的最小索引（保证组合不重复）
   - `path`：当前已选的数字列表
   - `remaining`：还差多少达到 target
3. **终止条件**：
   - `remaining == 0`：找到一个有效组合，加入结果（添加 `path[:]` 副本）
   - `remaining < 0`：超出目标，无效，直接返回
4. **遍历选择** — 从 `start` 到末尾：
   - `candidates[i] > remaining`：剪枝跳过
   - 选择：`path.append(candidates[i])`
   - 递归：`backtrack(i, path, remaining - candidates[i])`（注意传 `i` 不是 `i+1`，因为可重复选）
   - 回溯：`path.pop()`
5. **返回结果**

## 🔢 手动走一遍

输入: `candidates = [2, 3, 6, 7]`, `target = 7`

排序后：`[2, 3, 6, 7]`

### 决策树

```
                          start=0, remaining=7
                    /        |          |          \
                  选2        选3        选6         选7
                   |          |          |           |
              start=0      start=1     start=2     start=3
              rem=5        rem=4       rem=1       rem=0 ✅ [7]
            /  |  \        /  |  \      剪枝
          2    3   6      3   6   7
          |    |   |      |   |   |
         rem=3 rem=2 rem=-1 rem=1 rem=-2
        / | \   |    ✗     |    ✗
       2  3  6  3        剪枝
       |  |  |  |
      rem  rem rem
       =1  =0  =-3
       |   ✅   ✗
      剪枝 [2,2,3]
```

### 找到的所有组合

| 路径 | 组合 | sum | 说明 |
|------|------|-----|------|
| 2 -> 2 -> 3 | [2, 2, 3] | 7 | rem=0，收集 |
| 7 | [7] | 7 | rem=0，收集 |

`sorted` 后确保先走 2，走到 rem=1 时，3、6、7 都 >1，全部剪枝，效率很高。

## 💻 解法

> 回溯 + 剪枝：每个数字可重复选（递归传 i），排序后遇到大于目标的直接跳过。

```python
from typing import List


class Solution:
    def combinationSum(self, candidates: List[int], target: int) -> List[List[int]]:
        """
        用回溯法找出所有和等于 target 的组合。
        每个数字可重复使用。
        时间复杂度: O(n^(target/min_candidate))，最坏指数级
        空间复杂度: O(target/min_candidate) 递归栈深度
        """
        result = []
        # 排序便于剪枝：一旦 candidates[i] > remaining，后面的更大的数也可以跳过
        candidates.sort()

        def backtrack(start: int, path: List[int], remaining: int):
            """
            回溯函数

            参数：
                start: 当前可以选择的起始索引（保证组合去重）
                path: 当前已选的数字列表
                remaining: 距离 target 还差多少
            """
            # 终止条件1: 刚好找到目标和
            if remaining == 0:
                result.append(path[:])  # 添加副本
                return

            # 终止条件2: 超出目标和（由于剪枝，这步理论上不会触发）
            if remaining < 0:
                return

            # 从 start 开始尝试每个数字
            for i in range(start, len(candidates)):
                num = candidates[i]

                # 剪枝：如果当前数字已经大于剩余目标，后面的都跳过
                if num > remaining:
                    break

                # 选择当前数字
                path.append(num)
                # 递归：因为数字可以重复使用，所以 next_start 仍然是 i（不是 i+1）
                backtrack(i, path, remaining - num)
                # 回溯：撤销选择
                path.pop()

        backtrack(0, [], target)
        return result
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n^(t/m)) | n 是 candidates 大小，t 是 target，m 是最小元素。最坏指数级 |
| 空间 | O(t/m) | 递归栈的最大深度，即最多选 t/min_candidate 个最小数字 |

## ⚠ 易错点

- [ ] **重复使用 vs 去重**：因为每个数字可无限次使用，递归时传 `i`（不是 `i + 1`）。如果题目改成每个数字只能用一次，就传 `i + 1`
- [ ] **剪枝前必须排序**：如果不排序就 break，会漏掉后面可能更小的数字
- [ ] **添加副本**：`result.append(path[:])` 必须用 `[:]` 或 `.copy()`，直接 append path 会因后续 pop 导致结果被修改
- [ ] **去重策略**：用 `start` 索引导向，保证组合内部数字按非递减顺序排列，避免 `[2,3]` 和 `[3,2]` 重复出现
- [ ] 使用 break 而非 continue：因为排序后后面的数字更大，所以遇到 `num > remaining` 可直接 break

## 🎤 面试怎么说

```text
这道题是经典的回溯问题，求组合总和，每个数字可以重复使用。

核心思路是回溯法建一个决策树，每层选择一个数字。
为了让每个数字可以重复使用，递归时 start 参数不向前移动（传 i，不是 i+1）。
为了去重，用 start 索引保证组合内部数字按升序排列。

两个关键优化：
1. 先对数组排序，当当前数字大于剩余目标时直接 break，
   因为后面的数更大，不可能满足条件。
2. 收集结果时用 path[:] 添加副本，因为 path 是个列表，
   后续的 pop 操作会改变它，不加副本会导致最终结果全是空列表。

时间复杂度最坏是指数级的，但有了剪枝在实际中会快很多。
空间复杂度主要是递归栈的深度，最坏情况是 target / 最小元素。

如果面试官问变种——每个数字只能用一次，只需把递归的 start 从 i 改成 i+1 即可。
```
