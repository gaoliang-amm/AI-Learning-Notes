# 17. 电话号码的字母组合

## 🎯 人话翻译

> 手机九宫格输入法：按了几个数字键，列出所有可能的字母组合。

## 💡 核心思想

> **回溯法（Backtracking）遍历决策树：**
>
> 每个数字对应 3-4 个字母。问题本质是求多个集合的**笛卡尔积**。
>
> 决策树：第 `index` 层代表处理第 `index` 个数字，该层有 3-4 个分支（对应数字的字母）。从根到叶子的每条路径都是一个完整组合。
>
> 回溯过程：选择一个字母 -> 递归到下一数字 -> 递归返回后自动切换下一个字母（因为 Python 字符串是不可变的，`path + letter` 创建了新字符串，不需要显式撤销）。

用到：[[回溯]]、[[DFS]]

## 📝 解题步骤

1. **建立映射表** — 数字 `2-9` 到字母串的映射（如 `'2': 'abc'`）
2. **边界处理** — 输入空字符串直接返回空列表
3. **定义回溯函数 `backtrack(index, path)`**：
   - **终止条件**：`index == len(digits)`，说明处理完所有数字，将 path 加入结果
   - **遍历当前数字的所有字母**：对每个字母 `letter`，递归调用 `backtrack(index + 1, path + letter)`
   - **自动回溯**：因为 `path` 是字符串（不可变），每次 `path + letter` 创建新对象，不需要显式 pop
4. **启动回溯** — `backtrack(0, "")`
5. **返回结果**

## 🔢 手动走一遍

输入: `digits = "23"`，映射：`2->abc`, `3->def`

### 决策树

```
                    "" (index=0, 数字=2)
            /         |          \
           a          b           c       (选2的字母, index=1)
         / | \      / | \       / | \
        d  e  f    d  e  f     d  e  f   (选3的字母, index=2, 终止)
```

### 完整遍历路径

| 路径 | index=0 选 | index=1 选 | 结果 |
|------|-----------|-----------|------|
| 1 | 'a' | 'd' | "ad" |
| 2 | 'a' | 'e' | "ae" |
| 3 | 'a' | 'f' | "af" |
| 4 | 'b' | 'd' | "bd" |
| 5 | 'b' | 'e' | "be" |
| 6 | 'b' | 'f' | "bf" |
| 7 | 'c' | 'd' | "cd" |
| 8 | 'c' | 'e' | "ce" |
| 9 | 'c' | 'f' | "cf" |

最终结果：`["ad","ae","af","bd","be","bf","cd","ce","cf"]`

## 💻 解法

> 回溯法：每次选一个数字的一个字母，递归处理下一个数字，直到所有数字处理完毕。

```python
from typing import List


class Solution:
    def letterCombinations(self, digits: str) -> List[str]:
        # 时间复杂度 O(3^m × 4^n)，空间复杂度 O(L)

        # 边界情况：输入为空字符串，直接返回空列表
        if not digits:
            return []

        # 数字到字母的映射表（0和1没有对应字母）
        phone_map = {
            '2': 'abc',
            '3': 'def',
            '4': 'ghi',
            '5': 'jkl',
            '6': 'mno',
            '7': 'pqrs',
            '8': 'tuv',
            '9': 'wxyz'
        }

        result = []

        def backtrack(index: int, path: str):
            """
            回溯函数

            参数：
                index: 当前正在处理的数字在 digits 中的索引
                path: 当前已经组合的字符串
            """
            # 终止条件：已经处理完所有数字
            if index == len(digits):
                result.append(path)
                return

            # 获取当前数字对应的字母串
            letters = phone_map[digits[index]]

            # 遍历当前数字对应的每个字母
            for letter in letters:
                # 选择当前字母，继续处理下一个数字
                backtrack(index + 1, path + letter)
                # 这里不需要显式撤销操作，因为 path 是字符串不可变，
                # 每次递归传递的是新字符串，自动实现了回溯

        backtrack(0, "")
        return result
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(3^m x 4^n) | m 是对应 3 个字母的数字个数，n 是对应 4 个字母的数字个数（7 和 9） |
| 空间 | O(L) | L 是 digits 长度，递归栈的深度，不计结果集 |

## ⚠ 易错点

- [ ] 输入为空字符串 `""` 时，需要返回 `[]` 而不是 `[""]`，这是 LeetCode 的特殊要求
- [ ] 数字 7 和 9 对应 4 个字母（`pqrs` 和 `wxyz`），其他数字对应 3 个，映射表不能写错
- [ ] 因为使用了不可变字符串 `path + letter`，不需要显式回溯（pop），但要注意这种做法每次创建新字符串，略有开销
- [ ] 如果用列表（`list`）作为 path，必须记得 `path.append()` 后 `path.pop()` 撤销
- [ ] 递归的终止条件是 `index == len(digits)`，不是 `len(path) == len(digits)`（虽然值相同，但语义不同）
