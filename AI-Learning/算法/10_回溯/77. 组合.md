# 77. 组合

## 🎯 人话翻译

> 从 1 到 n 中选 k 个数，列出所有可能的选法（不讲顺序，`[1,2]` 和 `[2,1]` 算同一个）。

## 💡 核心思想

> **回溯法求解组合问题：用 `start` 索引保证内部数字按升序排列，自动去重。**
>
> 与排列不同，组合不关注顺序，所以每层决策不需要从 0 开始，而是从 `start` 开始（只往后选、不往前看）。
>
> **关键剪枝**：当剩余可选数字不够凑满 k 个时，提前终止。条件是 `剩余可选数量 < 还需要的数量`，即 `(n - i + 1) < (k - len(path))`。

用到：[[回溯]]、[[剪枝]]

## 📝 解题步骤

1. **初始化** — `result = []`
2. **定义回溯函数 `backtrack(start, path)`**：
   - `start`：当前可以选的最小数字（1-indexed）
   - `path`：当前已选的数字列表
3. **两个终止条件（顺序重要）**：
   - **剪枝**：如果 `n - start + 1 < k - len(path)`，剩余数字不够凑满 k 个，提前返回
   - **已选满**：如果 `len(path) == k`，收集结果 `path[:]`
4. **遍历选择** — 从 `start` 到 `n`：
   - 选择：`path.append(i)`
   - 递归：`backtrack(i + 1, path)`（i+1 保证不重复选同一个数且内部升序）
   - 回溯：`path.pop()`
5. **返回结果**

## 🔢 手动走一遍

输入: `n = 4, k = 2`

### 决策树

```
                        start=1, path=[]
            /           |           |           \
          选1          选2          选3          选4
          |            |            |            |
      start=2       start=3      start=4      start=5
      path=[1]      path=[2]     path=[3]     剪枝 (n-start+1=0 < k-len=1)
      /   |   \      /   \         |
    选2  选3  选4   选3  选4      选4
    |    |    |     |    |        |
  [1,2] [1,3] [1,4] [2,3] [2,4] [3,4]
    ✅    ✅    ✅    ✅    ✅    ✅
```

### 剪枝演示：当 start=4, path=[3] 时

- `n - start + 1 = 4 - 4 + 1 = 1`（剩余可选：只有 4）
- `k - len(path) = 2 - 1 = 1`（还需要 1 个）
- 刚好相等，不剪枝，选 4 -> `[3, 4]` ✅

当 start=5, path=[4] 时：
- `n - start + 1 = 4 - 5 + 1 = 0`（没有剩余可选）
- `k - len(path) = 2 - 1 = 1`（还需要 1 个）
- `0 < 1`，剪枝 ✅

结果：`[[1,2], [1,3], [1,4], [2,3], [2,4], [3,4]]`（共 `C(4,2) = 6` 个）

## 💻 解法

> 回溯 + 剪枝：使用 start 索引导向只往后选，剩余数字不够时提前终止。

```python
from typing import List


class Solution:
    def combine(self, n: int, k: int) -> List[List[int]]:
        # 时间复杂度 O(C(n,k) × k)，空间复杂度 O(k)

        result = []

        def backtrack(start: int, path: List[int]):
            """
            回溯函数

            参数：
                start: 当前可以选择的起始数字（保证不重复，只选后面的数字）
                path: 当前已选择的数字列表
            """
            # 剪枝：如果剩余数字不够凑满 k 个，提前返回
            # 还需要 k - len(path) 个数字
            # 剩余可用的数字数量 = n - start + 1
            # 条件：n - start + 1 < k - len(path)
            if n - start + 1 < k - len(path):
                return

            # 终止条件：已经选满了 k 个数字
            if len(path) == k:
                result.append(path[:])  # 添加副本，避免后续修改影响
                return

            # 尝试从 start 到 n 的每个数字
            for i in range(start, n + 1):
                # 选择当前数字
                path.append(i)
                # 递归：下一个数字从 i+1 开始（保证不重复）
                backtrack(i + 1, path)
                # 回溯：撤销选择，尝试下一个数字
                path.pop()

        backtrack(1, [])
        return result
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(C(n,k) x k) | C(n,k) 种组合，每种复制 O(k) |
| 空间 | O(k) | path 和递归栈的深度，不计结果集 |

## ⚠ 易错点

- [ ] **start 的语义**：`start` 是 1-indexed 的（因为数字从 1 到 n），不要用 0-indexed
- [ ] **递归传 i+1 不是 start+1**：因为当前选了 `i`，下一个可选的是 `i+1`，如果传 `start+1` 会导致重复或跳过选择
- [ ] **剪枝条件写对**：`n - start + 1 < k - len(path)`，每一项的含义要清楚。或者等价地 `n - i + 1 >= k - len(path)` 写在 for 循环条件中
- [ ] **必须用 `path[:]` 副本**：和所有回溯题一样，直接 append path 会导致后续 pop 影响结果
- [ ] **排列 vs 组合的核心区别**：排列每层从 0 开始遍历 + used 数组；组合使用 start 索引保证升序。混淆了会写出重复的组合或错误的排列

## 🎤 面试怎么说

```text
这道题是经典的组合问题，核心是用回溯法 + start 索引来去重。

和排列问题的关键区别：组合不关心顺序，[1,2] 和 [2,1] 是同一个组合。
所以我用 start 参数保证每次只选比当前数字大的数字，这样就自动避免了重复。

回溯框架：
- 从 start 开始遍历到 n
- 选一个数加入 path，递归时传 i+1
- 选满 k 个就收集结果（用 path[:] 副本）
- 递归返回后 pop 撤销

一个重要的优化是剪枝：如果剩余可选的数字数量已经不够凑满 k 个了，
就提前终止这个分支。具体条件是 n - start + 1 < k - len(path)。

时间复杂度是 O(C(n,k) × k)，也就是组合数乘以 k（复制路径的代价）。
空间复杂度 O(k) 是递归栈深度。

如果面试官问排列问题（46），我会说排列用 used 数组；如果是子集问题（78），
则是在决策树中给每个元素「选」和「不选」两种分支。
```
