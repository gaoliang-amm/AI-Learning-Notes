# 230. 二叉搜索树中第 K 小的元素

## 🎯 人话翻译

> 给你一棵 BST 和一个整数 k，返回树中第 k 小的节点值（1-indexed，即第 1 小就是最小值）。

## 💡 核心思想

> **中序遍历 BST 会产生升序序列。在遍历过程中计数，数到第 k 个访问的节点就是答案。**
>
> 利用 BST 的中序性质（左 < 根 < 右），不需要遍历整棵树，找到第 k 个就可以提前终止。
> 用 `self.count` 记录已访问的节点数，`self.result` 存储答案。一旦 `count == k`，记录结果并 `return`，不再继续往右子树深入（剪枝优化）。
>
> 进阶：如果 BST 频繁被修改且经常查询第 k 小，可以在节点中维护"左子树的节点数"字段，做到 O(h) 时间。

用到：[[二叉搜索树]]、[[中序遍历]]、[[二叉树]]、[[递归]]

## 📝 解题步骤

1. **初始化** — `self.count = 0`（已访问节点数），`self.result = None`（存储结果）
2. **中序遍历** — 左 → 当前 → 右
3. **剪枝优化** — 如果 `self.result is not None`（已经找到答案），直接返回不再深入
4. **访问当前节点** — `self.count += 1`，如果 `self.count == k`，记录 `self.result = node.val` 并返回
5. **遍历完成** — 返回 `self.result`

## 🔢 手动走一遍

以下面这棵 BST 为例，k = 3：

```
        5
       / \
      3   6
     / \
    2   4
   /
  1
```

中序遍历顺序：1 → 2 → 3 → 4 → 5 → 6

| 步骤 | 当前节点 | count 加 | count == k? | result  | 继续？             |
|------|---------|----------|-------------|---------|--------------------|
| 1    | 1       | 1        | 1 != 3      | None    | 是，走 2            |
| 2    | 2       | 2        | 2 != 3      | None    | 是，走 3            |
| 3    | 3       | 3        | 3 == 3！    | **3**   | 不再继续右子树和父级 |

所以第 3 小的元素 = 3。

## 💻 解法

```python
from typing import Optional


class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def kthSmallest(self, root: Optional[TreeNode], k: int) -> int:
        # 时间复杂度 O(n)，空间复杂度 O(h)

        self.count = 0  # 已访问的节点数
        self.result = None  # 存储第 k 小的结果

        def inorder(node):
            if not node or self.result is not None:
                return

            # 先遍历左子树
            inorder(node.left)

            # 访问当前节点
            self.count += 1
            if self.count == k:
                self.result = node.val
                return

            # 再遍历右子树
            inorder(node.right)

        inorder(root)
        return self.result
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明                                       |
|------|--------|--------------------------------------------|
| 时间 | O(h+k) | 最坏 O(n)（k=n 或树退化成链表）。找到后剪枝  |
| 空间 | O(h)   | 递归栈深度，最坏 O(n)，平均 O(log n)         |

## ⚠ 易错点

- [ ] **剪枝条件 `self.result is not None`** — 找到答案后立即停止，避免不必要遍历。注意 `result` 的值是节点的 val（int），不能用 `not self.result`（0 也是 falsy）
- [ ] **k 是 1-indexed** — 第一个访问的节点是 `count=1`，不是 `count=0`
- [ ] **`self.count` 和 `self.result` 必须用 `self.`** — 嵌套函数内修改变量
- [ ] **中序遍历的顺序是 左→根→右** — `inorder(node.left)` 在前，然后访问当前节点，最后 `inorder(node.right)`
