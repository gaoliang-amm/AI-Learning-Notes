# 103. 二叉树的锯齿形层序遍历

## 🎯 人话翻译

> 给你一棵二叉树，按层遍历，但方向要蛇形切换：第一层从左到右，第二层从右到左，第三层从左到右，以此类推。

## 💡 核心思想

> **层序遍历 + 方向标志。用 `left_to_right` 布尔变量控制当前层的输出方向。**
>
> 每层依然按从左到右的顺序收集节点值（因为队列天然保证从左到右入队）。
> 如果当前层应该从右到左，就在加入结果前把 `level_values` 反转。
> 每层处理完后，翻转 `left_to_right`。
>
> 替代方案：不用 `reverse()`，可以用 `deque` 的 `appendleft` 在收集时就从另一头插入，但 `reverse()` 更简洁。

用到：[[二叉树]]、[[BFS]]、[[层序遍历]]、[[队列]]

## 📝 解题步骤

1. **初始化** — `result = []`，`queue = deque([root])`，`left_to_right = True`
2. **外层循环** — `level_size = len(queue)`，`level_values = []`
3. **内层循环** — `for _ in range(level_size)`：
   - 出队节点，`level_values.append(node.val)`
   - 左右孩子非空则入队（始终先左后右）
4. **方向处理** — `if not left_to_right: level_values.reverse()`
5. **记录结果** — `result.append(level_values)`
6. **翻转方向** — `left_to_right = not left_to_right`

## 🔢 手动走一遍

以下面的树为例：

```
        3
       / \
      9  20
         / \
        15  7
```

锯齿形层序遍历过程：

| 层 | left_to_right | level_size | 出队节点 | level_values (原始) | 反转？ | 最终值       | 翻转后标志 |
|----|--------------|-----------|---------|-------------------|--------|-------------|----------|
| 1  | True         | 1         | 3       | [3]               | 否     | [3]         | False    |
| 2  | False        | 2         | 9, 20   | [9, 20]           | 是     | [20, 9]     | True     |
| 3  | True         | 2         | 15, 7   | [15, 7]           | 否     | [15, 7]     | False    |

结果：`[[3], [20, 9], [15, 7]]`

## 💻 解法

```python
from typing import List, Optional
from collections import deque


class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def zigzagLevelOrder(self, root: Optional[TreeNode]) -> List[List[int]]:
        # 时间复杂度 O(n)，空间复杂度 O(n)

        if not root:
            return []

        result = []
        queue = deque([root])
        left_to_right = True  # 当前层是否从左到右

        while queue:
            level_size = len(queue)
            level_values = []

            # 正常层序遍历收集节点值（从左到右）
            for _ in range(level_size):
                node = queue.popleft()
                level_values.append(node.val)

                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)

            # 如果不是从左到右，则反转当前层的结果
            if not left_to_right:
                level_values.reverse()

            result.append(level_values)
            left_to_right = not left_to_right  # 切换方向

        return result
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明                                         |
|------|--------|----------------------------------------------|
| 时间 | O(n)   | 每个节点访问一次，每层反转 O(level_size)，总计 O(n) |
| 空间 | O(w)   | 队列最多存一层的节点数 w，最坏 O(n)             |

## ⚠ 易错点

- [ ] **队列里始终是从左到右的顺序** — 入队顺序不变（先左后右），方向切换只在输出结果时反转
- [ ] **`reverse()` 操作在加入 `result` 之前** — 先反转（如果需要），再加入结果
- [ ] **`left_to_right` 每层都要翻转** — 别忘了 `left_to_right = not left_to_right`
- [ ] **不要试图改变入队顺序来达成锯齿** — 改变入队顺序会打乱队列结构，比直接反转更复杂

## 🎤 面试怎么说

```text
这道题在标准层序遍历的基础上加了方向切换。

核心思路：用 BFS 层序遍历，正常入队出队（队列顺序始终是由左到右）。
每层收集完节点值后，根据一个布尔标志决定是否需要反转，
然后加入结果并切换标志。

不需要改变队列的顺序，一个简单的 reverse 就能搞定方向切换。

时间 O(n)，空间 O(w)。如果面试官要求不反转，
可以用双端队列，根据方向选择 appendleft 或 append。
```
