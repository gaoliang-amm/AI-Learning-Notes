# 105. 从前序与中序遍历序列构造二叉树

## 🎯 人话翻译

> 给你二叉树的前序遍历（根-左-右）和中序遍历（左-根-右）的数组，让你把原始的那棵树重建出来。

## 💡 核心思想

> **前序的第一个元素一定是整棵树的根。在中序中找到根的位置，左边就是左子树，右边就是右子树。递归下去。**
>
> 关键在于理解两种遍历的顺序差异：
> - 前序：`[根] [左子树全部] [右子树全部]`
> - 中序：`[左子树全部] [根] [右子树全部]`
> - 左子树节点数 = 中序里根的位置 `mid`
> - 前序中 `[1:mid+1]` 是左子树，`[mid+1:]` 是右子树
>
> 每次通过根在中序中的位置，划分出左右子树的范围，然后递归构造。

用到：[[二叉树]]、[[递归]]、[[分治]]、[[哈希表]]

## 📝 解题步骤

1. **递归出口** — 如果前序或中序数组为空，返回 `None`（空树）
2. **确定根节点** — `preorder[0]` 就是当前子树的根，用它构建 `TreeNode`
3. **在中序中定位根** — 用 `inorder.index(root_val)` 找到根的位置 `mid`，这就是左子树的节点个数
4. **递归构造左子树** — 前序范围 `[1:mid+1]`，中序范围 `[:mid]`
5. **递归构造右子树** — 前序范围 `[mid+1:]`，中序范围 `[mid+1:]`

## 🔢 手动走一遍

输入：
- 前序 `preorder = [3, 9, 20, 15, 7]`
- 中序 `inorder = [9, 3, 15, 20, 7]`

构造过程：

```
第一层：preorder[0]=3 是根
中序里 3 的索引 mid=1（左边 1 个节点[9]，右边 3 个节点[15,20,7]）
左子树前序 [9]，中序 [9]
右子树前序 [20,15,7]，中序 [15,20,7]

第二层（左）：preorder[0]=9 是根，中序里 9 的索引 mid=0
左右都无 → 返回节点 9

第二层（右）：preorder[0]=20 是根，中序里 20 的索引 mid=1（左边 [15]，右边 [7]）
左子树前序 [15]，中序 [15]
右子树前序 [7]，中序 [7]

第三层：节点 15（叶子），节点 7（叶子）
```

最终结构：

```
        3
       / \
      9  20
         / \
        15  7
```

## 💻 解法

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def buildTree(self, preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
        """
        前序：根 → 左 → 右
        中序：左 → 根 → 右
        思路：
        1. 前序第一个就是根
        2. 在中序里找到根，左边是左子树，右边是右子树
        3. 递归构造左右子树
        """
        # 递归出口：空树
        if not preorder or not inorder:
            return None

        # 前序第一个节点就是根
        root_val = preorder[0]
        root = TreeNode(root_val)

        # 找到根在中序数组中的位置
        mid = inorder.index(root_val)

        # 左子树：前序[1:mid+1]，中序[0:mid]
        root.left = self.buildTree(preorder[1:mid + 1], inorder[:mid])

        # 右子树：前序[mid+1:]，中序[mid+1:]
        root.right = self.buildTree(preorder[mid + 1:], inorder[mid + 1:])

        return root
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明                                                         |
|------|--------|--------------------------------------------------------------|
| 时间 | O(n^2) | 每次 `index()` 需要 O(n)，可用哈希表优化到 O(n)，每个节点处理一次 |
| 空间 | O(n)   | 每次递归创建新的切片数组（Python 切片会复制），递归栈 O(h)  + 切片 O(n) |

如果使用索引 + 哈希表优化（推荐面试写法）：

| 项目 | 复杂度 | 说明                         |
|------|--------|------------------------------|
| 时间 | O(n)   | 哈希表 O(1) 查找，每个节点一次 |
| 空间 | O(n)   | 哈希表 + 递归栈               |

## ⚠ 易错点

- [ ] **前序切片的左子树范围是 `[1:mid+1]`** — 不是 `[1:mid]`，因为 Python 切片右边界不包含，左子树有 `mid` 个节点
- [ ] **中序中根的索引 `mid` 等于左子树的节点个数** — 这个性质是划分前序数组的关键依据
- [ ] **`index()` 查找是 O(n)** — 大型数据会超时，面试中要提可以用哈希表 `{val: index}` 优化到 O(1)
- [ ] **不要混淆前序和后序** — 这题和前序+中序联动，区别于 106 题的中序+后序
