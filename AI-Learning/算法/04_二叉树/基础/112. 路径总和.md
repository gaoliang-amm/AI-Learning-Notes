# 112. 路径总和

## 🎯 人话翻译

> 给一棵二叉树和一个目标值 `targetSum`，问：有没有一条从根到叶子的路径，路径上所有节点的值加起来恰好等于 `targetSum`。

## 💡 核心思想

> **DFS 递归，自顶向下减法。到叶子节点时，判断剩余值是否恰好减到了 0。**
>
> 每往下一层，把 `targetSum` 减去当前节点的值。当走到叶子节点（左右都为空），直接比较当前节点的值是否等于剩余目标值。
> 对于非叶子节点，递归检查它的左子树或右子树，只要有一条路径满足即可（OR 关系）。
>
> 关键：必须走到叶子节点才算一条完整的路径，不能在半路停下。

用到：[[二叉树]]、[[递归]]、[[DFS]]

## 📝 解题步骤

1. **空节点处理** — `root` 为空，不存在任何路径，返回 `False`
2. **叶子节点判断** — 如果当前节点左右都为空，比较 `root.val == targetSum`
3. **继续往下搜** — 计算 `remaining = targetSum - root.val`，递归搜索左子树和右子树
4. **OR 逻辑** — 只要左子树或右子树有一条满足，就返回 `True`

## 🔢 手动走一遍

以下面的树为例，`targetSum = 22`：

```
        5
       / \
      4   8
     /   / \
    11  13  4
   /  \       \
  7    2       1
```

DFS 过程（自顶向下减）：

| 当前节点 | 进入时的剩余值 | 是否为叶子 | 动作                                       |
|---------|--------------|-----------|--------------------------------------------|
| 5       | 22           | 否        | remaining=17，搜索左(4,17) 和 右(8,17)      |
| 4       | 17           | 否        | remaining=13，搜索左(11,13)                 |
| 11      | 13           | 否        | remaining=2，搜索左(7,2) 和 右(2,2)         |
| 7       | 2            | 是        | 7 != 2，返回 False                         |
| 2       | 2            | 是        | 2 == 2，返回 True → 整条路通了！             |

路径：5 → 4 → 11 → 2，和为 5+4+11+2 = 22，匹配！

## 💻 解法

```python
# 二叉树节点定义
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def hasPathSum(self, root: TreeNode, targetSum: int) -> bool:
        """
        判断二叉树中是否存在根到叶子的路径，其节点值之和等于 targetSum。
        递归 DFS：每层将 targetSum 减去当前节点值，到达叶子时判断剩余值是否为 0。

        时间复杂度: O(n)，每个节点访问一次。
        空间复杂度: O(h)，递归栈深度，最坏 O(n)（链表），最好 O(log n)（平衡树）。
        """
        # 空节点，不存在路径
        if not root:
            return False

        # 到达叶子节点，判断当前节点值是否等于剩余的目标和
        if not root.left and not root.right:
            return root.val == targetSum

        # 递归检查左子树和右子树，目标值减去当前节点值
        remaining = targetSum - root.val
        return self.hasPathSum(root.left, remaining) or self.hasPathSum(root.right, remaining)
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明                                      |
|------|--------|-------------------------------------------|
| 时间 | O(n)   | 最坏情况遍历所有节点                       |
| 空间 | O(h)   | 递归栈深度，最坏 O(n)，平均 O(log n)       |

## ⚠ 易错点

- [ ] **必须到叶子节点才判断** — 如果某中间节点值刚好等于 targetSum 但不是叶子，不算有效路径
- [ ] **空节点返回 False 不是 True** — `if not root: return False`，空树不存在任何路径（即使 targetSum=0）
- [ ] **二分递归用 OR 连接** — `self.hasPathSum(root.left, ...) or self.hasPathSum(root.right, ...)`，短路求值优化
- [ ] **减法 vs 加法** — 自顶向下做减法更简洁，叶子处直接比较 `root.val == targetSum`；如果用加法需要额外传 `cur_sum` 参数
