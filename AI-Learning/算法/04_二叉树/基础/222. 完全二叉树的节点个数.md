# 222. 完全二叉树的节点个数

## 🎯 人话翻译

> 给你一棵完全二叉树（除了最后一层可能不满，其余层全满，且最后一层从左到右填充），求节点总数。要求比 O(n) 更快。

## 💡 核心思想

> **利用完全二叉树的性质：如果某子树的左右深度相等，则该子树是满二叉树，节点数 = 2^h - 1；否则递归计算。**
>
> 普通递归 `1 + countNodes(left) + countNodes(right)` 是 O(n)，没有利用完全二叉树的性质。
>
> 优化思路：对每个子树，分别计算最左深度和最右深度：
> - 如果 `left_depth == right_depth`，说明这棵子树是**满二叉树**，节点数 = `2^h - 1`（直接公式算，不用递归）
> - 如果不等，说明这棵子树不满，需要递归统计左右子树再加 1
>
> 完全二叉树中至少有一半子树是满二叉树，所以时间复杂度可以降到 O(log^2 n)。

用到：[[二叉树]]、[[完全二叉树]]、[[分治]]

## 📝 解题步骤

1. **递归出口** — 空节点返回 0
2. **计算最左深度** — 一直往左走，数层数
3. **计算最右深度** — 一直往右走，数层数
4. **判断满二叉树** — 如果 `left_depth == right_depth`，返回 `2**h - 1`
5. **否则递归** — `1 + countNodes(root.left) + countNodes(root.right)`

## 🔢 手动走一遍

以下面的完全二叉树为例：

```
        1
       / \
      2   3
     / \  /
    4  5 6
```

| 节点 | 最左深度 | 最右深度 | 满二叉树？ | 处理方式                                                    | 结果 |
|------|---------|---------|-----------|-------------------------------------------------------------|------|
| 1    | 3 (1-2-4) | 2 (1-3-6) | 否 (3!=2) | 递归左+右+1                                                 | 6    |
| 2    | 2 (2-4)   | 2 (2-5)   | 是 (2==2) | 满二叉树，直接 2^2-1=3                                       | 3    |
| 3    | 2 (3-6)   | 1 (3)     | 否 (2!=1) | 递归左+右+1                                                 | 2    |
| 6    | 1         | 1         | 是 (1==1) | 满二叉树，直接 2^1-1=1                                       | 1    |
| 4/5  | 都是叶子的父节点处理时被公式覆盖    | 

最终：节点 1 = 1 + 3 + 2 = 6。正确！

## 💻 解法

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def countNodes(self, root: TreeNode) -> int:
        """
        统计完全二叉树的节点个数
        普通递归思路：当前节点 + 左子树 + 右子树
        时间 O(n)，能过大部分场景
        """
        if not root:
            return 0

        return 1 + self.countNodes(root.left) + self.countNodes(root.right)
```

### 进阶优化版（O(log^2 n)）：

```python
class Solution:
    def countNodes(self, root: TreeNode) -> int:
        if not root:
            return 0

        # 计算最左深度
        left_depth = 0
        node = root
        while node:
            node = node.left
            left_depth += 1

        # 计算最右深度
        right_depth = 0
        node = root
        while node:
            node = node.right
            right_depth += 1

        # 左右深度相等 → 满二叉树 → 公式直接算
        if left_depth == right_depth:
            return (1 << left_depth) - 1  # 2^h - 1

        # 否则递归
        return 1 + self.countNodes(root.left) + self.countNodes(root.right)
```

## ⏱ 复杂度

| 版本       | 时间          | 空间      | 说明                               |
|-----------|--------------|-----------|------------------------------------|
| 普通递归   | O(n)         | O(log n)  | 每个节点访问一次，平衡树高 log n      |
| 进阶优化   | O(log^2 n)   | O(log n)  | 每次递归计算深度 O(log n)，递归 log n 次 |

## ⚠ 易错点

- [ ] **最左深度和最右深度的计算** — 都是从当前节点出发一路走到底，左深度只走 left，右深度只走 right
- [ ] **满二叉树公式是 `2^h - 1`** — 不是 `2^h`，记得减 1（h 是层数，包括根）
- [ ] **`1 << left_depth` 是做 2^h** — 位运算更快，`1 << left_depth` 等于 2 的 left_depth 次方
- [ ] **不能对非完全二叉树用公式** — 进阶版只对完全二叉树有效，普通二叉树只能用 O(n)
