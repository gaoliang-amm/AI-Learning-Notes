# 114. 二叉树展开为链表

## 🎯 人话翻译

> 给你一棵二叉树，按照前序遍历的顺序把它"拉直"成一条链表（用节点的 right 指针做 next，left 全设为 null），原地修改。

## 💡 核心思想

> **后序遍历 + 指针重接。从下往上，把每个节点的左子树整个搬到右边，再把原来的右子树接到新右子树的末尾。**
>
> 采用后序遍历（左右根）的顺序：先把左右子树各自拉平，再处理当前节点。
> 处理当前节点的三步：
> 1. 保存原来的右子树（之后要接回去）
> 2. 把拉平的左子树移到 `root.right`
> 3. 顺着新的右侧走到尽头，把原来的右子树接上去
>
> 关键：必须后序遍历，因为需要等左右子树都处理好了，才能正确拼接。

用到：[[二叉树]]、[[递归]]、[[链表]]

## 📝 解题步骤

1. **递归出口** — 空节点直接返回，什么也不做
2. **递归拉平左右子树** — `self.flatten(root.left)` 和 `self.flatten(root.right)`（后序：先处理子树）
3. **暂存原来的右子树** — `temp_right = root.right`
4. **把左子树移到右边** — `root.right = root.left`，`root.left = None`
5. **走到新右子树的末尾** — 从 `root` 开始沿 `right` 指针走到尽头
6. **接上原来的右子树** — `cur.right = temp_right`

## 🔢 手动走一遍

以下面这棵树为例：

```
        1
       / \
      2   5
     / \   \
    3   4   6
```

后序遍历拉平过程：

| 步骤 | 当前节点 | 操作                                                 | 结果（当前子树）       |
|------|---------|------------------------------------------------------|----------------------|
| 1    | 3       | 叶子，无操作                                          | 3                    |
| 2    | 4       | 叶子，无操作                                          | 4                    |
| 3    | 2       | 左(3)移到右，原来右(4)接到 3 后面                      | 2 → 3 → 4            |
| 4    | 6       | 叶子，无操作                                          | 6                    |
| 5    | 5       | 无左，右(6)已拉平                                     | 5 → 6                |
| 6    | 1       | 左(2→3→4)移到右，原来右(5→6)接到末尾                   | 1 → 2 → 3 → 4 → 5 → 6 |

最终链表（沿 right 指针）：

```
1 → 2 → 3 → 4 → 5 → 6
(all left pointers are None)
```

## 💻 解法

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def flatten(self, root: TreeNode) -> None:
        """
        Do not return anything, modify root in-place instead.
        将二叉树按前序遍历展开成**右侧单链表**，左边全为 None
        思路：
        1. 把左右子树都拉平
        2. 把左子树接到 root.right
        3. 把原来的右子树接到最末尾
        """
        if not root:
            return

        # 先递归把左、右子树都拉平
        self.flatten(root.left)
        self.flatten(root.right)

        # 暂存原来的右子树
        temp_right = root.right

        # 把左子树挪到右边
        root.right = root.left
        root.left = None  # 左边置空

        # 走到现在右子树的末尾
        cur = root
        while cur.right:
            cur = cur.right

        # 把原来的右子树接上去
        cur.right = temp_right
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明                                                         |
|------|--------|--------------------------------------------------------------|
| 时间 | O(n)   | 每个节点访问一次，但最坏每次找末尾要遍历，总时间 O(n^2)        |
| 空间 | O(h)   | 递归栈深度，最坏 O(n)                                        |

进阶优化：可以维护一个全局 `prev` 指针，用"右-左-根"的逆后序遍历，把复杂度优化到 O(n)。面试中提一下即可，不一定需要写。

## ⚠ 易错点

- [ ] **遍历顺序必须是后序** — 如果先处理当前节点再递归，会把还没拉平的子树弄乱
- [ ] **不要忘记把 left 置为 None** — `root.left = None` 是必须的，否则不是链表
- [ ] **暂存 `temp_right` 的时机** — 必须在 `root.right = root.left` 之前保存，否则原来的右子树就丢了
- [ ] **找末尾时从 `root` 开始而不是从 `root.right` 开始** — 因为可能 root.left 就是空的，此时 root.right 没变化
- [ ] **函数不需要返回值** — 注意函数签名 `-> None`，原地修改

## 🎤 面试怎么说

```text
这道题要求把二叉树按前序遍历展开成右侧单链表。

采用后序遍历（左右根）自底向上处理：
1. 先把左右子树各自递归拉平
2. 保存原来的右子树
3. 把拉平的左子树移到 root.right，left 置空
4. 沿着新的右侧走到末尾，接上原来的右子树

时间 O(n^2)（每次找末尾），可以用变种后序遍历优化到 O(n)。
空间 O(h) 递归栈。

面试官可能会问优化：维护一个 prev 指针，用右-左-根的顺序做后序遍历，
这样不需要遍历找末尾。
```
