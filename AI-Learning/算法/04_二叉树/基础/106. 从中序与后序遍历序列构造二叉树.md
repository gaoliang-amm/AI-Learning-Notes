# 106. 从中序与后序遍历序列构造二叉树

## 🎯 人话翻译

> 给你二叉树的中序遍历（左-根-右）和后序遍历（左-右-根）的数组，把原始的那棵树重建出来。

## 💡 核心思想

> **后序的最后一个元素一定是整棵树的根。在中序中找到根的位置，左边是左子树，右边是右子树。递归构建。**
>
> 和 [[105. 从前序与中序遍历序列构造二叉树]] 的思路完全一致，唯一的区别是：
> - 105 题的根在 `preorder[0]`（开头）
> - 106 题的根在 `postorder[-1]`（末尾）
> - 后序数组的划分：左子树 `postorder[:mid]`，右子树 `postorder[mid:-1]`（去掉最后一个根）
>
> 两种遍历顺序的对称性：
> - 中序：`[左子树] [根] [右子树]`
> - 后序：`[左子树] [右子树] [根]`

用到：[[二叉树]]、[[递归]]、[[分治]]

## 📝 解题步骤

1. **递归出口** — 如果中序或后序数组为空，返回 `None`
2. **确定根节点** — `postorder[-1]` 是当前子树的根
3. **在中序中定位根** — 用 `inorder.index(root_val)` 找到根位置 `mid`，左子树节点数 = `mid`
4. **递归构造左子树** — 中序范围 `[:mid]`，后序范围 `[:mid]`
5. **递归构造右子树** — 中序范围 `[mid+1:]`，后序范围 `[mid:-1]`

## 🔢 手动走一遍

输入：
- 中序 `inorder = [9, 3, 15, 20, 7]`
- 后序 `postorder = [9, 15, 7, 20, 3]`

构造过程：

```
第一层：postorder[-1]=3 是根
中序里 3 的索引 mid=1（左边 [9]，右边 [15,20,7]）
左子树：中序 [:1]=[9]，后序 [:1]=[9]
右子树：中序 [2:]=[15,20,7]，后序 [1:4]=[15,7,20]

第二层（左）：postorder[-1]=9 是根，中序 mid=0 → 叶子节点 9

第二层（右）：postorder[-1]=20 是根，中序 mid=1（[15] 在左，[7] 在右）
左子树：中序 [:1]=[15]，后序 [:1]=[15] → 节点 15
右子树：中序 [2:]=[7]，后序 [1:2]=[7] → 节点 7
```

最终结构：

```
        3
       / \
      9  20
         / \
        15  7
```

| 递归层 | 中序范围         | 后序范围         | 根值 | mid | 左子节点数 |
|--------|-----------------|-----------------|------|-----|-----------|
| 1      | [9,3,15,20,7]   | [9,15,7,20,3]   | 3    | 1   | 1         |
| 2L     | [9]             | [9]             | 9    | 0   | 0         |
| 2R     | [15,20,7]       | [15,7,20]       | 20   | 1   | 1         |
| 3L     | [15]            | [15]            | 15   | 0   | 0         |
| 3R     | [7]             | [7]             | 7    | 0   | 0         |

## 💻 解法

```python
from typing import List, Optional


class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def buildTree(self, inorder: List[int], postorder: List[int]) -> Optional[TreeNode]:
        """
        中序：左 → 根 → 右
        后序：左 → 右 → 根
        思路：
        1. 后序最后一个是根
        2. 中序找到根，左边左子树，右边右子树
        3. 递归构造
        """
        if not inorder or not postorder:
            return None

        # 后序最后一个就是根
        root_val = postorder[-1]
        root = TreeNode(root_val)

        # 找到中序里根的位置
        mid = inorder.index(root_val)

        # 左子树：中序[:mid]，后序[:mid]
        root.left = self.buildTree(inorder[:mid], postorder[:mid])

        # 右子树：中序[mid+1:]，后序[mid:-1]
        root.right = self.buildTree(inorder[mid + 1:], postorder[mid:-1])

        return root
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明                                                         |
|------|--------|--------------------------------------------------------------|
| 时间 | O(n^2) | 每次 `index()` 需要 O(n)，可用哈希表优化到 O(n)               |
| 空间 | O(n)   | 递归创建切片数组，栈深度 O(h)，切片 O(n)。哈希表优化后为 O(n) |

## ⚠ 易错点

- [ ] **右子树的右边界是 `mid:-1`** — 后序的最后一个是根，已经被取出，所以右子树的后序范围截止到 `-1`（不包含最后一个）
- [ ] **后序的左子树和右子树范围不要搞反** — 后序是 `[左子树, 右子树, 根]`，所以左子树是 `[:mid]`，右子树是 `[mid:-1]`
- [ ] **`mid` 表示左子树的节点个数** — 和前序 105 题一样，这个数量关系是切分的关键
- [ ] **空数组的出口判断** — `if not inorder or not postorder` 两者都要检查，防止一个数组比另一个先耗完
