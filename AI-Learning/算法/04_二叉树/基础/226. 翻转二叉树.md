# 226. 翻转二叉树

## 🎯 人话翻译

> 给你一棵二叉树，把每个节点的左右孩子交换，返回翻转后的树根。

## 💡 核心思想

> **先交换当前节点的左右子节点，再递归翻转左右子树。**
>
> 翻转的本质：对每个节点，"左右孩子"互换位置。可以采用先序遍历思想——
> 先处理当前节点（交换左右），再递归处理左右子节点（此时它们已是交换后的状态）。
>
> 关键是：交换操作本身是 O(1)，递归保证整棵树的每个节点都处理到。

用到：[[二叉树]]、[[递归]]、[[DFS]]

## 📝 解题步骤

1. **空节点处理** — 如果 `root` 为 `None`，直接返回 `None`
2. **交换左右子节点** — `root.left, root.right = root.right, root.left`，利用 Python 的元组解包一步完成
3. **递归翻转左子树** — `self.invertTree(root.left)`，此时 `root.left` 是原来的右子树
4. **递归翻转右子树** — `self.invertTree(root.right)`，此时 `root.right` 是原来的左子树
5. **返回根节点** — `return root`

## 🔢 手动走一遍

以下面这棵树为例：

```
        4
       / \
      2   7
     / \ / \
    1  3 6  9
```

翻转过程（每一步先交换当前节点，再递归）：

| 步骤 | 当前节点 | 交换前左右             | 交换后左右             | 然后递归         |
|------|---------|----------------------|----------------------|------------------|
| 1    | 4       | left=2, right=7       | left=7, right=2       | 递归翻转左(7)和右(2) |
| 2    | 7       | left=6, right=9       | left=9, right=6       | 递归翻转左(9)和右(6) |
| 3    | 9       | left=None, right=None | left=None, right=None | 无需递归         |
| 4    | 6       | left=None, right=None | left=None, right=None | 无需递归         |
| 5    | 2       | left=1, right=3       | left=3, right=1       | 递归翻转左(3)和右(1) |
| 6    | 3       | left=None, right=None | left=None, right=None | 无需递归         |
| 7    | 1       | left=None, right=None | left=None, right=None | 无需递归         |

最终结果：

```
        4
       / \
      7   2
     / \ / \
    9  6 3  1
```

## 💻 解法

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def invertTree(self, root: TreeNode) -> TreeNode:
        """
        翻转二叉树（左右子树交换）
        思路：先交换当前节点的左右孩子，再递归——翻左、翻右
        时间复杂度：O(n)，空间复杂度：O(h)
        """
        # 空节点直接返回
        if not root:
            return None

        # 交换当前节点的左右子节点
        root.left, root.right = root.right, root.left

        # 递归翻转左右子树
        self.invertTree(root.left)
        self.invertTree(root.right)

        return root
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明                                      |
|------|--------|-------------------------------------------|
| 时间 | O(n)   | 每个节点访问一次                           |
| 空间 | O(h)   | 递归栈深度，最坏 O(n)，平均 O(log n)       |

## ⚠ 易错点

- [ ] **交换顺序要正确** — 先交换当前节点的左右孩子，再递归进入左右子树，不能倒过来
- [ ] **注意 return 值** — 递归返回的是翻转后的节点，但实际修改发生在引用上，调用的返回值一般不用于重新构建树（但也需要 return root 给顶层调用者）
- [ ] **空节点返回 None** — 忘记了会导致 `None.left` 报错
- [ ] **不要新建节点** — 题目要求原地翻转，不需要创建 `TreeNode` 副本
