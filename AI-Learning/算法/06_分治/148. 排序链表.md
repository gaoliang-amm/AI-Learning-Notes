# 148. 排序链表

## 🎯 人话翻译

> 给你一个单向链表，用 O(n log n) 时间、O(1) 空间（递归栈不计）把它排成升序。

## 💡 核心思想

> **归并排序（分治）。快慢指针找中点，断开链表，递归排序左右两半，然后合并两个有序链表。**
>
> 分治三步：
> 1. **分割**：快慢指针找到链表的中点，从中断开成两个子链表
> 2. **递归**：分别对左右两个子链表递归排序
> 3. **合并**：用"合并两个有序链表"的标准模板，把左右合并成一个有序链表
>
> 为什么是归并排序而不是快排？快排需要随机访问（数组下标），链表不支持。归并排序天然适合链表，合并过程利用了链表的指针灵活性。

用到：[[分治]]、[[归并排序]]、[[链表]]、[[快慢指针]]

## 📝 解题步骤

1. **递归出口** — 空链表或单节点链表，已经有序，直接返回 `head`
2. **快慢指针找中点** — `slow = head`，`fast = head.next`，fast 每次两步，slow 每次一步，fast 到头时 slow 就是中点（前中点）
3. **断开链表** — `mid = slow.next`，`slow.next = None`
4. **递归排序左右** — `left = self.sortList(head)`，`right = self.sortList(mid)`
5. **合并两个有序链表** — `return self.merge(left, right)`
6. **合并函数** — 标准双路归并，dummy 头 + cur 指针，谁小接谁

## 🔢 手动走一遍

输入：`head = 4 → 2 → 1 → 3`

归并过程：

```
切分阶段：
4 → 2 → 1 → 3
  → 快慢指针：slow=2, fast=3
  → 断开：左=[4→2], 右=[1→3]

左半 [4→2] 继续切分：
  4 → 2
    → slow=4, fast=None
    → 断开：左=[4], 右=[2]
    单节点 [4]：直接返回
    单节点 [2]：直接返回

右半 [1→3] 继续切分：
  1 → 3
    → slow=1, fast=None
    → 断开：左=[1], 右=[3]
    单节点 [1]：直接返回
    单节点 [3]：直接返回

合并阶段：
merge([4], [2]) → 2 → 4
merge([1], [3]) → 1 → 3
merge([2→4], [1→3]) → 1 → 2 → 3 → 4
```

结果：`1 → 2 → 3 → 4`

| 阶段 | 链表             | 操作                     | 结果              |
|------|-----------------|-------------------------|-------------------|
| 切分 | [4,2,1,3]       | slow=2（前中点），mid=1->3 | 左[4,2] 右[1,3]   |
| 切分 | [4,2]           | slow=4，mid=2            | [4] 和 [2]        |
| 合并 | [4] + [2]       | merge                    | [2,4]             |
| 切分 | [1,3]           | slow=1，mid=3            | [1] 和 [3]        |
| 合并 | [1] + [3]       | merge                    | [1,3]             |
| 合并 | [2,4] + [1,3]   | merge                    | [1,2,3,4]         |

## 💻 解法

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def sortList(self, head):
        """
        链表归并排序
        """

        # =========================
        # 递归终止条件
        # 空节点 或 只有一个节点
        # 已经天然有序
        # =========================
        if not head or not head.next:
            return head

        # =========================
        # 1、快慢指针找中点
        # =========================
        slow = head
        fast = head.next

        # fast走两步
        # slow走一步
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next

        # =========================
        # 2、断开链表
        # =========================
        mid = slow.next
        slow.next = None

        # =========================
        # 3、递归排序左右部分
        # =========================
        left = self.sortList(head)
        right = self.sortList(mid)

        # =========================
        # 4、合并两个有序链表
        # =========================
        return self.merge(left, right)

    def merge(self, l1, l2):
        """
        合并两个有序链表
        """

        # 虚拟头节点
        dummy = ListNode(0)

        # 当前尾节点
        cur = dummy

        # 两个链表都不为空
        while l1 and l2:

            # 谁小接谁
            if l1.val < l2.val:
                cur.next = l1
                l1 = l1.next
            else:
                cur.next = l2
                l2 = l2.next

            # cur向后移动
            cur = cur.next

        # 如果l1还有剩余
        if l1:
            cur.next = l1

        # 如果l2还有剩余
        if l2:
            cur.next = l2

        # 返回真正头节点
        return dummy.next
```

## ⏱ 复杂度

| 项目 | 复杂度       | 说明                                                         |
|------|-------------|--------------------------------------------------------------|
| 时间 | O(n log n)  | 每层递归合并 O(n)，递归深度 O(log n)，总 O(n log n)            |
| 空间 | O(log n)    | 递归栈深度 O(log n)。如果严格要求 O(1) 空间可以用自底向上的迭代归并 |

## ⚠ 易错点

- [ ] **找中点时 fast 的起点是 `head.next`** — 如果从 `head` 开始，偶数长度链表 slow 会停在"后中点"，导致无法断开成正确的前半段
- [ ] **断开操作** — `slow.next = None` 把链表切成两段，必须做，否则不是分治是死循环
- [ ] **`mid = slow.next` 在断开之前取值** — 如果先断开再取 `slow.next`，拿到的就是 `None` 了
- [ ] **merge 返回的 dummy.next** — 不能返回 dummy，dummy 是哨兵节点
- [ ] **合并后剩余节点直接接上** — `cur.next = l1 if l1 else l2`，不需要再遍历

```text
这道题要求 O(n log n) 时间排序链表，最适合用归并排序（分治）。

步骤：
1. 快慢指针找到链表的中点（fast 从头.next 出发，保证 slow 停在前中点）
2. 从中点断开链表
3. 递归对左右两半排序
4. 合并两个有序链表

时间 O(n log n)，空间 O(log n) 递归栈。

如果要求严格的 O(1) 空间：用自底向上的迭代归并，
按照步长 1, 2, 4, 8... 逐步合并，不需要递归。
面试中一般写出递归版就够了，提一下迭代优化。
```
