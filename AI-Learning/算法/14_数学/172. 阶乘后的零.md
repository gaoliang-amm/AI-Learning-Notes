# 172. 阶乘后的零

## 🎯 人话翻译

> 给你一个整数 n，问 n!（n 的阶乘）的末尾有几个连续的零。

## 💡 核心思想

> **末尾零由因子 10 产生，而 10 = 2 x 5。在阶乘中，因子 2 的数量远多于因子 5，所以末尾零的数量完全由因子 5 的数量决定。**
>
> 举例：`5! = 120`，有一个 0，因为 1-5 中只有一个 5。
>
> 注意：`25` 贡献两个 5（25 = 5 x 5），`125` 贡献三个 5。
>
> **公式：`count = n/5 + n/25 + n/125 + ...` = 累加 `n // 5^i` 直到 n < 5^i。**

用到：[[数学]]

## 📝 解题步骤

1. **初始化** — `count = 0`
2. **循环统计** — 当 `n >= 5` 时：
   - `n //= 5`（等价于 n 除以 5 取整，统计本轮有多少个 5 的倍数）
   - `count += n`（累加到结果）
3. **返回 count**

例如 n = 100：
- 第 1 轮：`n = 100 // 5 = 20`（20 个数至少有一个因子 5），`count = 20`
- 第 2 轮：`n = 20 // 5 = 4`（4 个数至少有两个因子 5，如 25, 50, 75, 100），`count = 24`
- 第 3 轮：`n = 4 // 5 = 0`（没有数有三个因子 5），停止

答案 = 24。

## 🔢 手动走一遍

输入: `n = 30`

| 轮次 | n | n //= 5 | count | 说明 |
|------|---|---------|-------|------|
| 1 | 30 | 6 | 6 | 1..30 中 5 的倍数：5, 10, 15, 20, 25, 30 (6 个) |
| 2 | 6 | 1 | 7 | 其中 25 的倍数：25 (1 个，额外再贡献一个 5) |
| 3 | 1 | 0 | 7 | 停止 |

验证：`30! = 265252859812191058636308480000000`，末尾有 7 个 0。

再验证 n = 10：
| 轮次 | n | n //= 5 | count |
|------|---|---------|-------|
| 1 | 10 | 2 | 2 |
| 2 | 2 | 0 | 2 |

`10! = 3628800`，末尾 2 个 0。

## 💻 解法

> 累加 n 除以 5 的幂次，统计所有因子 5 的总数。

```python
class Solution:
    def trailingZeroes(self, n: int) -> int:
        """
        计算 n! 中尾随零的个数。
        通过累加 n 除以 5 的各次幂的商来统计因子 5 的总数。

        时间复杂度: O(log n)，空间复杂度: O(1)。
        """
        count = 0
        # 循环直到 n 小于 5，每次统计当前 n 中有多少个 5 的幂次的倍数
        while n >= 5:
            n //= 5      # 求 n / 5 的整数部分，相当于计算 n 中包含的当前 5 的幂次的个数
            count += n   # 将这个个数累加到结果中
        return count
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(log n) | 每次 n 除以 5，约 log_5(n) 次循环 |
| 空间 | O(1) | 只用了计数器变量 |

## ⚠ 易错点

- [ ] **只统计因子 5**：不要同时统计 2 和 5 再取 min，因为因子 2 > 因子 5，零的数量由 5 决定。统计两者虽然正确但多余
- [ ] **注意 25、125 等**：25 包含两个因子 5，必须多次累加（`n/5 + n/25 + n/125 + ...`）。只做 `n // 5` 就错了
- [ ] **循环方式**：用 `while n >= 5: n //= 5; count += n` 比用 `i = 5; while i <= n: count += n // i; i *= 5` 更简洁
- [ ] **Python 整数除法**：`n //= 5` 是整除，自动向下取整，正确
- [ ] 不要用 `math.factorial(n)` 再数末尾零，阶乘值太大，n 稍大就会溢出或超时

## 🎤 面试怎么说

```text
n! 末尾的零由因子 10 产生，10 = 2 x 5。
在阶乘中，因子 2 的数量远远多于因子 5（每两个数就有一个偶数），
所以零的数量由因子 5 的数量决定。

关键是要注意像 25、125 这样的数包含多个因子 5：
- n/5：统计至少包含一个因子 5 的数的个数
- n/25：统计至少包含两个因子 5 的数的个数
- n/125：以此类推...

所以公式是：count = n/5 + n/25 + n/125 + ...

实现时不停地把 n 除以 5，累加商即可：
while n >= 5:
    n //= 5
    count += n

时间复杂度 O(log n)，空间复杂度 O(1)。
```
