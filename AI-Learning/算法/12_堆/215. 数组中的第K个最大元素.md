# 215. 数组中的第K个最大元素

## 🎯 人话翻译

> 给一个无序数组，找出第 K 大的元素（不是第 K 小的）。比如 [3,2,1,5,6,4] 中第 2 大的是 5。

## 💡 核心思想

> 维护大小为 K 的**最小堆**。遍历数组，将元素加入堆，当堆大小超过 K 时弹出堆顶（最小值）。遍历结束后，堆中保留了前 K 大的元素，堆顶就是第 K 大。Python 的 `heapq` 只能是[[堆|最小堆]]，所以直接用它维护前 K 大即可，不需要取负。

用到：[[堆]]、[[最小堆]]

## 📝 解题步骤

1. **初始化最小堆** — `min_heap = []`
2. **遍历数组** — 对每个 `num`：
   - `heapq.heappush(min_heap, num)` 加入堆
   - 如果 `len(min_heap) > k`，则 `heapq.heappop(min_heap)` 弹出最小的
3. **返回结果** — 堆顶 `min_heap[0]` 就是第 K 大的元素

## 🔢 手动走一遍

输入: `nums = [3, 2, 1, 5, 6, 4]`, `k = 2`

| num | 操作 | 堆内容(k=2) | 说明 |
|-----|------|------------|------|
| 3 | push | [3] | 大小1 |
| 2 | push | [2, 3] | 大小2(刚好k) |
| 1 | push, pop | [2, 3] | push后[1,2,3] > k，pop 1 |
| 5 | push, pop | [3, 5] | push后[2,3,5] > k，pop 2 |
| 6 | push, pop | [5, 6] | push后[3,5,6] > k，pop 3 |
| 4 | push, pop | [5, 6] | push后[4,5,6] > k，pop 4 |

最终堆顶 = `5`，第 2 大的元素。

## 💻 解法

> 大小为 k 的最小堆：堆顶始终保持为当前遇到的第 k 大元素。

```python
import heapq
from typing import List


class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        # 时间复杂度 O(n log k)，空间复杂度 O(k)

        min_heap = []

        for num in nums:
            # 将当前元素加入堆
            heapq.heappush(min_heap, num)
            # 保持堆的大小不超过 k
            if len(min_heap) > k:
                heapq.heappop(min_heap)

        # 堆顶就是第 k 大的元素
        return min_heap[0]
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n log k) | 每个元素入堆/出堆 O(log k)，共 n 个 |
| 空间 | O(k) | 堆中最多存 k 个元素 |

## ⚠ 易错点

- [ ] Python 的 `heapq` 是最小堆，要的是第 K **大**：直接用小顶堆即可（堆中保留前 K 大，堆顶就是第 K 大）。如果要用最大堆找第 K 大，需要取负数 `heappush(-num)`
- [ ] 如果 k = 1（第一大的，即最大值），复杂度退化到 O(n log n)? 注意每个元素入堆后如果大小超过 1 就弹出，每次 push 都是 O(1)，弹出也 O(1)，总体 O(n log 1) = O(n)——还是对的
- [ ] 也可以用快速选择（QuickSelect）做到平均 O(n) 时间，堆解法更简单稳定
- [ ] `heapq.heappop` 会修改堆，要确保操作后堆大小正确
