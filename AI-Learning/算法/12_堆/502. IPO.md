# 502. IPO

## 🎯 人话翻译

> 你有初始资本 w，最多能做 k 个项目。每个项目需要启动资金 capital[i]，完成后的利润是 profits[i]。每次完成一个项目，利润加到资本里，然后可以拿更多资本启动更贵的项目。目标是最终资本最大化。简单说就是：有钱了才能做项目，做了项目更赚钱。

## 💡 核心思想

> [[贪心]] + [[堆|最大堆]]。把项目按启动资金排序。每次在"当前资本能启动"的所有项目中，选利润最大的那个来做。这个"选利润最大"的操作正好用最大堆完成。Python heapq 是最小堆，所以利润存负数来模拟最大堆。

用到：[[堆]]、[[贪心]]、[[最大堆]]

## 📝 解题步骤

1. **预处理** — 把 `(capital[i], profits[i])` 打包成项目列表，按 capital 升序排序
2. **初始化最大堆** — `max_heap = []`（存利润的负数）
3. **循环 k 次** — 每次做最多一个项目：
   - 把所有 capital <= 当前资本 w 的项目从列表丢入最大堆（存 -profit）
   - 如果堆为空（没有可做的项目），提前结束
   - 从堆中弹出利润最大的项目：`w += -heapq.heappop(max_heap)`
4. **返回最终资本** — `return w`

## 🔢 手动走一遍

输入: `k = 2, w = 0, profits = [1, 2, 3], capital = [0, 1, 1]`

项目列表: [(0,1), (1,2), (1,3)]，排序后: [(0,1), (1,2), (1,3)]

| 轮次 | w | 可加入的项目 | 堆内容 | 选择 | w 更新 |
|------|---|-------------|--------|------|--------|
| 初始 | 0 | - | [] | - | - |
| 第1轮 | 0 | i=0: (0,1) 加入堆，i=1 | [(-1)] | 弹出 -1 | w = 0 + 1 = 1 |
| 第2轮 | 1 | i=1: (1,2) 加入堆，i=2: (1,3) 加入堆，i=3 | [(-2), (-3)] | 弹出 -3（最大利润） | w = 1 + 3 = 4 |

k 轮用完，返回 `4`。

## 💻 解法

> 项目按资本排序，最大堆选利润最高的：每次把所有能做项目入堆，弹出利润最大的执行。

```python
import heapq
from typing import List


class Solution:
    def findMaximizedCapital(self, k: int, w: int, profits: List[int], capital: List[int]) -> int:
        # 时间复杂度 O(n log n + k log n)，空间复杂度 O(n)

        n = len(profits)
        # 将项目按资本排序（最小堆）
        projects = [(capital[i], profits[i]) for i in range(n)]
        projects.sort()

        # 最大堆（存储利润，Python 的 heapq 是最小堆，存负数实现最大堆）
        max_heap = []

        i = 0
        for _ in range(k):
            # 将所有资本 <= 当前资本的项目加入最大堆
            while i < n and projects[i][0] <= w:
                heapq.heappush(max_heap, -projects[i][1])  # 存负数实现最大堆
                i += 1

            # 如果没有可做的项目，提前结束
            if not max_heap:
                break

            # 选择利润最大的项目
            w += -heapq.heappop(max_heap)  # 取出负数再取反

        return w
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n log n + k log n) | 排序 O(n log n)，每次堆操作 O(log n)，最多 n 次入堆 + k 次出堆 |
| 空间 | O(n) | 项目列表占 O(n)，堆最多 O(n) |

## ⚠ 易错点

- [ ] Python heapq 是最小堆，要实现最大堆必须**存负数**：`heappush(-profit)` 和 `w += -heappop()`
- [ ] 项目按 capital 排序后指针 `i` 不会回退：因为 w 只会增加，之前不满足 capital 条件的项目后面可能满足，但已满足的不会变回不满足
- [ ] 循环 k 次，但如果没有可做的项目（堆空）要提前 break，避免死循环
- [ ] 注意是"最多 k 个项目"：如果可做的项目不足 k 个，做完就停

## 🎤 面试怎么说

```text
这道题是贪心 + 优先队列的经典题。

思路是：把项目按启动资金排序。每一轮，
把当前资本能启动的所有项目扔进最大堆（按利润排序），
然后选利润最大的那个来做。做完后资本增加，解锁更多项目。

最大堆用 Python heapq 加负号模拟。
时间 O(n log n + k log n)，空间 O(n)。

面试官可能会问为什么贪心是对的？
因为每轮在当前可选的项目中选利润最大的，不会漏掉更优解：
资本只会增多不会减少，之前因为资本不够而跳过的项目
后面如果资本够了还是能选的。
```
