# 373. 查找和最小的 K 对数字

## 🎯 人话翻译

> 两个各自升序的数组 nums1 和 nums2，从两个数组各取一个数组成数对 (u, v)。找出所有数对中和最小的前 k 个数对。比如 nums1=[1,1,2], nums2=[1,2,3], k=2，答案是 [1,1]对(1+1=2) 和 [1,1]对(另一个1+1=2)。

## 💡 核心思想

> [[堆|最小堆]] + 多路归并。把问题看成：对 nums1 中每个元素，与 nums2 中的每个元素配对，形成多个有序序列。用最小堆维护"当前每路的最小候选"，每次弹出最小的数对 (i, j)，然后推入同路的下一对 (i, j+1)。

用到：[[堆]]、[[多路归并]]

## 📝 解题步骤

1. **初始化** — 将 `nums1` 的前 `min(k, len(nums1))` 个元素分别与 `nums2[0]` 配对，以 `(和, i, j)` 的形式放入最小堆
2. **循环 k 次**（或直到堆空）：
   - 弹出堆顶 `(和, i, j)`，记录 `[nums1[i], nums2[j]]`
   - 如果 `j + 1 < len(nums2)`，将同路的下一对 `(nums1[i] + nums2[j+1], i, j+1)` 入堆
3. **返回结果** — 收集到的 k 个数对

## 🔢 手动走一遍

输入: `nums1 = [1, 7, 11]`, `nums2 = [2, 4, 6]`, `k = 3`

初始化堆: (1+2=3, i=0, j=0), (7+2=9, i=1, j=0), (11+2=13, i=2, j=0)
堆: [(3,0,0), (9,1,0), (13,2,0)]

| 轮 | 弹出的和 | (i,j) | 结果数对 | 入堆下一对 | 堆状态 |
|----|---------|-------|---------|-----------|--------|
| 1 | 3 | (0,0) | [1,2] | (1+4=5, 0, 1) | [(5,0,1), (9,1,0), (13,2,0)] |
| 2 | 5 | (0,1) | [1,4] | (1+6=7, 0, 2) | [(7,0,2), (9,1,0), (13,2,0)] |
| 3 | 7 | (0,2) | [1,6] | j+1=3>=len=3 不入 | [(9,1,0), (13,2,0)] |

结果: `[[1,2], [1,4], [1,6]]`

## 💻 解法

> 最小堆多路归并：每路初始化第一个元素 (i, 0)，每次弹出最小后推入同路的 (i, j+1)。

```python
import heapq
from typing import List


class Solution:
    def kSmallestPairs(self, nums1: List[int], nums2: List[int], k: int) -> List[List[int]]:
        # 时间复杂度 O(k log k)，空间复杂度 O(k)

        if not nums1 or not nums2:
            return []

        # 最小堆存储 (和, 索引1, 索引2)
        min_heap = []

        # 将 nums1 的前 min(k, len(nums1)) 个元素与 nums2[0] 配对
        for i in range(min(k, len(nums1))):
            heapq.heappush(min_heap, (nums1[i] + nums2[0], i, 0))

        result = []

        while len(result) < k and min_heap:
            # 弹出当前最小的数对
            _, i, j = heapq.heappop(min_heap)
            result.append([nums1[i], nums2[j]])

            # 将 (nums1[i], nums2[j+1]) 加入堆
            if j + 1 < len(nums2):
                heapq.heappush(min_heap, (nums1[i] + nums2[j + 1], i, j + 1))

        return result
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(k log k) | 每次堆操作 O(log k)，共 k 次出堆，最多 k 次入堆 |
| 空间 | O(k) | 堆中最多存 min(k, len(nums1)) + k 个元素 |

## ⚠ 易错点

- [ ] 初始化时只取 `min(k, len(nums1))` 个起始元素：不需要全部入堆，因为最多只需要 k 个答案
- [ ] 堆中存储 `(和, i, j)`，和用于排序，i 和 j 用于扩展。注意三元组比较时 Python 会按第一个元素然后第二个、第三个比较，如果和相同会继续比 i 和 j，虽然最终回答可能乱序但正确性不受影响
- [ ] 弹出后推入的是 `(i, j+1)` 不是 `(i+1, j)`：每路是固定 nums1[i]，在 nums2 中滑动。初始化已经覆盖了所有 i，推入时只需往后移 j
- [ ] 不要忘记边界检查：`if not nums1 or not nums2: return []`

## 🎤 面试怎么说

```text
这道题是"多路归并 + 最小堆"的经典应用。

把 nums1 的每个元素看作一条路，每条路对应它和 nums2 所有元素的配对序列。
因为两个数组都升序，所以每条路内部按 j 递增的配对和也是递增的。

用最小堆维护每路当前最小的候选值。初始化时把每个 i 和 j=0 的配对入堆。
然后每次弹出最小值，把同路的下一个 j+1 入堆。
重复直到收集到 k 个。

时间 O(k log k)，空间 O(k)。这是解决"多有序序列合并取前 k"的通用模板。
```
