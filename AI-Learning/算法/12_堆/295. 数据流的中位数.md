# 295. 数据流的中位数

## 🎯 人话翻译

> 设计一个数据结构，支持两个操作：`addNum(num)` 添加一个整数，`findMedian()` 返回当前所有数中的中位数。数据是流式输入，随时可能查询中位数。

## 💡 核心思想

> 用**两个堆**把数据分成"较小的一半"和"较大的一半"：
> - **大顶堆 `max_heap`**（Python 用负数模拟）：存较小的一半，堆顶是这半的最大值
> - **小顶堆 `min_heap`**：存较大的一半，堆顶是这半的最小值
>
> 保持 `max_heap` 的大小等于 `min_heap` 或比它多 1。这样：
> - 奇数时，`max_heap` 堆顶就是中位数
> - 偶数时，两个堆顶的平均值就是中位数
>
> 每次插入新数时，先放 `max_heap`，再把最大值移到 `min_heap`，最后平衡两个堆的大小。

用到：[[堆]]、[[最大堆]]、[[中位数]]

## 📝 解题步骤

1. **初始化** — `max_heap = []`（大顶堆，存负数）、`min_heap = []`（小顶堆）
2. **addNum(num)**：
   - 将 `num` 取负放入 `max_heap`
   - 将 `max_heap` 堆顶（最大值）弹出，放入 `min_heap`
   - 如果 `max_heap` 的大小 < `min_heap`，从 `min_heap` 弹出一个元素放入 `max_heap`（保持平衡）
3. **findMedian()**：
   - 如果 `max_heap` 比 `min_heap` 多 1：返回 `-max_heap[0]`
   - 如果两堆等大：返回 `(-max_heap[0] + min_heap[0]) / 2.0`

更正：Python heapq 是最小堆，`heapq.heappush(max_heap, -2)` -> `max_heap = [-2, 1]`（堆顶-2）。`-heapq.heappop(max_heap)` = `-(-2)` = `2`。所以 `heapq.heappush(min_heap, 2)` -> `min_heap = [2]`。

| 操作 | max_heap | min_heap | 说明 |
|------|----------|----------|------|
| max < min? | 大小相等 | - | 不平衡不调整 |

两堆: max=[1], min=[2]，中位数 = (1 + 2) / 2 = 1.5。

**addNum(3):**

| 操作 | max_heap | min_heap | 说明 |
|------|----------|----------|------|
| push -3 到 max | [1, -3] 即内部 [-3, 1] | [2] | - |
| pop max(-3即值3) 到 min | [1] | [2, 3] | - |
| max < min? 是(1 < 2) | [1, -2] | [3] | 从 min 移 -min[0]=2 到 max |

两堆: max=[1, -2] 即内部 [-2, 1]，堆顶是 -2 = 最大值 2；min=[3]，堆顶 3。
中位数 = -max[0] = 2。

## 💻 解法

> 双堆法：大顶堆存较小一半，小顶堆存较大一半。插入时用"过一遍"的方式保持左右平衡。

```python
import heapq


class MedianFinder:
    # 时间复杂度 addNum: O(log n)，findMedian: O(1)
    # 空间复杂度 O(n)

    def __init__(self):
        # 小顶堆：存储较大的一半，堆顶是这部分的最小值
        self.min_heap = []
        # 大顶堆：存储较小的一半，堆顶是这部分的***\*最\**大值（存负数实现）
        self.max_heap = []

    def addNum(self, num: int) -> None:
        # 1. 先将元素加入大顶堆
        heapq.heappush(self.max_heap, -num)

        # 2. 将大顶堆的最大值移到小顶堆
        #    -heapq.heappop(self.max_heap) 取出最大值（取反后变回正数）
        #    再压入小顶堆
        heapq.heappush(self.min_heap, -heapq.heappop(self.max_heap))

        # 3. 保持两个堆的大小平衡：大顶堆的大小 >= 小顶堆的大小
        if len(self.max_heap) < len(self.min_heap):
            heapq.heappush(self.max_heap, -heapq.heappop(self.min_heap))

    def findMedian(self) -> float:
        # 如果元素总数为奇数，中位数在大顶堆堆顶
        if len(self.max_heap) > len(self.min_heap):
            return -self.max_heap[0]
        # 如果元素总数为偶数，中位数为两个堆顶的平均值
        return (-self.max_heap[0] + self.min_heap[0]) / 2.0
```

## ⏱ 复杂度

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 addNum | O(log n) | 最多 3 次堆操作，各 O(log n) |
| 时间 findMedian | O(1) | 直接取堆顶 |
| 空间 | O(n) | 存储所有元素 |

## ⚠ 易错点

- [ ] `max_heap` 存负数模拟大顶堆：放入时 `heappush(max_heap, -num)`，取出时 `-heappop(max_heap)`。注意 `-num` 是取反、`-heappop()` 也是取反
- [ ] 平衡条件：必须保证 `len(max_heap) >= len(min_heap)`（比 min 大 1 或相等），不能反过来
- [ ] 中位数计算：奇数时 `-max_heap[0]`；偶数时 `(-max_heap[0] + min_heap[0]) / 2.0`，注意用 `/ 2.0` 保证浮点结果
- [ ] 插入流程三步不能颠倒：先入 max -> 再移到 min -> 再平衡。如果跳过第二步直接平衡，可能出现 max 中有元素大于 min 中所有元素的情况

```text
这道题是数据流中位数的经典题，用两个堆解决。

维护一个大顶堆（max_heap）存储较小的一半，一个小顶堆（min_heap）存储较大的一半。
保持 max_heap 的大小等于 min_heap 或比它多 1。

插入时，先用"过一遍"法：新元素先入 max_heap，
然后把 max_heap 的堆顶（最大值）移到 min_heap。
最后如果 max_heap 比 min_heap 小，再移一个回来，保持平衡。

查询中位数时，如果总数为奇数，max_heap 堆顶就是中位数；
如果偶数，两个堆顶的平均值就是中位数。

添加 O(log n)，查询 O(1)，空间 O(n)。

Python 因为只有最小堆，所以 max_heap 用负数来模拟大顶堆。
```
